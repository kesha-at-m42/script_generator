## **‚úÖ STRUCTURE & TIMING CHECKLIST**

Scripts must:

* Include meaningful student interaction in all phases  
* Avoid disengagement invitations (e.g., ‚ÄúWant to keep going?‚Äù)  
* Stage vocabulary (concept before term)  
* Use remediation logic in Practice (Light ‚Üí Medium ‚Üí Heavy)  
* Ensure visual/metaphor continuity unless flagged  
* Introduce visuals only in Lesson phase (unless tagged \[Minimum\_Expectation?\])  
* Use directive, not optional phrasing (e.g., ‚ÄúLet‚Äôs try this‚Ä¶‚Äù)  
* Fraction coverage and symbolic integration match Full List tab and Visualization Specs  
* Warm-Up includes at least two: visual interest, judgment task, mini anchor, or real-world context  
* Tool naming, affordances, constraints, and misuse risks must match Visualization Specs. Any noncanonical use must be tagged \[Variant\_Use\] and documented.

If Spiral Tool Onboarding occurs (late path reroute to unfamiliar tool), follow Queue Rulebook spec: \[Tool\_Intro\] \+ review-level spiral task from original module \+ forward bridge to current module.
