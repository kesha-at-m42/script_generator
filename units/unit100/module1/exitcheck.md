### EC Rectangle Selection

Per Tool Flow Document M02 §5: EC.1 uses canonical-adjacent multiplication facts to test transfer of the reversed-orientation Focus skill beyond the instructional 4 × 5 = 20 rectangle. The selected facts are:

- **EC1:** 5 × 8 = 40 (paired with rectangle display)
- **EC2:** 7 × 6 = 42 (equation given; rectangle companion display optional)
- **EC3:** 36 = 9 × 4 (reversed-orientation given; rectangle companion display optional)

These three facts are intentionally outside the instructional rectangle to confirm that the relational-equal-sign insight transfers to facts the student has not seen in M02's lesson copy. The instructional canonical 4 × 5 = 20 fact is NEVER used in EC sub-items.

### Transition Frame

*(Teacher language moving from §1.7 lesson → §1.8 Exit Check. No student action during the frame.)*

Tutor narration (student-facing): "You just wrote the same fact two ways. Now I'm going to show you a different rectangle and a different fact. Show me you can do the same thing — two ways around — with a fact you haven't seen yet today. Three quick questions. Take your time."

*(The frame deliberately names "a different rectangle" and "a different fact" so the student is oriented to the transfer task. Required phrase "two ways around" carries forward from the lesson. Balance-scale framing is not re-invoked here unless Heavy-tier remediation triggers post-EC.)*

### EC.1: 3-Item Exit Check [APPLICATION]

*(EC.1 is delivered as a single Exit Check arc containing three sub-items — EC1, EC2, EC3 — per the Mini contract. The Mini Design Doc and Tool Flow Document consistently treat the EC arc as one beat. Sub-items are sequenced; mastery is computed across all three.)*

**Purpose:** Test the reversed-orientation Focus skill bidirectionally. The L.3 operations-on-both-sides recognition exposure is NOT exit-checked — EC.1 tests only reversed-orientation recognition and production.

**Toy:** §2.10 Drop Down (production sub-items EC2, EC3) + §2.11 Multiple Choice / Checkbox (recognition sub-item EC1) + Hundred Grid Mode B/C companion display.

**Introduction (student-facing):** "Now show what you know. Three quick questions about the same fact, both ways around."

| EC Sub-Item | Display | Student Task | Skill Tested | Mastery Contribution |
|---------|---------|-------------|-------------|---------------------|
| **EC1** | `5 × 8 = 40` rectangle displayed (canonical-adjacent fact, NOT the 4 × 5 instructional rectangle). §2.11 Checkbox. Candidates: `5 × 8 = 40` ✓, `40 = 5 × 8` ✓, `5 + 8 = 40` ✗, `8 × 40 = 5` ✗. | Select ALL correct equations. | Recognition: both reversed orientations, both must be selected | Strong requires both correct selected with no distractors |
| **EC2** | `7 × 6 = 42` displayed as given. §2.10 Drop Down stem: `The same rectangle shows ___ × ___ = ___ AND ___ = ___ × ___ because the equal sign means both sides have the same value.` | Complete the stem — fill both orientations. | Production: reversed form from given standard | Needs Practice if EC1+EC3 correct but EC2 wrong |
| **EC3** | `36 = 9 × 4` displayed as given. §2.10 Drop Down stem: `The same rectangle shows ___ × ___ = ___ AND ___ = ___ × ___ because the equal sign means both sides have the same value.` | Complete the stem — fill both orientations. | Production: standard form from given reversed | Still Gathering if 0–1 items correct OR if EC1 selects only `a × b = c` and rejects `c = a × b` |

**Vocabulary check (EC items):** No "factor," "factor pair," "prime," "composite," "height," "handle," bare "squares," or WODB language. "Multiple" / "is a multiple of" is permitted in EC items (Abstract section). No operations-both-sides items in EC.1.

**Required phrase (verbatim in EC2 and EC3 production stems):**
`The same rectangle shows ___ × ___ = ___ AND ___ = ___ × ___ because the equal sign means both sides have the same value.`

### EC Verification Checklist

- [ ] **EC-Alignment:** All three EC sub-items map to LG1 + LG2 + Module Goal (per Alignment Check table)
- [ ] **EC-Independence:** Student completes EC.1 with toy support but no tutor cueing; no Mode C pulse / balance-scale invocation pre-Check
- [ ] **EC-Language:** EC2 and EC3 stems use the required-phrase verbatim production stem; no preferred-orientation language ("backward," "reversed," "normal") appears
- [ ] **EC-Transfer:** EC sub-items use canonical-adjacent facts only (5 × 8, 7 × 6, 9 × 4); the instructional canonical 4 × 5 = 20 fact does not appear in EC
- [ ] **EC-Scope:** No operations-both-sides items (no `a × b = b × a` and no `a × b = c × d`) appear in EC.1
- [ ] **EC-U1.6 detection:** EC1 detection logic for U1.6-active flag is wired (selects-only-`a × b = c` pattern triggers Heavy-tier ladder per §1.4.1)

## 1.8.5 PRACTICE INPUTS

### Practice Phase Overview

M02 operates under the Mini contract (7 interactions, no mid-module calibration). Practice depth is intentionally bounded: the student's primary practice opportunities for the reversed-orientation Focus skill occur (1) inside L.4 (the supported production beat), (2) inside EC.1 (three sub-items), and (3) downstream in M03+ post-Check displays where Mode C dual-orientation rendering provides ongoing implicit practice on every fact-equation interaction. There is no separate within-module practice phase in M02.

*(For the canonical model of a Practice Phase with a within-module pipeline of generated practice problems, see M03 — the next module — which expands the practice surface alongside its broader factor-pair search.)*

### Dimension Constraints

M02 operates with one instructional rectangle (4 × 5 = 20) and three canonical-adjacent transfer facts (5 × 8 = 40, 7 × 6 = 42, 9 × 4 = 36). No other dimensions are introduced. No factor-pair search occurs in M02; the dimension space is intentionally minimal so that the relational-equal-sign insight is not crowded out by visual or numerical variety. Dimension expansion begins in M03.

### Mastery Routing

*(Module-specific subsection. Per Mini Design Doc §8 and Tool_Flow_Document_M02.md §5.2. Skeleton-permitted within Practice Inputs — see skeleton lines 568–570.)*

| Exit Check State | Response |
|-----------------|---------|
| **Strong** | EC1 correct (both correct orientations selected, no distractors) AND EC2 correct AND EC3 correct. → Advance to M03 normally. |
| **Needs Practice** | EC1 correct OR EC2 correct OR EC3 correct, but not all three. → Re-deliver Beats 5–6 (L.4 production + EC.1) with canonical fact reused. Re-enter Exit Check. If second pass returns Needs Practice: route to M03 with `U1.6-active` flag — every fact-equation post-Check display in M03 invokes balance visual with extra emphasis. M03 entry is NOT blocked. |
| **Still Gathering** | Zero or one item correct, OR EC1 selects only `a × b = c` and rejects `c = a × b`. → Re-deliver the full M02 arc (all 7 beats) with canonical fact reused. Re-enter Exit Check. If second pass still returns Still Gathering: block M03 entry; surface to SME / human tutor. |
| **U1.6-active flag** | EC1 selects only `a × b = c` and rejects `c = a × b` → tag `U1.6-active`; route through Heavy remediation tier (§1.4.1) before re-entering Exit Check. |
