## 1.0 THE ONE THING

**The equal sign is relational, so a multiplication equation can be written with the product on either side — `a × b = c` and `c = a × b` are equivalent forms of the same fact, and neither is preferred.**

**CRA Stage:** Concrete → Relational → Abstract (Mini-module arc). In Concrete (Beats 1–2), the student re-encounters the canonical 4 × 5 = 20 M01 rectangle and sees both equation forms revealed simultaneously in Mode C. In Relational (Beats 3–4), the student recognizes both reversed orientations and a recognition-only operations-both-sides form (`4 × 5 = 5 × 4`), confirming that the equal sign means both sides have the same value. In Abstract (Beat 5), the student produces both equation orientations in a guided stem using the same canonical rectangle. The Exit Check (Beat 6 / EC.1) and Synthesis (Beat 7 / S.1) complete the arc. CRA distribution: 20 / 50 / 30 (Concrete / Relational / Abstract).

**Module designation:** Mini-module. M02 carries full Required-track status — Mini designation does not reduce the U1.6 instructional stakes. Mini contract: 7 interactions (W.1, L.1, L.2, L.3, L.4, EC.1, S.1), within the 4–8 band. No mid-module calibration.

**Primary Misconception Target:** U1.6 (M6) — reversed-equation rejection / equal-sign-as-operator. Cross-national data ([Powell 2012](https://pmc.ncbi.nlm.nih.gov/articles/PMC3374577/); [Li et al. 2008](https://files.eric.ed.gov/fulltext/EJ1184961.pdf)) shows U.S. students accept reversed equations at ~28% vs. ~98% in China. This gap does not self-correct ([Knuth et al. 2008](https://alibali.psych.wisc.edu/wp-content/uploads/sites/371/2018/02/KnuthAlibaliHattikudurMcNeilStephens2008.pdf)). M02 is the targeted intervention.

**Hybrid acknowledgment:** Per Mini Design Doc §3 and §9.1 (SME 2026-04-27, reaffirmed SME 2026-04-28), M02 includes a single recognition-only exposure to operations-on-both-sides form (`a × b = b × a`) at L.3 / Beat 4. This acknowledges [McNeil et al. 2006](https://cladlab.nd.edu/assets/250420/mcneiletal06.pdf) evidence that operations-on-both-sides equations promote relational equal-sign understanding more strongly than reversed-orientation alone. Production of operations-on-both-sides equations remains out of scope. The L.3 beat is NOT exit-checked — EC.1 tests only the reversed-orientation Focus skill.

**Canonical rectangle:** ALL M02 instructional beats use the 4 × 5 = 20 rectangle the student built in M01. No new rectangle builds occur in M02. EC.1 uses canonical-adjacent facts (5 × 8, 7 × 6, 9 × 4) to test transfer.

**Success Indicator:** After M02, the student (a) reads both `4 × 5 = 20` and `20 = 4 × 5` as correct, equivalent names for the same rectangle; (b) produces the reversed-orientation form when given the standard form (and vice versa) for any fact; (c) explains that the equal sign means both sides have the same value, not "write the answer here."

**Biggest Risk:** If Mode C is not enforced in M03+ post-Check displays, M02's remediation work for U1.6 is undone. Both-orientation post-Check display from M03 onward must be a hard engineering constraint (RISK-M02-1).

## 1.4 MISCONCEPTION COVERAGE

### 1.4.1 U1.6: Reversed-Equation Rejection / Equal-Sign-as-Operator (PRIMARY)

**Global ID:** U1.6 (M6)
**Sources:** Phase 4 Research Summary §3 M6; Module Mapping v2 §6 Misconception Coverage Matrix; Mini Design Doc §2 Trigger Evidence; Tool_Flow_Document_M02.md §4.

U1.6 is the sole and primary misconception target for M02. The student treats `=` as a left-to-right compute directive ("write the answer here") rather than as a relational symbol meaning both sides have the same value. Consequently, the student rejects `c = a × b` as "backwards" or "wrong" — the reversed orientation does not fit the mental template of "number-times-number-equals-answer."

**Observable signs:**
- Student selects only `a × b = c` and rejects `c = a × b` in a multi-select recognition item
- Student completes a production stem by writing the standard orientation when asked for the reversed form, or vice versa
- Student describes `=` as "makes" or "gives you" rather than "is the same as" or "balances"
- Exit Check EC1: student selects `4 × 5 = 20` but NOT `20 = 4 × 5` from the candidate set (U1.6-active flag trigger)

**Three-Tier Remediation Ladder:**

| Tier | Trigger pattern | Tutor move | Surface | Return condition |
|------|----------------|------------|---------|-----------------|
| **Light — visual flip** | Student hesitates or selects only one orientation in an early recognition prompt | Tutor pulses the right-panel `c = a × b` equation in the Mode C strip and animates attention across the two equations on the same rectangle. Sentence stem: `Both forms show the same rectangle because ___.` Re-cue without giving the answer. No new rectangle. | Hundred Grid Mode C highlight pulse + Drop Down stem | Student accepts both forms on the next recognition move |
| **Medium — balance scale** | Rejection pattern persists across two consecutive recognition items; student treats `=` as a one-way signal | Re-invoke balance framing: "The equal sign is not a 'go' button — it's a balance. This side and this side have the same value." Tap-and-hold cross-emphasis on Mode C confirms both equations point to the same canonical rectangle. | Static balance-scale tutor asset + Hundred Grid Mode C cross-emphasis | Student names the equal sign as showing a balance |
| **Heavy — modeled both-forms walkthrough** | Exit Check (EC.1) response shows U1.6-active — selects only `a × b = c` in EC1 and rejects `c = a × b` | Tutor presents the canonical 4 × 5 = 20 rectangle again with BOTH equations already displayed in Mode C (pre-filled, not student-built), narrates step-by-step: "The area is 20. The side lengths are 4 and 5. `4 × 5` makes 20 — so `4 × 5 = 20`. And 20 is the same as `4 × 5` — so `20 = 4 × 5`." Student then completes a near-copy stem. Re-delivers L.1 with the same canonical fact, then re-enters Exit Check. If second pass still returns Still Gathering, surface to SME / human tutor. | Mode B + Mode C pre-filled display + Drop Down scaffolded production | Student completes the next stem with both forms present |

*(Per Research_Summary §7.5, U1.6 is a 4-tier expansion candidate if prototype data shows persistent rejection. Heavy tier above is Tier 3; Tier 4 escalation = SME / human tutor. Engineering must implement all three tiers fully — placeholder remediation codes are forbidden per BrainLift DOK3.)*

### 1.4.2 Hybrid Beat 4 — Operations-Both-Sides Recognition (PREVENTION / ENRICHMENT)

The recognition-only L.3 / Beat 4 exposure to `4 × 5 = 5 × 4` is not a standalone misconception target — it is a bounded enrichment beat that addresses the equal-sign-as-operator root of U1.6 using the stronger evidence base from [McNeil et al. 2006](https://cladlab.nd.edu/assets/250420/mcneiletal06.pdf). By seeing that operations can appear on both sides of `=`, the student's relational-equal-sign framing is strengthened beyond what reversed-orientation recognition alone provides. Beat 4 does NOT test this enrichment in EC.1 — the Focus skill tested in EC.1 is reversed-orientation only.

## 1.6 WARMUP (3–5 minutes)

### Core Purpose

The Warmup orients the student's attention to equation orientation as a dimension of variation before the lesson formally introduces both-orientation equivalence. It re-encounters the canonical 4 × 5 = 20 rectangle from M01 (Concrete prelude) so that the lesson's Mode C first reveal at L.1 lands on already-familiar ground.

### Warmup Type Rationale

**Type:** [EXPOSURE] — re-encounter + dimension-of-variation orientation.

The student already trusts the 4 × 5 = 20 rectangle from M01. Re-encountering it in W.1 capitalizes on established prior knowledge and frees cognitive bandwidth for the relational-equal-sign insight that Mode C will deliver at L.1. Using equation-orientation as the discriminative dimension (three candidates with the product on the right; one with the product on the left) primes the student to attend to where the product sits — the precise dimension the lesson's One Thing will reframe as "two equivalent forms of the same fact." The post-Check bridge statement ("That form is today's topic — and you'll see that it is just as correct as the others") immediately neutralizes the risk that "breaks the pattern" reads as "is wrong."

### Interaction W.1: Re-encounter the 4 × 5 = 20 Rectangle [REVIEW]

**Setup:**
Four equation candidates appear via §2.11 Multiple Choice. All four candidates involve the canonical 4 × 5 = 20 fact. Three share a pattern: they are written with the product on the right side (`a × b = c`). One breaks the pattern: it is written with the product on the left side (`c = a × b`). Hook: "Look at the rectangle you built last lesson. Four equations all describe it. Three share a pattern. One breaks it. Which one?"

Candidate set (authoring constraint, single canonical fact only): `4 × 5 = 20` ✓-pattern, `5 × 4 = 20` ✓-pattern, `4 × 5 = 20` ✓-pattern (presented with neutral variation in display order so visual placement is not the discriminator), `20 = 4 × 5` ✗-pattern-breaker (breaks the "product on right" pattern).

*(Authoring note 1: To preserve four distinct candidates while keeping the single canonical fact discipline, content authors may use the four equivalent canonical-fact equations: `4 × 5 = 20`, `5 × 4 = 20`, `20 = 4 × 5`, `20 = 5 × 4`. ## 1.7 LESSON STRUCTURE

### Core Purpose + Pedagogical Flow

The M02 lesson delivers the One Thing — the equal sign is relational, so both `a × b = c` and `c = a × b` are equivalent forms of the same fact — across a compressed Concrete → Relational → Abstract arc using the canonical 4 × 5 = 20 rectangle the student already built in M01.

- **Section 1 (CONCRETE):** The student re-encounters the 4 × 5 = 20 rectangle in Mode B (reuse, no new build). Mode C activates post-Check and reveals both equation forms for the first time. The static balance-scale tutor asset appears and names the equal sign as a balance. These two beats establish the empirical ground: same rectangle, two legitimate equation forms.
- **Section 2 (RELATIONAL):** The student recognizes both reversed-orientation forms (L.2) and the operations-on-both-sides form `4 × 5 = 5 × 4` (L.3 / Hybrid Beat). Recognition-before-production is enforced. L.3 required phrase "same value on both sides" anchors the relational equal-sign framing in its strongest form. Neither L.2 nor L.3 requires production.
- **Section 3 (ABSTRACT):** The student produces both equation orientations for the canonical rectangle using a guided Drop Down stem (L.4). The word "multiple" / "is a multiple of" may now appear. This is the first production beat.

Required Phrases anchor each section. Forbidden Phrases prevent vocabulary-fence violations. Purpose Frame opens the lesson before any student action. Every beat serves a math content purpose — no beat exists solely to teach a UI affordance.

### Interaction L.1: Mode C First Reveal + Balance Framing (CONCRETE → RELATIONAL) [CONNECT]

**Setup:**

**Common Detour:**

## 1.10 KEY DESIGN DECISIONS (KDDs)

### KDD-1: One Thing — Equal Sign as Relational; Both-Forms Equivalence

The M02 One Thing directly targets U1.6 (reversed-equation rejection / equal-sign-as-operator) with precise scope: the student learns that the equal sign is relational — both sides have the same value — and therefore both `a × b = c` and `c = a × b` are equivalent, legitimate forms of the same fact, neither preferred. This is narrower than general "relational reasoning" instruction and broader than any single fact. The One Thing wording is verbatim in §1.0, §1.1 LG2, and §1.2 Must Teach item 1, ensuring it propagates through the full authoring chain.

Cross-national data grounds the urgency: U.S. ~28% vs. China ~98% acceptance of reversed equations ([Powell 2012](https://pmc.ncbi.nlm.nih.gov/articles/PMC3374577/); [Li et al. 2008](https://files.eric.ed.gov/fulltext/EJ1184961.pdf)). The gap does not self-correct across grades ([Knuth et al. 2008](https://alibali.psych.wisc.edu/wp-content/uploads/sites/371/2018/02/KnuthAlibaliHattikudurMcNeilStephens2008.pdf)), making M02 a required, not optional, intervention.

### KDD-2: Vocabulary Fence Rationale (M02-Specific)

M02 suppresses four categories of fenced terms:

1. **factor / factor pair / prime / composite** — These are M03 and M04 territory respectively. Introducing them here would cross-contaminate concept formation before the systematic factor-pair search (M03) or prime/composite classification (M04) establishes the definitional context. M03 is named as the FIRST appearance of "factor" / "factor pair" in student-facing copy.

2. **height / handle / bare "squares"** — System-wide forbidden terms per Vocabulary Guidance §5 and M01 corrections #2, #2a, #4. These apply in M02 as in all modules.

3. **WODB / "doesn't belong"** — System-wide replacement with "Which one breaks the pattern" per Vocabulary Guidance §4 / correction #3. If any discrimination routine is used in M02, only "Which one breaks the pattern" is permitted.

4. **"multiple" / "is a multiple of" (Beats 1–4)** — Gated to Abstract (L.4 / Beat 5 onward). The M01 vocabulary anchor "multiple" is load-bearing in M01 but is NOT the M02 One Thing. Introducing it in the Concrete/Relational sections of M02 would dilute the relational-equal-sign framing that is M02's sole focus. The gate is explicit: "multiple" is forbidden in student-facing copy at W.1, L.1, L.2, and L.3.

### KDD-3: Hybrid Beat 4 Inclusion Rationale

The L.3 / Beat 4 recognition-only exposure to `4 × 5 = 5 × 4` (operations-on-both-sides, multiplicative-commutative form) is included per SME 2026-04-27 Hybrid decision (reaffirmed 2026-04-28) to address [McNeil et al. 2006](https://cladlab.nd.edu/assets/250420/mcneiletal06.pdf) evidence: "After exposure to the operations-both-sides equations, students understood the equal sign relationally almost twice as often as after exposure to operation-right-side equations alone."

The Hybrid is bounded by four constraints:
- **Recognition only:** Student selects the correct equation from two candidates; no production of `a × b = b × a` form is required.
- **Multiplicative-commutative form only:** Only `a × b = b × a` (e.g., `4 × 5 = 5 × 4`). Equivalent-product form (`a × b = c × d`, e.g., `4 × 5 = 2 × 10`) is M03 territory and never appears in M02 L.3.
- **Not exit-checked:** EC.1 tests only the reversed-orientation Focus skill. The L.3 enrichment exposure is not gated.
- **Required phrase:** "same value on both sides" must appear in teacher move and/or post-Check stem at L.3. "multiple" / "is a multiple of" is forbidden at L.3.

This bounded Hybrid keeps M02 narrow (Mini contract preserved; atomic skill scope: reversed-orientation read/produce) while acknowledging the stronger evidence base for operations-both-sides instruction.

### KDD-4: Balance Visual as Relational Anchor (Static Asset)

The balance-scale framing for the equal sign is the primary instructional vehicle for addressing U1.6 at the conceptual level. The decision (OQ-M02-BAL-1, RESOLVED SME 2026-04-28) is to use a **static tutor-layer asset** — a balance-scale image rendered in the tutor explanation zone at L.1 (Beat 2) — rather than a Mode C strip icon or a dynamic animation.

Rationale:
- A static image in the tutor explanation layer has a clear, unambiguous host (tutor zone) and does not compete with the Mode C dual-panel equation display for student attention.
- A Mode C icon at the `=` symbol would require the Mode C component to carry both the equation display and the balance framing — two instructional loads on one UI element.
- A dynamic animation is higher engineering cost and higher cognitive load than needed for a first-encounter framing beat.

The balance visual is re-invoked in Medium-tier remediation (§1.4.1) and in the S.1 synthesis tutor narration. Its function is to make "same value on both sides" concrete and repeatable — not ornamental.

### KDD-5: Single Canonical Rectangle Rationale

Per OQ-M02-RECT-1 (RESOLVED SME 2026-04-28), the canonical 4 × 5 = 20 rectangle is used for ALL M02 instructional beats (Beats 1–5 and 7). No swapping among a bank of known rectangles.

Rationale:
- **Cognitive simplicity:** M02's One Thing is the equation-orientation insight. Adding visual variety across different rectangle shapes introduces perceptual load that is unrelated to the Focus skill. The single canonical rectangle lets the student focus entirely on what changes (equation form) vs. what stays constant (the same rectangle, the same area, the same side lengths).
- **M01 continuity:** The 4 × 5 = 20 rectangle is the rectangle the student built and checked in M01. Reusing it in M02 capitalizes on established prior knowledge — the student already trusts this rectangle.
- **RISK-M02-3 accepted:** SME chose maximum cognitive simplicity over visual variety. Stem variation and Mode C reveal pacing provide freshness; EC.1 introduces canonical-adjacent facts (5 × 8, 7 × 6, 9 × 4) for transfer.

### KDD-6: No New Rectangle Builds in M02 — Reuse-Only Purity

No M02 interaction asks the student to build a new rectangle. Mode B build affordances are DISABLED for all instructional beats in M02 (Beats 1–5, 7). EC.1 presents canonical-adjacent facts as given (displayed, not built).

Rationale: M02's Focus skill is equation orientation, not rectangle construction. Every cognitive step spent building a new rectangle is a step not spent on the relational-equal-sign insight. Reuse-only purity also eliminates any risk of the student forming a new M01-style "multiple" association with an unfamiliar rectangle while the relational-equal-sign concept is still being established.

Engineering enforcement: Every Mode B surface in M02 carries a reload-only flag. Any authored activity that sets build affordances to ENABLED in M02 is a build error (QC-04 check).

### KDD-7: CRA Relational (Not Representational) — BrainLift v2

The middle CRA stage in this system is named **Relational**, not "Representational." This follows BrainLift v2's explicit rename (DOK3 CRA section): "The stage rename from Representational to Relational reflects that the middle stage is about relating quantities, not just picturing them."

In M02, the Relational stage (L.2 and L.3) is about the student relating both equation forms to the same rectangle and recognizing that the equal sign expresses a relationship of equal value — not about creating a visual diagram or picture. The rename is load-bearing for the authoring framing: "RELATIONAL" in §1.7 section headers and in every CRA label in this document. "Representational" does not appear outside this KDD as the named-and-rejected alternative.

### KDD-8: Mode C M02-First Activation — M03+ Inheritance

Mode C (Hundred Grid Dual-Orientation Paired Post-Check Display) is introduced for the FIRST TIME in M02. M01 suppressed Mode C at the toy layer via a per-activity flag (M01 Tool Flow v4 §4 / §7). M02 activates Mode C for the first time; M03+ inherits it as the unit-wide standard.

The "M02-only" label refers to FIRST INTRODUCTION — not an exclusive lock. Once a student has established both-orientation fluency through M02's arc, every fact-equation post-Check display in M03–M08 shows both `a × b = c` AND `c = a × b` in Mode C. This is a unit-wide, hard engineering constraint (RISK-M02-1): M03 Tool Flow must explicitly activate Mode C in its post-Check displays. If Mode C is omitted from M03+ post-Check displays, M02's U1.6 remediation is undone by the downstream experience.

Phase 6 v4 reference: `Digital_Toys_Interactables_v4.md` §2.13 Mode C.

### KDD-9: "Which One Breaks the Pattern?" Routine Inheritance from M01

M01 established "Which one breaks the pattern" as the system's discrimination routine name, replacing the IM-inherited "Which One Doesn't Belong?" per Vocabulary Guidance §4 (correction #3). M02 inherits this convention at W.1.

In M02, the routine serves a different discriminative dimension than in M01: instead of distinguishing rectangles by multiple-of-N structure, M02's W.1 uses equation-orientation as the discriminative dimension — three standard-orientation equations vs. one reversed-orientation equation. The routine structure is preserved; only the domain changes.

If any future M02 activity uses a discrimination routine, only "Which one breaks the pattern?" is permitted. "Which One Doesn't Belong," "WODB," and "doesn't belong" are forbidden in all student-facing copy system-wide.

### KDD-10: Mini Contract — 7 Beats, No Mid-Module Calibration, Open Questions

M02 operates under the Mini contract: 7 interactions (W.1, L.1, L.2, L.3, L.4, EC.1, S.1) within the 4–8 band. No mid-module calibration. One concentrated equation-orientation rehearsal arc with one bounded operations-both-sides recognition exposure (Hybrid, not mid-module calibration).

The Mini contract does NOT reduce instructional rigor. U1.6's stakes are high (cross-national gap; gap does not self-correct). The 7-beat structure is the minimum viable arc to move a student from re-encountering the M01 rectangle (Concrete) through recognition-before-production (Relational) to guided production (Abstract) with an Exit Check and Synthesis closure — all within the word-count band (800–1,800).
