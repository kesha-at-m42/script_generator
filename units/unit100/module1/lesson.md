## 1.7 LESSON STRUCTURE

### Core Purpose + Pedagogical Flow

The M02 lesson delivers the One Thing — the equal sign is relational, so both `a × b = c` and `c = a × b` are equivalent forms of the same fact — across a compressed Concrete → Relational → Abstract arc using the canonical 4 × 5 = 20 rectangle the student already built in M01.

- **Section 1 (CONCRETE):** The student re-encounters the 4 × 5 = 20 rectangle in Mode B (reuse, no new build). Mode C activates post-Check and reveals both equation forms for the first time. The static balance-scale tutor asset appears and names the equal sign as a balance. These two beats establish the empirical ground: same rectangle, two legitimate equation forms.
- **Section 2 (RELATIONAL):** The student recognizes both reversed-orientation forms (L.2) and the operations-on-both-sides form `4 × 5 = 5 × 4` (L.3 / Hybrid Beat). Recognition-before-production is enforced. L.3 required phrase "same value on both sides" anchors the relational equal-sign framing in its strongest form. Neither L.2 nor L.3 requires production.
- **Section 3 (ABSTRACT):** The student produces both equation orientations for the canonical rectangle using a guided Drop Down stem (L.4). The word "multiple" / "is a multiple of" may now appear. This is the first production beat.

Required Phrases anchor each section. Forbidden Phrases prevent vocabulary-fence violations. Purpose Frame opens the lesson before any student action. Every beat serves a math content purpose — no beat exists solely to teach a UI affordance.

### Required Phrases

**Required Phrases (Concrete + Relational sections — Beats 1–4, W.1 through L.3):**

- "Look at the rectangle you built last lesson."
- "[N] square tiles in each row."
- "I built a 4-by-5 rectangle." *(referenced from M01 build)*
- "The same rectangle. The same fact. Two ways to write it."
- "Look at the equal sign."
- "The equal sign is not a 'go' button — it's a balance."
- "Both sides have the same value." *(required for balance framing, L.1)*
- "The equal sign means both sides have the same value." *(required phrase, L.1)*
- "Tap and hold either equation to see the rectangle light up."
- "Choose ALL the equations that correctly describe this rectangle."
- "Both correct equations must be selected for a green check."
- "Two ways to write the same fact." / "Both ways around."
- "Both of these equations describe the same rectangle." *(L.3 setup)*
- **"Same value on both sides."** *(L.3 REQUIRED — must appear in Beat 4 tutor copy and/or post-Check stem)*
- "Which one breaks the pattern?" — discrimination routine inherited from M01 if used.

**Required Phrases (Abstract section — Beat 5 onward, L.4 through S.1):**

These phrases are gated to Section 3 and may not appear in student-facing copy before L.4.

- "I built a 4-by-5 rectangle, so I can write `4 × 5 = 20` AND `20 = 4 × 5` — they say the same thing." *(required production stem template)*
- "Finish the sentence. This rectangle lets me write the same fact in two ways: `___ × ___ = ___` AND `___ = ___ × ___`."
- "The same rectangle shows `___ × ___ = ___` AND `___ = ___ × ___` because the equal sign means both sides have the same value."
- "They say the same thing, and the equal sign means both sides have the same value."
- "Two ways to write the same fact." *(also permitted in Section 2 when not combined with "multiple" language)*
- "Neither equation is preferred." *(locks the orientation claim — explicit One Thing closure phrase)*
- "20 is a multiple of 4 and 5." *(optional Abstract-only bridge phrase — permitted from L.4 onward if a light bridge back to M01 multiple-language is needed)*

### Purpose Frame

**Tutor narration (student-facing):** "In the last lesson, you built a rectangle with 4 square tiles in each row and 5 rows. You checked it and saw that the area was 20 square tiles. Today you will look at that same rectangle again — and learn two different ways to write what you already know about it. The equal sign is the key. Let's look at what it really means."

### Section 1: CONCRETE (L.1)

*(Note: W.1 is now hosted in §1.6 WARMUP per skeleton compliance; §1.7 Section 1 begins with L.1.)*

### Interaction L.1: Mode C First Reveal + Balance Framing (CONCRETE → RELATIONAL) [CONNECT]

**Setup:**
Hundred Grid Mode B reloads the canonical 4 × 5 = 20 M01 rectangle. Build affordances are DISABLED — the student sees the rectangle but cannot place new square tiles. Tutor prompt (student-facing): "Here is the rectangle you built last lesson. It has 4 square tiles in each row and 5 rows. The area is 20 square tiles."

Student taps Check. **Mode C activates automatically post-Check.** Two-panel strip appears below the rectangle: left panel `4 × 5 = 20`, right panel `20 = 4 × 5`. A bridge element (double-headed arrow or `=`) between panels signals equivalence. Both panels render at identical font size, identical background, identical visual weight.

**Simultaneously**, the static balance-scale tutor-layer asset appears in the tutor explanation zone alongside (not inside) the Mode C strip. Tutor narration (student-facing): "Look at the equal sign. It is not a 'go' button. It is a balance. What is on this side has the same value as what is on that side. The equal sign means both sides have the same value."

**Student Action:**
Student taps and holds the left panel (`4 × 5 = 20`) — the rectangle highlights on Mode B above. Student taps and holds the right panel (`20 = 4 × 5`) — the same rectangle highlights. The student observes that both equations point to the same rectangle.

Then the student fills the Drop Down stem: `The equal sign means ___.` Pre-filled options (student selects): "both sides have the same value" / "the answer goes on the right" / "I should multiply." Only "both sides have the same value" is correct.

**Teacher Move:**
After stem completion: "You got it. The equal sign means both sides have the same value — it is a balance, not a 'go' button." *(Observable language only. Balance-scale asset remains visible during the stem.)*

**Key Observation:**
Student selects "both sides have the same value" in the stem AND demonstrates the tap-and-hold cross-highlighting on both panels. Both behaviors confirm early relational-equal-sign framing.

**Common Detour:**
Student selects "the answer goes on the right" — the tutor redirects to the balance visual: "Look at the balance. Both sides of a balance have the same weight. Both sides of the equal sign have the same value. Which option matches the balance?" Re-offer the stem.

**Intervention (Light):**
If student hesitates on which equation is "correct," pulse the Mode C right panel (`20 = 4 × 5`) and animate attention across both panels. Prompt: "This equation and that equation both show the same rectangle. Can you tap and hold each one to see?" Do not reveal the answer; return student to the stem.

**Next Move:**
Proceed to Section 2 (Relational), Interaction L.2.

**Duration:** 3–4 minutes.

### Section 2: RELATIONAL (L.2 and L.3)

### Interaction L.2: Both Orientations Match One Rectangle (RELATIONAL) [IDENTIFY]

**Setup:**
Hundred Grid Mode B companion display shows the canonical 4 × 5 = 20 rectangle (build affordances disabled). §2.11 Multiple Choice / Checkbox surfaces four equation candidates. Tutor prompt (student-facing): "Look at this rectangle. Choose ALL the equations that correctly describe it." (Both correct equations must be selected for a green check.)

Candidate set: `4 × 5 = 20` ✓, `20 = 4 × 5` ✓, `4 + 5 = 20` ✗ (addition distractor), `4 × 20 = 5` ✗ (product/side-length swap distractor).

Post-Check Drop Down stem: `These equations are correct because ___.` Pre-filled options: "they show two ways to write the same fact" / "the equal sign means both sides have the same value" / "one of them must be wrong." Both first two options are reinforcing; student selects one (either accepted).

**Student Action:**
Student selects equations via checkbox taps, then taps Check. Post-Check: student completes the Drop Down stem.

**Teacher Move:**
If both correct options are selected: "Both of those describe the same rectangle — two ways to write the same fact." If only one correct option is selected: Light-tier intervention (pulse the unselected panel in Mode C; prompt "Is there another equation that describes this same rectangle?"). If an incorrect option is also selected: "Look at that equation. Does `4 + 5` equal 20? What operation does this rectangle show?"

**Key Observation:**
Student selects BOTH `4 × 5 = 20` AND `20 = 4 × 5` — bidirectional recognition of reversed orientations as both correct. This is the primary observable for L.2.

**Common Detour:**
Student selects only `4 × 5 = 20` — U1.6 pattern. Tutor invokes Light-tier (Mode C right-panel pulse). If pattern persists: Medium-tier (balance scale re-invocation).

**Intervention:**
Light → Medium per §1.4.1 remediation ladder. Use the same canonical 4 × 5 = 20 rectangle — no new rectangle.

**Next Move:**
Proceed to Interaction L.3 (Hybrid Beat).

**Duration:** 2–3 minutes.

### Interaction L.3: Operations-Both-Sides Recognition — Hybrid Beat (RELATIONAL) [IDENTIFY]

*(Engineering-internal note: This beat is the L.3 NEW Hybrid per SME 2026-04-28. Recognition-only. Multiplicative-commutative form `a × b = b × a` ONLY. NOT exit-checked. The word "multiple" is FORBIDDEN in student-facing copy at this beat.)*

**Setup:**
Mode C hosts a paired-equation display: left panel `4 × 5 = 20` (the reversed-orientation fact already established in L.1/L.2), right panel `4 × 5 = 5 × 4` (the operations-on-both-sides multiplicative-commutative form). The canonical 4 × 5 = 20 Mode B rectangle is visible above the paired display.

§2.11 Multiple Choice surfaces two candidates. Tutor prompt (student-facing): "Both of these equations describe the same rectangle. Tap the one that says it correctly."

Candidate set: `4 × 5 = 5 × 4` ✓ (operations-on-both-sides, multiplicative-commutative, correct), `4 × 5 = 5 + 4` ✗ (mixed-operation distractor, incorrect).

Post-Check Drop Down stem: `Both sides of `4 × 5 = 5 × 4` show the ___.` Target completion: "same value." Pre-filled options: "same value" / "same number of square tiles only" / "same number of rows only."

**Student Action:**
Student taps `4 × 5 = 5 × 4`, then taps Check. Completes the Drop Down stem.

**Teacher Move:**
Post-Check: "You got it. Both sides of `4 × 5 = 5 × 4` show the same value — 20. The equal sign means same value on both sides, no matter how many operations are on each side." *(Required phrase: "same value on both sides.")*

**Key Observation:**
Student selects the correct multiplicative-commutative form AND completes the stem with "same value." This confirms the relational equal-sign framing extends to forms with operations on both sides.

**Common Detour:**
Student selects the mixed-operation distractor (`4 × 5 = 5 + 4`). Tutor: "Does `5 + 4` equal 20? Let's check: 5 + 4 = 9, not 20. The equal sign means same value on both sides — which equation has the same value on both sides?" Re-offer the choice.

**Intervention:**
If student is unsure, the tutor highlights each panel of the Mode C display and narrates: "Left side: `4 × 5` — what is that?" (student mentally confirms: 20). "Right side: `5 × 4` — what is that?" (student mentally confirms: 20). "Same value on both sides." Then re-offer the choice.

**Misconception Prevention Note:**
This beat does NOT test `4 × 5 = 2 × 10` or any equivalent-product form — that is M03 territory. The distractor uses `5 + 4` to avoid equivalent-product confusion. The word "multiple" / "is a multiple of" does NOT appear in any student-facing copy at this beat. The word "commutative" does NOT appear in student-facing copy (engineering-internal only).

**Next Move:**
Proceed to Section 3 (Abstract), Interaction L.4.

**Duration:** 2–3 minutes.

### Section 3: ABSTRACT (L.4)

### Interaction L.4: Supported Production — Both Orientations (RELATIONAL → ABSTRACT) [CONNECT]

**Setup:**
Hundred Grid Mode B companion display shows the canonical 4 × 5 = 20 rectangle. Mode C paired display is visible (both `4 × 5 = 20` and `20 = 4 × 5`). §2.10 Drop Down stem unlocks. Tutor prompt (student-facing): "Finish the sentence. This rectangle lets me write the same fact in two ways."

Drop Down stem (required): `I built a 4-by-5 rectangle, so I can write ___ × ___ = ___ AND ___ = ___ × ___ — they say the same thing, and the equal sign means both sides have the same value.`

*(Stem slots: first blank set = 4, 5, 20; second blank set = 20, 4, 5. Drop Down constrains slots — no free text. Both equation orientations must be completed for a green check.)*

**Student Action:**
Student fills the six numeric slots in the Drop Down stem to produce `4 × 5 = 20 AND 20 = 4 × 5`. Taps Check.

**Teacher Move:**
Post-Check (both orientations correct): "You wrote both of them. `4 × 5 = 20` and `20 = 4 × 5` — the same fact, two ways. The equal sign means both sides have the same value." *(Observable only; no identity praise.)*

If only one orientation is completed correctly: Light-tier (Mode C pulse on the incomplete panel; re-cue the stem). If repeated partial completion: Medium-tier (balance scale re-invocation with the stem visible).

**Key Observation:**
Student successfully fills BOTH equation orientations in the stem — first genuine production beat. Both numeric slots correct (4 × 5 = 20; 20 = 4 × 5).

**Common Detour:**
Student fills the second set as `20 = 4 × 5` but inverts to `4 × 5 = 20` again (copies the first form). Tutor: "You wrote the same form twice. Look at the right panel in the display — what numbers go on the left side of the equal sign there?"

**Intervention:**
Light → Medium per §1.4.1 ladder. Mode B + Mode C cross-emphasis; balance scale reappears if needed.

**Next Move:**
Proceed to §1.8 Exit Check (EC.1).

**Duration:** 3–4 minutes.

### Misconception Prevention

- **U1.6 at every beat:** The equal sign is narrated as a balance (relational) — never as a compute-and-write directive. Mode C's equal-weight dual-panel display enforces this visually. The static balance-scale asset appears at L.1 and is re-invoked in Medium-tier remediation.
- **Preferred-orientation language is forbidden:** No prompt, stem, or teacher narration uses "standard," "reversed," "backward," "normal," or "the answer goes here." Both orientations are named equally throughout.
- **No new builds:** Every Mode B surface in M02 reloads the canonical 4 × 5 = 20 rectangle with build affordances DISABLED. No authored task constructs a fresh rectangle.
- **"Multiple" gated to Beat 5 (L.4) onward:** The word "multiple" / "is a multiple of" does not appear in student-facing copy in Beats 1–4. This prevents the M01 vocabulary anchor from cross-contaminating the M02 relational-equal-sign framing.
- **L.3 operations-both-sides scope:** Only `a × b = b × a` form appears. No `a × b = c × d` (equivalent-product) form. No production asked. Not exit-checked.

### Incomplete Script Flags

- **ISF-M02-1:** Warmup W.1 candidate set is illustrative. Content authors must verify the four equation options meet the orientation-variation constraint before authoring. Do not author a warmup where the pattern-breaker is distinguishable by a non-mathematical visual attribute.
- **ISF-M02-2:** L.3 Drop Down stem post-Check wording must be reviewed against the "multiple" gate — "same value" is the target, not "same multiple." Any stem revision must be re-screened for vocabulary-fence violations.
- **ISF-M02-3:** EC.1 item density (OQ-M02-EC-1) is unresolved pending SME confirmation. Do not migrate items without SME approval.
- **ISF-M02-4:** L.1 Drop Down stem options screen — the L.1 stem options ("both sides have the same value" / "the answer goes on the right" / "I should multiply") use the verb "multiply," not the fenced noun "multiple," and are within fence as authored. However, any revision to the L.1 stem options must be screened explicitly against the "multiple" / "is a multiple of" gate (forbidden in Beats 1–4 student-facing copy per §1.3) before re-authoring. Content authors expanding L.1 options in the future must not introduce "it makes the multiple," "is a multiple of," or any "multiple"-fenced phrasing. Cross-reference: §1.7 Forbidden Phrases.
- **ISF-M02-5:** W.1 "breaks the pattern" framing risk — the W.1 candidate set positions `20 = 4 × 5` (reversed orientation) as the pattern-breaker, which can inadvertently signal that the reversed orientation is aberrant — precisely the opposite of what M02 teaches. **RESOLUTION:** The W.1 post-Check teacher move includes a verbatim required bridge statement: "That form is today's topic — and you'll see that it is just as correct as the others." This statement is REQUIRED — not optional — in W.1 authoring. Authors must not omit, paraphrase, or soften the "just as correct" claim. Cross-reference: §1.7 W.1 Teacher Move.

### Success Criteria

By the end of §1.7, the student:
1. Has seen the canonical 4 × 5 = 20 rectangle produce both `4 × 5 = 20` AND `20 = 4 × 5` in Mode C.
2. Can select both correct reversed-orientation forms for the canonical rectangle in a multi-select recognition item (L.2).
3. Can identify `4 × 5 = 5 × 4` as correct (same value on both sides) in a single-select recognition item (L.3).
4. Has completed the guided Drop Down stem producing both orientations for the canonical rectangle (L.4).
5. Has produced a sentence naming the equal sign as relational ("both sides have the same value").
