## **1.6 WARMUP (3-4 minutes)**

**Detail Level:** Structure + parameters (AI writes dialogue)

---

### **Purpose**

Activate prior knowledge of scaled graphs (1:2) from M2 while introducing the key M3 concept: scale of 5 is more efficient for larger numbers. Students collect personal data, then OBSERVE the same data displayed as BOTH 1:2 and 1:5 picture graphs side-by-side. The discovery that "scale of 5 uses fewer symbols" creates curiosity and sets up Lesson's formalization of the skip-counting connection.

### **Parameters**

| Element | Specification |
| :---- | :---- |
| Interactions | 4 interactions (Game + Scale Comparison Display + Efficiency Discovery + Bridge) |
| Active Participation | 3 of 4 require student response |
| Format | Data Collection Game → Side-by-Side Scale Comparison → Efficiency Discovery → Bridge |
| Cognitive Load | 20-30% (observation and comparison are accessible) |
| Remediation | Light only |
| Tone | Medium-High energy, curious, builds anticipation |
| Engagement Anchors | Narrative Setup (Minis), Personalization (their data), Curiosity Gap (which is better?) |

### **Constraints**

| USE | AVOID |
| :---- | :---- |
| Informal language ("Check this out," "Look at this") | Formal phrasing ("Observe the comparison") |
| "Scale of 2" and "Scale of 5" (familiar from M2) | Formal teaching of new vocabulary |
| Observable acknowledgments ("You counted 20") | Assumed states ("You noticed the difference") |
| 3 categories | 2 or 4+ categories |
| Multiples of 5 only (10, 15, 20, 25, 30) | Non-multiples |
| Values that show clear efficiency difference | Values too small to see benefit (e.g., 5) |
| Half-symbols acceptable on 1:2 graph (reinforces M2) | Avoiding half-symbols unnecessarily |

---

### **Warmup Type Rationale**

**Standard Playbook Types:** Binary Choice, WODB, Number Talk, Notice & Wonder, Estimation 180, Visual Patterns, Mystery Reveal

**M3 Approach:** Data Collection Game + Scale Efficiency Comparison

**Why this serves the concept:**

* Unit 1 is about DATA—personal data creates investment
* M3's core insight is EFFICIENCY: same data, fewer symbols with scale of 5
* Side-by-side comparison makes abstract concept visible and concrete
* Students SEE the benefit before Lesson teaches the HOW (skip-counting)
* Follows Playbook PRINCIPLES (low cognitive load, engagement anchors, curiosity gap) while adapting STRUCTURE for the efficiency discovery

**Test:** If we removed this Warmup, would students lose mathematical preparation for Lesson? YES—they would miss the motivation for WHY scale of 5 matters (efficiency), making Lesson feel like arbitrary new content rather than a meaningful improvement.

**Key Difference from M2 Warmup:**

* M2: Introduces SCALING concept (1 symbol = multiple items) via grouping animation
* M3: Introduces EFFICIENCY concept (same data, fewer symbols) via side-by-side comparison
* M3's grouping animation moves to Lesson Section 1 (skip-counting connection is the new learning)

---

**Interaction W.1: Data Collection Game**

* **Purpose:** Generate personalized data. Create investment through Minis context. Provide concrete numbers for scale comparison.
* **Hook (first 15-20 seconds):**
  * **Visual:** Minis characters appear in animated counting scenario.
  * **Guide:** "The Minis are having a busy day! Help count what they're up to."
* **Engagement Anchor:** Narrative Setup (Minis context), Personalization (their counts)
* **Game Specifications:**

| Element | Specification |
| :---- | :---- |
| Style | Mario Party-style counting game |
| Categories | 3 (e.g., Red Minis, Blue Minis, Green Minis) |
| Values | Multiples of 5 only: 10, 15, 20, 25, or 30 per category |
| Duration | ~60 seconds |
| Student Interaction | Count items; confirm/input count for each category |

* **Task:** Student participates in counting activity, tracking count for each of 3 categories.
* **On Complete:** Game ends; counts recorded for next interaction.

**Design Note:** All values MUST be multiples of 5 AND large enough to show efficiency difference. Target range: 10-30. Values like 20, 25, 30 show the clearest difference between scales.

---

**Interaction W.2: Scale Comparison Display (THE KEY MOMENT)**

* **Purpose:** Visually demonstrate that the SAME data looks different with different scales. Set up the efficiency discovery.
* **Engagement Anchor:** Curiosity Gap (what's different about these two graphs?)
* **Visual:** Picture Graphs (side-by-side comparison). Same student data shown as 1:2 picture graph AND 1:5 picture graph. Both vertical. Keys visible on both: "Each ⭐ = 2" and "Each ⭐ = 5." Data Table not visible. Half-symbols acceptable on 1:2 graph.
* **Major Beats:**
  * LEFT graph builds first
  * **Guide:** "Here's your data with scale of 2. Each symbol shows 2 items."
  * RIGHT graph builds (noticeably fewer symbols)
  * **Guide:** "And here's the SAME data with scale of 5. Each symbol shows 5 items."
  * Both graphs now visible side-by-side
  * **Guide:** "Same data. Same totals. But look at the graphs… What do you notice about the two graphs?"
* **Prompt:** "What do you notice about the two graphs?"
* **Student Action:** \[Multiple choice:
  * "The scale-of-5 graph has fewer symbols"
  * "The scale-of-5 graph has more symbols"
  * "They have the same number of symbols"
  * "The graphs show different data"\]
* **Correct Answer:** "The scale-of-5 graph has fewer symbols"
* **On Correct:** "Fewer symbols for the same data. When each symbol shows MORE, you need FEWER of them."

**Design Note:** This is the conceptual payoff. Students SEE the efficiency before Lesson explains the mechanism (skip-counting).

---

**Interaction W.3: Efficiency Discovery**

* **Purpose:** Solidify the efficiency insight. Students quantify the difference.
* **Visual: Picture Graphs (side-by-side comparison).** Same display as W.2. System highlights symbol counts for comparison.
* **Guide:** "Look at \[category name\]. It has \[X\] items."
* **Guide:** "With scale of 2... that's \[X÷2\] symbols."
* **Guide:** "With scale of 5... that's only \[X÷5\] symbols. Which scale is more efficient for showing \[X\] items?"
* **Prompt:** "Which scale is more efficient for showing \[X\] items?"
* **Student Action:** \[Multiple choice: Scale of 2, Scale of 5\]
* **Correct Answer:** Scale of 5
* **On Correct:** "Scale of 5. When numbers get bigger, scale of 5 saves space. Fewer symbols, same information."

---

**Interaction W.4: Bridge to Lesson**

* **Purpose:** Create anticipation for Lesson without teaching the mechanism.
* **Visual: Picture Graphs (1:5 graph).** Scale-of-5 graph from W.2 remains. 1:2 graph fades. Data Table not visible.
* **Guide:** "You discovered something important: Scale of 5 uses fewer symbols for the same data."
* **Guide:** "But HOW do you know how many symbols to use? There's a skill that helps..."
* **Guide:** "Do you remember how to count by 5s?"
* **Bridge Statement:** "Let's learn how skip-counting and scale of 5 work together."
* **No student prompt**

---

### **Verification Checklist**

**Structure:**

- [ ] Hook appears in first 15-20 seconds (Minis appear)
- [ ] 2+ engagement anchors (Narrative, Personalization, Curiosity Gap)
- [ ] 3 active student interactions (Game, Comparison Noticing, Efficiency Selection)
- [ ] Side-by-side comparison is the key visual moment
- [ ] Clear bridge to Lesson creates anticipation (skip-counting teased)
- [ ] Total time 3-4 minutes

**Content Alignment:**

- [ ] 3 categories (not 2 or 4)
- [ ] All values multiples of 5 (10, 15, 20, 25, 30 range)
- [ ] No non-multiples (half-symbols appear on 1:2 graph only, demonstrating why 1:5 is more efficient)
- [ ] Both scales shown with same data (1:2 AND 1:5)
- [ ] Efficiency discovery is explicit ("fewer symbols, same data")
- [ ] Skip-counting teased but NOT taught (save for Lesson)
- [ ] Keys visible on both graphs ("Each ⭐ = 2" and "Each ⭐ = 5")

**Constraints:**

- [ ] No student graph creation (observation and discovery only)
- [ ] No formal skip-counting instruction (just teased in bridge)
- [ ] No grouping animation (moves to Lesson Section 1)
- [ ] Cognitive load light (20-30%)
