## **1.9 SYNTHESIS (\~6-7 minutes)**
**Purpose**
Consolidate understanding that highlighting and rotation are two proofs of the same property ("same dots, same product"), generalize the commutative property beyond specific arrays to ALL multiplication, connect the property to its strategic value (every fact you know gives you its partner), and reflect on what made this discovery stick. This is the final module in the multiplication unit — closure names the M7→M12 arc, not just M12. This is NOT more practice — students have demonstrated mastery. It's connection-making, generalization, and unit culmination.
### **Parameters**
| Element | Specification |
| --- | --- |
| Interactions | 4 connection tasks + reflection + closure (6 beats total) |
| Task Types | Type A (Strategy Consolidation), Type B (Generalization), Type C (Real-World Bridge), Type 4 (Key Takeaway) |
| Remediation | Light only — mastery assumed post-Practice |
| Tone | Reflective, warm, discovering together |
| Energy | Medium (reflective, not exhausted) |

### **Opening Frame**
- **Visual:** Clean transition. Practice clears.
- **Guide:** "You've been using the commutative property to solve problems — with arrays, without arrays, in all kinds of situations. Let's step back and connect what you discovered."
---
### **Interaction S.1: Strategy Consolidation — Two Proofs, One Idea (Type A)**
- **Purpose:** Explicitly consolidate the two proof methods taught in Lesson. Students see that highlighting and rotation are two routes to the same conclusion — and the shared anchor is "same dots."
- **Visual:** Split screen. Left side: A 4×5 array with highlighting demo — rows highlighted (4 × 5 = 20), then columns highlighted (5 × 4 = 20). Right side: Same 4×5 array with rotation animation — turns 90° to become 5×4, expression updates (5 × 4 = 20). Both sides display their equations.
- **Guide:** "Two ways to prove the same thing. On the left, highlighting. On the right, rotation. What's the SAME about both?"
- **Prompt:** "What do both proofs have in common?"
- **Options (MC):** A) Same dots, nothing is added or removed \| B) <span discussion-urls="discussion://3215917e-ac52-800a-ad60-001c4fefcbd7">~~The arrays look the same~~</span> Both methods give the same expression\| C) Both use columns only {color="yellow_bg"}
- **Correct Answer:** A) Same dots, nothing is added or removed
- **Guide (on correct):** "Same dots. That's the anchor. Whether you read it a different way or turn it, no dots change. Same dots means same total. Always."
- **Remediation:** Pipeline
- **Answer Rationale:**
	- A) Same dots, nothing is added or removed = Correct. Student identifies the structural anchor shared by both proofs.
	- B) Both methods give the same expression = Student confuses "same product" with "same expression." Highlighting produces two *different* expressions (4 × 5 and 5 × 4) for the same array; rotation also produces two different expressions. What stays the same is the *dots and the product*, not the expressions. 
	- C) Both use columns only = Factually incorrect. Highlighting uses rows AND columns; rotation swaps them.
---
### **Interaction S.2: Generalization — Does This ALWAYS Work? (Type B)**
- **Purpose:** Push understanding from "it works for the arrays I've seen" to "it works for ALL multiplication." This is the principle extraction moment — students move from empirical observation to mathematical generalization.
- **Visual:** Display<span discussion-urls="discussion://3215917e-ac52-8066-99ad-001c93a6db51">: "If 9 × 7 = 63, what's 7 × 9?"</span>
- **Guide:** "Here's one you haven't seen as an array today. 9 times 7 is 63. What's 7 times 9?"
- **Prompt:** "If 9 × 7 = 63, what's 7 × 9?"
- **Options (MC):** A) 63 \| B) 79 \| C) 56 \| D) I'd need to see the array
- **Correct Answer:** A) 63
- **Guide (on correct):** "63. You didn't need an array. You used the turn-around facts."
- **Remediation:** Pipeline
- **Answer Rationale (application):**
	- A) 63 = Correct. Property applied without visual support.
	- B) 79 = Concatenation error. Student combined digits rather than applying the property.
	- C) 56 = Incorrect. Other math fact in close proximity to product (7 × 8)
	- D) I'd need to see the array = Partial understanding. Student has connected the property to arrays but hasn't generalized beyond the visual. Indicates the property hasn't fully internalized.
*(Immediately followed by generalization:)*
- **Guide:** "Now here's the big question. Will this work for ANY two numbers you multiply, not just the ones we tried?"
- **Prompt:** "Does 'multiply factors in any order and get the same product' work for ALL numbers?"
- **Options (MC):** A) Yes, it always works \| B) Only for small numbers \| C) Only when you have an array  \| D) Only when I know the first multiplication fact 
- **Correct Answer:** A) Yes, it always works
- **Guide (on correct):** "Always. Any two numbers. You could build an array for 9 × 7 and rotate it. Same dots, same product. You can imagine an array even when you don't see one. That's why mathematicians call it  a property, it's always true."
- **Guide: **”And here's something interesting, this isn't true for every operation. 8 minus 3 is NOT the same as 3 minus 8. But with multiplication, you can always switch the factors and get the same product. You already knew addition worked this way: 3 plus 5 equals 5 plus 3. Now you've proven multiplication works the same way." {color="yellow_bg"}
- **Remediation:** Pipeline
- **Answer Rationale (generalization):**
	- A) Yes, it always works = Correct. Student has generalized from specific examples to a universal principle.
	- B) Only for small numbers = Incomplete generalization. Student may believe the property depends on array size or visual confirmability.
	- C) Only when you have an array = Representation-dependent understanding. Student hasn't separated the property from the proof tool. Needs the conceptual leap: arrays PROVE the property, but the property EXISTS independently.
	- D) Only when I know the first multiplication fact = Student is still not convinced. They may think they need to know the fact to prove that it will be true. If they don't know the fact, they cannot state the property with certainty.
---
### **Interaction S.3: Strategy Value — Every Fact Has a Partner (Type C)**
- **Purpose:** Connect the commutative property to its practical value: every multiplication fact you learn automatically gives you its partner. This is the real-world payoff — the property isn't just interesting, it's useful.
- **Visual:** Display showing three facts with their partners revealed:
	- 3 × 8 = 24 → 8 × 3 = 24 ✓
	- 6 × 5 = 30 → 5 × 6 = 30 ✓
	- 4 × 9 = 36 → 9 × 4 = 36 ✓
	- Below: "3 facts learned → 6 facts known"
- **Guide:** "Look at this. You learned 3 facts. But because you can multiply the factors in any order, you actually know 6 facts. Every fact gives you its partner for free."
- **Guide:** "Which of these is a good way to use this strategy?"
- **Prompt:** "Which is a good use of the strategy?"
- **Options (MC):** A) If I know 5 × 7, I already know 7 × 5 — no extra work \| B) If I know 5 × 7, I should still compute 7 × 5 separately to be sure \| C) The strategy only helps when I have an array in front of me
- **Correct Answer:** A) If I know 5 × 7, I already know 7 × 5 — no extra work
- **Guide (on correct):** "No extra work. One fact, two answers. That's what makes this a strategy, it saves you time and effort."
- **Remediation:** Pipeline
- **Answer Rationale:**
	- A) If I know 5 × 7, I already know 7 × 5 — no extra work = Correct. Student sees the property as a time-saving strategy.
	- B) I should still compute 7 × 5 separately to be sure = #13 residual / #20 risk. Student states the property but doesn't trust it enough to apply it without verification. May need more exposure in future practice, but this is diagnostic, not remediation-worthy.
	- C) Only helps when I have an array = Representation-dependent. Same pattern as S.2 option C — student hasn't separated property from proof tool.
---
### **Interaction S.4: Key Takeaway (Metacognitive Reflection — Type 4)**
- **Purpose:** Students identify what they want to remember from today. Module-specific options ensure reflection is grounded in M12 content.
- **Visual:** Clean display — no array. Reflection moment.
- **Guide:** "If you had to remember ONE thing from today, what would it be?"
- **Prompt:** "What's the one thing to remember?"
- **Options (MC):**
	- A) You can multiply in any order and always get the same product
	- B) Arrays can be read by rows or columns
	- C) Rotation proves that rows become columns
	- D) Knowing one multiplication fact gives you its partner
- **No incorrect answer** (all are valid takeaways).
- **Guide responses:**
	- A) "That's the big idea. The order doesn't change the product and you proved it."
	- B) "That's where it all started. Two descriptions, same array and same product."
	- C) "Rotation made it impossible to deny. Same dots, just turned."
	- D) "That's the strategy. One fact, two answers. That saves you real time."

### **Identity-Building Closure (Unit Culmination)**
- **Purpose:** M12 is the final module in the multiplication unit. Closure names the M7→M12 arc — not just M12 accomplishments — and sends students off with a sense of the full journey.
- **Visual:** Brief montage or summary display showing the progression:
	- Equal groups (bags/circles) → Expressions → Equations → Arrays → Property
- **Guide:** "Think about where you started. You found equal groups in bags and circles. You learned that multiplication means groups of the same size. You built expressions, then equations. You discovered arrays: organized groups with rows and columns. And today, you proved something powerful: you can multiply in any order and always get the same answer."
- **Guide:** "You didn't just memorize that. You proved it with highlighting and rotation. You used it as a strategy. That's real mathematical thinking."
- **Guide:** "Get ready."
