# **Module 12: Arrays for Problems and Properties**

# BACKBONE

---

## **1.0 THE ONE THING**

When you change the order of the factors in a multiplication expression, the product stays the same — and arrays prove it visually.

**Critical Misconception:** #13 — Order Changes Answer (Believing 3×4 ≠ 4×3). Students may have internalized from M11 that "3 rows of 4" and "4 columns of 3" are different descriptions and conclude they must produce different answers. *(Decision Q6)*

**Success Indicator:** Given a multiplication fact (e.g., 4 × 6 = 24), student can immediately state 6 × 4 = 24 without computing, AND can explain WHY using arrays (rotation or dual interpretation).

**Biggest Risk:** Students memorize "you can multiply in any order" as a rule without understanding WHY it works. If the visual proof (highlighting + rotation) doesn't land, students have a stated rule without a mental model — exactly the #20 pattern (Memorization Without Relationships).

### **1.1.2 Module Bridges**

**From M11 (Arrays as Multiplication):** Students can identify rows and columns in any dot array, write a multiplication expression from a row description AND from a column description, recognize that both descriptions are valid for the same array, compute products from array-based expressions, and specify arrays using word-based description. M11's Synthesis deliberately surfaced the observation that both expressions give the same product ("Interesting, right? Same product. That's something worth remembering.") and its closure teased M12: "You noticed something in those two products. Next time, we'll figure out WHY that happens — and whether it's always true." The 4×3 array briefly rotated 90° as a visual tease with no narration.

**This Module (M12):** Students prove that the two expressions for the same array always give the same product through two visual methods: (1) highlighting — same array read as rows gives one expression, read as columns gives another, both have same total because neither adds nor removes items; (2) rotation — physically turning the array 90° swaps rows and columns but preserves every item, proving equivalence. Students name this discovery ("you can multiply in any order" / commutative property) and apply it strategically (if you know 3 × 7 = 21, you immediately know 7 × 3 = 21 without computing). *(Decision Q3)*

**To Future Learning:** The commutative property is the first formal multiplication property students encounter. It establishes that multiplication has structural rules worth discovering and applying — a foundation for the associative and distributive properties (also referenced in 3.OA.B.5). Strategic use of the property ("I know 2 × 7 because I know 7 × 2") doubles the effective size of students' fact repertoire, directly supporting the 3.OA.C.7 fluency goal.

## **1.2 SCOPE BOUNDARIES**

### **✅ Must Teach**

**Highlighting Review (Lesson Section 1): *****(Decision Q1)***

* Same array can be described two ways (rows or columns) — M11 consolidation
* Both descriptions produce expressions that evaluate to the same product — M12 NEW
* Highlighting (row emphasis → column emphasis) demonstrates this visually without adding or removing items
* Guide explicitly names the pattern: "Same array, same items, same product — no matter which way you describe it"
**Rotation Discovery (Lesson Section 2): *****(Decision Q1)***

* Rotation animation: a 3×4 array physically turns 90° to become a 4×3 array
* Items don't change — what were rows become columns and vice versa
* This is a DIFFERENT proof of the same property: rotation proves that 3×4 and 4×3 are related transformations of the same arrangement
* Hybrid understanding: highlighting shows "two ways to read ONE array"; rotation shows "two arrays that are the SAME array turned"
* Both proofs lead to the same conclusion: order doesn't change the product
**Property Naming and Application (Lesson Section 3): *****(Decision Q1)***

* Name the discovery: "You can multiply in any order" (primary) / "commutative property" (exposure, 1-2 uses) *(Decision Q2)*
* Strategic application: if you know one fact, you know its partner without computing
* Mixed tasks: highlighting, rotation prediction, property application ("If 6 × 4 = 24, what's 4 × 6?")
* Equivalence discrimination: distinguish rotated arrays (same total, same items) from genuinely different arrays (same total, different dimensions — e.g., 4×6 vs 3×8, both = 24)
**Vocabulary:** strategy (NEW — purposeful method for solving problems)

### **⚠️ Scope Warnings**

⚠️ **PROPERTY PROOF ≠ RULE MEMORIZATION:** M12's central job is to make the commutative property *visually undeniable* — not to state it as a rule students memorize. If students leave M12 saying "you can multiply in any order" but can't explain WHY using an array, the module has failed. The visual proof (highlighting + rotation) must be the anchor, with the verbal rule as the label for what they've already seen.

⚠️ **CONTEXT VARIETY IN PRACTICE:** While Lesson sections use dot arrays for clean visual proof, Practice MUST include concrete word-problem contexts per L19's problem-solving orientation (e.g., "There are 5 rows of chairs with 4 chairs in each row"). OUR L19 uses contexts like "rows of crayons" and "rows of chairs." Exit Check and Practice should also include some problems WITHOUT arrays visible — testing whether students have internalized the property rather than just reading the visual. Arrays should be available as remediation scaffolding when students struggle with array-free problems. This fulfills Q4's resolution: dot arrays primary for proof, concrete contexts for application, array-free problems for internalization, arrays as remediation support.

⚠️ **EQUIVALENCE ≠ IDENTITY:** 3×4 and 4×3 produce the same PRODUCT but are NOT the same EXPRESSION. "3 rows of 4" is a different description than "4 columns of 3." The property says the products are equal, not that the expressions are identical. Guide language must maintain this precision: "Different expressions, same product" — not "they're the same thing."

⚠️ **AVOID 12 AS A PRODUCT:** Per Andrea's SME feedback, avoid using 12 as a product in key teaching moments. 12 has many factor pairs (1×12, 2×6, 3×4, 4×3, 6×2, 12×1) which can create confusion when the goal is to show one specific rotation relationship. Use products with fewer factor pairs for cleaner demonstrations (e.g., 3×5=15, 2×7=14, 3×7=21).

## **1.4 MISCONCEPTIONS**

### **1.4.1 #13: Order Changes Answer (PRIMARY)**

**Trigger Behavior:** Student sees 3 × 4 = 12 and insists 4 × 3 must be different (or hesitates/recomputes instead of applying the property). When shown a rotated array, student says "that's a different array" or "you can't just switch the numbers." In Exit Check, student correctly identifies both expressions for an array but doesn't connect them to the same product.

**Why It Happens:** M11 carefully taught that "3 rows of 4" and "4 columns of 3" are *different descriptions*. Students correctly learned that the description determines the expression. Some students over-generalize this to: "different expressions → different answers." This is a natural and reasonable inference from M11's precision emphasis — the student is being consistent with what they learned, just extending it too far. Additionally, students' experience with addition and subtraction shows that order matters in subtraction (8 - 3 ≠ 3 - 8), so assuming order matters in multiplication is a reasonable default.

**Where It Surfaces:** Lesson Section 2 (rotation — students may resist the proof), Exit Check (equivalence recognition problem), Practice (property application tasks).

**Prevention Strategy:**

* **Dual proof approach:** Highlighting AND rotation both demonstrate the same conclusion. Students who don't find one proof compelling may connect with the other.
* **Counting verification:** In early Section 1 tasks, students verify by counting/computing that both expressions give the same product. The property is DISCOVERED through repeated observation before being stated.
* **Explicit contrast with subtraction:** Brief acknowledgment that "this doesn't work for subtraction — multiplication is special this way" prevents students from doubting the property.
* **Multiple examples:** Students see 3+ different arrays all confirming the same pattern before the property is named. Discovery through pattern, not assertion.
**Remediation Strategy:**

* If student resists equivalence: Return to a simple array (2×3). Guide highlights rows: "2 rows of 3. Count all the dots." Student counts 6. Guide highlights columns: "3 columns of 2. Count all the dots." Student counts 6. Guide: "Did any dots appear or disappear? No. Same dots, same total." Then rotation animation for the same array.
* If student computes correctly but doesn't generalize: Guide asks "Do you think this will work for EVERY array, or just this one?" Then shows one more example to confirm.
### **1.4.2 #20: Memorization Without Relationships (SECONDARY)**

**Trigger Behavior:** Student can state "you can multiply in any order" but cannot explain why, cannot connect the rule to an array visual, and cannot apply it to unfamiliar facts. Student treats the commutative property as another rule to memorize rather than a structural feature of multiplication they've proven.

**Why It Happens:** Students may focus on the verbal label ("you can multiply in any order") rather than the visual proof that supports it. If the highlighting and rotation don't create a lasting mental image, the property becomes disconnected from understanding. This is particularly likely for students who are strong memorizers but weaker visual-spatial thinkers.

**Where It Surfaces:** Synthesis (if student selects the property as their key takeaway but can't elaborate), Practice (if student applies property correctly but struggles when asked "how do you know?").

**Prevention Strategy:**

* **Proof before label:** Students see multiple visual proofs BEFORE the property is named. The name is a label for something they've already experienced, not a rule they're told to remember.
* **"How do you know?" follow-ups:** After property application tasks, Guide occasionally asks "How do you know 7×3 = 21?" and validates array-based reasoning: "Right — you could rotate the array, or read it by columns instead of rows."
* **Strategic application framing:** Property isn't just a fact — it's a STRATEGY. "If you know 2 × 8 = 16, you already know 8 × 2. That's one less fact to memorize." This gives the property practical value beyond abstract truth.
**Remediation Strategy:**

* If student states rule without understanding: Return to array. "Show me on this array why 3 × 5 equals 5 × 3." If student cannot, Guide walks through highlighting: "Watch the rows... now watch the columns... did any dots change?"
### **1.4.3 #8: Rows/Columns Confusion (RESIDUAL FROM M11)**

**Note:** This is not a new M12 misconception but may still be active in some students. M11 introduced rows and columns; M12 assumes this knowledge. If #8 surfaces during M12, it will manifest as incorrect expression writing (confusing which number goes first), which undermines the commutative property proof because students need to write BOTH expressions correctly to see that they're equal.

**Residual Prevention:** Lesson Section 1 begins with highlighting review that visually reinforces row vs. column distinction. Sequential highlighting (rows one at a time, then columns one at a time) serves double duty: reviewing M11 vocabulary AND setting up the commutative property observation.

**Remediation:** Same as M11 — sequential highlighting with explicit "rows go across, columns go up and down" anchoring. Brief; M12 assumes basic row/column proficiency.

## **1.6 WARMUP (~3 minutes)**

**Core Purpose**

Re-activate M11's dual interpretation of arrays (row expression + column expression) and surface the "same product" observation from M11 Synthesis as a question worth investigating. Students arrive having seen that two expressions can describe the same array; Warmup confirms this skill and then frames the central M12 question: "Does this always happen? Why?"

**Key Function:** This Warmup positions M12 as the payoff to M11's tease. M11 Synthesis deliberately surfaced product invariance ("Interesting, right? Same product.") and its closure promised: "Next time, we'll figure out WHY that happens — and whether it's always true." M12 Warmup picks up that exact thread: re-activate the dual interpretation skill (W.1), re-surface the product observation with a *new* array (W.2), and bridge to the Lesson's proof work.

### **Parameters**

| Element | Specification |
| --- | --- |
| Interactions | 2 meaningful interactions + bridge |
| Active Participation | 2 of 2 require student response |
| Format | Recognition Callback (M11 skill) → Notice & Wonder (product observation) → Bridge |
| Cognitive Load | 20-30% (W.1 is M11 recognition; W.2 is low-demand noticing) |
| Remediation | Light only |
| Tone | Medium-High energy, curious, building toward "today we prove it" |
| Engagement Anchors | Narrative Setup (Hook — "something hiding in the numbers"), Personalization (Hook — callback to M11 accomplishment) |

### **Constraints**

| USE | AVOID |
| --- | --- |
| Session-relative language ("Last time / this time") | Temporal language ("yesterday/today") |
| "Array" / "row" / "column" / "expression" / "product" (all M11-owned) | "Commutative" / "property" / "strategy" (Lesson introduces) |

### **Opening Hook (15-20 seconds)**

* **Visual:** Arrays with Pictures (dot mode, neutral emphasis): A 2×7 dot array displayed centrally. No emphasis highlighting yet.
* **Guide:** "Here's an array. You got really good at reading these last time — rows and columns, two expressions. But this array is hiding something in its numbers. Let's see if you can spot it."
* **No student action** (activation only).
> **Design Note:** "You got really good at reading these last time" validates prior accomplishment and activates array schema without re-teaching.

> **Design Note:** "This array is hiding something in its numbers" frames the mathematical discovery (product invariance) as something worth finding.

---

### **Interaction W.1: Row Expression Recognition**

* **Purpose:** Confirm M11 dual interpretation fluency. Student identifies which expression matches the row description of the array. Quick success with established skill.
* **Visual:** Arrays with Pictures: 2×7 dot array. System applies row emphasis (horizontal lines highlighted).
* **Guide:** "I'm highlighting the rows. See them going across? There are 2 rows, and each row has 7 dots. Which expression matches what you see?"
* **Prompt:** "Which expression describes the rows?"
* **Options (MC):** A) 2 × 7 | B) 7 × 2 | C) 7 + 2 | D) 2 + 7
* **Correct Answer:** A) 2 × 7
* **Guide (on correct):** "2 times 7. 2 rows of 7. Nice."
* **Remediation:** Pipeline
---

### **Interaction W.2: Product Observation (Notice & Wonder)**

* **Purpose:** Surface the "same product" observation from M11 Synthesis with a new array. Frame it as something worth investigating, not something already explained.
* **Visual:** Same 2×7 array. System transitions to show column emphasis. Both expressions now displayed with computed products:
  * **Row expression:** 2 × 7 = 14
  * **Column expression:** 7 × 2 = 14
* **Guide:** "Now check this out — here's the row expression AND the column expression, both with their products. Look at those two products. What do you notice?"
* **Prompt: **"What do you notice about the two products?"
* **Options (MC): **A) They're both 14 — the same product| B) They're different because the expressions are different| C) The second product is bigger| D) You can't tell without counting again
* **Correct Answer: **A) They're both 14 — the same product
* **Answer Rationale:**
  * A = Correct. Observes product equivalence.
  * B = #13 preview. Student assumes different expressions → different products. Diagnostically valuable.
  * C = Numerosity distractor. Student may expect the "bigger first number" expression to yield more.
  * D = Uncertainty. Student doesn't trust the displayed computation.
* **Guide (on correct):** "Same product! Two different expressions, but they both equal 14. Hmm. You noticed something like that last time too..."
* **Remediation:** Pipeline
> **Design Note:** Using 2×7 instead of a familiar array from M11 serves the "is this ALWAYS true?" question better — students see the pattern hold for a new case, not just recall a specific example. 2×7 is on the recommended teaching array list, product = 14 (avoids 12 per SME), not square, and small enough for low cognitive load.

---

### **Bridge to Lesson (10-15 seconds)**

* **Visual:** Both expressions (2 × 7 = 14 and 7 × 2 = 14) remain displayed. Array visible.

## **1.7 LESSON (~10-14 minutes)**

**Core Purpose**

Students prove that two expressions describing the same array always give the same product — through two visual methods (highlighting and rotation) — then name this discovery ("you can multiply in any order" / commutative property) and apply it as a strategy. By the end of the Lesson, students can explain WHY order doesn't change the product using arrays, predict the result of rotating an array, distinguish rotated arrays from genuinely different arrays, and apply the property to derive unknown facts from known ones.

### **Pedagogical Flow**

Students arrive from Warmup having re-activated dual interpretation (M11 skill) and observed that 2 × 7 and 7 × 2 both equal 14 — without explanation. The Lesson delivers the explanation through three sections.

Section 1 consolidates M11's dual interpretation and extends it: students don't just write two expressions for the same array, they explicitly verify both expressions give the same product. This is grounding — students see the equivalence pattern across multiple arrays before any explanation of WHY.

Section 2 introduces the new element: rotation animation. The array physically turns 90°, making it visually undeniable that rows become columns, columns become rows, and no dots appear or disappear. Rotation is the "proof" — it explains WHY both expressions give the same product (same dots, just reorganized). Students predict rotation outcomes before seeing them, building understanding rather than passive observation.

Section 3 names what students have discovered and applies it strategically. Vocabulary ("you can multiply in any order" / "commutative property" / "strategy") comes AFTER the visual proof creates mental hooks. Students apply the property to derive facts, and discriminate between rotated arrays (same product because same dots) and genuinely different arrays (same product but different dimensions).
