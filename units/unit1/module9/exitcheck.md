### Parameters
| Element | Specification |
| --- | --- |
| Problems | 3 (standard) |
| Cognitive Types | IDENTIFY, CREATE, CREATE (no COMPARE — not taught in Lesson) |
| Time | 3-4 minutes |
| Tone | Calm, low-stakes, neutral |

---
### Constraints
| MUST | MUST NOT |
| --- | --- |
| Use Equation Builder only (primary M9 tool) | Use Context Visualization, Products Strip, or Picture Graph (assessment, not discovery) |
| Use Methods A (MC) and C/D (tiles) — both practiced in Lesson | Introduce new interaction types |
| Test all three families (×2, ×5, ×10) | Test only one family |
| Use second factor as 2, 5, or 10 (M9 structure) | Use second factor outside 2/5/10 |
| Use first factors in 3-9 range | Use values outside Lesson range or values already used in Lesson |
| Match Lesson difficulty | Increase complexity (no first factors \>10, no multi-step) |
**Value Selection Note:** Lesson used 4×2, 5×2, 3×5, 6×5, 9×5, 3×10, 6×10, 7×10. Exit Check uses different first factors (7×5, 6×2, 4×10) to test computation, not recall.

### Transition into Exit Check
- **Visual:** Lesson Equation Builder clears. Clean transition.
- **Guide:** "You built equations for 2s, 5s, and 10s. Let's see what you know."
> **Voice Note:** Low-stakes framing. One sentence. No "test" or "quiz" language. References what student DID (built equations), not what they should know.
---
### Problem EC.1: Compute ×5 Product (IDENTIFY)
- **Purpose:** Simplest problem — MC method, factors pre-filled; tests only ×5 computation.
- **Visual:** Equation Builder: `7 × 5 = [___]`. Factors pre-filled. MC options in product slot: 12, 30, 35, 40.
- **Guide:** "7 times 5. Pick the product."
- **Prompt:** "What is the product of 7 × 5?"
- **Student Action:** Equation Builder Method A (MC on product)
- **Options:**
	- 12 (sum of factors)
	- 30 (6 counts)
	- 35 ✓ (7 counts)
	- 40 (8 counts)
- **Correct Answer:** 35
- **Answer Rationale:**
	- 35 = Correct (skip-count by 5s seven times: 5, 10, 15, 20, 25, 30, 35)
	- 12 = Added factors instead of multiplying
	- 30 = Skip-count error — stopped at 6 counts
	- 40 = Skip-count error — went to 8 counts
- **On Correct:** "35. It ends in a 5 so we know it's a multiple of 5. 7 times 5 equals 35."
- **Remediation:** Pipeline
> **Remediation Note:** Have student skip-count by 5s seven times to compute the product (5, 10, 15, 20, 25, 30, 35).
> **Design Note:** Simplest problem — MC method, factors pre-filled. Tests only computation. Uses ×5 (middle difficulty family — 5s require more counts than 10s but smaller jumps than 2s at high first factors). Guide dialogue is 5 words. Feedback is result-only, no re-teaching.
---
### Problem EC.2: Build Full from Visual ×2 (CREATE)
- **Purpose:** Moderate-to-hard problem; test factor identification from visual + ×2 computation + tile placement.
- **Visual:** Context Visualization: 6 groups with 2 dots each. Equation Builder: `[___] × [___] = [___]` (all three slots empty). Methods C/D active. Tile palette: 2, 4, 5, 6, 8, 10, 12, 14.
- **Guide:** "Build the equation that matches the picture."
- **Prompt:** "Build the equation represented by this picture."
- **Student Action:** Equation Builder Methods C or D (tile placement for all slots)
- **Correct Answer:** `6 × 2 = 12`
- **Answer Rationale:**
	- `6 × 2 = 12` = Correct
	- Common errors: factor order reversal (2 × 6), wrong product (10 = 5 counts OR 14 = 7 counts instead of 6), wrong product (8 because added the factors instead of multiplying), wrong factors (4 or 5 because miscounted)
- **On Correct:** "6 groups of 2 is 6 times 2 which equals 12."
- **Remediation:** Pipeline
> **Remediation Note:** If student misidentifies factors, prompt: first factor = number of groups (6), second factor = items in each group (2). Then skip-count by 2s six times for product (12).
> **Design Note:** Most complex problem — tile placement instead of MC. Student must extract factors from visual, compute, AND place all three tiles. Tests ×2 family. ×2 computation was taught in Lesson 2.1-2.2 (MC method); visual-to-equation from W.1 and 3.4; tile placement was practiced with ×5 and ×10 (2.3, 3.1-3.3). EC.2 combines a practiced family (×2) with a practiced method (tiles) in a new pairing — appropriate for assessment. Per SME feedback: giving factors verbally while calling this a "harder" problem is backwards — extracting factors from a visual is the genuinely harder demand, which is why EC.2 uses a visual while EC.3 drops to verbal only.
---
### Problem EC.3: Build Full ×10 Equation (CREATE)
- **Purpose:** Moderate difficulty; test all three slots with ×10 (simplest pattern).
- **Visual:** Equation Builder: `[___] × [___] = [___]` (all three slots empty). Methods C/D active. Tile palette: 4, 5, 10, 14, 20, 30, 40, 50.
- **Guide:** "4 groups of 10. Build the equation."
- **Prompt:** "Build the equation for 4 groups of 10."
- **Student Action:** Equation Builder Methods C or D (all three slots)
- **Correct Answer:** `4 × 10 = 40`
- **Answer Rationale:**
	- `4 × 10 = 40` = Correct
	- Common errors: factor order reversal (10 × 4 — NOT accepted; M9 constrains second factor to the skip-counting unit per §1.2 Scope Warnings; commutativity is M12), wrong product (50 = 5 counts or 30 = 3 counts instead of 4), wrong product (14 = added factors instead of multiplying), wrong second factor (5 instead of 10)
- **On Correct:** "4 times 10 equals 40. That ends in a 0 which is what we expect with multiples of 10."
- **Remediation:** Pipeline
> **Remediation Note:** Prompt factor identification from verbal description: 4 groups (first factor = 4), each group of 10 (second factor = 10). Skip-count by 10s four times to find product (40).
> **Design Note:** Moderate difficulty — no visual reference, all three slots empty, student builds the entire equation. Tests factor identification + computation + construction. Uses ×10 (simplest pattern, but full construction raises interaction demand). Same interaction method used in Lesson 3.1 and 3.4. Tile palette includes 5 as distractor (plausible second-factor error).
