# **Module 11: Arrays as Multiplication**

# BACKBONE

---

## 1.0 THE ONE THING

Students must understand that **an array is a rectangular arrangement of objects in rows and columns, and the SAME array can be described two different ways — by its rows OR by its columns — and each description maps to a different multiplication expression where the language MUST match the notation.** M11 is the first module where a single visual produces two valid expressions: a 3×4 array is "3 rows of 4" → 3 × 4 AND "4 columns of 3" → 4 × 3. The learning is that HOW YOU DESCRIBE an array determines WHICH expression you write — and precision matters.

**Critical Misconception:** Rows/Columns Confusion (\#8) — Students mix up horizontal rows and vertical columns. When shown a 3×4 array and asked "How many rows?", they answer "4" (counting columns instead). This confusion cascades: if students can't reliably distinguish rows from columns, they can't accurately describe what they see, and every expression they write becomes unreliable. Unlike Groups vs. Size Confusion (\#10, which reverses factor order), \#8 produces structurally wrong descriptions — the student isn't just reversing numbers, they're misidentifying the physical structure.

**Secondary Misconception:** Array Dimensions as Addition (\#9) — Students who see a 3×4 array and write "3 + 4 = 7" instead of "3 × 4 = 12." This reveals incomplete transition from additive to multiplicative thinking — the student sees two numbers (rows and columns) and defaults to adding them rather than multiplying. This is especially dangerous because 3 + 4 = 7 looks "right" (it IS a correct addition) while being fundamentally wrong about what the array represents.

**Success Indicator:** Given a 4×5 dot array, student:

1. Identifies "4 rows" and "5 columns" correctly (row/column distinction — NEW)
1. Describes as "4 rows of 5" and writes expression 4 × 5 (row description → expression — NEW)
1. Describes the SAME array as "5 columns of 4" and writes expression 5 × 4 (column description → expression — NEW)
1. Recognizes that 4 × 5 and 5 × 4 are BOTH valid for this array (dual interpretation — NEW)
1. Computes product: 4 × 5 = 20 (carried from M9-M10 — NOT new)
**Biggest Risk:** Students who learn "rows first" convention and then treat column descriptions as wrong rather than as a second valid interpretation. M11 walks a tightrope: it must teach the mathematical convention (first factor = rows in "rows of" language) clearly enough that students write precise expressions, but flexibly enough that students understand column descriptions are equally valid — just different. If Section 1 over-anchors rows as "the" way to read an array, students will resist the column interpretation in Section 2, and M12's commutative property proof loses its foundation. The balance is: convention gives you a starting point, but arrays give you TWO valid starting points.

*Verbatim from OUR Curriculum — Script Must Achieve These*

Exit Check Tests:

* Can student correctly identify the number of rows AND columns in an array?
* Can student write a multiplication expression that matches a row description of an array (e.g., "3 rows of 4" → 3 × 4)?
* Can student write a multiplication expression that matches a column description of the same array (e.g., "4 columns of 3" → 4 × 3)?
* Can student compute the product from an array-based expression (e.g., 3 × 4 = 12)?

### 1.1.2 Module Bridges

**From Module 10:** Students can build complete multiplication equations with unknowns in any position, using diverse contexts (bags, rows, stacks, groups). They understand = means "same value as" and can write equations in both orientations (3 × 4 = 12 AND 12 = 3 × 4). They have computational fluency with 2s, 5s, and 10s and can reason about broader factor ranges using Equal Groups with Pictures. M10's Synthesis closure previewed M11: *"Next time, you'll see equal groups arranged in a new way"* with a visual teaser showing dots in rows and columns. The word "array" was NOT used. That's M11's job to introduce.

**This Module:** Students discover that ORGANIZING equal groups into rectangular arrangements (arrays) creates a structure that can be described in two ways: by rows or by columns. Key learning:

* **Array as organized equal groups:** Items arranged in aligned rows AND aligned columns forming a rectangle. Every row has the same number; every column has the same number.
* **Row and column vocabulary:** Rows go horizontal (left-right); columns go vertical (up-down). Formally taught and practiced.
* **Row description → expression:** "3 rows of 4" maps to 3 × 4. First number = how many rows, second number = how many in each row.
* **Column description → expression:** "4 columns of 3" maps to 4 × 3. First number = how many columns, second number = how many in each column.
* **Dual interpretation:** The SAME 3×4 array produces BOTH expressions (3 × 4 and 4 × 3). The description determines the expression. Language must match notation.
* **Computation from arrays:** Students compute products from array-based expressions, applying M9 fluency.
* **Abstraction progression:** Concrete objects (brief real-world grounding) → dot arrays (primary working representation).
**To Module 12:** M11 established that the same array produces two different expressions (3 × 4 and 4 × 3). M12 asks the inevitable question: "Do they give the SAME answer?" M12 uses arrays to PROVE the commutative property, that a × b = b × a, through rotation animation (a 3×4 array rotated 90° becomes a 4×3 array with the same items) and side-by-side comparison. M11's dual interpretation is the foundation; M12's proof is the payoff. Without M11's clear establishment that rows ≠ columns but both are valid descriptions, M12 has nothing to prove equivalent.

## 1.2 SCOPE BOUNDARIES

### **✅ Must Teach**

Array Structure (Lesson Section 1):

* What an array IS: objects arranged in aligned rows and aligned columns forming a rectangle. Every position filled, no gaps.
* Vocabulary: **array** (NEW — formal term for rectangular arrangement), **row** (NEW — horizontal line of items), **column** (NEW — vertical line of items)
* How arrays differ from equal groups: equal groups are clustered (3 bags of 4); arrays are aligned in a grid (3 rows of 4). Same multiplication, different spatial organization.
* Row convention: "rows of" language maps to expression with first factor = rows. "3 rows of 4" = 3 × 4.
Dual Interpretation (Lesson Section 2):

* Column description: same array can also be described by columns. "4 columns of 3" = 4 × 3.
* Language-expression precision: the description you USE determines the expression you WRITE. If you say "rows of," the first number is rows. If you say "columns of," the first number is columns.
* Both descriptions are VALID for the same array — neither is "wrong."
* Guide-controlled row and column highlighting to demonstrate both interpretations visually.
Array Specification and Computation (Lesson Section 3):

* Student specifies arrays using word-based Equation Builder: "Making \*\*\_ rows of \*\*\*" or "Making \*\*\* columns of \*\*\_"
* Student writes matching multiplication expression using Equation Builder (expression mode primary)
* Student computes products from array expressions (extending to full equation mode)
* Dot arrays as primary representation for flexibility
Abstraction Progression:

* Concrete objects for initial grounding (1-2 worked examples with realistic contexts — window panes, ice cube trays, sticker sheets — brief)
* Transition to dot arrays within Section 1 (primary working representation throughout)
* Mixed concrete + dots in Section 2 for bridge (side-by-side showing same structure)

### **⚠️ Scope Warnings**

⚠️ **DUAL INTERPRETATION ≠ COMMUTATIVE PROPERTY:** M11 teaches that a 3×4 array can be described as "3 rows of 4" (→ 3 × 4) OR "4 columns of 3" (→ 4 × 3). M11 does NOT prove or teach that 3 × 4 and 4 × 3 give the same product. Students may notice this — and that's fine — but the Guide does NOT make this the teaching point. M12 owns that proof. If a student says "they're both 12!" the Guide validates without teaching: "Interesting! We'll explore that idea next time."

⚠️ **EXPRESSION MODE PRIMARY, EQUATION MODE LATE:** M11's core cognitive demand is array → expression mapping. Equation Builder starts in expression mode (2-slot: `[___] × [___]`) to isolate this new skill. Full equation mode (3-slot: `[___] × [___] = [___]`) is available in late Section 3 and Practice, since students already own equations from M9-M10. But the teaching focus stays on "which expression matches this description?" not "what's the total?"

⚠️ **CONCRETE OBJECTS ARE BRIEF, NOT BELABORED:** Real-world array contexts (window panes, ice cube trays, sticker sheets, seating arrangements) appear for 1-2 worked examples only. These must be REALISTIC configurations — no 5×7 ice cube trays. Avoid food items that encourage unhealthy eating. The purpose is to ground the concept that arrays exist in the real world, then quickly transition to dot arrays for instructional flexibility. Don't spend extended time on concrete objects that constrain factor ranges.

## 1.4 MISCONCEPTIONS

### **1.4.1 Rows/Columns Confusion (#8) — PRIMARY**

**What It Looks Like:** Student sees a 3×4 array (3 horizontal rows of 4 items). When asked "How many rows?", student answers "4." When asked to write the expression for "3 rows of 4," student writes "4 × 3."

**Why It Happens:** "Row" and "column" are new vocabulary with no prior anchor in the multiplication sequence. Students may have encountered these words casually but haven't needed to distinguish them precisely. Additionally, arrays present BOTH dimensions simultaneously — unlike equal groups (where you see bags first, then items inside), arrays show rows and columns at the same time, creating a "which is which?" confusion.

**Where It Surfaces:** Section 1 (first array description tasks), Section 2 (when columns are introduced alongside rows), Exit Check.

Prevention Strategy:

* Physical gesture anchoring: Guide explicitly associates "rows go ACROSS" (horizontal gesture cue in language) and "columns go UP AND DOWN" (vertical gesture cue).
* Sequential highlighting: System highlights rows one at a time (horizontal bands), then columns one at a time (vertical bands) during initial teaching. Students SEE the difference before being asked to name it.
* "Rows first" convention provides a consistent starting point.
Remediation Strategy:

* If student reverses row/column count: System uses sequential row highlighting ("Watch — I'm going to highlight the ROWS. See how they go across?"), counts aloud, then sequential column highlighting ("Now the COLUMNS. See how they go up and down?").
* Recount with visual support before re-asking.
### **1.4.2 Array Dimensions as Addition (#9) — SECONDARY**

**What It Looks Like:** Student sees a 3×4 array and writes "3 + 4 = 7" instead of "3 × 4 = 12."

**Why It Happens:** Student sees two numbers (3 rows, 4 columns) and defaults to additive thinking. This is a regression to pre-M7 thinking — the student hasn't fully internalized that "groups OF items" is multiplication, not addition of the group count and group size.

**Where It Surfaces:** Section 1-2 (expression building tasks), Exit Check.

Prevention Strategy:

* Explicit connection to equal groups: "This array has 3 rows. Each row has 4 items. That's 3 groups of 4 — just like the bags and boxes you've been working with. 3 groups of 4 is 3 TIMES 4."
* The "groups of" → "times" bridge from M7-M8 is re-activated in array context.
Remediation Strategy:

* If student writes addition: Guide draws the equal groups connection explicitly. "You wrote 3 + 4. But look at this row — it has 4 items. And there are 3 rows JUST LIKE IT. That's 3 groups of 4. What operation do we use for groups of?" (Student should recall ×.) Then rebuild expression.
### **1.4.3 Groups vs. Size Confusion (#10 — Carried from M8)**

**What It Looks Like:** Student sees 3 rows of 4 and writes "4 × 3" when describing by rows (reversing factors).

**Why It Happens:** This is the same reversal from M8 but in array context. Student doesn't consistently map "how many groups" to the first factor. In arrays, this becomes more subtle because BOTH 3 × 4 AND 4 × 3 are valid for the same array (just from different descriptions). The confusion is whether the student is intentionally describing by columns (valid) or accidentally reversing the row description (error).

**Where It Surfaces:** Section 2-3 (when both descriptions are available), Practice.

Diagnostic Approach:

* When student writes 4 × 3 for a 3×4 array, the system must determine intent: "You wrote 4 × 3. Are you describing 4 COLUMNS of 3, or 3 ROWS of 4?" If student says "columns" → correct (column description). If student says "rows" → Misconception \#10 is active (factor reversal). This diagnostic is critical because the response differs entirely based on intent.
Prevention Strategy:

* Language-expression matching drilled throughout: "Say what you see, then write what you say." Description → expression, always in that order.
* Word-based Equation Builder makes the language explicit: "Making ____ rows of_____" forces the student to commit to a description before writing the numeric expression.

## 1.7 LESSON

---

### Purpose

Students learn that equal groups arranged in a rectangular grid form an **array** with **rows** (horizontal) and **columns** (vertical), that the SAME array can be described two different ways — by rows OR by columns — and that each description maps to a different multiplication expression where the language MUST match the notation. By the end of the Lesson, students can independently identify rows and columns, write multiplication expressions from both row and column descriptions, and compute products from array-based expressions.

---

### **Pedagogical Flow**

M11 bridges from CLUSTERED equal groups (M7-M10) to ORGANIZED equal groups (arrays). Students arrive from the Warmup having built an equation from a clustered equal groups image (M10 callback) and seen the same items rearranged into a rectangular grid. The Lesson teaches array structure, vocabulary, and expression mapping through three deliberate sections.

Unlike M10 (which introduced three layered demands: = sign meaning, context variety, unknowns), M11 introduces one spatial structure (arrays) but must teach three new vocabulary terms (array, row, column) AND establish that the same structure generates two valid expressions. The graduated section design prevents cognitive overload by introducing rows-only first (Section 1), then columns as a second interpretation (Section 2), then student specification + computation (Section 3).

CRA Sequence for M11:

| CRA Phase | M11 Application |
| --- | --- |
| **Concrete** | Section 1.1: Sticker sheet (3×4) — what an array IS. Guide names structure. Vocabulary (array) introduced AFTER student sees the rectangular arrangement. |
| **Relational** | Section 1.2-1.3: Row/column vocabulary with highlighting → identification in new arrays. Section 2.1-2.3: Same array, two descriptions, two expressions. |
| **Abstract** | Section 1.4: Row description → expression mapping formalized. Section 2.4: Student chooses description, writes expression, computes. |
| **Application** | Section 3: Student specifies arrays by description, writes equations, computes products. Full cycle: language → structure → expression → product. |

### 1.7.1 LESSON SECTION 1: Arrays Are Organized Equal Groups

**Purpose:** Establish what an array IS, introduce three vocabulary terms (array, row, column), teach row identification, and map row descriptions to multiplication expressions. Section 1 uses ROW descriptions only. Column interpretation is deferred to Section 2.

---

### Interaction 1.1: Transition from Warmup + Introduce Array Concept

* **Visual:** Arrays with Pictures (concrete mode): Sticker sheet showing 3 rows of 4 stickers in a clear rectangular arrangement.
* **Guide:** "In the Warmup, you saw equal groups lined up into a rectangle. See this sticker sheet? The stickers aren't bunched up in bags or scattered around — they're lined up. Every sticker has a spot. Let me show you what I notice."
* **Guide:** "See how the stickers are lined up? They go across, and they go up and down. Everything lines up into a rectangle."
* **Visual:** System briefly outlines the full rectangle shape around the sticker sheet.
* **Guide:** "When objects are arranged like this — lined up in a rectangle, with every spot filled — that's called an ARRAY."
* **Vocabulary:** **array** — formally introduced here.
* **No student action.**

* **Guide:** "See this line going across? That's a **row**. Rows go across from left to right."
* **Visual:** System highlights row 2, then row 3 sequentially.
* **Guide:** "Row 1... row 2... row 3. Three rows, and they all go across."
* **Visual:** All row highlights clear. System highlights column 1 (left vertical line of 3 stickers).

### 1.7.3 LESSON SECTION 3: Build and Compute Arrays

**Purpose:** Shift student role from OBSERVER/DESCRIBER to CONSTRUCTOR/COMPUTER. Students specify array structure, build it by adding rows or columns (not individual dots), write matching equations, and compute products. Construction operationalizes the row/column vocabulary through action — students don't just label structure, they create it. This parallels the Equal Groups Builder arc from M7-M8 (observe → describe → build).

**New demand:** Array construction (incrementally building arrays by adding rows or columns). Computation via equation mode was introduced at 2.4 and is extended here, not new.

---

### Required Phrases (Must Appear in Script)**

* "**array**" — formal introduction in 1.1
* "**row**" / "**rows**" — formal introduction in 1.2
* "**column**" / "**columns**" — formal introduction in 1.2
* "**rows of**" — pattern language starting in 1.4
* "**columns of**" — pattern language starting in 2.1
* "**Your description determines your expression**" — core principle stated in 2.1
* "**expression**" — used naturally throughout (carried from M8)
* "**equation**" — used with equation mode at 2.4+ (carried from M9-M10)
* "**Same array, two different expressions**" or equivalent — stated in 2.1/2.2
* "**times**" — when reading × symbol aloud
### **Forbidden Phrases (Create Misconceptions)**

* ❌ "**They're the same**" / "**They give the same answer**" / "**3 × 4 equals 4 × 3**" (M12)
* ❌ "**commutative**" / "**property**" (M12)
* ❌ "**The right way**" / "**The correct description**" when both are valid
* ❌ "**Rows are better**" / "**Start with rows**" as a rule
* ❌ "**Array means rows**"
* ❌ "**Unknown**" / "**missing number**" / "**☐**" (M10)
* ❌ "**Solve**" in the sense of finding an unknown
* ❌ "**Repeated addition**" as a method
### **Commutative Property Deflection Script**

If a student builds both expressions and notices the products are the same:

* **Guide:** "You noticed something interesting — those two expressions gave you the same answer. That's a great observation. We'll explore that idea more next time."
Do NOT elaborate, prove, or name the property. Acknowledge → validate → defer.

Problem EC.1: Build Row Expression

* **Visual:** Arrays with Pictures (dot mode): 2×7 dot array. Equation Builder in expression mode: `[___] × [___]`. Tile palette: 2, 3, 4, 5, 6, 7, 9, 14.
* **Guide:** "Look at this array. Describe it by rows and build the expression."

* **Correct Answer:** Expression must be `2 × 7`.
* **On Correct:** "2 rows of 7. 2 times 7. "
* **Remediation:** Pipeline
* **Answer Rationale:**
  * `2 × 7` = Correct (2 rows of 7)
  * `7 × 2` = Column expression — valid but doesn't match "row expression" prompt
  * Using 9 tile = Misconception #9 (2+7 additive)
  * Using 14 tile = Computing product instead of writing expression
  * Using 3, 4, 5 or 6 tiles = Miscount

Problem EC.2: Identify Column Expression

* **Visual:** Arrays with Pictures (dot mode): 5×4 dot array. System highlights rows briefly.
* **Guide:** "This array is described as 5 rows of 4. But there's another way to describe this array, by columns. Which expression matches the column description?"
* **Prompt:** "Which expression describes this array by columns?"
* **Student Action:** MC (A: 4 × 5, B: 5 × 4, C: 4 × 4, D: 5 × 5)
* **Correct Answer:** A (4 × 5)
* **On Correct:** "4 times 5. 4 columns of 5. Same array, different description, different expression."
* **On Incorrect (B):** "5 times 4 is the row expression. Columns go up and down — how many columns does this array have?"
* **On Incorrect (C or D):** "Count the columns going up and down. Then count how many dots are in one column. That gives you the expression."
* **Answer Rationale:**
  * A: 4 × 5 = Correct. 4 columns, 5 in each column.
  * B: 5 × 4 = Repeats row expression. Doesn't understand dual interpretation (The One Thing).
  * C: 4 × 4 = Column count correct but used for both factors.
  * D: 5 × 5 = Row count used for both factors.

* **On Correct (if 3 × 6 = 18):** "3 times 6 equals 18. 3 columns of 6."

## 1.8.5 Practice Phase Notes/Inputs

---

**Equation orientation:** Practice MUST include both standard (`[___] × [___] = [___]`) and reversed (`[___] = [___] × [___]`) equation orientations. Reversed orientation reinforces M10's "equals means same value as" concept and prevents regression to "equals means answer." Suggested distribution: ~70% standard, ~30% reversed.

**Description direction:** Practice problems should include both row-description and column-description prompts, plus open-choice problems where the student selects the description direction. Suggested distribution: ~40% row-prompted, ~30% column-prompted, ~30% student choice.

**Factor range:** Factors 2-9, products within 100 per 3.OA.A.3. Early items use Lesson-range values (factors 2-6) for warm-up; progression extends to full standard range. All students reach products within 100.

Task S.2: Same Array, Two Products

* **Purpose:** Surface the observation that both descriptions of the same array produce the same product. This is the key insight M12 will formalize. Students NOTICE it here; M12 PROVES it.
* **Visual:** One 3×5 dot array (center). Below it, two equations displayed side by side: Left: `3 × 5 = 15` (labeled "rows: 3 rows of 5"). Right: `5 × 3 = 15` (labeled "columns: 5 columns of 3").
* **Guide:** "Here's an array you've worked with. Described by rows, it's 3 times 5 equals 15. Described by columns, it's 5 times 3 equals 15. Look at both equations. What do you notice about the products?"
* **Prompt:** "What's true for these two equations?"
* **Student Action:** MC (A: "The products are both 15", B: "The expressions are the same", C: "They have different arrays", D: "The factors are different")
* **Correct Answer:** A ("The products are both 15")
* **On A:** "You noticed it’s the same array, two descriptions, same product. Interesting, right? We'll dig into that next time."
* **On B:** "The expressions are actually different: 3 times 5 and 5 times 3. But look at the answers. What happened?"
* **On C:** "It's actually the same array, just described two ways. Look at the products on the right side of each equation."
* **On D:** "The factors are in a different order, but they are the same numbers, 3 and 5.  What about the products? Look at both answers."
**Connection:** "Two different expressions. Same product. That's something worth remembering."

**Commutative boundary:** Guide does NOT say "they always give the same answer" or name the property. Guide says "same array, two descriptions, same product" (specific to THIS observation) and "we'll dig into that next time" (deferred to M12).

## **1.10 KEY DESIGN DECISIONS SUMMARY**

### KDD 1: Three Sections With Graduated Cognitive Load

Section 1 introduces arrays with row-only description (one interpretation, three vocabulary terms). Section 2 adds column description (dual interpretation). Section 3 adds student specification + computation. Each section adds ONE new demand. Section 1 is expanded to 5 interactions because it must introduce three vocabulary terms, establish the row/column distinction, AND teach expression mapping.

### KDD 2: Concrete Objects — Brief and Realistic

Real-world array contexts appear briefly in two places: early Section 1 (sticker sheet — introduces array concept) and one mixed concrete+dots comparison in Section 2 (ice cube tray alongside dot array — abstraction bridge per CRA progression). Both are brief (1-2 interactions each). Contexts must be REALISTIC configurations that kids genuinely encounter — window panes, ice cube trays, sticker sheets. Dot arrays become the primary representation within Section 1 and remain so throughout Sections 2-3, Exit Check, and Practice.

### KDD 3: Expression Mode Primary, Equation Mode as Extension

Equation Builder operates in expression mode (2-slot) for Sections 1-2 to isolate the NEW cognitive demand (array → expression mapping). Equation mode (3-slot) becomes available in late Section 3 and Practice as a natural extension.

### KDD 4: Word-Based Specification for Array Construction

When students build arrays in Section 3, they first commit to a description in words ("Making ___ rows of ___" or "Making ___ columns of ___") and then build it by adding rows and columns as separate dimensions. Dots appear at intersections — only where both a row and a column exist. The first dimension named in the description is built first (as empty placeholders); the second dimension fills in the dots. System enforces sequential build order.
This construction mechanic makes multiplicative structure visible through action: each press of the second-dimension button adds one dot to every placeholder in the first dimension. A student building "4 rows of 6" watches 4 dots appear with each column-add — the multiplication is literally happening on screen. Building column-first in 3.3 flips the experience, reinforcing that both descriptions produce valid arrays.
This parallels the Equal Groups Builder arc from M7-M8 (observe → describe → build) while introducing a key distinction: arrays are two-dimensional, so building requires two separate actions. The word template serves as a plan; the construction is where the learning happens.

### KDD 5: Computation Enters in Late Section 2, Anchors Section 3

Section 1 focuses purely on array recognition + row/column identification (no computation). Computation enters at 2.4 once expression mapping is established, then is present throughout Section 3 (3-4 computation interactions total).

### KDD 6: Highlighting Is Guide-Controlled

Row/column emphasis switching is a demonstration and remediation tool only. Students do NOT toggle emphasis themselves. They explore both interpretations through structured tasks.

### KDD 7: Dual Interpretation Without Equivalence Proof

M11 establishes one array → two descriptions → two expressions. It does NOT prove these expressions yield the same product. If students notice, the Guide acknowledges without teaching: "That's an interesting observation — we'll explore that idea next time."

### KDD 8: Factor Ranges Not Confined to 2.OA.C.4

The Building On standard limits to 5×5. M11 addresses 3.OA.A.1, which is broader. Early examples use smaller arrays for clarity but are not capped at 5×5.

### KDD 9: Array Connection to Equal Groups Is Explicit, Not Assumed

The Warmup bridges from M10's equal groups to M11's arrays with an explicit comparison: same items, different arrangement. Guide names the difference: equal groups are clustered; arrays are aligned.

### KDD 10: Warmup Vocabulary Withheld

Backbone specified Guide names "array," "rows," "columns" in bridge. Departure: Bridge does NOT name these terms. Per Warmup Phase Playbook §4B, formal vocabulary is prohibited in Warmup. All three terms relocated to Lesson (array at 1.1, row/column at 1.2). The bridge TEASES the structure (visual highlighting) and PROMISES the vocabulary ("It has a name") without delivering it.

### KDD 11: Synthesis S.2 Surfaces Product Invariance Without Proving It

S.2 deliberately shows both equations pre-computed (`3 × 5 = 15` and `5 × 3 = 15`) so students NOTICE the matching products. Guide acknowledges ("Interesting, right? We'll dig into that next time.") without claiming universality or naming commutativity. This seeds M12's central question: "Is this always true?"

### KDD 12: Exit Check EC.2 Typed as IDENTIFY

EC.2 provides the row expression pre-filled and asks students to identify the column expression from MC options. Guide names the row expression and frames the column question — more context than typical EC feedback. Rationale: Lesson taught dual interpretation by always showing one description first (rows → columns), so scaffolding the row expression feels aligned with instructional pattern.
