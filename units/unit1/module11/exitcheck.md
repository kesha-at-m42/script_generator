## 1.8 EXIT CHECK

---

### Purpose

Verify Lesson understanding before Practice. Tests whether student can build a multiplication expression from an array's row description, identify the column expression for the same array, and build a complete equation from an array — all skills explicitly taught in Lesson Sections 1-3.

---

### Parameters

| Element | Specification |
| --- | --- |
| Problems | 3 (standard) |
| Cognitive Types | CREATE, IDENTIFY, CREATE |
| Time | 3-4 minutes |
| Tone | Calm, low-stakes, neutral |
| Remediation | Light only (brief redirect, no re-teaching). Full L-M-H added via Remediation Pipeline. |

---

### Constraints

| MUST | MUST NOT |
| --- | --- |
| Use Arrays with Pictures (dots) + Equation Builder (same toys as Lesson) | Introduce new visual models or interaction types |
| Use dot arrays only (Lesson exits concrete at 1.2) | Use concrete objects (sticker sheets, ice cube trays) |
| Include at least one row expression task and one column/dual-interpretation task | Test only one description direction |
| Include at least one equation (not just expression) | Test only expressions |
| Use factors within Lesson range (2-7, products within 50) | Use values outside Lesson range or products > 50 |
| Use values NOT identical to Lesson interactions at same cognitive demand | Repeat exact Lesson value combinations at same task type |
| Match Lesson difficulty | Increase complexity beyond Lesson |

**Value Selection Rationale:** Lesson used 3×4, 2×6, 3×5, 4×2, 2×5, 4×3, 4×5, 2×6, 4×6,2×5 and product=20. Exit Check uses 2×7, 5×4, and 6×3 — all within range, none identical to Lesson at same cognitive demand level.

---

### Alignment Check

| Problem | Cognitive Type | Tests | Lesson Source | Array |

### Transition into Exit Check**

* **Guide:** "You've learned to describe arrays by rows or columns, write expressions for each description, and find products. Let's see what you know."
---

Problem EC.1: Build Row Expression

* **Visual:** Arrays with Pictures (dot mode): 2×7 dot array. Equation Builder in expression mode: `[___] × [___]`. Tile palette: 2, 3, 4, 5, 6, 7, 9, 14.
* **Guide:** "Look at this array. Describe it by rows and build the expression."
* **Prompt:** "Build the expression based on rows for this array."
* **Student Action:** Equation Builder Methods C/D (tiles).
* **Correct Answer:** Expression must be `2 × 7`.
* **On Correct:** "2 rows of 7. 2 times 7. "
* **Remediation:** Pipeline
* **Answer Rationale:**
  * `2 × 7` = Correct (2 rows of 7)
  * `7 × 2` = Column expression — valid but doesn't match "row expression" prompt
  * Using 9 tile = Misconception #9 (2+7 additive)
  * Using 14 tile = Computing product instead of writing expression
  * Using 3, 4, 5 or 6 tiles = Miscount
---

Problem EC.2: Identify Column Expression

* **Visual:** Arrays with Pictures (dot mode): 5×4 dot array. System highlights rows briefly.
* **Guide:** "This array is described as 5 rows of 4. But there's another way to describe this array, by columns. Which expression matches the column description?"
* **Prompt:** "Which expression describes this array by columns?"
* **Student Action:** MC (A: 4 × 5, B: 5 × 4, C: 4 × 4, D: 5 × 5)
* **Correct Answer:** A (4 × 5)
* **On Correct:** "4 times 5. 4 columns of 5. Same array, different description, different expression."
* **On Incorrect (B):** "5 times 4 is the row expression. Columns go up and down — how many columns does this array have?"
* **On Incorrect (C or D):** "Count the columns going up and down. Then count how many dots are in one column. That gives you the expression."
* **Answer Rationale:**
  * A: 4 × 5 = Correct. 4 columns, 5 in each column.
  * B: 5 × 4 = Repeats row expression. Doesn't understand dual interpretation (The One Thing).
  * C: 4 × 4 = Column count correct but used for both factors.
  * D: 5 × 5 = Row count used for both factors.
---

Problem EC.3: Build Equation from Array

* **Visual:** Arrays with Pictures (dot mode): 6×3 dot array. Equation Builder in equation mode: `[___] × [___] = [___]`. Tile palette: 3, 4, 6, 7, 9, 15, 16, 18, 21.
* **Guide:** "Describe this array by rows or columns — your choice. Build the equation."
* **Prompt:** "Build the equation for this array."
* **Student Action:** Equation Builder Methods C/D (equation mode — 3 slots).
* **Correct Answer:** `6 × 3 = 18` (rows) OR `3 × 6 = 18` (columns). Both valid.
* **On Correct (if 6 × 3 = 18):** "6 times 3 equals 18. 6 rows of 3."
* **On Correct (if 3 × 6 = 18):** "3 times 6 equals 18. 3 columns of 6."
* **Remediation:** Pipeline
* **Answer Rationale:**
  * `6 × 3 = 18` = Correct (row description)
  * `3 × 6 = 18` = Correct (column description)
  * Using 9 tile in expression = Misconception #9 (6+3 additive)
  * Expression correct but product wrong = Computation error (M9 fluency gap)
---

## 1.8.5 Practice Phase Notes/Inputs

---

**Equation orientation:** Practice MUST include both standard (`[___] × [___] = [___]`) and reversed (`[___] = [___] × [___]`) equation orientations. Reversed orientation reinforces M10's "equals means same value as" concept and prevents regression to "equals means answer." Suggested distribution: ~70% standard, ~30% reversed.

**Description direction:** Practice problems should include both row-description and column-description prompts, plus open-choice problems where the student selects the description direction. Suggested distribution: ~40% row-prompted, ~30% column-prompted, ~30% student choice.

**Factor range:** Factors 2-9, products within 100 per 3.OA.A.3. Early items use Lesson-range values (factors 2-6) for warm-up; progression extends to full standard range. All students reach products within 100.
