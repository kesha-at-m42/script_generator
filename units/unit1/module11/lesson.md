## 1.7 LESSON

---

### Purpose

Students learn that equal groups arranged in a rectangular grid form an **array** with **rows** (horizontal) and **columns** (vertical), that the SAME array can be described two different ways — by rows OR by columns — and that each description maps to a different multiplication expression where the language MUST match the notation. By the end of the Lesson, students can independently identify rows and columns, write multiplication expressions from both row and column descriptions, and compute products from array-based expressions.

---

### **Pedagogical Flow**

M11 bridges from CLUSTERED equal groups (M7-M10) to ORGANIZED equal groups (arrays). Students arrive from the Warmup having built an equation from a clustered equal groups image (M10 callback) and seen the same items rearranged into a rectangular grid. The Lesson teaches array structure, vocabulary, and expression mapping through three deliberate sections.

Unlike M10 (which introduced three layered demands: = sign meaning, context variety, unknowns), M11 introduces one spatial structure (arrays) but must teach three new vocabulary terms (array, row, column) AND establish that the same structure generates two valid expressions. The graduated section design prevents cognitive overload by introducing rows-only first (Section 1), then columns as a second interpretation (Section 2), then student specification + computation (Section 3).

CRA Sequence for M11:

| CRA Phase | M11 Application |
| --- | --- |
| **Concrete** | Section 1.1: Sticker sheet (3×4) — what an array IS. Guide names structure. Vocabulary (array) introduced AFTER student sees the rectangular arrangement. |
| **Relational** | Section 1.2-1.3: Row/column vocabulary with highlighting → identification in new arrays. Section 2.1-2.3: Same array, two descriptions, two expressions. |
| **Abstract** | Section 1.4: Row description → expression mapping formalized. Section 2.4: Student chooses description, writes expression, computes. |
| **Application** | Section 3: Student specifies arrays by description, writes equations, computes products. Full cycle: language → structure → expression → product. |

### 1.7.1 LESSON SECTION 1: Arrays Are Organized Equal Groups

**Purpose:** Establish what an array IS, introduce three vocabulary terms (array, row, column), teach row identification, and map row descriptions to multiplication expressions. Section 1 uses ROW descriptions only. Column interpretation is deferred to Section 2.

---

### Interaction 1.1: Transition from Warmup + Introduce Array Concept

* **Visual:** Arrays with Pictures (concrete mode): Sticker sheet showing 3 rows of 4 stickers in a clear rectangular arrangement.
* **Guide:** "In the Warmup, you saw equal groups lined up into a rectangle. See this sticker sheet? The stickers aren't bunched up in bags or scattered around — they're lined up. Every sticker has a spot. Let me show you what I notice."
* **Guide:** "See how the stickers are lined up? They go across, and they go up and down. Everything lines up into a rectangle."
* **Visual:** System briefly outlines the full rectangle shape around the sticker sheet.
* **Guide:** "When objects are arranged like this — lined up in a rectangle, with every spot filled — that's called an ARRAY."
* **Vocabulary:** **array** — formally introduced here.
* **No student action.**
---

### Interaction 1.2: Introduce Row/Column Vocabulary with Highlighting

* **Visual:** Arrays with Pictures: Same sticker sheet from 1.1 (3×4). System highlights rows and columns sequentially during Guide narration.
* **Guide:** "Now let me teach you the names for what you're seeing. Watch this."
* **Visual:** System highlights row 1 (top horizontal line of 4 stickers).
* **Guide:** "See this line going across? That's a **row**. Rows go across from left to right."
* **Visual:** System highlights row 2, then row 3 sequentially.
* **Guide:** "Row 1... row 2... row 3. Three rows, and they all go across."
* **Visual:** All row highlights clear. System highlights column 1 (left vertical line of 3 stickers).
* **Guide:** "Now watch this direction. See this line going up and down? That's a **column**. Columns go up and down."
* **Visual:** System highlights column 2, column 3, column 4 sequentially.
* **Guide:** "Column 1... column 2... column 3... column 4. Four columns, all going up and down."
* **Visual:** All highlights clear. Array returns to neutral display.
* **Guide:** "So think about the difference between rows and columns. How many rows that go across does this array have?"
* **Prompt:** "How many rows? (Rows go across.)"
* **Student Action:** MC (2, 3, 4, 7)
* **Correct Answer:** 3
* **On Correct:** "3 rows — the lines that go across. So this array has 3 rows and 4 columns. An array is equal groups organized into rows and columns."
* **Vocabulary:** **row** ("rows go across — left to right"), **column** ("columns go up and down") — formally introduced here.

### Interaction 1.3: Identify Rows and Columns in New Dot Array

* **Visual:** Arrays with Pictures (dot mode): 2×6 dot array. Strong asymmetry between row count (2) and column count (6) makes the distinction visually obvious.
* **Guide:** "Here's a new array, this time with dots. How many rows does this array have? Remember, rows go across."
* **Prompt:** "How many rows? (Rows go across.)"
* **Student Action:** MC (2, 6, 8, 12)
* **Correct Answer:** 2
* **On Correct:** "2 rows. The lines going across."
[Immediately followed by:]

* **Guide:** "Now how many columns? Columns go up and down."
* **Prompt:** "How many columns? (Columns go up and down.)"
* **Student Action:** MC (2, 6, 8, 12)
* **Correct Answer:** 6
* **On Correct:** "6 columns. The lines going up and down. So this array has 2 rows and 6 columns."

### Interaction 1.4: Row Description → Expression

* **Visual:** Arrays with Pictures (dot mode): 3×5 dot array. Equation Builder in expression mode: `[___] × [___]`. Tile palette available.
* **Guide:** "This array has 3 rows, and each row has 5 dots. So we can say: 3 rows of 5."
* **Visual:** System highlights rows sequentially — row 1 (5 dots), row 2 (5 dots), row 3 (5 dots).
* **Guide:** "Here's how that becomes a multiplication expression. Watch."
* **Guide:** "'3 rows of 5.' The first number is how many rows: that's 3."
* **Visual:** 3 tile places into first slot of Equation Builder.
* **Guide:** "The second number is how many in each row: that's 5."
* **Visual:** 5 tile places into second slot. Expression shows `3 × 5`
* **Guide:** "3 rows of 5. That's just like saying 3 groups of 5 — and you know equal groups means multiplication. 3 rows of 5 becomes 3 times 5."
* **Guide:** "Now here's a new array."
* **Visual:** Equation Builder clears. New dot array appears: 4×2. Tile palette: 2, 3, 4, 5, 6, 7, 8, 9.
* **Guide:** "Build the expression — the first number is how many rows, the second is how many in each row."
* **Prompt:** "Build the expression for this array."
* **Student Action:** Equation Builder Methods C/D (tiles).
* **Correct Answer:** Expression must be `4 × 2`.
* **On Correct:** "4 times 2. 4 rows of 2. The words match the expression."
---

### Interaction 1.5: Independent Row Expression

* **Visual:** Arrays with Pictures (dot mode): 2×5 dot array. Equation Builder in expression mode. Tile palette: 2, 3, 4, 5, 6, 7, 8, 10.
* **Guide:** "One more. Look at this array and build the expression for the rows."
* **Prompt:** "Build the row expression for this array."
* **Student Action:** Equation Builder Methods C/D.
* **Correct Answer:** Expression must be `2 × 5`.
* **On Correct:** "2 times 5. 2 rows of 5.. You've got it."

* **No student action.**

### 1.7.2 LESSON SECTION 2: Two Ways to See an Array

**Purpose:** Introduce column description as a SECOND valid way to describe the same array, establish the language-expression precision principle (your description determines your expression), and introduce computation as a natural extension. Section 2 adds ONE new demand (column interpretation) to Section 1's foundation.

---

### Interaction 2.1: Introduce Column Description

* **Visual:** Arrays with Pictures (dot mode): 2×5 dot array (returns ). Equation Builder in expression mode.
* **Guide:** "You just described this array by ROWS: 2 rows of 5. That gave us 2 times 5."
* **Visual:** System highlights rows briefly. Expression `2 × 5` appears in Equation Builder, then fades.
* **Guide:** "But watch what happens when I look at the COLUMNS instead."
* **Visual:** System highlights column 1 (2 dots going up and down), then column 2, column 3, column 4, column 5 sequentially.
* **Guide:** "I see 5 columns. Each column has 2 dots. So I can also say: 5 columns of 2."
* **Guide:** "5 columns of 2 becomes 5 times 2. The first number is how many columns. The second number is how many in each column."
* **Visual:** Expression `5 × 2` appears in Equation Builder.
* **Guide:** "Same array, but a different description which gives us  a different expression. 2 rows of 5 gave us 2 times 5. 5 columns of 2 gives us 5 times 2. Both describe THIS array."
* **Visual:** Both expressions display briefly: `2 × 5` and `5 × 2` with the same dot array between them.
* **Guide:** "The rule is: your DESCRIPTION determines your EXPRESSION. What you SAY about the array is what you WRITE. Let's try one."
* **Visual:** Equation Builder clears. New 4×3 dot array appears. Word description active: ___ columns of ___. Tile palette: 2, 3, 4, 5, 6, 7, 8, 9.
* **Guide:** "This array has 4 rows of 3. But what's the column description? Count the columns going up and down, then count how many in each column."
* **Prompt Step 1:** "Describe this array by columns."
* **Student Action (Step 1):** Word description template: "___ columns of ___". Student fills two number slots.
* **Correct State:** "3 columns of 4."
* **Guide:** "3 columns of 4. Now build the expression to match."
* **Visual:** Same array. Same tile palette: 2, 3, 4, 5, 6, 7, 8, 9. “3 columns of 4” remains. Equation Builder slots appear ____ `×` ___ 
* **Prompt Step 2:** "Build the expression for 3 columns of 4."
* **Student Action (Step 2):** Equation Builder Methods C/D (expression mode).
* **Correct Answer:** `3 × 4`
* **On Correct:** "3 columns of 4. 3 times 4. The description matches the expression."

### Interaction 2.4: Choose Description, Write Expression, Compute

* **Visual:** Arrays with Pictures (dot mode): 3×4 dot array (callback to 1.1). Equation Builder switches to equation mode: `[___] × [___] = [___]`. Tile palette: 2, 3, 4, 5, 6, 7, 8, 10, 12, 14.

* **Student Action:** Student selects "rows" or "columns"
* **Prompt:** "Now describe this array by words and build the equation."
* **Student Action:** 2 Equation Builders, word mode [”__ rows of __ items” or “__ columns of __ items”] and equation mode [”__ x __ = __].
* **Correct Answer (if rows):** 3 rows of 4.  `3 × 4 = 12`.
* **Correct Answer (if columns):** 4 columns of 3.`4 × 3 = 12`.
* **On Correct (if columns):** " You described it by columns. 4 columns of 3. 4 times 3 equals 12."
* **On Correct (if rows):** "You described it by rows. 3 rows of 4. 3 times 4 equals 12."

### 1.7.3 LESSON SECTION 3: Build and Compute Arrays

**Purpose:** Shift student role from OBSERVER/DESCRIBER to CONSTRUCTOR/COMPUTER. Students specify array structure, build it by adding rows or columns (not individual dots), write matching equations, and compute products. Construction operationalizes the row/column vocabulary through action — students don't just label structure, they create it. This parallels the Equal Groups Builder arc from M7-M8 (observe → describe → build).

**New demand:** Array construction (incrementally building arrays by adding rows or columns). Computation via equation mode was introduced at 2.4 and is extended here, not new.

---

### Interaction 3.1: Worked Example — Build Array by Rows-First + Equation

* **Visual:** Arrays with Pictures build area empty.  Two buttons visible: "Row" with "-/+" buttons (active, guide-controlled) and "Items per Row" with "-/+" buttons (dimmed/disabled).
* **Guide:** "Now YOU get to build arrays. Watch how this works."
* **Guide:** "I want to make 3 rows of 5. I said ROWS first, so I build the rows first."
* **Prompt:** S"Make 3 rows of 5."
* **Guide:** "I add my first row."
* **Visual:** Guide activates "Row" and "+" button highlighted. An empty horizontal placeholder appears — a faint line or shelf indicating where a row will be. No dots yet.
* **Guide:** "Second row."
* **Visual:** Second empty row placeholder appears below the first.
* **Guide:** "Third row. I've set up 3 rows but there are no dots yet. I need to add columns to fill them in."
* **Visual:** Third row placeholder appears. "Row +/-" button dims/disables. "Items per Row +/-" button activates.
* **Guide:** "Now I add items to each row. Watch what happens. Each "+" puts one dot in every row."
* **Visual:** Guide activates "Items per Row +/-." First column appears: one dot in each of the 3 rows (3 dots total, forming a vertical line at the left).
* **Guide:** "One dot in each row. That's 3 dots."
* **Visual:** Guide adds column 2. Three more dots appear, one per row.
* **Guide:** "Two dots in each row, 6 dots. Keep going."
* **Visual:** Guide adds columns 3, 4, 5 in sequence. With each press, 3 more dots fill in across all rows.
* **Guide:** "3, 4, 5 in each row. 3 rows of 5. 15 dots. I built it."
* **Visual:** Complete 3×5 array visible. Items per Row +/-" button dims.
* **Guide:** "Now I write the equation."
* **Visual:** Equation Builder switches to equation mode: `[___] × [___] = [___]`. Tile palette: 3, 4, 5, 8, 12, 15, 18, 20. Guide demonstration: tiles `3`, `5`, `15` place into slots.
* **Prompt:** "Write the equation that matches the array based on rows."
* **Guide:** "3 rows of 5. 3 times 5 equals 15. I planned it, built it, and found the answer. Your turn."
* **No student action.**

### Interaction 3.2: Build Array by Rows-First + Write Equation

* **Visual:** "Row +/-" button active. "Items per Row +/-" button dimmed. Arrays with Pictures build area empty.
* **Guide:** "Build an array with 4 rows of 6. Start with the rows."
* **Prompt:** "Build 4 rows of 6. Add the rows first."
**Step 1 — Add Rows:**

* **Student Action:** Student presses "Row +" 4 times. Each press creates an empty horizontal placeholder.
* **Correct State:** 4 empty row placeholders visible. No dots yet.
* **Visual:** After 4th press, "Row +/-" dims. "Items per Row +/-" activates.
* **Guide:** "4 rows. Now add the items into every row."
**Step 2 — Add Columns (Items per Row):**

* **Prompt:** "Add 6 dots into every row"

* **Correct State:** After 6 presses, complete 4×6 dot array visible (24 dots). "Items per Row +/-" dims.
* **On Complete:** "4 rows of 6. You built it."
**⚠️ UX Implementation Note:** Construction is sequential — "Items per Row +/-" activates only after the specified number of rows is placed (and vice versa for column-first builds in 3.3). System prevents exceeding specified counts for either dimension. See revised KDD #4 and UX Flag #1.

**Step 3 — Equation:**

* **Visual:** Equation Builder equation mode: `[___] × [___] = [___]`. Tile palette: 4, 5, 6, 10, 12, 20, 24, 28.
* **Guide:** "Now write the equation for 4 rows of 6."
* **Prompt:** "Build the equation for 4 rows of 6."
* **Student Action:** Equation Builder Methods C/D (equation mode).
* **Correct Answer:** `4 × 6 = 24`
* **On Correct:** "4 times 6 equals 24. You built the array and found the product."

### Interaction 3.4: Open Specification + Build — Given Total

* **Visual:** Two multiple choice options saying "Row" and "Column".  
* **Guide:** "Last challenge. I want an array with 20 dots total. You choose: build it with rows first or columns first, pick your numbers, and find the equation. There's more than one right answer."
* **Prompt:** "Choose rows or columns to build an array with 20 dots."
**Step 1 — Specification:**

* **Student Action:** Student selects "rows" or "columns".
* **Visual:** On valid specification, the buttons activate (matches whichever the student named, "Rows & Items per Row" or "Columns & Items per Column").
**Step 2 — Construction:**

* **Student Action:** Student builds first dimension (placeholders) and second dimension (dots fill in at intersections).
* **Correct State:** Completed array has exactly 20 dots (2 × 10, 10 × 2, 4 × 5, 5 × 4).
**Step 3 - Words:**

* **Visual:** Equation Builder words mode. The array from previous step still on the screen.
* **Guide:** "You built it. Now write the word description for your array”.
* **Prompt:** "Describe your array using words."
* **Correct State:** Any valid specification that matches the row or column array which = 20 (e.g., "4 rows of 5," "5 columns of 4," "2 rows of 10," “10 rows of 2”.).
**Step 4 — Equation:**

* **Visual:** Equation Builder equation mode. The array and the word equation from the previous step still on the screen.
* **Guide:** "Now write the equation. What multiplication does your array show?"
* **Prompt:** "Build the equation for your array."
* **Student Action:** Equation Builder Methods C/D.
* **Correct Answer:** Equation must match the word mode description AND product must equal 20.
* **On Correct (example: 4 × 5 = 20):** "4 times 5 equals 20. You chose 4 rows of 5. That's one way to build 20."
* **On Correct (example: 2 × 10 = 20):** "10 times 2 equals 20.  10 columns of 2. That’s an array with 20 dots."

### Required Phrases (Must Appear in Script)**

* "**array**" — formal introduction in 1.1
* "**row**" / "**rows**" — formal introduction in 1.2
* "**column**" / "**columns**" — formal introduction in 1.2
* "**rows of**" — pattern language starting in 1.4
* "**columns of**" — pattern language starting in 2.1
* "**Your description determines your expression**" — core principle stated in 2.1
* "**expression**" — used naturally throughout (carried from M8)
* "**equation**" — used with equation mode at 2.4+ (carried from M9-M10)
* "**Same array, two different expressions**" or equivalent — stated in 2.1/2.2
* "**times**" — when reading × symbol aloud
### **Forbidden Phrases (Create Misconceptions)**

* ❌ "**They're the same**" / "**They give the same answer**" / "**3 × 4 equals 4 × 3**" (M12)
* ❌ "**commutative**" / "**property**" (M12)
* ❌ "**The right way**" / "**The correct description**" when both are valid
* ❌ "**Rows are better**" / "**Start with rows**" as a rule
* ❌ "**Array means rows**"
* ❌ "**Unknown**" / "**missing number**" / "**☐**" (M10)
* ❌ "**Solve**" in the sense of finding an unknown
* ❌ "**Repeated addition**" as a method
### **Commutative Property Deflection Script**

If a student builds both expressions and notices the products are the same:

* **Guide:** "You noticed something interesting — those two expressions gave you the same answer. That's a great observation. We'll explore that idea more next time."
Do NOT elaborate, prove, or name the property. Acknowledge → validate → defer.

### 1.7.5 Success Criteria

**The One Thing:** Students understand that an array is a rectangular arrangement with rows and columns, that the SAME array can be described two ways (by rows or by columns), and that each description maps to a different multiplication expression where the language MUST match the notation.

**Ready for Module 12:** Students can identify rows and columns in any dot array, write a multiplication expression from a row description AND from a column description, recognize that both descriptions are valid for the same array, compute products from array-based expressions, and specify arrays using word-based description. M12 builds on this by asking: "Those two expressions for the same array — do they always give the same answer?"
