## **1.6 WARMUP (3-4 minutes)**

**Detail Level:** Structure + parameters (AI writes dialogue)

---

### **Purpose**

Activate prior knowledge of all four scales (1, 2, 5, 10) from M1-M4 while introducing the key M5 question: "Which scale should you CHOOSE?" Students collect personal data, then use the Scale Preview System to see that multiple scales can work for the same data. The discovery that "you have options" sets up Lesson's teaching of selection criteria.

---

### **Parameters**

| Element | Specification |
| :---- | :---- |
| Interactions | 3 interactions + Bridge |
| Active Participation | 3 of 4 require student response |
| Format | Data Collection Game → Scale Preview Exploration → Efficiency Comparison → Bridge |
| Cognitive Load | 20-30% (exploration and comparison are accessible) |
| Remediation | Light only (or none—low stakes) |
| Tone | Medium-High energy, curious, inviting |
| Engagement Anchors | Narrative Setup (Minis), Personalization (their data), Choice/Agency (which scale?) |

---

### **Constraints**

| USE | AVOID |
| :---- | :---- |
| Informal language ("Check this out," "Let's see") | Formal phrasing ("Observe the following") |
| All four scale names (familiar from M1-M4) | Teaching selection criteria (save for Lesson) |
| Observable acknowledgments ("You picked Scale of 10") | Assumed states ("You noticed the efficiency") |
| 3-4 categories | 2 or 5+ categories |
| **Multiples of 5 only** (10, 15, 20, 25, 30, 35, 40, 45) | Non-multiples (save for Lesson Section 2) |
| Values where both Scale 5 and Scale 10 work | Values that force only one scale option |
| "works" / "fits" / "easier to read" (informal) | "efficient" / "appropriate" / "precision" (formal—save for Lesson) |

---

### **Warmup Type Rationale**

**Standard Playbook Types:** Binary Choice, WODB, Number Talk, Notice & Wonder, Estimation 180, Visual Patterns, Mystery Reveal

**M5 Approach:** Data Collection Game + Scale Preview Exploration

**Why this serves the concept:**

* Unit 1 is about DATA—personal data creates investment
* M5's core question is CHOICE: which scale should you use?
* Scale Preview System is the new tool—Warmup introduces it informally with low stakes
* Students experience "multiple options work" before Lesson teaches decision criteria
* Follows Playbook PRINCIPLES (low cognitive load, engagement anchors, judgment task, bridge) while adapting STRUCTURE for strategic thinking

**Test:** If we removed this Warmup, would students lose mathematical preparation for Lesson? YES—they would miss the framing that scale selection involves choices, making Lesson feel like arbitrary rules rather than useful decision-making.

---

**Interaction W.1: Data Collection Game**

* **Purpose:** Generate personalized data. Create investment through Minis context. Provide concrete numbers for scale comparison.
* **Hook (first 15-20 seconds):**
  * **Visual:** Minis characters appear in animated counting scenario
  * **Guide:** "The Minis are organizing their collections! Let's help them count."
* **Engagement Anchor:** Narrative Setup (Minis), Personalization (their counts)
* **Game Specifications:**

| Element | Specification |
| :---- | :---- |
| Style | Counting/sorting game |
| Categories | 4 (e.g., Red blocks, Blue blocks, Green blocks, Yellow blocks) |
| Values | **Multiples of 5 only:** 10, 15, 20, 25, 30, 35, 40, 45 |
| Duration | ~45-60 seconds |
| Student Interaction | Count items per category; confirm counts |

* **On Complete:** Counts recorded. Guide: "You counted [results]. Now let's make a graph with your data."

---

**Interaction W.2: Scale Preview Exploration**

* **Purpose:** Introduce Scale Preview System informally. Students discover that multiple scales can work for the same data.
* **Visual:**
  * Data Table showing student's collected data
  * Scale Preview System with four clickable buttons: "Scale of 1" / "Scale of 2" / "Scale of 5" / "Scale of 10"
  * Vertical bar graph displays as the scale button is clicked
* **Guide:** "You know how to use different scales. But which one should you use for YOUR data?"
* **Guide:** "Let's find out. Click each scale to see what the graph would look like."
* **Student Action:** Explore all four scales freely
  * Scale of 1: ✓ All fit (but lots of tick marks!)
  * Scale of 2: ✓ All fit
  * Scale of 5: ✓ All fit
  * Scale of 10: ✓ All fit
* **Guide (after exploration):** "All four scales work for this data! You have choices."
* **Engagement Anchor:** Choice/Agency (student controls exploration)

---

**Interaction W.3: Efficiency Comparison**

* **Purpose:** Surface the idea that some scales are easier to read than others—without formally teaching "efficiency."
* **Visual:** Side-by-side display showing student's data at Scale of 5 AND Scale of 10
* **Guide:** "Both of these work. Take a look at them side by side. Which graph is easier to read?"
* **Prompt:** "Which graph is easier to read?"
* **Student Action:** [Multiple choice: "Scale of 5" / "Scale of 10" / "They're the same"]
* **On Any Selection:** Guide validates without declaring a "right" answer:
  * (Scale of 10): "You picked Scale of 10. Nice and clean!"
  * (Scale of 5): "You picked Scale of 5. More detail!"
  * (Same): "You think they're both easy to read. Both do work!"
* **Guide:** "When more than one scale works, you get to choose."
* **Engagement Anchor:** Choice/Agency (genuine preference question)

---

**Interaction W.4: Bridge to Lesson**

* **Purpose:** Create anticipation for Lesson by posing the problem Lesson will solve.
* **Visual:** Scale Preview System remains visible
* **Guide:** "You just saw that all four scales worked for your data. Nice to have options!"
* **Guide:** "But what if the numbers are bigger? Or what if you need to figure out which scale is BEST?"
* **Bridge Statement:** "Let's learn how to figure out which scale to choose—and what tells us that a scale won't work for the data we have."
* **No student prompt** (transition)

---

### **Verification Checklist**

**Structure:**

- [ ] Hook appears in first 15-20 seconds (Minis appear, counting context)
- [ ] 2+ engagement anchors (Narrative, Personalization, Choice/Agency)
- [ ] 3 active student interactions (Game, Preview Exploration, Efficiency Selection)
- [ ] Scale Preview System introduced informally (no formal tool demo)
- [ ] Clear bridge to Lesson creates anticipation ("what if scales DON'T work?")
- [ ] Total time 3-4 minutes

**Content Alignment:**

- [ ] 4 categories
- [ ] All values multiples of 5 (10-45 range)
- [ ] NO non-multiples (save for Lesson Section 2)
- [ ] All four scales shown working (✓ on each)
- [ ] Side-by-side comparison for efficiency question
- [ ] No "correct" answer for efficiency—validates all choices
- [ ] Selection criteria NOT taught (just experienced)

**Constraints:**

- [ ] No formal vocabulary ("efficient," "appropriate," "precision")
- [ ] No teaching of decision rules (save for Lesson)
- [ ] No "wrong" scale demonstrated (save for Lesson 1.2)
- [ ] Light cognitive load (20-30%)
- [ ] Warm, inviting tone throughout
