## **1.9 SYNTHESIS (5-6 minutes)**

**Detail Level:** Structure \+ parameters (AI writes dialogue)

### **Purpose**

Consolidate understanding that scaled graphs (1:2) use fewer symbols than 1:1 graphs for the same data. Connect scale concept to efficiency. Confirm half-symbol understanding. Reflect on what "scale" means. This is NOT more practice—it's connection-making and consolidation.

### **Parameters**

| Element | Specification |
| :---- | :---- |
| Interactions | 4 (3 connection tasks \+ 1 metacognitive reflection) |
| Task Types | Type D (Representation Transfer), Type A (Pattern Discovery), Metacognitive Reflection |
| Remediation | Light only—mastery assumed |
| Tone | Reflective, warm, discovering together |
| Energy | Medium (not exhausted, not high-energy) |

### **Constraints**

| DO | DO NOT |
| :---- | :---- |
| Connection tasks (pattern, transfer, comparison) | More practice problems |
| Compare 1:1 and 1:2 representations | Introduce new scales (5, 10\) |
| Confirm half-symbol understanding | Teach new half-symbol content |
| Light remediation if needed | Extended teaching or heavy remediation |
| Specific identity closure | Generic praise ("Great job\!") |
| Visual support for every task | Abstract discussion without visuals |
| Use scaled picture graphs from Lesson | Introduce bar graphs or new formats |

### **Opening Frame**

- **Purpose:** Signal shift from Exit Check to reflection/connection.
- **Visual:** Clean transition screen.
- **Guide:** "You made scaled picture graphs. You used half-symbols for odd numbers. Now let's see WHY scaled graphs are useful."

---

**Task S.1: Representation Transfer (Type D) — Scale Toggle Exploration**

- **Purpose:** Surface understanding that scaled graphs use fewer symbols for the same data. Connect scale to efficiency. Student toggles between scales themselves — agency in switching views reinforces metarepresentational competence (English 2010/2012) more powerfully than static side-by-side observation.
- **Visual:** Numeric Data Table visible (Cats=8, Dogs=6, Birds=10, Fish=4). Picture Graph displayed alongside, defaulting to scale of 2 (1:2). Scale toggle control visible with options: "Each ⭐ \= 1" and "Each ⭐ \= 2." Key updates dynamically when toggled.
- **Guide:** "Here's data you know — and a graph with scale of 2\. But watch what happens when you change the scale."
- **Prompt:** "Toggle the scale to 1\. What happens to the graph?"
- **Student Action:** Click scale toggle from 2 to 1\. Graph animates: symbols double in count (4 symbols → 8 for Cats, 3 → 6 for Dogs, 5 → 10 for Birds, 2 → 4 for Fish). Key updates to "Each ⭐ \= 1."
- **Guide:** "Look at that\! Cats went from 4 symbols to 8\. Same 8 cats — but now every cat gets its own symbol."
- **Prompt:** "Toggle back to scale of 2\. Which scale uses fewer symbols?"
- **Student Action:** Click scale toggle from 1 back to 2\. Graph animates: symbols halve. Then select answer: \[Multiple choice: "Scale of 1" / "Scale of 2"\]
- **Correct Answer:** Scale of 2
- **On Correct:** "Scale of 2 — fewer symbols, same data. That's what makes scaled graphs efficient."
- **On Incorrect:** "Toggle back and forth one more time. Count the symbols each way. Scale of 2 always uses fewer."
- **Connection (Guide states):** "You just saw the same data change shape. The INFORMATION stays the same — only the number of symbols changes. That's the power of scale."

**Design Notes:**

- Student toggles TWICE (2→1, then 1→2). The back-and-forth is the point — it makes the scale-representation relationship tangible and repeatable, not a one-time observation.
- Animation between states should be smooth enough to show symbols appearing/disappearing, reinforcing that the same data produces different visual density. A brief transition (\~0.5s) is ideal — fast enough to feel responsive, slow enough to see the change.
- The toggle remains active only during S.1. It is NOT available during S.2–S.4.
- This replaces the static three-way display (data table \+ 1:1 graph \+ 1:2 graph) with a two-state toggle (1:1 ↔ 1:2) plus data table. The data table serves as the stable anchor while the graph transforms.
- The MC question at the end confirms understanding — it's not just about playing with the toggle.

---

**Task S.2: Pattern Discovery (Type A) — Half-Symbol Mastery**

- **Purpose:** Confirm understanding of when half-symbols are needed. Students must apply even/odd reasoning without visual scaffolding.
- **Visual: Picture Graphs (Mode 2: Creating).** NEW graph context. Key visible: "Each \[symbol\] \= 2." Mix of even and odd values displayed. Data Table not visible.
- **Guide:** "You learned about half-symbols. If you made a graph using this key, which numbers would need a half-symbol?"
- **Prompt:** "If you made a graph with scale of 2, which numbers would need a half-symbol? Select all that apply."
- **Student Action:** \[Multi-select: ☐ 8  ☐ 7  ☐ 12  ☐ 9\]
- **Correct Answer:** 7 and 9 (odd numbers)
- **On Correct:** "7 and 9 are ODD numbers. They don't split evenly into groups of 2, so they need a half-symbol for the leftover 1."
- **On Incorrect (missed one):** "Check again. Which numbers are odd? Odd numbers always need a half-symbol with scale of 2."
- **Guide (Follow-up):** "What does a half-symbol show?"
- **Prompt:** What does a half-symbol show?
- **Student Action:** \[Multiple choice:
  - "Half of 2, which is 1"
  - "Half of the total"
  - "2 items"
  - "Nothing—it's decoration"\]
- **Correct Answer:** "Half of 2, which is 1."
- **Connection (Guide states):** "Half of 2 is 1\. A half-symbol shows 1 item. That's how we show odd numbers exactly—the full symbols show groups of 2, the half shows the 1 left over."

**Design Note:** Key provides context without scaffolding the answer. Students must reason about even/odd rather than just visually identifying half-symbols.

---

**Task S.3: Metacognitive Reflection**

- **Purpose:** Build awareness of what "scale" means. Student articulates understanding.
- **Visual: Picture Graphs (Mode 1: Reading).** Same side-by-side display from S.1 (1:1 and 1:2). Data Table not visible.
- **Guide:** "You've been working with 'scale of 2' all lesson. Which of these describes what a 'scale' on a graph means?"
- **Prompt:** "Which of these describes what a 'scale' on a graph means?"
- **Student Action:** \[Multiple choice:
  - "How many items each symbol shows"
  - "How many symbols are on the graph"
  - "How many categories there are"
  - "The size of the symbols"\]
- **Correct Answer:** "How many items each symbol shows"
- **On Correct:** "Scale tells you how many items each symbol shows. Scale of 1 means each symbol is 1\. Scale of 2 means each symbol is 2."
- **Connection (Guide states):** "When you see a graph, always check the key first. The key tells you the scale."

---

**Task S.4: Identity-Building Closure \+ Bridge to M3**

- **Purpose:** Specific, growth-oriented affirmation. Connect to future learning.
- **Visual: Grouping Animation (static frame).** Student's Warmup data transformation result. Brief display.
- **Visual: Picture Graphs (Mode 2: Creating).** A completed 1:2 graph from the Lesson. Visual bookend. Data Table not visible.
- **Guide:** "You started by watching your data transform. Now you can CREATE scaled graphs yourself. You know that each symbol shows 2, and half-symbols show 1."
- **Guide:** "You discovered something important: scaled graphs need fewer symbols. That makes them easier to read when numbers get big."
- **Bridge:** "Next time, you'll use a scale of 5\. Instead of groups of 2, you'll group by 5s. You already know how to count by 5s—that will help\!"
- **No student prompt** (closure)
