## **1.6 WARMUP (3-4 minutes)**

**Detail Level:** Structure \+ parameters (AI writes dialogue)

---

### **Purpose**

Activate prior knowledge of picture graphs from M1 while introducing the critical new concept: scaled representation. Students collect personal data, PREDICT how grouping will work, WATCH their prediction confirmed through animation, then VERIFY they can read the result. This creates ownership ("my data"), cognitive engagement ("was I right?"), and confidence ("I can do this") before Lesson formalizes vocabulary and has students create.

### **Parameters**

| Element | Specification |
| :---- | :---- |
| Interactions | 4 interactions (Game \+ Prediction \+ Animation \+ Verification/Bridge) |
| Active Participation | 3 of 4 require student response |
| Format | Data Collection Game → Prediction → Grouping Animation → Verification \+ Bridge |
| Cognitive Load | 20-30% (prediction and verification are accessible) |
| Remediation | Light only (prediction has gentle correction) |
| Tone | Medium-High energy, curious, builds anticipation |
| Engagement Anchors | Narrative Setup (Minis), Personalization (their data), Curiosity Gap (prediction check) |

### **Constraints**

| USE | AVOID |
| :---- | :---- |
| Informal language ("Watch this," "Check it out") | Formal phrasing ("Observe the transformation") |
| Contractions naturally | Stiff language |
| "Groups of 2" (casual, sets up Lesson) | "Scale of 2" (formal vocabulary—save for Lesson) |
| Observable acknowledgments ("You counted 8") | Assumed states ("You paid attention") |
| 3 categories | 2 or 4+ categories |
| Even values only (6-14 range) | Odd values (half-symbols are Lesson content) |

---

### **Warmup Type Rationale**

**Standard Playbook Types:** Binary Choice, WODB, Number Talk, Notice & Wonder, Estimation 180, Visual Patterns, Mystery Reveal

**M2 Approach:** Data Collection Game \+ Grouping Animation

**Why this serves the concept:**

* Unit 1 is about DATA—personal data creates investment
* The ANIMATION is the critical teaching tool for scaled graphs (effect size 0.50-0.77)
* Watching transformation with THEIR data makes abstract concept concrete
* "Watch then do" structure: Warmup \= observe transformation, Lesson \= create graphs
* Follows Playbook PRINCIPLES (low cognitive load, engagement anchors, curiosity gap) while adapting STRUCTURE for the critical conceptual leap

**Test:** If we removed this Warmup, would students lose mathematical preparation for Lesson? YES—they would miss the animated transformation that makes "each symbol \= 2 items" visible and concrete.

---

**Interaction W.1: Data Collection Game**

* **Purpose:** Generate personalized data. Create investment through Minis context. Provide concrete numbers for animation.
* **Hook (first 15-20 seconds):**
  * **Visual:** Minis characters appear in animated counting scenario.
  * **Guide:** "The Minis need your help counting\! Let's see what we find."
* **Engagement Anchor:** Narrative Setup (Minis context), Personalization (their counts)
* **Game Specifications:**

| Element | Specification |
| :---- | :---- |
| Style | Mario Party-style counting game |
| Categories | 3 (e.g., Red Minis, Blue Minis, Yellow Minis) |
| Values | Even numbers only: 6, 8, 10, 12, or 14 per category |
| Duration | \~60 seconds |
| Student Interaction | Count items; confirm/input count for each category |

* **Task:** Student participates in counting activity, tracking count for each of 3 categories.
* **On Complete:** Game ends; counts recorded for next interaction.

**Design Note:** All values MUST be even (6, 8, 10, 12, 14). Odd numbers introduce half-symbols, which is Lesson Section 2 content. Warmup animation shows clean grouping only.

---

**Interaction W.2: Review \+ Prediction**

- **Purpose:** Confirm data, then create cognitive engagement BEFORE animation through prediction. No feedback—the animation reveals the answer.
- **Visual:** Results displayed clearly: "\[Category A\]: \[X\], \[Category B\]: \[Y\], \[Category C\]: \[Z\]"
- **Guide:** "You counted \[X\] \[Category A\], \[Y\] \[Category B\], and \[Z\] \[Category C\]. That's *your* data."
- **Guide:** "Here's something new: what if each symbol showed 2 items instead of 1?"
- **Guide:** "You have \[X\] \[Category A\]. If you group them into 2s..."
- **Prompt:** "How many groups of 2 can you make from \[X\]?"
- **Student Action:** \[Multiple choice: X÷2 (correct), X, X+2, X-2\]
- **On Submit:** No correct/incorrect feedback. Student's selection is recorded.
- **Guide:** "You think \[student's answer\] groups. Let's see if you're right..."
- **Engagement Anchor:** Curiosity Gap (will my prediction be right?)
- **Duration:** 30-45 seconds

**Design Note:** This is a prediction, not an assessment. Withholding feedback creates genuine curiosity—the animation becomes the "answer." Students watch actively to check their thinking.

---

**Interaction W.3: Grouping Animation (THE KEY MOMENT)**

* **Purpose:** Visually demonstrate the critical leap—items grouping into 2s, then becoming symbols. This is the conceptual foundation for all of M2.
* **Engagement Anchor:** Curiosity Gap (what will happen to my data?)
* **Visual:** Animation workspace showing student's collected data on a picture graph.
* **Animation Sequence (45-60 seconds):**

**Phase 1: Items Appear**

* First category items appear (e.g., 8 Red Minis)
* Guide: "You counted 8 Red Minis. Watch what happens..."

**Phase 2: Grouping**

* Items visually pair up: 2, 2, 2, 2
* Guide: "8 items... break/split/show into 4 groups of 2..."

**Phase 3: Transformation**

* Each group of 2 becomes one symbol on the graph
* Guide: "Each group of 2 becomes ONE symbol."
* "4 groups... 4 symbols."

**Phase 4: Key Reveal**

* Key appears: "Each ⭐ \= 2"
* Guide: "Each symbol shows 2 items, not 1\. That's different from before\!"

**Phase 5: Remaining Categories (Condensed)**

* Second and third categories transform (faster, same pattern)
* Guide: "\[Y\] items... \[Y÷2\] groups of 2... \[Y÷2\] symbols."
* Guide: "\[Z\] items... \[Z÷2\] groups of 2... \[Z÷2\] symbols."

**Phase 6: Complete Graph**

* Full scaled picture graph visible with all 3 categories
* Guide: "Your data is now a SCALED picture graph\!"
* **No student prompt** (observation interaction)
* **On Complete:** Scaled picture graph with student's data remains visible.

---

**Interaction W.4: Verification \+ Bridge**

- **Purpose:** Confirm students can read the scaled graph. Create anticipation for Lesson.
- **Visual: Picture Graphs (Mode 2: Creating).** Completed 1:2 scaled picture graph from Warmup animation. 3 categories, all even values. Key visible: "Each \[symbol\] \= 2." Data Table not visible.
- **Guide:** "Now you've seen how grouping works\! Each group of 2 became one symbol."
- **Prompt:** "Look at \[Category B\]. How many symbols do you see?"
- **Student Action:** \[Multiple choice showing symbol counts\]
- **Correct Answer:** Actual symbol count for Category B (Y÷2)
- **On Correct:** "\[Y÷2\] symbols. And each one shows 2 items. So that's \[Y\] total\!"
- **Guide (Bridge):** "You can already read a scaled graph\! Now let's learn more about how they work."
- **Duration:** 30-40 seconds

---

### **Verification Checklist**

**Structure:**

- [ ] Hook appears in first 15-20 seconds (Minis appear)
- [ ] 2+ engagement anchors (Narrative, Personalization, Curiosity Gap)
- [ ] 3 active student interactions (Game, Prediction, Verification)
- [ ] Animation is observation but with prediction stakes
- [ ] Clear bridge to Lesson creates anticipation
- [ ] Total time 3-4 minutes

**Content Alignment:**

- [ ] 3 categories (not 2 or 4\)
- [ ] All values even (6, 8, 10, 12, 14 range)
- [ ] No odd values (half-symbols are Lesson Section 2\)
- [ ] Prediction asks "how many groups of 2"
- [ ] Animation shows grouping into 2s explicitly
- [ ] "Groups of 2" language used (not "scale of 2" formally)
- [ ] "Each symbol shows 2 items" stated in animation
- [ ] Verification confirms student can count symbols
- [ ] Key appears visually but isn't formally taught

**Constraints:**

- [ ] No student graph creation (prediction \+ verification only)
- [ ] No formal vocabulary teaching (casual use only)
- [ ] No half-symbols shown
- [ ] Cognitive load light (20-30%)
