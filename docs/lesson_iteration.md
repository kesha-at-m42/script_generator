# Lesson Script Iteration Process

## Overview

Scripts are generated by AI, reviewed and edited in Notion, then pulled back as a new versioned file. Scene config changes are flagged for manual follow-up.

---

## Workflow

**1. Generate**
```bash
python cli/run_pipeline.py -p lesson_generator -u 1 -m 1
```
Outputs to `outputs/unit1/lesson_generator_module_1/v{N}/step_01_lesson_generator/lesson_generator.json`

**2. Push to Notion**
```bash
python cli/push_to_notion.py outputs/unit1/lesson_generator_module_1/v{N}/step_01_lesson_generator/lesson_generator.json
```
- Creates or updates the Notion page for this unit/module (same page across all versions)
- Renders as readable script: `💬` dialogue · `❔` prompt · `🎬` scene · `✅/❌` validator states

**3. Review in Notion**

Editable fields:
- `💬` **Dialogue text** — edit the callout text directly
- `❔` **Prompt text** — edit the second line of the callout
- `🎬` **Scene description** — edit the callout text (flags for manual config update on pull)

Do not add or remove callout blocks — positional matching breaks.

**4. Pull changes**
```bash
# Overwrite current file
python cli/push_to_notion.py outputs/unit1/lesson_generator_module_1/v{N}/step_01_lesson_generator/lesson_generator.json --pull

# Save as new version (recommended)
python cli/push_to_notion.py outputs/unit1/lesson_generator_module_1/v{N}/step_01_lesson_generator/lesson_generator.json --pull --new-version
```
- Auto-applies: dialogue text, prompt text
- Auto-applies + flags: scene `description` field
- Saves flagged scene changes to `notion_flags.json` next to the output file

**5. Handle scene flags**

If `notion_flags.json` is non-empty, the scene description changed in Notion — meaning the visual state changed. Manually update the relevant `params` fields (e.g. `highlight_categories`, `values`, `mode`) in the pulled JSON to match the new description, then delete the flags file.

**6. Re-push**

After editing, push the new version to update the Notion page.

---

## Registry

`config/notion_pages.json` maps each pipeline+unit+module to a Notion page ID. The key is version-agnostic — all versions of the same module share one page.
