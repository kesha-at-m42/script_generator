**Grade 3 Unit 1: Toy Flow for SME Review**
===========================================

**MODULE 1: Data Sense & 1:1 Graphs**
-------------------------------------

### **Learning Goal**

Interpret picture graphs and bar graphs to generate questions about data (review/activate Grade 2 knowledge)

**Key Teaching Points:**

-   Picture graphs use symbols to represent data, each symbol = 1 item

-   Bar graphs use bars to show data, scale counts by 1s

-   Both graphs have key parts: title, categories, scale/key

-   Both orientations (horizontal and vertical) show the same information

-   Graphs help us see patterns: most, least, same, different

-   Address \"reading the picture not the data\" misconception: Students must use key/scale, not count visual elements

### **What Students DO**

-   READ pre-made picture graphs and bar graphs (1:1 scale, both orientations)

-   IDENTIFY information from graphs (most, least, total, difference)

-   COMPARE data across categories

-   ANSWER \"how many more/fewer\" questions

### **Cognitive Focus**

-   IDENTIFY (Conceptual) - recognizing data patterns

-   COMPARE (Conceptual) - determining relationships

### **Toy Flow**

**WARM-UP**

-   **\[DATA COLLECTION GAME - \~1 min\]**

    -   Mario-party style counting activity

    -   Student counts animals across 3 categories (Cats, Dogs, Birds)

    -   Results: values 3-8 range (appropriate for 1:1)

-   **Guide reviews results (\~30 sec):**

    -   \"You counted 5 cats, 7 dogs, and 3 birds\"

    -   **Symbol Selection:**

        -   Guide: \"Let\'s pick a symbol to represent animals on our graph\"

        -   Student selects from palette (paw prints, bones, hearts, stars, etc.)

        -   Same symbol will be used for ALL categories

    -   \"Each \[symbol\] will represent one animal\"

-   Students create 1:1 picture graph with THEIR collected data

-   Same symbol used across all three categories

-   Category labels identify what each row represents

-   **Purpose:** Graphs show information we collect

**LESSON**

-   **Data Tables** → **Picture Graphs (Reading)** → **Bar Graphs (Reading)**

-   Students practice reading graphs in both orientations

-   Click categories to highlight, click two to compare

-   System shows difference calculations

-   Guide asks: \"What do you notice? What\'s the same/different?\"

-   Multiple graphs, varied contexts

-   Interactions: View, Click, Compare, Discuss observations

**EXIT CHECK**

-   **Bar Graph** → Students answer questions about data

-   Simple reading comprehension check

**PRACTICE**

-   **Multiple graphs** → Students answer comparison questions

-   Focus on \"how many more/fewer\" language

-   Both orientations mixed

**SYNTHESIS**

-   **Side-by-side graphs** → Students compare picture vs. bar representations

-   Discussion: \"When would each graph be useful?\"

-   Bridge: \"Tomorrow we\'ll see symbols that represent MORE than one\...\"

### **Scaffolding Notes**

-   Heavy use of familiar Grade 2 formats

-   Both orientations from start (build flexibility early)

-   No graph creation yet - pure reading and interpretation

-   Guide models comparison language explicitly

### **SME Review Questions**

1.  Is full lesson of review appropriate, or should M1 be shorter/combined with M2?

    a.  Andrea - totally appropriate. Unless we have previous data that the student has mastered this concept and doesn't need it, it is activating prior knowledge and ensuring they have a solid understanding foundation to make the jump to scaled representations. (This could possibly be a shorter module if that real time assessment is showing mastery, and only involves longer explanations in remediation? Unless the goal is to have all modules be about the same length?) Answering how many more/fewer, in total/combined, type questions is also such an important skill that continues to cycle in as the representations get more complex (in this unit and higher grades).

2.  Does starting with both orientations simultaneously create confusion?

    a.  Andrea - students should already be exposed to tables and graphs in both orientations from grades 1 and 2, so it should not create confusion and should be an activation of prior knowledge where they continue to use both orientations and don't default to only vertical.

**MODULE 2: Introduction to 1:2 Scale**
---------------------------------------

### **Learning Goal**

Interpret scaled picture graphs where each symbol represents 2 items

**Key Teaching Points:**

-   THE CRITICAL LEAP: Each symbol represents MULTIPLE items (scale of 2)

-   Scale means \"each symbol = 2 items\"

-   Even numbers group cleanly by 2s (no half-symbol needed)

-   Odd numbers have one left over (half-symbol shows the remainder)

-   **Mathematical precision: Half of 2 = 1** (clean, understandable)

-   Students can use even/odd knowledge to predict when half-symbols needed

-   Address \"symbol counting\" misconception: Count by 2s, don\'t count individual symbols

### **What Students DO**

-   WATCH animated grouping (10 items → 5 groups of 2 → 5 symbols)

-   CREATE picture graphs by placing symbols (calculating how many needed)

-   IDENTIFY scale meaning (\"each symbol = 2\")

-   COMPARE data values (answer \"how many more/fewer\" questions)

-   CONNECT to even/odd numbers (even = whole symbols only, odd = need half-symbol)

### **Cognitive Focus**

-   CREATE (Procedural) - placing symbols correctly

-   IDENTIFY (Conceptual) - understanding scale

-   COMPARE (Conceptual) - reading completed graphs to answer questions

-   CONNECT (Transfer) - applying even/odd understanding to predict half-symbols

### **Toy Flow**

**WARM-UP (Extended for both collection + transformation)**

-   **\[DATA COLLECTION GAME - \~1 min\]**

    -   Counting activity with results in even numbers (2, 4, 6, 8, 10, 12)

    -   3-4 categories

-   **Guide reviews (\~30 sec):**

    -   \"You counted \[results\]\"

-   **Grouping Animation (NEW):**

    -   Takes student\'s collected data

    -   Shows items → groups of 2 → symbols on graph

    -   \"Watch your data transform! Instead of showing ALL items, we can group them\...\"

-   **Purpose:** Their data + scale introduction via transformation

**LESSON**

*Early activities:*

-   **Symbol/Picture Table** → **Grouping Animation** → **Picture Graph (Creating)**

-   Students read table, watch grouping, place symbols

-   Pre-completion: 3 of 4 categories already done

-   **All values are EVEN numbers** (8, 10, 12, 14)

-   Guide narrates: \"8 votes needs how many symbols? Let\'s see: Because 8 is an even number, it can be split into 4 groups of 2\"

-   Data table visible on side as reference

-   Interactions: Read → Watch → Drag/Click symbols → Submit

-   **After creation:** Answer 1-2 questions about the completed graph

    -   \"Which got the most votes?\"

    -   \"How many more chose X than Y?\"

*Mid activities:*

-   Pre-completion: 2 of 4 categories

-   **Introduce FIRST odd number** (7 votes)

-   **HALF-SYMBOL INTRODUCTION:**

    -   Animation shows: 7 items → 6 items (3 groups of 2) + 1 remaining → 3 full symbols + ?

    -   Guide: \"7 votes needs 3 full symbols for 6, plus HALF a symbol for 1 more\"

    -   Guide: \"Half of 2 is 1, so this half-symbol shows 1 vote\"

    -   Student practices placing half-symbols

-   Table can be minimized

-   **After creation:** Answer comparison questions

    -   \"How many fewer chose X than Y?\"

    -   \"How many in all for these two categories?\"

*Late activities:*

-   **Data Table** → **Picture Graph (Creating)**

-   No animation, 0 pre-filled

-   **Multiple odd numbers** (7, 9, 11, 13)

-   Students place half-symbols independently

-   Full independence expected

-   **After creation:** Answer 2-3 questions independently

    -   Mix of \"how many more/fewer\" and \"in all\" questions

    -   Students use completed graph to solve problems

**EXIT CHECK**

-   **Data Table** → **Picture Graph**

-   4 categories, mix of even and odd numbers

-   Half-symbols required for odd values

-   Independent work, system feedback only

**PRACTICE**

-   Multiple graphs for fluency

-   Mix of even and odd numbers

-   Both horizontal and vertical half-symbol cuts available

-   Building comfort with half-symbols

**SYNTHESIS**

-   **Multi-Format Display** → Same data shown three ways: Table \| Graph (1:2) \| Graph (1:1)

-   Students compare: \"Which needs fewer symbols? Why?\"

-   **Half-Symbol Mastery Check:** \"What does a half-symbol mean with scale of 2?\"

-   Reflection: \"Tell me what \'scale of 2\' means in your own words\"

-   Bridge: \"Next we\'ll group by 5s instead of 2s. Watch how skip-counting helps\...\"

### **Scaffolding Notes**

-   Animation is NON-NEGOTIABLE early (research-backed: effect size 0.50-0.77)

-   Pre-fill decreases: 3→2→0 categories

-   Animation availability: Auto-play → Optional → Removed

-   **Even numbers only until mid activities (8, 10, 12, 14)**

-   **Half-symbol introduction: Mid activities with explicit teaching**

-   **Mathematical precision: Half of 2 = 1 (clean, understandable for 3rd graders)**

-   Odd numbers increase in late activities and practice

-   **Question complexity:** Simple identification (early) → Comparison (mid) → Multiple questions independently (late)

### **SME Review Questions**

1.  Animation removal by late activities - too early for most difficult graphing concept?

    a.  Andrea - since it will be re-introduced with 5s and 10s, I think it can be removed in late activities as students are building fluency with 2s. I would suggest to include animation as a primary support for remediation though.

2.  Should there be 1-of-4 pre-filled middle step (3→2→1→0)?

    a.  Andrea - not necessary. Still plenty of scaffolding with the jump from 2 to 0.

3.  Half-symbol introduction in mid activities - appropriate timing or should it wait until late activities?

    a.  Andrea - mid-activities seems appropriate. See my comment for possible scaffolding of building with half symbols only to building with a combination of whole and half symbols

**MODULE 3: Picture Graphs with 1:5 Scale**
-------------------------------------------

### **Learning Goal**

Represent data using scaled picture graphs where each symbol represents 5 items

**Key Teaching Points:**

-   Scale of 5: Each symbol represents 5 items

-   Connect to skip-counting by 5s (familiar from Grade 2)

-   Connect to hands/fingers: 5 on each hand

-   **All values are multiples of 5** (no half-symbols in this module)

-   Scale of 5 uses fewer symbols than scale of 2 (more efficient for larger numbers)

-   Pattern: Count by 5s to read the graph

-   Address \"skip-counting errors\": Must count all symbols, not stop early

### **What Students DO**

-   WATCH animated grouping with 5s

-   CREATE picture graphs (multiples of 5 only)

-   COMPARE same data with different scales (1:2 vs 1:5)

-   IDENTIFY patterns (skip-counting, efficiency)

-   CONNECT to skip-counting by 5s (familiar from Grade 2)

### **Cognitive Focus**

-   CREATE (Procedural) - placing symbols for scale of 5

-   COMPARE (Conceptual) - evaluating scale efficiency

-   IDENTIFY (Conceptual) - recognizing skip-counting patterns

-   CONNECT (Transfer) - linking to skip-counting

### **Toy Flow**

**WARM-UP**

-   **\[DATA COLLECTION GAME - \~1 min\]**

    -   Counting activity with results in multiples of 5 (5, 10, 15, 20, 25)

    -   3-4 categories

-   **Guide reviews (\~30 sec):**

    -   \"Last time: scale of 2. Today: scale of 5!\"

    -   Brief comparison: \"What\'s different?\"

-   Students create scale 5 picture graph with collected data

-   **M2 Scale Review:** First lesson activity shows one scale-2 example briefly

**LESSON**

*Early activities:*

-   **Symbol/Picture Table** → **Grouping Animation** → **Picture Graph (Creating)**

-   **All values are multiples of 5** (10, 15, 20, 25)

-   Pre-fill: 2 of 4 categories

-   Guide connects to hands/fingers: \"5 on each hand, easy to count\"

-   Guide connects to skip-counting: \"You know how to count by 5s: 5, 10, 15\...\"

-   Interactions: Read → Watch → Drag/Click symbols → Submit

*Mid activities:*

-   **Symbol/Picture Table** → **Picture Graph (Creating)**

-   **Still multiples of 5** (5, 10, 15, 20, 25, 30)

-   Pre-fill: 1 of 4 categories

-   Guide prompts: \"How many groups of 5 in 20?\"

-   Skip-counting emphasized

-   **Format introduction:**

    -   **Multi-Format Display** → Same data shown as Picture (1:5) AND Bar (1:5)

-   Guide: \"Same data, different way to show it. Bars are continuous, not separate symbols.\"

-   **Students create BOTH:**

    -   Given data → Create picture graph

    -   Same data → Create bar graph

    -   Compare: Which was easier? Which is clearer?

*Late activities:*

-   **Multi-Format Display** → Picture graph (1:5) alongside Bar graph (1:5)

-   **Multiples of 5 only** (larger values: 25, 30, 35, 40)

-   Guide: \"Same data, new format. Bars are continuous, not separate symbols.\"

-   Students CREATE bar graphs with scale 5

-   Multiple problems for practice

-   Builds comfort with bar format before scale change

-   Ghost gridlines available

-   Building comfort with continuous bars

-   Focus on efficiency: \"Scale of 5 needs fewer symbols than scale of 2!\"

**EXIT CHECK**

-   **Data Table** → **Picture Graph**

-   4 categories, all multiples of 5

-   Values in range 10-40

**PRACTICE**

-   Multiple graphs for fluency

-   All multiples of 5

-   Building speed with scale of 5

-   Both orientations

**SYNTHESIS**

-   **Side-by-side:** Same data shown as Picture (1:5) and Bar (1:5)

-   Discussion: \"When would bars be better than pictures?\"

-   **Skip-counting connection:** \"You\'ve been skip-counting! That\'s what scale of 5 shows!\"

-   **Hand reference:** \"5 fingers on each hand - easy to remember\"

-   Bridge: Guide: \"For REALLY big numbers, bars work even better. Tomorrow we\'ll use scale of 10\...\"

### **Scaffolding Notes**

-   **NO half-symbols in M3** - students learned half-symbols with scale of 2 in M2

-   All values are multiples of 5 throughout module (simpler, cleaner)

-   Focus entirely on grouping by 5s and skip-counting connection

-   Pre-fill decreases: 2→1→0 categories

-   Animation: Auto-play early → Optional mid → Removed late

-   Scale of 5 conceptual leap supported by skip-counting familiarity (Grade 2 skill)

### **SME Review Questions**

1.  Is keeping M3 multiples-only appropriate given scale of 5 complexity?

    a.  Andrea - If we were counting by 4s (which we aren't), then a quarter of a symbol might be appropriate. I think it is much too complex to do fifths of a symbol, unless that symbol naturally has 5 parts (a hand with 5 fingers, a flower or leaf with 5 parts, etc). It's too complex and doesn't add much value to split an apple into 5 parts to represent 12 on the pictograph, for instance. Maybe we could have a challenge/extension type question that does involve a symbol with 5 parts just to see if students can apply the idea from half symbols in M2 to partial symbols in M3, but it's not imperative.

2.  Skip-counting connection - emphasized enough to support learning?

    a.  Andrea - I say yes!

3.  Should we include ANY non-multiples (like 23) even without half-symbols, showing as \"about 4.5 symbols\"?

    a.  Andrea - see my comment under \#1. I don't think we should estimate it to approximately half a symbol. It is either the exact fifth or we don't use it at all. We don't want kids approximating bar height as in the middle of 20 and 25 when they have 23, we would want it to clearly be more than half even as an approximation

**MODULE 4: Bar Graphs & 1:10 Scale**
-------------------------------------

### **Learning Goal**

Represent data using scaled bar graphs where the scale increments by 10

**Key Teaching Points:**

-   THE CRITICAL LEAP: Bars are continuous, not individual symbols

-   Scale of 10: Tick marks at 10, 20, 30, 40\...

-   Halfway points show multiples of 5: 25 is halfway between 20 and 30

-   **Half of 10 = 5** (reinforces half-symbol concept from M2)

-   **Precision matters:** We place bars exactly at marks or halfway points, no estimation

-   **All values are multiples of 5** - no estimation required

-   Same data can be shown as picture graph OR bar graph

-   Address \"symbol counting\" misconception: Students must read the scale, not count gridlines

### **What Students DO**

-   WATCH transformation (picture graph → bar graph)

-   CREATE bar graphs by adjusting heights

-   IDENTIFY values at tick marks (multiples of 5)

-   COMPARE discrete symbols vs continuous bars

-   APPLY understanding to answer questions

### **Cognitive Focus**

-   CREATE (Procedural) - setting bar heights

-   CONNECT (Transfer) - understanding symbols → bars transition

-   IDENTIFY (Conceptual) - reading scaled axis

-   COMPARE (Conceptual) - determining relationships between values

### **Data Constraints**

**All data values in Module 4 are multiples of 5:**

-   Examples: 10, 15, 20, 25, 30, 35, 40, 45, 50\...

-   NO values like 23, 37, 41, 48 (would require estimation)

-   Students place bars at tick marks (multiples of 10) or halfway points (multiples of 5)

-   Mathematical precision maintained: half of 10 = 5

**Rationale:** Keeps focus on understanding bar graphs and scale of 10 without introducing estimation complexity inappropriate for Grade 3.

### **Toy Flow**

**WARM-UP**

-   **\[DATA COLLECTION GAME - \~1 min\]**

    -   Counting activity, results in multiples of 5 (10, 15, 20, 25, 30)

    -   3-4 categories

-   **Guide reviews results (\~30 sec):**

    -   \"You counted \[results\]\"

    -   \"You learned bar graphs with scale 5 last time. Watch this with YOUR data\...\"

-   **Transformation Animation (Single: 1:5 picture → 1:5 bar):**

    -   Student\'s collected data shown as picture graph (scale 5)

    -   Transforms to bar graph (scale 5)

    -   Symbols fade, bars emerge, scale/labels stay same

    -   Guide: \"Same data, different format - just like you learned!\"

-   **Bridge to Scale 10:**

    -   Guide: \"Today we learn an even BIGGER scale: 10!\"

    -   \"Helpful when numbers get really large\...\"

**LESSON**

*Activities 1-2 (Observation Phase):*

-   **Picture Graph (1:10)** → **Bar Graph (1:10)** transformation

-   **Transformation Animation** → Students watch 1:10 picture become 1:10 bar

-   Guide: \"You know bars from scale 5. Same format, bigger scale: 10.\"

-   Focus: Only NEW element is scale of 10 (format already familiar)

-   No creation with 1:10 pictures (skip straight to bars)

-   **Compare bar graphs: Scale 5 vs Scale 10 for same data**

-   Discussion: What\'s different? What stayed the same?

-   NO CREATION YET - building conceptual understanding first

-   Guide: \"Bars are continuous. No individual symbols to count.\"

-   Less time needed here since bar format is familiar

*Activities 3-4 (Guided Creation):*

-   **Data Table** (static reference) → **Bar Graph (Creating)**

-   Student adjusts 4 bar heights

-   Pre-set: 3 bars correct, student completes 1

-   All values are multiples of 10

-   Ghost gridlines visible, snap-to-10 enabled

-   Interactions: Read table → Drag bar OR Click-set height → Submit

-   **After creation:** Answer 1-2 comparison questions

    -   \"How many more X than Y?\"

    -   Questions require reading between tick marks

*Activities 5-7 (Building Independence):*

-   All 4-5 bars need adjustment (0 pre-set)

-   **Introduce halfway points: values that are multiples of 5 but NOT multiples of 10** (25, 35, 45)

-   **Half-symbol reinforcement:** \"Remember half-symbols from scale of 2? Same idea here.\"

-   **Mathematical precision:** \"Half of 10 is 5, so bar at 25 is halfway between 20 and 30\"

-   Ghost gridlines still available

-   Snap-to-10 now optional (snap-to-five available)

-   Guide helps with interpolation for first non-multiple, then students independent

-   **After creation:** Answer 1-2 questions requiring interpolation

    -   \"How many chose X?\" (where X is at 25 or 35)

    -   Focus on reading halfway points correctly

*Activities 8-9 (Full Independence):*

-   **Mix of multiples of 10 and halfway points** (e.g., 20, 25, 30, 45, 50)

-   Both orientations practiced

-   Gridlines off by default (can enable if needed)

-   Students place bars at tick marks and halfway points

-   Students apply half-symbol understanding to continuous bars

-   **After creation:** Answer 1-2 simple comparison questions

    -   \"Which has the most/least?\"

    -   \"How many more X than Y?\" (direct comparison)

**EXIT CHECK**

-   **Data Table** → **Bar Graph**

-   4 bars, vertical orientation

-   Mix of multiples and non-multiples

**PRACTICE**

-   Both orientations

-   Emphasis on mix of multiples

-   Building estimation skill

**SYNTHESIS**

-   **Multi-Format:** Same data as Bar (1:10), Picture (1:5), Picture (1:2)

-   Discussion: \"Which is most efficient for large numbers? Why?\"

-   Guide: \"Bars work great for big numbers. But how do you CHOOSE which scale?\"

-   Bridge to M5: \"Tomorrow you\'ll decide which scale to use\...\"

### **Scaffolding Notes**

-   TWO shifts at once (symbols→bars AND scale 5→10) require extra support

-   Observation activities before creation (build understanding before doing)

-   **Half-symbols: Reinforcement not introduction** (learned in M2, now applied to scale of 10)

-   **Mathematical precision maintained:** Half of 10 = 5 (clean for interpolation)

-   Non-multiples introduced after multiple-only practice (Activities 5-7)

-   Gridlines gradually removed as scaffolding

### **SME Review Questions**

1.  Two simultaneous shifts - is multi-format display sufficient scaffolding?

    a.  Andrea - I think students need to see the shift of symbols to bars with both 1:2 and 1:5 before starting 1:10.

    b.  Andrea - could be worth having students create a pictograph with 1:10 before jumping to the bar graph with 1:10. This could be in the warm up or in the observation phase activities 1-2 of the lesson. They see how a table with larger values becomes a pictograph with a scale of 1:10 and then bar graph. Maybe that reduces the double shift of symbols→bars AND scale 5→10

    a.  Andrea - Valuable with the double shift. Unless the early part of the lesson is creating a bar graph with a scale of 1:2 or 1:5 which they already mastered, then move to observation phase. My concern is that lengthens this lesson when there is already a lot in here.

3.  Non-multiple introduction (Activity 5) - too early?

    a.  Andrea - No, especially if we are using a constraint of multiples of 5. But they need to adjust ALL bars with multiples of 10 before dealing with non-multiples. If other non-multiples come in, add in activities 8-9 but WITH the gridlines.

**MODULE 5: Scale Selection Strategy**
--------------------------------------

### **Learning Goal**

Choose an appropriate scale for a bar graph based on the data set range and justify the choice

**Key Teaching Points:**

-   Scale must be large enough for your largest data value

-   Different scales work with different types of numbers:

    -   Scale of 2: works for any numbers (whole and half symbols)

    -   Scale of 5: works best when numbers are multiples of 5

    -   Scale of 10: works when numbers are multiples of 5 (we can show halves)

-   Preview helps you check: \"Will everything fit?\"

-   Multiple scales can work - choose based on efficiency AND fit

-   Pattern discovery: Small numbers (under 25) → scale of 5; Large numbers (50+) → scale of 10

-   **All values are multiples of 5** - we can read graphs precisely without estimating

-   Clicking two categories can mean different things:

    -   \"How many altogether?\" → add them

    -   \"How many more?\" → find the difference

### **Cognitive Focus**

-   COMPARE (Conceptual) - evaluating scale efficiency

-   APPLY (Transfer) - strategic decision-making

-   CONNECT (Transfer) - relating data characteristics to scales

### **Data Constraints**

**All values are multiples of 5** (maintains no-estimation principle):

-   Early activities: 5, 10, 15, 20, 25, 30, 35, 40, 45, 50

-   Examples for scale selection:

    -   Small range: 5, 10, 15, 20, 25 → scale of 2 or 5

    -   Medium range: 10, 20, 25, 35, 45 → scale of 5 or 10

    -   Large range: 20, 35, 50, 65, 85 → scale of 10

**Rationale for multiples of 5:**

-   Students can read values precisely from graphs (no estimation)

-   Supports answering questions directly from graph (no table needed)

-   Consistent with M4 constraint (precision over real-world messiness)

### **Toy Flow**

**WARM-UP**

-   **\[DATA COLLECTION GAME - \~1 min\]**

    -   Counting activity designed to produce data range \~5 to 30

    -   4 categories with VARIED range (intentional design)

-   **Guide reviews (\~30 sec):**

    -   \"Look at your numbers. Biggest is \[30\]. Smallest is \[5\].\"

    -   \"Which scale would work best?\"

-   **Scale Preview System:**

    -   Student\'s collected data shown in scale-5 preview

    -   Then scale-10 preview

    -   \"Which fits better? Which is easier to read?\"

-   **Purpose:** Real decision-making with THEIR data

**LESSON**

*Early activities:*

-   **Data Table** → **Scale Preview System**

-   Example 1: Student sees data range: 5, 10, 20, 35, 45

    -   Limited choices: scale of 5 OR 10

    -   Click scale → see preview of what graph would look like

    -   Guide: \"Look at your highest number (45). Which scale makes sense?\"

    -   Students explore both options with preview function

    -   After selecting scale: Answer 1-2 questions about the data

        -   \"How many chose X?\" (value at 20)

        -   \"How many chose A and B altogether?\" (values at 10 and 35)

-   Example 2: Student sees data range: 20, 35, 50, 70, 90

    -   Same limited choices: scale of 5 OR 10

    -   Click scale → see preview

    -   Scale of 10 is clearly more efficient (scale of 5 would require many more tick marks)

    -   Guide: \"Now compare. What changed? Why does scale of 10 work better for this data?\"

    -   Students recognize: bigger numbers → bigger scales needed

    -   After selecting scale: Answer comparison question

        -   \"How many more chose X than Y?\" (values at 50 and 35)

*Mid activities:*

-   All three scale options (2, 5, 10)

-   **Guide demonstrates poor choice first**

    -   \"Let\'s try scale of 2 with this data. Watch what happens\...\"

    -   System shows: tallest bar (87) doesn\'t fit on graph with scale of 2

    -   Guide: \"Oh no! The bar went off the graph. What went wrong?\"

    -   Students see: scale too small for the data

-   **Now student\'s turn:**

-   Multiple defensible choices possible

-   **Interactive Analysis Toy** → Preview function

-   Student selects scale → preview shows if all data fits

-   Visual feedback: ✓ \"All data fits!\" or ⚠️ \"Tallest bar won\'t fit - try a bigger scale\"

-   Student must explain choice (Multiple Choice) AND answer 1 question about the data

-   Guide asks: \"Can all data fit? Is it easy to read?\"Now tell me: which category has the most?\"

*Late activities:*

-   Explain tradeoffs in detail

-   Multiple datasets to evaluate

-   Compare student\'s choice to alternatives

-   Discussion of when each scale is best

-   Students answer 1-2 questions using their created graph

**EXIT CHECK**

-   Given dataset → Choose scale → Justify briefly

**PRACTICE**

-   Multiple datasets

-   Variety of \"best\" scales

-   Building decision-making fluency

**SYNTHESIS**

-   **Pattern Discovery:** Small numbers (0-20) → scale of 2; Medium (0-50) → scale of 5; Large (50+) → scale of 10

-   Students articulate the patterns they\'ve discovered

-   Reflection: \"How do you decide which scale to use?\"

-   Bridge: \"Now you can CREATE graphs and CHOOSE smart scales. Let\'s USE these graphs to solve problems\...\"

### **Scaffolding Notes**

-   Meta-learning: WHEN to use which scale (not just HOW)

-   Scaffolded choice: 2 options → 3 options

-   Preview function prevents wrong choices (students see consequences)

-   Both orientations used to build flexibility

### **SME Review Questions**

1.  Is meta-cognitive load appropriate for 3rd grade after just learning scales?

    a.  Andrea - it's good as long as there isn't an over-emphasis on justifying their scale (answers like it looks better, it's clearer, I can see the data, etc should be acceptable). The more important skill is to use scales appropriately and interpret them accurately.

2.  Should justification be verbal only, or require written explanation?

    a.  Andrea - a more in-depth justification of scale choice seems to be 4th or 5th grade skill. A 3rd graders justification might be, \"I used a scale of 5 because the numbers are big\" or \"A scale of 10 would make it easier to count\".

**MODULE 6: Problem Solving with Data**
---------------------------------------

### **Learning Goal**

Solve one- and two-step \"how many more/fewer\" problems using data from scaled bar graphs

**SYNTHESIS**

-   **Reading Multiplication:**

    -   Display: 5 × 2

    -   Guide: \"When you see just numbers and symbols like this, the first number means rows.\"

    -   \"So 5 × 2 means 5 rows of 2.\"

-   **Describing Arrays:**

    -   Display: Array with 5 rows and 2 columns

    -   Guide: \"When you look at an array, you can describe it two ways:\"

    -   \"You can say: 5 rows of 2 → write 5 × 2\"

    -   \"OR you can say: 2 columns of 5 → write 2 × 5\"

    -   \"Same array. Different descriptions.\"

-   **The Important Rule:**

    -   \"Your words must match what you write\"

    -   \"If you say \'rows,\' write the rows first\"

    -   \"If you say \'columns,\' write the columns first\"

-   **The Amazing Discovery:**

    -   Display both equations: 5 × 2 = 10 and 2 × 5 = 10

    -   \"Both equal 10! The order doesn\'t change the total.\"

    -   \"That\'s called the commutative property\"

    -   \"Or we can call it: turn-around facts\"

-   **Being a Smart Mathematician:**

    -   \"Both 7 × 2 and 2 × 7 equal 14\"

    -   \"But which is easier to count?\"

    -   \"Skip-count by 2s: 2, 4, 6, 8, 10, 12, 14 ← We practiced this!\"

    -   \"Skip-count by 7s: 7, 14 ← We haven\'t practiced this much\"

    -   \"Same answer, but smart mathematicians choose the easier way!\"

**KEY TRANSITION POINTS**
-------------------------

### **Transition 1: Scale Introduction (M1 → M2)**

**Pedagogical Shift:** 1:1 correspondence → scaled representation WITH half-symbols

**Cognitive Load:** HIGH (most difficult elementary graphing concept, plus half-symbols)

**Support Needed:**

-   Animated grouping (non-negotiable)

-   Explicit half-symbol teaching (mid-lesson)

-   Heavy practice with both even and odd numbers

-   Mathematical precision (half of 2 = 1)

**Why introduce half-symbols here:** Scale of 2 provides mathematically clean introduction. Students need this foundation before moving to larger scales.

### **Transition 2: Scale Increase (M2 → M3)**

**Pedagogical Shift:** Scale of 2 → scale of 5 (larger grouping)

**Cognitive Load:** MODERATE (scale increase only, no new symbol types)

**Support Needed:**

-   Skip-counting connection (familiar from Grade 2)

-   Hand/finger reference (5 per hand)

-   Multiples-only values (no half-symbols needed)

-   Focus on efficiency (fewer symbols needed)

**Why multiples-only:** Scale of 5 is conceptual leap. Keeping values clean allows focus on grouping pattern and skip-counting connection without half-symbol complexity.

**Transition 3: Symbol Type Introduction (M3 Mid → M3 Late → M4)**

**Pedagogical Shift:** Discrete symbols → continuous bars (format change), THEN scale 5 → 10 (scale change)

**Cognitive Load:** MODERATE (distributed across M3 late and M4)

**Support Needed:**

-   Transformation animation with scale 5 (M3 introduction)

-   Side-by-side comparison at same scale (M3 mid activities)

-   Practice with bars at familiar scale before scale jump (M3 late)

-   Bar format already comfortable when scale 10 introduced (M4)

-   Dual transformation in M4 warm-up (reinforcement)

**Why distributed**: Separating format change (M3) from scale change (M4) reduces cognitive load. Students master bars with familiar scale before larger scale.

### **Transition 4: Strategic Thinking (M4 → M5)**

**Pedagogical Shift:** Following given scale → choosing appropriate scale

**Cognitive Load:** MODERATE (meta-learning)

**Support Needed:**

-   Preview function (see consequences of choices)

-   Justification prompts

-   Comparison opportunities

-   Pattern discovery guidance

### **Transition 5: THE BIG BRIDGE (M6 → M7)**

**Pedagogical Shift:** Data representation → multiplication meaning

**Cognitive Load:** MODERATE (concept is present in M1-6, just reframed)

**Support Needed:**

-   Extended opening (5-7 min)

-   Explicit callback to graphs

-   New animation framing (symbols → groups → equation)

-   Multiple representations (graphs, bags, boxes)

**Critical Success Factor:** Students must see \"I was already doing multiplication!\"

**Transition 6: Symbolic Representation - Expressions (M7 → M8)**

**Pedagogical Shift:** Visual groups (no symbols) → expressions (× symbol)

**Cognitive Load:** LOW-MODERATE (single symbol introduction)

**Support Needed:**

-   Guide shows notation first (M7), students create in M8

-   Progressive method unlocking (multiple choice → tiles → all methods)

-   Bidirectional practice (visual → expression AND expression → visual)

-   Explicit structure teaching: first number = groups, second = items per group

-   Consistent contexts (bags/boxes only)

Critical Success Factor: Students understand × symbol MEANING, not just placement

**Transition 7: Factor Fluency & Graph Connection (M8 → M9)**

**Pedagogical Shift:** Expressions with any factors → fluency with 2s, 5s, 10s

**Cognitive Load:** LOW (consolidation and pattern discovery)

**Support Needed:**

-   Pattern recognition (2s even, 5s end in 0/5, 10s end in 0)

-   Explicit graph callback (scales WERE multiplication)

-   Repetition with familiar factors builds automaticity

-   Connection to skip-counting reinforced

Why this sequence: Building fluency with common factors BEFORE tackling equations with unknowns provides foundation for problem-solving

**Transition 8: Equations & Variety Introduction (M9 → M10)**

**Pedagogical Shift:** Expressions with familiar factors → equations with variety of contexts

**Cognitive Load:** MODERATE (= sign meaning + variety + unknowns)

**Support Needed:**

-   Fluency from M9 supports unknown factor work

-   = sign flexibility taught from start (12 = 3×4 AND 3×4 = 12)

-   EXPLICIT variety announcement mid-lesson

-   Structure language maintained: \"\_\_\_ groups/rows/stacks of \_\_\_\"

-   Students already know unknowns in various positions (grades 1-2 with +/-)

Critical Success Factor: Fluency from M9 reduces cognitive load when introducing equations with unknowns

**Transition 9: Spatial Organization (M10 → M11)**

**Pedagogical Shift:** Unorganized groups (bags, stacks, rows) → organized arrays (spatial structure)

**Cognitive Load:** MODERATE (new spatial structure + both row/column interpretations)

**Support Needed:**

-   Clear row AND column language from start

-   Language must match expression (precision emphasized)

-   Context progression: Concrete objects → Abstract dots

-   Both interpretations validated (rows OR columns, both correct)

-   Connection to equal groups (each row/column = one group)

Why both interpretations: Respects student diversity (some see rows, some columns); sets up M12 property discovery naturally

**Transition 10: Property Discovery (M11 → M12)**

**Pedagogical Shift:** Arrays with two interpretations → commutative property proof

**Cognitive Load:** LOW-MODERATE (builds on M11 understanding)

**Support Needed:**

-   Highlighting review (M11 knowledge)

-   Rotation animation introduction (new element)

-   Hybrid approach: Both highlighting AND rotation prove same property

-   Side-by-side displays

-   Discovery-based approach (students see equivalence)

Critical Success Factor: M11 established both interpretations exist; M12 proves they\'re EQUIVALENT (property revelation, not interpretation introduction)

**CRITICAL DESIGN DECISIONS**
-----------------------------

### **Half-Symbol Sequencing Rationale**

1.  **M2 (Scale of 2):** Introduce with mathematical precision (half of 2 = 1)

2.  **M3 (Scale of 5):** Skip entirely - focus on skip-counting and scale efficiency

3.  **M4 (Scale of 10):** Reinforce with continuous bars (half of 10 = 5)

**Why this works:**

-   Two mathematically clean teaching moments

-   M3 complexity reduced (scale increase only)

-   Students apply learned concept in new context (bars vs symbols)

**Module Order Rationale (M9 ↔ M10 Swap)**

NEW ORDER: M9 Factor Fluency → M10 Equations

-   WHY:

    -   Factor fluency (M9) builds automaticity with 2s, 5s, 10s before solving unknowns

    -   Reduces cognitive load in M10: students have automatic recall for 3×☐=15 when they know 3×5=15

    -   Graph connection (M9) happens before variety expansion (M10) - Vocabulary scaffolding: \"factors\" and \"product\" introduced in M9, applied in M10

    -   = sign flexibility can be emphasized throughout M10, not rushed at end

**CROSS-MODULE PATTERNS**
-------------------------

### **Major Scaffolding Arcs**

**M1-M6: Graph Scaffolding**

-   Progression: Read → Create (partial) → Create (full) → Choose scales → Use for problems

-   Pre-completion decreases: 3→2→0 categories filled (no 1-of-4 step)

-   Animation support: Auto → Optional → Removed

-   Scale progression: 1:1 → 1:2 (with half-symbols) → 1:5 (multiples only) → 1:10 (half-symbols reinforced) → Choice

-   **Half-symbol arc:** Introduced M2 (scale 2) → Skipped M3 → Reinforced M4 (scale 10)

-   **Mathematical precision:** Half of 2 = 1, Half of 10 = 5 (both clean)

-   Purpose bridge: Creates authentic context for multiplication introduction

-   Real-world scenarios make abstract math concepts meaningful

-   Students see \"why\" for skip-counting and equal groups before formal instruction

**M7-M10: Symbolic Notation Introduction**

-   M7: Visual only, guide SHOWS notation (students see ×)

-   M8: Expressions, students CREATE notation (3 × 4)

-   M9: Factor fluency + graph connection (consolidation)

-   M10: Complete equations with variety (3 × 4 = 12, unknowns, variety)

-   Contexts: Consistent (M7-M8) → Fluency focus (M9) → Variety explicit (M10)

-   = sign: Introduced in M10 with flexibility from start (both orientations)

**M11-M12: Spatial Organization & Properties**

-   M11: Array structure (rows AND columns both taught)

-   M12: Array properties (commutativity via highlighting + rotation)

-   Contexts: Concrete → Abstract (dots) in M11; Primarily abstract in M12

-   Interpretation: Both valid (M11) → Proven equivalent (M12)

### **Consistent Design Elements**

**Warm-Ups:**

-   Always conceptual engagement

-   Visual, non-failable

-   Bridge from prior module explicit

-   Interest-driven where possible

**Lesson Activities:**

-   Grouped by scaffolding level (Early/Mid/Late)

-   Progressive complexity within each module

-   Multiple interactions per concept

-   Guide role decreases over activities

**Synthesis:**

-   Always includes consolidation

-   Often includes cross-module connection

-   Always includes bridge to next module

-   Reflection or discussion component

**Scaffolding Removal:**

-   Gradual across activities (not sudden)

-   Animation: Auto-play → Replay → Removed

-   Pre-completion: More → Less → None

-   Guide: Narrates → Prompts → Minimal

**SCAFFOLDING PHILOSOPHY ACROSS UNIT**
--------------------------------------

### **Early Modules (M1-M3): Building Foundations**

-   Heavy scaffolding (partial completion, calculation support)

-   Consistent structures (same graph types repeatedly)

-   Explicit teaching of new concepts

-   Guided practice dominates

### **Middle Modules (M4-M6): Developing Independence**

-   Moderate scaffolding (fewer pre-filled elements)

-   Increased variety (both orientations, multiple scales)

-   Strategic decision-making introduced

-   Balance of guided and independent practice

**Bridge Modules (M7-M10): Connecting Ideas**

-   Consistent contexts to focus on NEW concepts (M7-M8: equal groups, expressions)

-   Consolidation with familiar factors (M9: 2s, 5s, 10s fluency + graph connection)

-   Then explicit variety introduction (M10: equations with diverse contexts)

-   Multiple input methods for differentiation

-   Discovery balanced with explicit teaching

-   = sign flexibility emphasized throughout M10

-   Fluency foundation (M9) supports problem-solving (M10)

**Advanced Modules (M11-M12): Deepening Understanding**

-   Minimal scaffolding (independent work expected)

-   Full variety of contexts (concrete → abstract)

-   Both interpretations validated (rows AND columns)

-   Property discovery through multiple proofs (highlighting + rotation)

-   Student agency in mathematical thinking

-   Precision in language-to-expression matching

**ORIENTATION MIXING STRATEGY**

M1: Both orientations equally (build flexibility from start)

M2-M3: Both orientations throughout (flexibility reinforced during scale introduction)

-   Interpretation questions work with both orientations

-   Students practice reading graphs regardless of layout

M4: Both orientations in observation phase (Activities 1-2), then practice both

-   Transformation animation shown for both orientations in warm-up

-   Students create bars in both horizontal and vertical layouts

M5: Both orientations strategically (students choose appropriate format)

-   Scale selection practice includes both layouts

-   Orientation becomes one of many design choices

M6: Both orientations mixed (full flexibility before multiplication units)

-   Problem-solving works equally well with both

-   No orientation preference established

M7-M8: Context-focused (bags/boxes), orientation not emphasized

-   Equal groups and expressions don\'t have inherent orientation

-   Spatial layout less critical than structure understanding

M9: Horizontal emphasis when connecting to graphs (visual bridge clearer)

-   Picture graphs callback uses horizontal layout

-   Connection: \"These scales were equal groups!\"

-   Fluency practice can use any layout

M10: Orientation less relevant (equations focus)

-   Variety of contexts don\'t have fixed orientations

-   Expression and equation work independent of spatial layout

M11-M12: Arrays have inherent orientation (rows vs columns matter)

-   BUT both interpretations are valid

-   Rotation changes orientation (critical for property)

-   Horizontal/vertical distinction matters for row/column language
