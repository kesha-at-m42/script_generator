### Open Questions / Flags

1. **InterpretHalfScale as standalone vs embedded:** M1 backbone embedded comparison within read/create. Here, InterpretHalfScale is a spine-designated skill with its own INT status. Should half-symbol placement get dedicated templates, or should it be a dimension within CreatePicGraph templates? **Recommendation:** Dedicated templates — the spine treats it as a distinct node, and the Toy Flow gives it an explicit teaching moment with animation. Folding it in would lose the ability to track mastery of half-symbol understanding independently.

2. **Even-only vs mixed tiers:** The Toy Flow explicitly scaffolds even-only → first odd → multiple odd. This maps cleanly to confidence → baseline → stretch for CreatePicGraph, but InterpretHalfScale by definition requires odd numbers. InterpretHalfScale confidence tier should use the simplest odd case (single odd value, animation available).
