### Misconception Targeting

| Misconception | ID | Relevant Skills | Priority | Template Coverage |
|---------------|----|-----------------|---------|--------------------|
| Rows/Columns Confusion | 8 | BuildArray | PRIMARY | Student confuses rows (horizontal) and columns (vertical). MC distractor: reversed row/column labels. Template should show array and ask "How many rows?" with distractor = column count. |
| Array Dimensions as Addition | 9 | BuildArray | PRIMARY | Student adds row count + column count. "3 rows of 4" → writes "3 + 4 = 7" instead of "3 × 4 = 12." MC distractor: sum of dimensions. |
| Groups vs. Size Confusion | 10 | BuildArray | SECONDARY | Carries over from equal groups. "3 rows of 4" → writes "4 × 3." MC distractor: reversed expression. |

### Open Questions / Flags

1. **Real-world object constraints:** Egg cartons are 2×6 or 2×3 IRL. Muffin tins are 2×6 or 3×4. Dot arrays have no constraint. SME flagged that egg cartons/muffin tins "aren't real world" at non-standard sizes. Templates using concrete objects should respect realistic configurations.

2. **Column de-emphasis vs equal treatment:** SME debate about whether columns should get equal attention. "Some students will naturally picture columns." The TF teaches both from start. Templates should offer both but not penalize row-first convention.

3. **Convention vs flexibility tension:** Mathematical convention says "rows first" for reading expressions (5 × 2 = 5 rows of 2). But DESCRIBING an array, either perspective is valid. Templates testing expression READING should use row convention. Templates testing array DESCRIPTION should accept either.
