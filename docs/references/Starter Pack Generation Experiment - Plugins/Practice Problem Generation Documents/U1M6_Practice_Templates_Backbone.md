### Open Questions / Flags

1. **SolveMultiStepData is single-module (M6 only):** Like SelectScale, all tiers within one module. The scaffolding progression (full prompts → partial → none) maps directly to confidence → baseline → stretch.

2. **Creation as context-setting:** Activities 1-3 and 7-9 have students CREATE a graph THEN answer questions about it. This is a two-part interaction. Templates could model this as: creation step (lower tier) → problem-solving step (higher tier) within the same template, or separate creation templates from problem-solving templates.

3. **One-step vs two-step identification:** Activities 10-12 mix one-step and two-step WITHOUT labeling. This is an important metacognitive skill — "Is this one step or two?" Consider stretch/challenge templates that require students to identify problem structure before solving.

4. **Struck-through content in TF:** The TF has struck-through sections for an earlier version of M6 (scale selection focused). The current version is problem-solving focused. Use only non-struck content.
