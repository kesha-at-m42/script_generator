### Open Questions / Flags

1. **Square arrays as special case:** 4×4 rotated is still 4×4. This is a special case of commutative property — same expression both ways. Include 1-2 templates that surface this: "What happens when you rotate a 4×4 array?"

2. **Property naming:** SME prefers "you can multiply in any order" over "turn-around facts" or "commutative property." Templates should use this language. The formal name "commutative property" can appear at stretch/challenge tier.

3. **Strategic application depth:** Activities 6-8 ask "Which way is easier to count?" This requires metacognitive judgment. Templates at stretch/challenge should include strategy questions, not just "what's the answer?"

4. **M12 as unit capstone:** This is the final module. Synthesis references the full unit journey. Consider a small number of challenge templates that bridge back to data domain: "An array of 5 rows of 4 shows the same total as a scale-5 graph with 4 symbols. Both show 5×4=20."
