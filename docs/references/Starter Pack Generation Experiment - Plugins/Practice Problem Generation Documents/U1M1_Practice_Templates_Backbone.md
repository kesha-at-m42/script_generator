## §PT.1 Source Analysis

### Toy Inventory

*From SP §1.5. Only toys available in Practice are listed with full detail.*

**Picture Graph — Mode 1: Reading (PRACTICE-AVAILABLE)**

- **Orientation:** Both horizontal and vertical
- **Scale:** 1:1 only. Key on every graph: "Each [symbol] = 1 [item]"
- **Categories:** 3–4 per graph (Lesson/EC range)
- **Values:** 3–10 per category
- **Interactions:** Hover → value displays. Click category → highlight. Click two categories → compare (difference displays).

**Bar Graph — Mode 1: Reading (PRACTICE-AVAILABLE)**

- **Orientation:** Both vertical and horizontal
- **Scale:** 1:1 only. Axis counts by 1s, tick marks at each whole number.
- **Categories:** 3–4 per graph
- **Values:** 3–10 per category
- **Interactions:** Click bar → highlight. Click two bars → compare (difference displays). Helping line animation (system-triggered).

**Data Table — Display Only (PRACTICE-AVAILABLE when specified)**

- **Structure:** 2-column (Category name | Value). Not clickable.
- **Visibility:** Per-interaction specification. Pattern for Practice: visible in confidence/support tiers (scaffold), hidden in baseline+ (independent reading).
- **Cross-Toy Highlighting:** App-controlled animation, not student-triggered.

**Picture Graph — Mode 2: Creating (NOT in Practice)** — Warmup only.

**Data Collection Game (NOT in Practice)** — Warmup only.

> **Interaction Constraints (All Toys):** NO verbal responses. NO keyboard/text input. All responses via click/tap/drag or MC selection. Questions answered through on-screen action.

### Constraint Extraction

*From SP §1.2 Scope Boundaries.*

**IN SCOPE:**
- Reading picture graphs (1:1 scale, both orientations)
- Reading bar graphs (1:1 scale, both orientations)
- Graph components: title, categories, key, scale, labels
- Comparison language: "most," "least," "fewer," "fewest"
- Problem-solving language: "how many more," "how many fewer," "in all," "total"
- Explicit: "how many more/fewer" = compare = subtract
- Explicit: "in all" = combine = total = add
- Orientation vocabulary: "horizontal," "vertical"

**OUT OF SCOPE:**
- Scaled graphs where symbol > 1 (M2+)
- Half-symbols or fractional representations (M2+)
- Scale selection or scale comparison (M5)
- Graph creation in Practice (Warmup only)
- Line plots (separate unit)
- Formal "axis/axes" vocabulary (M4)

### Vocabulary

**Practice-Available Terms** (all terms introduced through Lesson Sections 1–3): bar graph, picture graph, data, category, scale, key, title, label, vertical, horizontal, most, least, total, difference, compare, combine, fewer, fewest

**Terms to Avoid:** "scaled graph" (M2), "each symbol represents [>1]" (M2), "half-symbol" (M2), "skip counting" in graph context (M3), "axis/axes" (M4)

**Notes on decomposition:**

**S3 (most/least) is a distinct skill, classified as COMPARE.** The SP §1.8.5 folds it into S1/S2, but the Lesson teaches it explicitly across 5 interactions (1.1, 1.2, 2.2a, 2.3, 2.4) and the EC tests it directly (EC.2). The SP's own EC alignment table classifies EC.2 as cognitive type COMPARE — finding "most" or "fewest" requires evaluating a relationship across all categories, not just reading a single value. Keeping it separate enables targeted templates.

**Two-step problems are stretch/challenge variants of S4 and S5,** not a separate skill. The student isn't learning a new cognitive operation — they're chaining S5 (combine) then S4 (compare), or vice versa, with sequential submissions. Stretch/challenge tier S4 and S5 templates can use two-step designs. This avoids inflating skill counts in every module that introduces multi-step problems.

**Selective data use (Lesson 3.4a) is folded into S4/S5 distractor design,** not a standalone skill. It was taught in a single metacognitive interaction — not enough to justify dedicated templates. Instead, S4 and S5 MC distractors should include "all data" answers that catch Misconception #17. The misconception_remediation template for #17 handles the remediation path.

**Metacognitive assessment opportunity:** Lesson 3.4a (multi-select: "which categories do you need?") could be assessed in Practice. Evaluated but not included as a standalone skill — single teaching moment, and #17 distractor design covers the same cognitive ground. If assessment data later shows #17 persists despite distractor coverage, a metacognitive template could be added.

**SP §1.8.5 Cross-Reference:** SP S1 = our S1 (+ S3 folded in), SP S2 = our S2 (+ S3 folded in), SP S3 = our S4, SP S4 = our S5. Our S3 is the only addition from the decomposition framework.

### Misconception Coverage

| ID | Name | Priority | Observable Behavior | Template Strategy |
|----|------|----------|-------------------|-------------------|
| 16 | Graph as Picture | **PRIMARY** (SP §1.4.1) | Student focuses on visual appearance rather than data values. Describes graph shapes ("it goes up") instead of data meaning ("more people chose cats"). | Detection in S1, S2 standard templates. **1 dedicated misconception_remediation template.** |
| 17 | All Data Must Be Used | **SECONDARY** (SP §1.4.2) | When asked about specific categories, student incorporates ALL categories into answer. | Detection in S4, S5 templates via "all data" MC distractors. **1 dedicated misconception_remediation template.** |
| 6 | "More Than" Means Add | **ADDRESSED** (SP §1.2 teaches explicitly; not in SP §1.4) | For "How many more chose A than B?" adds values instead of subtracting. Lesson evidence: 3.1 distractor 14 (8+6), 3.2 distractor 13 (8+5), 3.4b distractor 11 (7+4). | Detection in ALL S4 templates — sum of values as MC distractor. Detection in S5 templates — difference as MC distractor (the inverse: student subtracts when they should add). **1 dedicated misconception_remediation template.** |

**Remediation template designs:**

**#16 (PRIMARY) — Graph as Picture:** Graph where visual appearance contradicts data. For example, a bar graph where two bars look similar in height but values differ by 3+. Student must read the actual value, not judge by appearance. Uses confidence/support tier parameters so computation isn't a barrier.

**#17 (SECONDARY) — All Data Must Be Used:** Question asks about 2 specific categories from a 4–5 category graph. Error answer = sum or operation on ALL categories. Correct answer clearly involves only the named categories. Parameters chosen so the "all data" answer is visibly different from the correct answer.

**#6 (ADDRESSED) — "More Than" Means Add:** Two manifestations, both detected:
- *In S4:* "How many more A than B?" — student adds instead of subtracting. Distractor = sum (A+B). Example: A=8, B=2, correct=6, distractor=10.
- *In S5 (inverse):* "How many cats and dogs in all?" — student subtracts instead of adding. Distractor = difference (A−B). Example: A=7, B=3, correct=10, distractor=4. Same underlying confusion about which operation comparison-adjacent language requires.
- *Remediation template:* Uses parameters where sum and difference are obviously different. For example, A=8, B=2: difference=6, sum=10.

**Remediation structure (per RDR v3):**

All M1 templates use MC interaction → **MC Remediation Track** (RDR §3): per-distractor Medium branching + shared Heavy with `[Modeling]`. Exception: EC.2 uses click-to-select (non-MC track, RDR §2). Confidence-tier templates use Light only (RDR §11).

Each template defines a **Distractor Types pool** (4–6 types) specifying the conceptual error each wrong-answer category represents and a Medium remediation direction per type. The system selects 3 distractors from the pool per problem instance. Distractor types draw from SP Answer Rationale entries (EC.1, EC.3, EC.4 document distractor logic) and Misconception Table Observable Behaviors.

**Remediation content sources (in authority order):**
1. **RDR** — Structural authority: track, escalation, word counts, `[Modeling]` placement, post-modeling language, validator tags
2. **SP §1.4** — Content authority for #16 and #17: prevention strategies ("What does this graph TELL us?" not "What do you SEE?"; model question analysis; visually dim irrelevant categories)
3. **SP §1.7 Lesson Answer Rationales** — EC.3 distractors (used Dan's value only, used Ava's value only, added instead of subtracted); EC.4 distractors (used Cara's value only, subtraction/misread, included additional category). These are validated distractor patterns.
4. **SP §1.2** — "how many more/fewer = compare = subtract"; "in all = combine = total = add"
5. **SP §1.5.2** — Helping line animation (system-triggered visual scaffold for bar graph reading)
