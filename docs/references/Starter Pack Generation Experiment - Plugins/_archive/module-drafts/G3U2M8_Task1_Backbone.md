## 1.0 THE ONE THING

**Testable Core:** By the end of M8, students can **find the area of a rectangle from its dimensions by multiplying** (forward) AND **identify multiple sets of side lengths that produce a given area** (reverse), demonstrating that multiplication for area works in both directions.

**CRA Stage: Abstract (Application/Transfer).** M8 is fully at dimensions-only level — no grids, no tick marks. Students consolidate M7's abstraction into fluency (forward) and extend it in a new cognitive direction (reverse). This is not a new CRA stage but a deepening of Abstract through bidirectional reasoning.

**Critical Misconception: #5: Not Multiplying Side Lengths** (PRIMARY, HIGH PRIORITY)
- **Observable Behavior:** Student reads dimension labels correctly but adds them instead of multiplying (e.g., "7 + 4 = 11" for a 7 ft × 4 ft rectangle), states dimensions as two separate numbers without calculating, or omits the square unit from the answer. In reverse problems, student guesses dimensions randomly rather than using multiplication reasoning to verify that selected factor pairs produce the target area.
- **Why It Persists into M8:** Students who didn't fully internalize M7's dimension-multiplication connection during grid fading may have developed surface-level compliance (e.g., multiplying because the Equation Builder required it) without conceptual understanding. M8's extended dimensions-only practice — without any grid scaffold — exposes whether the connection is real or procedural.
- **Surfaces:** Forward problems (Early/Late phases); reverse problems where students select pairs that don't multiply to the target area (Mid phase)

**Success Indicator:** Given a rectangle with dimensions labeled (no grid), the student independently multiplies the side lengths and states the area with the correct square unit. Given an area, the student identifies at least two valid sets of side lengths from MC options and can explain that both produce the target area through multiplication. The student can do both forward and reverse problems in text-only format with no visual support.

**Biggest Risk: Reverse problems become guessing games.** If students don't connect reverse problems to multiplication reasoning — if they just pick number pairs that "look right" rather than verifying that the factors produce the target area — M8's core insight (one area → many rectangles through factor pairs) collapses into random selection. The instructional design must require verification (student sees rectangle appear with selected dimensions, confirming the area) and build toward systematic factor-pair thinking (seed for M10).

## 1.2 SCOPE BOUNDARIES

### ✅ Must Teach

- **Forward area fluency (dimensions-only):** Extended practice multiplying labeled dimensions to find area, across all four unit types (sq cm, sq in, sq ft, sq m). No grid support. Both visual (rectangle outline + dimension labels) and text-only formats. This consolidates M7's abstraction into automaticity.
- **Reverse reasoning (area → dimensions):** Given an area, students identify possible sets of side lengths. Students discover that one area maps to multiple rectangles (e.g., area 24 → 3×8, 4×6, 2×12). Factor-pair reasoning introduced.
- **Commutativity in context:** Explicit check that 3×4 and 4×3 represent the SAME rectangle (rotation equivalence), not two different rectangles. One dedicated problem addresses this.
- **Text-only problems (forward AND reverse):** Both problem types use the same text format so visual presentation does not signal problem type. Students must read to determine direction.
- **Vocabulary formalization:** length, width (used casually since M4, now central to text-only problem structure). Factor pair (new compound term — students know "factor" and "product" from M4/M7).
- **All four unit types:** Square centimeters, square inches, square feet, square meters — used freely across all phases.
- **Shape-reasoning extension (STRETCH tier, per TVP Edit 84):** 1-2 problems in Late phase asking "Which rectangle would be longer and skinnier? Which would be closer to a square?" — connects factor pairs to geometric intuition, previews M9 reasoning about which rectangle works best for a context. Tagged STRETCH, not BASELINE — counts toward mastery but does not penalize students still solidifying core forward/reverse skills.

### ❌ Must Not Include

- **Grids, partial grids, or tick marks as instructional tools** — M8 is fully abstract per Decision #3. Rectangle outlines with dimensions only. Exception: visual confirmation when student selects factor pair (displays rectangle with those dimensions), which is verification, not scaffold.
- **Perimeter concept** — Per Decision #5. If student adds instead of multiplying, address through multiplication reasoning, not perimeter contrast.
- **Ruler measurement** — Dropped from OUR L9. Not in engineering specs. No downstream dependency.
- **Composite figures** — Simple rectangles only. Composite figures begin at M11.
- **New units** — All four unit types established in M5-M6. No new units.
- **Open-ended text input** — All responses via click/tap/drag or MC selection. No typed answers.

## 1.3 VOCABULARY ARCHITECTURE

**Assessment Vocabulary (appears on state tests and Exit Check):** area, side length, dimensions, length, width, multiply, factor, product, square inch, square centimeter, square foot, square meter

**FORMALIZED in M8:** length, width (used casually since M4; now central to text-only problem structure where students parse "[number] [unit] long and [number] [unit] wide")

**INTRODUCED in M8:** factor pair (compound term — students know "factor" and "product" from M4/M7; "factor pair" names two factors that produce a given product; central to reverse problem reasoning)

**REVIEW from M7:** side length, dimension, area, square inch, square centimeter, square foot, square meter, multiply, factor, product

### Vocabulary Staging by Phase

| Phase | Terms | Introduction Approach |
|---|---|---|
| **Warm-Up** | area, dimensions, multiply (natural use) | No formal introduction; familiar terms in familiar context. Students calculate area of two rectangles — vocabulary is assumed from M7. |
| **Lesson Early (Forward)** | length, width (formalized), all unit types | After 2 forward problems establish fluency, formalize length/width: "The length tells you one side. The width tells you the other. You multiply them to find area." All four unit types used naturally in problem contexts. |
| **Lesson Mid (Reverse)** | factor pair (introduced), length, width (reinforced) | After first reverse problem, introduce "factor pair": "3 and 8 are a factor pair for 24 — they multiply to make 24. 4 and 6 are ANOTHER factor pair for 24." |
| **Lesson Late (Mixed)** | All terms consolidated | Students encounter all vocabulary in text-only problems. "Length," "width," and "factor pair" used in both Guide dialogue and problem stems. |
| **EC** | All assessment vocabulary | Terms appear in problem stems and answer options. Factor pair not directly named in EC stems but the concept is tested (selecting valid dimension pairs). |
| **Synthesis** | All terms — vocabulary review | "The side lengths are the factor pair. The area is the product. You can go forward — factor pair to product. Or reverse — product to factor pairs." |

### Vocabulary Notes

**Length and Width:** The two dimensions of a rectangle. Either side can be called "length" — what matters is multiplying them. In M8's text-only problems, "long" maps to one dimension and "wide" maps to the other: "A rug is 7 feet long and 4 feet wide." The transition from "side length" (M7) to "length and width" (M8) reflects the shift from dimension labels on a visual to dimensional language in text.

**Factor Pair:** Two numbers that multiply to make a given product. M8's reverse problems center on finding factor pairs for a given area. Example: For area 24, the factor pairs are (1,24), (2,12), (3,8), and (4,6). The term "factor pair" connects M8's reverse reasoning to M10's multiplication table (where every cell is a product and the row/column headers are the factor pair).

**Commutativity Convention:** 3×8 and 8×3 are the SAME factor pair (same rectangle, rotated). M8 treats these as equivalent, not separate. This is explicitly taught through a dedicated commutativity check problem.

### Terms to Avoid (Save for Later Modules)

- **"Formula"** (standalone) — Suggests rote memorization. Use: "Multiply the dimensions to find area." (Same constraint as M7.)
- **"Perimeter"** — Do not use, even for contrast. Per Decision #5.
- **"Decompose"** — Not applicable to M8. Composite figures and decomposition begin in M11.
- **"Feet squared," "Meters squared"** — Ambiguous; use "Square feet," "Square meters." (Same constraint as M7.)
- **"Equation"** — Students build multiplication expressions, not equations. The Equation Builder is a tool name, not a vocabulary term for students.

## 1.4 MISCONCEPTIONS

### 1.4.1 #5: Not Multiplying Side Lengths (PRIMARY)

**Trigger Behavior:** Student reads dimension labels correctly but does not multiply them. Observable patterns: (1) Adds dimensions instead ("7 + 4 = 11, so the area is 11 square feet"); (2) States dimensions as two separate numbers without calculating ("The area is 7 and 4"); (3) Multiplies correctly but omits the square unit ("28" instead of "28 square feet"); (4) In reverse problems, selects dimension pairs that don't multiply to the target area (indicating multiplication is not the reasoning mechanism).

**Why It Persists into M8:** M7 taught the dimension-multiplication connection through grid fading, but some students may have achieved surface-level compliance (multiplying because the Equation Builder required it) without deep understanding. M8's extended dimensions-only practice — without any grid scaffold or Equation Builder template (Late phase) — exposes whether the connection is conceptual or procedural. The reverse direction (area → dimensions) particularly tests this: students who don't think in terms of multiplication will guess randomly.

**Prevention Strategy:**
1. **Forward consolidation (Early):** 4 dimensions-only problems across all unit types. Students who add instead of multiply are caught immediately. Error-specific feedback loops to M7's vocabulary anchor: "The dimensions are the factors. Multiply them."
2. **Equation Builder scaffolding (Mid):** Reverse problems use `___ × ___ = [area]` template, forcing the multiplication frame. The blank slots and × symbol make multiplication the expected operation.
3. **Visual verification (Mid):** When student selects a factor pair, a rectangle with those dimensions appears. This concrete confirmation bridges abstract factor-pair reasoning back to the visual — students SEE that their multiplication produces a real rectangle.
4. **Text-only independence (Late):** All problems text-only, no visual or EB support. Students must extract dimensions from text AND multiply. This is the highest-demand test of the dimension-multiplication connection.
5. **Vocabulary anchor (Throughout):** "The dimensions are your factors. Multiply them to find the product, which is the area." Repeated from M7.

**Anti-Pattern Detection:** Student's calculation includes addition of dimensions, repetition of separate numbers without multiplication, correct multiplication but omitted unit, or (in reverse) selection of factor pairs that don't produce the target area. These indicate the dimension-multiplication link has not been fully constructed.

---

### 1.4.2 Module-Specific: Picking Numbers Without Understanding (SECONDARY)

**Trigger Behavior:** In reverse problems (given area, find dimensions), student selects dimension pairs without verifying they multiply to the target area. Observable patterns: (1) Selects pairs with one factor matching a correct pair but wrong second factor (e.g., for area 24, selects 3×7 because 3 appears in the correct pair 3×8); (2) Selects pairs that are arithmetically close to the target (e.g., 5×5=25 instead of 4×6=24); (3) Selects only one pair when multiple valid pairs exist; (4) Cannot explain WHY selected dimensions give the target area.

**Why It Happens:** Reverse reasoning is new. Students have spent M1-M7 going forward (dimensions → area). The reverse direction (area → dimensions) requires a different cognitive strategy: "What two numbers multiply to make ___?" Students without strong multiplication fluency may not have ready access to factor pairs, leading to trial-and-error or guessing.

**Prevention Strategy:**
1. **Equation Builder as verification frame (Mid):** The `___ × ___ = [area]` template makes the verification explicit — students can CHECK their selection by seeing if the equation is true.
2. **Visual confirmation (Mid):** Selected factor pair produces a rectangle on screen. This grounds abstract number selection in concrete geometric reality.
3. **"Are there any others?" prompt (Mid):** After students find 2+ pairs, Guide asks: "Are there any others? How could you check?" This plants the seed for systematic factor-pair thinking without requiring it.
4. **Distractor design (MC):** Distractors must NOT share factors with correct answers (per TVP Edit 83). This prevents anchoring on a familiar factor and guessing the second.
5. **Mixed practice (Late):** Students encounter both forward and reverse problems in the same format, requiring them to determine the problem type AND apply the appropriate reasoning direction.

**Anti-Pattern Detection:** Student consistently selects exactly one correct pair out of multiple valid options, suggesting they stop after finding one rather than checking for others. Or student selects pairs where one factor is valid but the other produces a product ≠ target area.

---

### 1.4.3 #3: Confusing Area and Perimeter (LATENT — Monitor Only)

**Trigger Behavior:** Student adds side lengths when asked for area. Per Misconceptions database: "When asked for area, student adds side lengths; may surface if students have prior exposure to perimeter."

**Why It's Listed:** M8's text-only problems use "length" and "width" language, which could trigger perimeter associations in students with prior exposure to perimeter from other contexts. Per Decision #5, we do NOT teach perimeter and do NOT proactively address this confusion.

**Prevention Strategy:** If a student adds dimensions instead of multiplying, address through multiplication reasoning: "We multiply to find area, not add." Do NOT introduce perimeter language to explain the difference. M8's consistent use of "multiply the dimensions" serves as passive prevention.

**Priority:** LOW (only if student-introduced). Not a design target for M8.

### Data Constraints

**Forward problems:**
- Factors: 2–10 each
- Products: ≤ 100
- All four unit types represented across the phase
- Examples: 7×4=28 sq ft, 8×5=40 sq cm, 9×3=27 sq m, 6×6=36 sq in

**Reverse problem stem constraint (load-bearing, per TVP Edit 91):** All reverse problem prompts MUST specify "whole number dimensions" or "whole number side lengths." This closes the fractional answer loophole — without it, a student could answer "area 24 → 2.4 × 10" which is valid but outside M8's scope.

**Reverse problems — target areas with multiple clean factor pairs:**

| Target Area | Valid Factor Pairs (unique, both factors ≤12) | Notes |
|---|---|---|
| 12 | (1,12), (2,6), (3,4) | 3 non-trivial pairs |
| 18 | (1,18), (2,9), (3,6) | Good for initial reverse practice |
| 20 | (1,20), (2,10), (4,5) | |
| 24 | (1,24), (2,12), (3,8), (4,6) | Best for key beat — 4 non-trivial pairs |
| 30 | (1,30), (3,10), (5,6) | 2×15 excluded (factor 15 beyond useful range) |
| 36 | (1,36), (3,12), (4,9), (6,6) | 2×18 excluded (factor 18 beyond useful range); includes perfect square |

**Factor cap:** Both factors ≤ 12. 1×n always included (trivial multiplication). Factors 13+ excluded — beyond useful 3rd grade range.

**1×n factor pairs (per TVP Edit 91):** Include in multi-select MC options alongside all other valid pairs. Selecting 1×n is correct. Seeing 1×n validated as a correct selection normalizes it and reinforces that 1 is a legitimate factor. 1×n should NOT be the only correct option for any problem — choose target areas with multiple non-trivial factor pairs.

**Mastery tracking / problem-type tagging (per TVP Edit 85 — to be documented fully in §1.8.5 Practice Inputs):**
- Forward application (dimensions → area) = BASELINE
- Reverse (area → factor pairs) = BASELINE
- Missing-dimension ("area is 28, width is 4, find length") = TRANSFER (APPLY verb)
- Shape comparison of factor pairs ("longer and skinnier? closer to a square?") = STRETCH
- Design problems = STRETCH
- Multi-step problems = STRETCH
- Constraint problems = CHALLENGE (bonus only)

**Avoid:**
- Prime-number areas (7, 11, 13, 17, 19) — only one non-trivial factor pair, defeats the purpose of multi-select
- Areas > 50 — too many factor pairs, overwhelming for 3rd graders

**Reverse problem MC distractors:**
- Must have products ≠ target area
- Must NOT share a factor with any correct answer (per TVP Edit 83)
- Should be plausible multiplication facts, not random numbers
- Example for area 18: correct pairs are (1,18), (2,9), (3,6) — correct factors are {1, 2, 3, 6, 9, 18}. Distractors must use factors OUTSIDE that set: (4×5)=20 ✓, (5×7)=35 ✓. NOT (2×7)=14 — shares factor 2 with correct pair (2,9), enabling anchoring.

**Text-only problem template:** "A [object] is [number] [unit] long and [number] [unit] wide. What is its area?" (rigid template from M7, used for both forward and reverse in Late)

### Interaction Constraints

- Student cannot drag/manipulate rectangles (display only)
- Student can tap/click to enlarge for clarity (accessibility)
- Student responds via MC selection (forward: select area value + unit; reverse: multi-select factor pairs) or Equation Builder (Mid only)
- Side-by-side rectangle comparisons are view-only — no student action on the comparison display itself

### 1.5.3 Secondary Tools

**Multiple Choice (MC) — All Phases**
- **Forward problems:** Standard MC — student selects area value with correct unit from 4 options. Distractors include: addition of dimensions, wrong unit, partial calculation, off-by-one multiplication error.
- **Reverse problems (Multi-select MC):** "Select ALL sets of WHOLE NUMBER side lengths that give an area of [target]." Options include all valid unique factor pairs (both factors ≤12, plus 1×n) alongside 2-3 distractors. Number of correct answers varies by target area — do NOT tell students how many to find. Scoring: all correct with no incorrect = full credit; some correct with no incorrect = partial credit; any incorrect = incorrect.
- **Commutativity check:** Standard MC (single-select) — 4 text options, student selects one. ⚠️ No dichotomous (Yes/No/Agree/Disagree) framing — each option must be a standalone claim requiring reading to evaluate (per TVP Edit 89). See TVP for reference options.
- **Text-only problems (Late/EC):** MC for forward problems; for reverse, MC with dimension pair options.

- No rulers (dropped from OUR L9)

### Core Purpose

**Purpose:** Activate M7 dimensions-only fluency and surface the one-to-many relationship (one area → multiple rectangles) before the Lesson formally introduces reverse reasoning.

**Key Function:** This Warmup positions the Lesson by presenting students with two different rectangles that have the SAME area — a concrete experience of the key insight M8 will formalize. Students calculate both areas using familiar forward multiplication, then encounter the surprise: different dimensions, same answer. This creates the conceptual question ("How is that possible?") that the Lesson's reverse reasoning answers.

**Why this serves the concept:**

- **Activates forward fluency from M7:** Students multiply dimensions to find area — the exact skill M8 consolidates — using the dimensions-only display they mastered in M7. This confirms readiness for the Lesson's extended forward practice.
- **Surfaces the one-to-many relationship without formalizing it:** Students discover that 3×8 and 4×6 both equal 24 through their own calculation, creating genuine surprise. The Lesson's reverse reasoning ("given an area, what could the dimensions be?") directly extends this discovery.
- **Establishes the visual context for Lesson key beats:** Side-by-side rectangles with the same area appear again in Lesson Mid (after factor-pair identification) and Synthesis. The Warmup's first exposure to this display primes students to interpret it meaningfully later.

**Test:** If we removed this Warmup and went straight to the Lesson, students would lack the visceral "wait, those are DIFFERENT but the answer is the SAME?" moment that makes reverse reasoning feel motivated. They'd encounter reverse problems as an arbitrary new task rather than the natural extension of a genuinely surprising observation.

### Warmup Type Rationale

**Selected Type:** Visual Pattern (adapted — comparison discovery variant)

**Why not Number Talk?** A pure Number Talk ("What's 4 × 6?") would activate multiplication fluency but miss the geometric dimension — M8's insight is that one AREA maps to multiple RECTANGLES, not just that one product has multiple factor pairs. Students need to SEE two different shapes and notice they share an area. The visual is load-bearing.

**Why not straight Binary Choice?** Binary Choice ("Which rectangle has area 24?") tests recognition but doesn't create the surprise of discovering the match through independent calculation. The Warmup needs students to compute BOTH areas before they see the connection.

**Adapted structure:** Two rectangles displayed side-by-side with different dimensions. Students calculate each area independently (two interactions). The surprise — same answer — emerges from their own work. Bridge asks the question the Lesson answers.

*Design Note: This is a Type A interaction (teaching only, no prompt). **Interaction 2.5: "Are There Any Others?" [WORKED EXAMPLE]**

* **Purpose:** Required TVP guide beat. After seeing 4 factor pairs for 24, Guide asks: "Are there any others? How could you check?" This plants the seed for systematic factor-pair thinking (M10 prerequisite) without requiring students to answer.

*Design Note: This is a Type A teaching-only interaction. **Interaction 2.6: Commutativity Check [CONCEPTUAL CHECK]**

* **Purpose:** Explicitly address that 3×8 and 8×3 are the SAME rectangle (rotation equivalence, not two different rectangles). One dedicated problem per §1.2.
* **Visual:** Text display. "Kiran found two rectangles with area 24: one is 3 × 8 and the other is 8 × 3. How many DIFFERENT rectangles did Kiran find?"
* **Guide:** "Kiran says they found two rectangles with area 24: one that's 3 by 8, and one that's 8 by 3. How many different rectangles did Kiran actually find?"
* **Prompt:** "Kiran says 3 × 8 and 8 × 3 are two different rectangles with area 24. How many DIFFERENT rectangles did Kiran actually find?"
* **Student Action:** MC selection (single-select)
  * **Options:** A) "Two — the dimensions are in different order, so they're different rectangles.", B) "One — turning the same rectangle doesn't make a new one.", C) "Zero — neither one has an area of 24.", D) "It depends on whether you turn it sideways."
* **Correct Answer:** B
* **Answer Rationale:**
  - B = Correct — 3×8 and 8×3 are the same rectangle rotated. Commutativity means a×b = b×a, same product, same shape.
  - A = Commutativity misconception — treating order as meaningful for rectangle identity
  - C = Computation error — both DO have area 24
  - D = Partially correct reasoning but not the best answer — it always produces the same rectangle regardless of orientation
* **On Correct:** "One rectangle. Turning it doesn't make a new one. 3 × 8 and 8 × 3 are the same factor pair."
* **Remediation:** Pipeline

*Design Note: Per Edit 89 — no dichotomous (Yes/No/Agree/Disagree) framing. All four options are standalone claims requiring reading to evaluate. Option D is the "nuanced wrong" — it hedges rather than committing, which tests whether students have internalized the commutativity principle.*

### Misconception Prevention

**#5: Not Multiplying Side Lengths (PRIMARY):**

* **Section 1 (Forward):** Interactions 1.2-1.5 all include an addition-error distractor (e.g., 13 for 8+5, 12 for 9+3). Pipeline remediation for these targets the "dimensions are factors — multiply them" anchor from M7. Worked example 1.1 explicitly models "what do I multiply? The two dimensions."
* **Section 2 (Reverse):** Equation Builder in 2.2 and 2.3 displays `___ × ___ = [area]`, making multiplication the visible frame. Students who select pairs that don't multiply to the target area are caught by the "product ≠ target" check in MC scoring. Visual confirmation (rectangle appearing with selected dimensions) provides concrete verification.
* **Section 3 (Text-only):** Addition distractors remain in 3.1 and 3.3. Text-only format is the highest-demand test — no visual or EB support to cue multiplication.
* **Cross-section:** "Multiply the dimensions" language appears in 1.1 (WE), On Correct for 1.2, and 2.7 (vocabulary consolidation).

**Module-Specific: Picking Numbers Without Understanding (SECONDARY):**

* **Section 2:** Multi-select MC with Edit 83 distractor rules prevents anchoring on familiar factors. Visual confirmation (rectangle appearing) grounds selection in geometry. "Are there any others?" beat (2.5) models systematic checking.
* **Section 3:** Unscaffolded reverse problems (3.2, 3.4) require students to generate factor pairs without EB template. Distractors are plausible multiplication facts with products ≠ target area.

**#3: Confusing Area and Perimeter (LATENT):**

* No perimeter language anywhere in the Lesson. If addition errors surface, remediation redirects to multiplication reasoning without mentioning perimeter. "We multiply to find area" serves as passive prevention.

## §1.8.5 PRACTICE INPUTS

### Practice Phase Overview

Practice is generated by the Pipeline (adaptive engine), not fully scripted in the SP. The SP provides inputs that constrain the Pipeline's problem generation. These inputs define what skill types, interaction modalities, dimension ranges, unit distributions, and mastery tags the Pipeline uses.

M8 Practice builds fluency in two directions: forward (dimensions → area) and reverse (area → factor pairs). Forward practice maintains the automaticity M9 depends on. ### Connection Task S.2: When Would You Go Forward? When Reverse? [Type C — Real-World Bridge]

* **Purpose:** Connect forward and reverse reasoning to real-world purposes. Students identify WHEN each direction is useful — moving from procedure to judgment.
* **Task Type:** Type C (Real-World Bridge) — linking abstract math to contexts.
* **Visual:** Text display with two scenarios:

> Scenario A: "Mia is painting a wall. The wall is 8 feet long and 3 feet wide. She needs to know how much paint to buy."
>
> Scenario B: "Jamal has 30 square tiles. He wants to make a rectangle. He needs to figure out what shapes are possible."

* **Guide:** "Two people, two problems. Which one goes forward, and which one goes reverse?"
* **Prompt:** "Which person needs to go FORWARD (dimensions to area), and which needs to go REVERSE (area to dimensions)?"
* **Student Action:** MC selection (matching)
  * **Options:** A) "Mia goes forward, Jamal goes reverse.", B) "Mia goes reverse, Jamal goes forward.", C) "Both go forward.", D) "Both go reverse."
* **Correct Answer:** A
* **Answer Rationale:**
  - A = Correct — Mia knows dimensions (8×3), needs area (paint coverage). Jamal knows area (30 tiles), needs dimensions (possible rectangles).
  - B = Reversed — student may be confused about which direction is which
  - C = Fails to recognize Jamal's problem as reverse
  - D = Fails to recognize Mia's problem as forward
* **On Correct:** "Mia knows the wall's dimensions, so she goes forward to find the area. Jamal knows the area, so he goes reverse to find the dimensions. Same math, two directions."
* **Connection:** Forward and reverse aren't just classroom exercises — they answer different real-world questions. "How much space?" = forward. "What shapes could work?" = reverse.
* **Remediation:** Pipeline (light redirect: "Mia already knows her wall is 8 by 3. Does she need to find the area, or find the dimensions?")

> **Design Note:** Type C (Real-World Bridge) connects abstract forward/reverse reasoning to concrete purposes. Mia's scenario is a classic forward problem (dimensions, find area for paint). Jamal's scenario mirrors Lesson 3.4 (Tyler's tiles) but with a different name and number. The MC format requires students to CLASSIFY the problems, not solve them — this is connection work, not more practice. Correct answer at position A. "Same math, two directions" is the takeaway phrase.

## §1.10 KEY DESIGN DECISIONS (KDD)

### Purpose

KDD documents the pedagogical design choices that make this module work. Each entry explains WHAT was decided and WHY it matters for student learning. A writer seeing this module for the first time should understand why it works this way from the KDDs alone.

---

### KDD-1: Dimensions-Only Throughout — No Grids, No Tick Marks

M8 is fully at the Abstract CRA stage per Decision #3. No grids, no partial grids, no tick marks appear in any phase from Warmup through Synthesis. Students multiply labeled dimensions to find area, relying entirely on the dimension-multiplication connection established in M7's grid fading. The one exception: when a student selects a factor pair in a reverse problem, a dimensions-only rectangle appears as visual confirmation — this is verification, not scaffold.

---

### KDD-2: Within-Lesson CRA Adaptation — Forward Consolidation Before Reverse Introduction

M8's Lesson uses a non-standard internal CRA ordering: Application (forward consolidation, S1) → Relational (same-area discovery, S2 opening) → Abstract (vocabulary formalization, S2 mid) → Application/Transfer (mixed text-only, S3). This reversal of the typical Concrete→Relational→Abstract sequence is justified because forward area calculation is already mastered from M7 — Section 1 is not teaching a new concept but consolidating an existing one to automaticity. The genuinely new learning (reverse reasoning) requires the relational insight first ("one area maps to multiple rectangles"), then vocabulary ("factor pair"), then independent application.

---

### KDD-3: Two Worked Examples with Cognitive-Novelty-Matched Fading

The Lesson contains two worked examples: 1.1 (full WE, forward) and 2.1 (partial WE, reverse). The fading sequence matches cognitive novelty: forward calculation is familiar from M7 and needs only one full demonstration before student practice. Reverse reasoning is genuinely new and gets a partial WE — the Guide models the thinking but the next interaction (2.2) asks students to perform the same task with scaffold support. This asymmetric fading prevents over-scaffolding the known skill while adequately supporting the new one.

---

### KDD-4: Warmup-Lesson-Synthesis Area 24 Threading

Area 24 appears three times across the module with deliberate unit variation: Warmup (3×8 and 4×6, sq cm), Section 1.4 (6×4, sq m), and Section 2.3 key beat (all four factor pairs, sq in). The Warmup surfaces the "same area" surprise, forward practice coincidentally produces 24 in a different unit, and the key beat formalizes the insight with all four factor pairs visible. Different units prevent repetition feel while maintaining conceptual continuity. The Warmup's 3×8 reappears as one of 2.3's correct options, rewarding students who remember the connection.

---

### KDD-5: EC Area 36 Reuse — Same Number, Different Direction and Unit

EC.2 uses area 36 (sq m, reverse) despite area 36 appearing in Lesson 1.5 (6×6=36 sq in, forward). This is intentional: the same numeric area in a completely different cognitive direction (reverse vs. forward) and unit (sq m vs. sq in) tests genuine understanding, not recall. Students who remember "6×6=36" have an entry point to the reverse problem — EC.2 then tests whether they can ALSO find the other three factor pairs (3×12, 4×9, 1×36). The reuse probes depth, not surface.

---

### KDD-6: Vocabulary Staging — Experience Before Naming

"Factor pair" is introduced at 2.1 (during the reverse worked example, after students see that multiple pairs produce the same product). "Length" and "width" are formalized at 2.7 (after 7+ interactions of multiplying dimensions). Both follow the experience-first principle: students have concrete mental hooks for the terms before the Guide names them. This ordering means the Warmup and early Lesson use only M7-established vocabulary (area, dimensions, multiply), keeping cognitive load low while building toward the new terms.

---

### KDD-7: 2:1 Forward-to-Reverse EC Ratio

The Exit Check contains 2 forward problems and 1 reverse problem. Forward fluency confirmation is prioritized because M9 depends on automatic dimensions-to-area calculation — a student who hesitates on forward problems will struggle with M9's multi-step reasoning. A single reverse problem is sufficient to flag students who haven't grasped factor-pair thinking; Practice provides ample reverse repetition for those who need it.

---

### KDD-8: Correct Answer Position Randomization

Correct answer positions are distributed across A/B/C/D roughly equally (A=3, B=2, C=2, D=2 across 9 Lesson single-select MCs, plus C and D in EC) with no consecutive duplicates. A test-wise student who always picks one letter succeeds on at most ~25% of items. Script-generating Claude must maintain this distribution; no more than 2 consecutive interactions should share the same correct position.

---

### KDD-9: Edit 83 Distractor Constraints — Strict Compliance

All reverse-problem distractors have both factors entirely outside the correct factor set for that target area. This prevents the anchoring behavior where a student sees a familiar factor, assumes the option is correct, and selects without verifying the product. Area 20's distractors are the tightest constraint case: both use factor 3 (7×3 and 9×3) because the correct factor set {1,2,4,5,10,20} covers most small numbers. This pattern is a coincidence of the mathematical constraint, not a pedagogical choice, and is documented for author awareness.

---

### KDD-10: No Grids in Reverse Confirmation Display

When a student selects a factor pair in a reverse problem, a dimensions-only rectangle appears with those dimensions labeled. This is visual CONFIRMATION, not a grid scaffold — the student has already committed to the factor pair before the rectangle appears. The display serves two purposes: (1) it grounds abstract number selection in concrete geometry (students SEE that 1×24 is dramatically different from 4×6), and (2) it provides the side-by-side display used in the key beat (2.4) and Synthesis. No grid squares appear in these confirmation rectangles — only outlines and dimension labels, consistent with Decision #3.

---

### KDD-11: 1×n Inclusion in All Multi-Select MCs (Edit 91 Reversal)

Edit 91 reversed the original guidance ("accept 1×n if offered, but don't include it in MC options") to a strict inclusion requirement: every multi-select reverse MC must include 1×n as a correct option, and every reverse problem stem must include the "whole number" constraint language. This affects every multi-select MC in the module (2.2, 2.3, 3.2, 3.4, EC.2). The rationale is twofold: (1) normalizing 1 as a legitimate factor prevents the misconception that "1 doesn't count," which would undermine M10's multiplication table work, and (2) the "whole number" language in stems is necessary precisely because 1×n is included — without it, students might wonder whether fractional factor pairs are expected. This was a SME-driven change that reversed a prior position, and it changes what correct answer sets look like: area 18 has 3 correct options (including 1×18), not 2.

### KDD-12: M7 Reverse Preview Does Not Reduce M8 Novelty

M7 Practice includes ~15% reverse problems tagged EXPOSURE (not TRANSFER or BASELINE), using a single-missing-dimension format ("area is X, one side is Y, find the other side"). M8's reverse problems are a genuinely different cognitive task: open factor-pair identification with no dimensions given, multi-select MC requiring students to find ALL valid pairs. The structural difference — M7 asks "what's the one missing number?" while M8 asks "what are ALL the number pairs?" — means M8's claim of reverse as new instructional content is clean. The 15% EXPOSURE preview in M7 actually helps M8: students arrive with faint familiarity that makes the reverse concept less alien, without having practiced the multi-select factor-pair format enough to treat M8 as review.
