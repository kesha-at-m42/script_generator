## CROSS-REFERENCE TABLE B — TVP EXTRACTION

```
TVP: MODULE 4 — Area and Multiplication Connection
===================================================
OUR Lessons: L5 | Standard: 3.MD.C.7a, 3.MD.C.7b

⚠️ BRIDGE MODULE — Students transition from "6 rows of 4 = 24" (M3 skip-counting) to "6 × 4 = 24" (multiplication notation). Research shows this is where many students struggle — they can count but cannot see multiplication as the structure itself.

Learning Goal: Understand that the area of a rectangle can be found by multiplying the side lengths; use multiplication to find the area of rectangles with whole-number side lengths.

Key Teaching Points:
1. Multiplication is a way to describe the row-by-column structure already learned
2. "4 rows of 6" can be written as "4 × 6"
3. Both "4 rows of 6" AND "6 rows of 4" describe the same rectangle → both orientations valid
4. Area = length × width names the structure students have already been using — it's not a new trick, it's a label for what they already do
5. SME Insight: Don't motivate multiplication primarily through efficiency ("faster than skip-counting"). Skip-counting by 2s, 5s, 10s is already fast — students may not feel the need. Instead, motivate through recognition: "You already know this from Unit 1. 6 rows of 4 IS 6 × 4. The × symbol describes the structure you can already see." Efficiency becomes obvious with larger numbers; the initial motivation should be connection, not speed.

CRA Stage: REPRESENTATIONAL → ABSTRACT transition

Cognitive Focus:
- CONNECT (Transfer) — linking structure to symbolic notation
- IDENTIFY (Conceptual) — recognizing dimensions as factors
- CREATE (Procedural) — writing multiplication expressions
- APPLY (Transfer) — using multiplication to solve area problems

Misconceptions Targeted:
- M3: Formula as arbitrary rule (PRIMARY) — Students must understand multiplication emerges from structure, not a random trick
- M10: Dimension confusion (MODERATE) — Both orientations taught explicitly to prevent "which number goes first?" errors

PHASE-BY-PHASE FLOW:

WARM-UP:
- Grid Rectangles (full grid, display) → 4×6 rectangle with row highlighting
- M3 callback: "How many rows? How many in each row? Skip-count to find the area."
- Key beat: After student skip-counts "6, 12, 18, 24" — Guide writes "4 × 6 = 24" alongside. "4 rows of 6. Mathematicians write that as 4 TIMES 6." — the × symbol appears for the first time as a LABEL for what they already did.
- Grid Rectangles (full grid, display) → 6×4 rectangle with row highlighting (same rectangle orientation as M3 synthesis callback)
- M3 callback: "How many rows? How many in each row? Skip-count to find the area."
- Key beat: After student skip-counts "4, 8, 12, 16, 20, 24" — Guide writes "6 × 4 = 24" alongside. "6 rows of 4. You already know how to write that — 6 TIMES 4. The × symbol describes the rows and columns you can already see." — multiplication notation as recognition, not introduction. Students know × from Unit 1 M7-M8.

LESSON:
Early activities (Connecting language to notation):
- Grid Rectangles (full grid, row/column highlighting on tap) + Equation Builder (scaffolded)
- Student identifies structure ("___ rows of ___"), then builds matching multiplication expression
- Equation Builder scaffolded: slots pre-labeled ("Rows" × "In each row" = "Area")
- Key beat — both orientations: Show rectangle, student writes 6 × 4 = 24 from rows. Then system rotates the rectangle 90° (animated) so rows become columns. Student writes 4 × 6 = 24. "SAME rectangle, just turned. SAME area. Two ways to describe it." The rotation makes the equivalence physical, not just verbal — reinforces Unit 1 M12 commutativity work with arrays. Engineering note: Grid Rectangles must support animated 90° rotation with row/column highlighting updating to match new orientation.
- Key beat — both orientations: Show same rectangle, student writes 4 × 6 = 24 from rows, then 6 × 4 = 24 from columns. "SAME area. Two different ways to describe it." — prevents rigid thinking about factor order
- Skip-counting available as computation strategy (not just verification)
- 3-4 rectangles with small, familiar factors (2×4, 3×5, 4×6)

Mid activities (Grid still visible, less scaffolding):
- Grid Rectangles (full grid visible, NO highlighting) + Equation Builder (unlabeled slots)
- Student identifies dimensions from grid, builds expression independently
- Guide prompts decrease
- After building expression: skip-count to verify answer matches
- Vocabulary bridge: start using "length" and "width" alongside "rows" and "columns"
- 3-4 rectangles with moderate factors (3×7, 5×6, 4×8)

Late activities (Full independence — grid stays, scaffolding removed):
- Grid Rectangles (full grid visible, no highlighting) + Equation Builder (no scaffolding)
- Key beat: "Now YOU write the whole thing. Look at the rectangle, figure out the length and width, write the multiplication, and find the area."
- Student writes expression from scratch, calculates product, states area with units
- Both orientations accepted (4×6 and 6×4 both valid)
- Vocabulary: "length × width" becomes primary language
- 5+ rectangles with factors up to 10

EXIT CHECK:
3 problems testing Equation Builder independence with grid support:
1. Grid Rectangles (full grid visible) + Equation Builder (unlabeled slots) → write expression, find area (tests: can they build an expression with slot scaffolding?)
2. Grid Rectangles (full grid visible) + Equation Builder (no scaffolding) → write expression, find area (tests: can they build an expression independently?)
3. Grid Rectangles (full grid visible, no highlighting, no Equation Builder) → state area with correct units from MC (tests: can they determine area without any notation support?)
Assessing: visual-to-symbolic connection, independent expression building, multiplication fluency for area. Grid is visible in ALL problems — grid abstraction is M7's assessment, not M4's.

PRACTICE:
- All problems with full grid visible
- No Equation Builder scaffolding
- Some word problems: "A rectangle is 7 units long and 3 units wide. What is the area?" (grid still shown alongside). All M4 problems use generic "square units" only. Named units (cm, in, ft, m) are introduced in M5-M6.
- Some comparison: given two rectangles with different dimensions, which has greater area?
- Emphasis on independent expression building across varied factor sizes

SYNTHESIS:
- Key beat — formula emerges: Display several completed problems. "What pattern do you notice? How did we find area EVERY time?" → "Area = length × width." This isn't NEW — it's naming what students have been doing since M1.
- Progression callback: M1 (tiled and counted) → M2 (tiled correctly) → M3 (skip-counted rows) → M4 (multiplied) — same concept, increasingly efficient
- Connection: "Every multiplication fact is an area!"
- Bridge to M5: "You can find area of ANY rectangle now. But we've been using 'square units' — tomorrow we'll learn about SPECIFIC square units that everyone uses."

TRANSITION IN (from M3):
Key Transition: M3 → M4
- Pedagogical Shift: Skip-counting structure → Multiplication notation
- What carries forward: Row/column structure identification, "equal rows" understanding
- What's new: × symbol, equation builder, vocabulary shift to "length × width"

TRANSITION OUT (to M5):
Key Transition: M4 → M5
- What M4 establishes: Area = length × width; independent expression building (no Equation Builder scaffolding); multiplication fluency for area WITH grid support
- What M5 builds: Standard units (square inches, square centimeters); same formula, but unit matters
- Bridge: "You can find area of ANY rectangle. But what SIZE are those square units?"

SCAFFOLDING PROGRESSION:
```

## DESIGN CONSTRAINTS (from Important Decisions)

```
DESIGN CONSTRAINTS (from Important Decisions)
=============================================

Decision 1: Unified Path with Adaptive Pacing
  Rule: Use a single instructional path following CRA progression. Differentiate through pacing and representation duration, not different approaches.
  Applies to M4? YES
  Constraint: M4 follows a single CRA progression (Representational → Abstract). No alternate paths. Struggling learners get more time at each scaffold level, not different tools.

Decision 2: Explicit Spatial Structuring as Central Focus
  Rule: Make row-by-column "seeing" an explicit instructional target with dedicated activities. Delay multiplication formula until students demonstrate row-column structuring.
  Applies to M4? PARTIAL
  What applies: M4 ASSUMES spatial structuring from M3. The "delay multiplication" constraint is satisfied — M3 taught structure, M4 introduces multiplication. M4 must verify structure is present (via Warmup callback) before teaching notation.
  What doesn't: M4 does not need dedicated structuring activities — that was M3's job. But M4 should not SKIP structure verification.

Decision 3: Progressive Grid Removal (Concreteness Fading)
  Rule: Full grid → Partial grid → Tick marks → Measurement
  M1-M4: Full grids
  Applies to M4? YES
  Constraint: FULL GRIDS in ALL M4 phases. No grid fading, no partial grids, no tick marks. Grid stays because it's useful — students can verify multiplication by counting. Grid fading begins M7.

Decision 4: Array Knowledge is Helpful but Not Required
  Rule: Design to be self-contained. Leverage array/multiplication knowledge IF students have it.
  Applies to M4? YES
  Constraint: M4 CAN reference Unit 1 (students know × from M7-M8). But the multiplication-area connection must be taught fresh — assume M3 skip-counting as the bridge, not Unit 1 multiplication. TVP explicitly references Unit 1 M7-M8 and M12 commutativity.

Decision 5: Perimeter Kept Separate - Do Not Introduce
  Rule: Do NOT introduce perimeter in this unit.
  Applies to M4? YES
  Constraint: No perimeter language. No "distance around." No area-vs-perimeter comparisons.

Decision 6: Dragging Over Tapping for Manipulatives
  Rule: Prioritize drag-to-place for virtual manipulatives. Exception: simple selection tasks can use tap.
  Applies to M4? YES
  Constraint: Equation Builder should use drag-and-drop for number tiles (OR click-and-place per TVP — touchpad-friendly). MC selections use tap. Grid Rectangles is display-only (no manipulation).

Decision 7: Standard Units Get Full Scaffolding (L6 and L7 Separate)
  Applies to M4? NO — M4 uses generic "square units" only. Standard units are M5-M6.

Decision 8: Include Multiplication Table Connection as Module
  Applies to M4? NO — this is M10.

Decision 9: Keep Application Module Standalone
  Applies to M4? NO — this is M14/M15.
```
