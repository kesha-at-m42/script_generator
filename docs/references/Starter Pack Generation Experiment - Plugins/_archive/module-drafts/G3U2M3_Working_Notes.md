Learning Goal: Identify rows and columns in tiled rectangles; use row and column structure

found by counting equal rows or equal columns.

Key Teaching Points:
1. Tiles in a rectangle naturally form ROWS (horizontal) and COLUMNS (vertical)
2. Every row has the SAME number of tiles; every column has the SAME number
3. We can skip-count by rows instead of counting every tile
4. Skip-counting by columns gives the SAME answer
5. "How many rows? How many in each row?" is the key question
6. This is the BRIDGE from counting to multiplying
7. CRA Stage: CONCRETE → REPRESENTATIONAL transition (structured seeing)

Cognitive Focus:
- IDENTIFY (Conceptual) — seeing structure in arrays

  skip-count by 5 (5, 10), columns → skip-count by 2 (2, 4, 6, 8, 10). Both

  student taps rows OR columns, records structure ("___ rows of ___" or

- Grid Rectangles (full grid, NO highlighting available)

- Scaffolding within Late: highlighting available on tap (student choice) → no

- Rectangles with areas 20-50 square units (larger to show efficiency gain)
- Students find area BOTH ways (rows and columns) for some problems to confirm

  3. "Skip-count by rows to find the area" (validates skip-counting)
  4. "Can you find the area by counting columns too?" (tests flexible structure-seeing)
- Assessing: structure identification, skip-counting (not count-all), row AND column
  flexibility

## DESIGN CONSTRAINTS (from Important Decisions)

### Decision 1: Unified Path with Adaptive Pacing
**Rule:** Use a single instructional path following CRA progression. Differentiate through pacing and representation duration, not different approaches.
**Applies to M3?** YES
**What it constrains:** M3 follows a single CRA path (CONCRETE → REPRESENTATIONAL transition per TVP). No alternative instructional sequences.

### Decision 2: Explicit Spatial Structuring as Central Focus
**Rule:** Make row-by-column "seeing" an explicit instructional target with dedicated activities, not an assumed skill.
**Applies to M3?** YES — THIS IS M3's PRIMARY CONSTRAINT
**What it constrains:** M3 MUST include dedicated activities targeting "How many rows? How many in each row?" Row-column structuring is the explicit instructional focus, not a byproduct. Delay multiplication formula until students demonstrate row-column structuring (multiplication is M4's domain). Use highlighting/animation to make row-column structure visible.

**Rule:** M1-M4: Full grids.
**Applies to M3?** YES
**What it constrains:** All M3 rectangles display full grids. This is consistent with M2. Grid Rectangles in "full" grid state. Grid fading doesn't begin until M7.

**Applies to M3?** YES — PARTIAL

M3's spatial structuring can serve as array introduction (for students new to it) OR reinforcement (for students who had Unit 1). The instruction must stand alone — no hard dependencies on Unit 1 vocabulary like "equal groups" or "factors." Brief activation is OK; assumed mastery is not.

### Decision 5: Perimeter Kept Separate — Do Not Introduce
**Rule:** Do NOT introduce perimeter in this unit.
**Applies to M3?** YES
**What it constrains:** No perimeter language. Same as M1-M2.

### Decision 6: Dragging Over Tapping for Manipulatives

Decision #6's drag requirement applies when manipulatives are present — M3 uses display-only Grid Rectangles with tap-to-highlight, not manipulatives.

**Applies to M3?** NO — Standard units are M5-M6. M3 uses generic "square unit."

### Decision 9: Keep Application Module Standalone

**⚠️ AUTHOR FLAG #4 (Skip-Counting Display/Input):** RESOLVED — Two-part answer: (a) Skip-counting CAN be said in Guide dialogue (Guide narrates: "5... 10... 15!"); (b) System highlights sequential groups within the grid based on rows or columns (system-driven animation synced with Guide narration). This means skip-counting is a SYSTEM-DRIVEN demonstration: system highlights Row 1 → Guide says first count → system highlights Row 2 → Guide says next count → etc. Student's cognitive work is observing the pattern and then answering structure/area questions. **⚠️ AUTHOR FLAG #5 (Highlighting — Scaffolding Progression):** REVISED after Gate 2 author review. Original resolution removed tap-to-highlight entirely ("ALL highlighting system-driven"). Gate 2 evaluation identified this as a MAJOR finding: removing the tap-to-highlight step eliminates the critical TRANSFER mechanism between passive observation and independent performance. For the 69% of students who can't structure arrays without instruction, the jump from "system shows you" directly to "find it yourself with no help" is too steep.

**REVISED RESOLUTION:** Restore TVP's three-step scaffolding: system auto-highlight (S1) → student tap-to-highlight (S2 transfer) → no highlighting (S3). Tap-to-highlight is student-INITIATED but system-EXECUTED. The key pedagogical insight: passive observation doesn't reliably transfer to independent performance — students need an active practice step where they use the tool to find structure themselves.

Updated progression:
- S1 (Early): System auto-highlights ROWS (with skip-counting narration)
- S2 (Mid — first column exposure): System auto-highlights COLUMNS (first exposure needs system support)
- S2 (Mid — transfer): Student tap-to-highlight rows AND columns (2 interactions: 2.3, 2.4)
- S3 (Late): NO highlighting — student identifies structure independently
- YAML interaction_tools updated to "MC, Tap-to-Highlight"

## SECTION PLAN

### Warmup
- **Prior knowledge activated:** M2 tiling + one-by-one counting. Students can tile accurately. M2 Synthesis showed highlighted row: "Is there a FASTER way?"
- **Warmup type:** Counting Motivation (TVP specifies a counting task that creates felt inefficiency). Student counts large tiled rectangle one by one.
- **Bridge target:** "That took a while! Let's find a FASTER way." Creates need for structured counting without teaching it.
- **Key design:** 6×4 rectangle (24 squares). Orientation: 6 rows of 4 (tall). This exact rectangle returns in Synthesis for the callback payoff.

### Lesson Sections
- **Section 1 (Lesson Early — Discovering Rows):** ~3-4 interactions. Auto-highlighted rows on 3×5 rectangle. Students count per row, discover equal count, see skip-counting demo. Practice on 2-3 Early rectangles (dimensions from {2,5,10}). CRA: Concrete → beginning Representational.
  - Key vocabulary introduced: row (during observation), "rows of" (during skip-counting)
  - Interaction types: Pattern 2 (observe auto-highlight), Pattern 1 (MC: "How many in each row?", "What's the area?")

- **Section 2 (Lesson Mid — Discovering Columns + Both Directions):** 5 interactions. Three sub-phases: (a) System auto-highlights columns for first exposure (2.1-2.2), (b) Student tap-to-highlight for transfer practice (2.3-2.4), (c) Array vocabulary capstone (2.5). Key moment: 2×5 rectangle where rows AND columns both give the same area (2.1). Column vocabulary introduced (2.2). Both-directions practice with tap-to-highlight (2.3-2.4). "Array" activated as callback from Unit 1 AFTER both-directions practice — capstone naming.
  - Key vocabulary introduced: column (2.2, during column discovery), "columns of" (2.2, during practice), array (2.5, capstone after practice)
  - Interaction types: Pattern 2 (observe system column highlighting + vocabulary), Tap-to-Highlight + MC (transfer practice), Pattern 2 (array naming)
  - Scaffolding shift: system auto-highlight → tap-to-highlight within this section

- **Section 3 (Lesson Late — Independence):** ~3-4 interactions + EC bridge. No highlighting. Students identify structure and skip-count independently. Larger rectangles (areas 20-50). Some problems require BOTH row and column descriptions.
  - Interaction types: Pattern 1 (MC structure + area, no highlighting support)

### Exit Check
- **Skills to assess:** (1) Independent structure identification, (2) Structure description construction, (3) Skip-counting validation, (4) Column flexibility
- **Cognitive types:** IDENTIFY (EC.1, EC.3, EC.4) + CREATE (EC.2) — M1-3 guideline; COMPARE deferred to Practice
- **4 problems per TVP** (see AF#2, KDD-4)

### Synthesis
- **Connections:** Callback to Warmup rectangle (now easy with skip-counting). Structure consolidation (rows = horizontal, columns = vertical, either works).
- **Task types:** Pattern Recognition (before/after counting comparison), Metacognitive Reflection (strategy identification), Structure Consolidation
- **Closure/bridge:** Celebrate skip-counting achievement. NO M4 teaser (resolved SME decision). M4 bridge is M4's responsibility.
