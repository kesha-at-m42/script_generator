## 2. CROSS-REFERENCE TABLE B — TVP Extraction

### TVP Learning Goal
Describe square units based on different linear units of measurement. Measure the area of the same figure in square inches and square centimeters.

### TVP Key Teaching Points

1. A square inch is a square with side lengths of 1 inch; a square centimeter is a square with side lengths of 1 centimeter
2. These are standard units — everyone agrees on their size
3. **THE BIG INSIGHT:** Same shape measured in both units gives DIFFERENT numbers — "It takes more of a smaller unit to cover the same area"
4. Students know inches and centimeters from Grade 2 linear measurement (2.MD.A.1) — NEW: the square version (area units, not length units)
5. Dedicated practice on linear vs. square distinction is required. The transition from linear units to square units is a known difficulty.

### TVP CRA Stage
Representational — full grids return because GRID SIZE is the new concept

### TVP Cognitive Focus
- **IDENTIFY (Conceptual)** — recognizing different-sized standard units
- **COMPARE (Conceptual)** — understanding why same shape gives different numbers
- **CONNECT (Transfer)** — linking linear units to area units
- **APPLY (Transfer)** — using multiplication with correct unit labels

### TVP Misconceptions Targeted
- **M8: Square Unit Confusion (PRIMARY)** — Students think all square units are equal, or that 12 sq cm = 12 sq in
- **Bigger number = bigger shape (MODERATE)** — 80 sq cm vs 12 sq in for same rectangle

### TVP What Students DO

**WARM-UP:** Grid Rectangles (side-by-side display, two grid sizes, NO unit labels yet) → Notice & Wonder. Students observe same rectangle tiled two ways.

**LESSON:**
- *Early activities (Naming the units):* Grid Rectangles 1×1 side-by-side inch/cm. Dimension labels show "1 in" or "1 cm". Key beat: "You know what an INCH is from measuring length. A SQUARE INCH is a square with sides of 1 inch."
  - Dedicated linear vs. square unit practice (SME requirement — 3-4 MC problems)
  - Bridge from "square units" to named standard units
- *Mid activities (The critical comparison):* Same rectangle with inch grid then cm grid. Side-by-side display. Key beat: "SAME rectangle. Different numbers! The rectangle didn't change — but it takes MORE of a smaller unit to cover the same space."
- *Late activities (Independent practice):* Single unit type per problem. Must include unit in answer. Unit Square Reference Display for identification problems.

**EXIT CHECK:** 3 problems using SAME rectangle for problems 1-2:
1. Find area with inch grid, state in square inches
2. SAME rectangle with cm grid — find area, state in square centimeters
3. Given SAME rectangle with two area measurements (e.g., "Area = 12" and "Area = 80"), student selects which is sq in and which is sq cm. MC format.

**PRACTICE:** Grid Rectangles (mixed), same rectangle both units, Type A (direction prediction 4-option MC), Type B (misconception trap 4-option MC). Also 2-3 linear vs. square unit distinction problems.

**SYNTHESIS:** Callback to warmup rectangle pair with unit names. Consolidate inverse relationship. Vocabulary review. Bridge to M6.

### TVP Data Constraints
- **Square inch problems:** Factors 2–10, products within 100
- **Square centimeter problems (independent):** Factors 3–10, products within 100
- **Comparison pairs:** Both inch and cm measurements involve clean multiplication facts. All cm products within 100.
  - Example: 4in×3in=12 sq in → ~10cm×8cm=80 sq cm
  - Example: 2in×6in=12 sq in → ~5cm×15cm=75 sq cm
  - Example: 4in×4in=16 sq in → ~10cm×10cm=100 sq cm

### TVP Vocabulary
- **New:** square inch, square centimeter, standard unit
- **Abbreviations:** NOT introduced in M5. Full unit names required in all student responses.

### TVP Tool Requirements
- **Critical:** Side-by-side display of same rectangle with two different grid overlays
- **1×1 Grid Rectangle with unit type set** serves as individual unit square display

### TVP Key Transitions
- **M4→M5:** Area = length × width; independent expression building; multiplication fluency with grid support → **Standard units; same formula but unit matters**
- **M5→M6:** Standard units exist; different units → different numbers; smaller unit → bigger number → **Larger standard units (sq ft, sq m); choosing appropriate units for context**

### TVP SME Resolved Decisions
- *Approximate vs exact cm:* **RESOLVED** — use clean approximate cm values
- *Square distinction dedicated practice:* **RESOLVED** — dedicated practice required
- *Multiplication vs counting:* **RESOLVED** — allow both
- *Grid return:* **RESOLVED** — moot, M4 keeps grids, fading begins M7
- *Abbreviations:* **RESOLVED** — No abbreviations in M5
- *Number range for cm:* **RESOLVED** — all areas ≤ 100

## 4. DESIGN CONSTRAINTS — Important Decisions Extraction

### Important Decision #1: Unified Path with Adaptive Pacing
**Rule:** Single CRA path; pacing adapts to student needs but path does not branch.
**Applies to M5?** YES
**M5 Implication:** All students progress through Concrete → Representational → Abstract using Grid Rectangles (concrete model). No branching into alternative tools. Pacing within this path per student readiness.

### Important Decision #2: Explicit Spatial Structuring
**Rule:** Teach students to structure area visually (rows/columns) before relying on formula alone.
**Applies to M5?** PARTIAL
**M5 Implication:** M5 doesn't introduce spatial structuring as a new concept (students know this from M4). However, M5 leverages row/column structure to highlight that grid SIZE (unit) changes the count. Side-by-side grids emphasize: SAME structure, DIFFERENT unit = DIFFERENT count.

### Important Decision #3: Progressive Grid Removal
**Rule:** Grids fade progressively: M4 (full grids throughout) → M5–M6 (full grids return for unit concepts) → M7+ (grids fade toward abstraction).
**Applies to M5?** YES
**M5 Implication:** M5 maintains FULL grids throughout (all activities). Fading begins M7. This is intentional: unit comparison requires visible grid SIZE difference.

### Important Decision #4: Array Knowledge Helpful, Not Required
**Rule:** Prior multiplication/array knowledge is beneficial context but not a prerequisite.
**Applies to M5?** YES
**M5 Implication:** Students should have multiplication fluency from M4 (area formula context). M5 assumes students can multiply but doesn't require advanced array work. Multiplication remains WITH grid support.

### Important Decision #5: Perimeter Kept Separate
**Rule:** Perimeter is not taught alongside area in this sequence.
**Applies to M5?** YES
**M5 Implication:** M5 focuses exclusively on area. No perimeter problems, no perimeter vocabulary.

### Important Decision #6: Dragging Over Tapping
**Rule:** Physical/digital manipulatives favor drag actions over tap/click (better for spatial reasoning).
**Applies to M5?** YES
**M5 Implication:** Grid Rectangles tool should support dragging (e.g., positioning rectangles, adjusting grids) over single-click interactions where possible.

### Important Decision #7: Standard Units Get Full Scaffolding (L6/L7 Separate)
**Rule:** Square inches and square centimeters (L6/M5) receive dedicated module. Square feet and square meters (L7/M6) receive dedicated module. Do not combine.
**Applies to M5?** YES (DIRECTLY)
**M5 Implication:** M5 teaches ONLY square inches and square centimeters. Square feet and square meters explicitly deferred to M6. This decision ensures sufficient practice time and prevents unit overload.

### Important Decision #8: Multiplication Table Connection
**Rule:** Multiplication facts and area are connected later (M10+), not in M5–M6.
**Applies to M5?** NO
**M5 Implication:** M5 uses multiplication to find area (length × width) but does NOT reference multiplication tables, facts drills, or times-table mastery. Multiplication in M5 is TOOL, not CONTENT.

### Important Decision #9: Application Module Standalone
**Rule:** Application/real-world context modules (e.g., "Area in Garden Design") are standalone, not embedded in core modules.
**Applies to M5?** NO
**M5 Implication:** M5 is conceptual core (square units). Real-world applications deferred to dedicated application module (M14+). M5 does not include "design a bedroom" or similar applied contexts.

## 5. SECTION PLAN — High-Level Phase Outline

### Phase 1: Warm-Up (Notice & Wonder)
- **Tool:** Grid Rectangles (side-by-side display, two grid sizes, NO unit labels)
- **Student Action:** Observe same rectangle tiled two ways; identify differences in number of tiles
- **Cognitive Move:** IDENTIFY (Conceptual) — "Why does this rectangle need different numbers of tiles?"
- **Output:** Conjecture about grid size mattering

### Phase 2: Naming the Units (Early Lesson)
- **Tool:** Grid Rectangles with dimension labels ("1 in" / "1 cm")
- **Instructional Beat:** "You know what an INCH is. A SQUARE INCH is a square with 1-inch sides. Here's a SQUARE CENTIMETER."
- **Cognitive Move:** IDENTIFY (Conceptual) + CONNECT (Transfer) — link linear units to square units
- **Student Action:** Count tiles; label grid size; recognize 1×1 dimensions
- **Dedicated Practice:** 3-4 MC problems on linear vs. square distinction (SME requirement)
- **Output:** Students name units correctly; distinguish 1 inch from 1 square inch

### Phase 3: The Critical Comparison (Mid Lesson)
- **Tool:** Grid Rectangles showing SAME rectangle with inch grid, then cm grid (side-by-side)
- **Instructional Beat:** "SAME rectangle. Different numbers. The rectangle didn't change — but it takes MORE of a smaller unit to cover the same space."
- **Cognitive Move:** COMPARE (Conceptual) — inverse relationship between unit size and count
- **Student Action:** Count tiles in both grids; calculate area using multiplication; compare counts
- **Output:** Recognition that 12 sq in ≠ 80 sq cm, but SAME area

### Phase 4: Independent Practice (Late Lesson)
- **Tool:** Grid Rectangles (mixed sizes)
- **Structure:** Single unit type per problem; must include unit label in answer
- **Unit Square Reference Display:** Available for students to check 1×1 dimensions
- **Cognitive Move:** APPLY (Transfer) — using multiplication with correct unit labels
- **Student Action:** Find area; state with unit
- **Output:** Consistent use of unit labels

### Phase 5: Exit Check
- **Problem 1:** Rectangle with inch grid → area in square inches
- **Problem 2:** SAME rectangle with cm grid → area in square centimeters
- **Problem 3:** MC: Two area measurements (12 and 80); select which is sq in, which is sq cm
- **Cognitive Move:** IDENTIFY + COMPARE at mastery level

### Phase 6: Synthesis & Bridge
- **Callback:** Return to warm-up rectangle pair; now label units
- **Consolidation:** "Smaller units make bigger numbers. Both measure the SAME rectangle. Both are correct."
- **Vocabulary Review:** square inch, square centimeter, standard unit
- **Bridge to M6:** "Next, we'll learn about even bigger standard units."

### EC Skills (Exit Criteria)
- Recognizes square inch and square centimeter visually
- Measures area using correct standard unit
- Explains why same shape gives different numbers in different units
- Includes unit label in all area measurements

## 7B. MASTERY TRACKING GUIDANCE (for Task 3 Script Writers)

Per TVP mastery tracking note, M5 errors fall into two categories with different intervention routing:

| Error Type | Description | Example | Classification | Intervention Loop |
|-----------|-------------|---------|----------------|-------------------|
| **Unit-label error** | Correct area calculation, wrong or missing unit name | Student writes "20" or "20 inches" instead of "20 square inches" | **CONCEPTUAL** | Loop to M5 Early activities (vocabulary reinforcement — Section 1) |
| **Calculation error** | Wrong area calculation, correct unit name | Student writes "24 square inches" for a 5×4 rectangle | **PROCEDURAL** | Address multiplication fluency (M4 review) |
| **Both** | Wrong calculation AND wrong/missing unit | Student writes "24 inches" | Tag both; prioritize CONCEPTUAL (unit first, then fluency) | M5 Early → M4 review |

**This distinction drives intervention routing in Practice and EC problem tagging.** Task 3 must tag each EC problem and Practice problem type with the error classification it primarily diagnoses. The "correct number, wrong unit" MC distractors used throughout the Lesson (1.1b, 1.2, 2.1, 2.2) are CONCEPTUAL diagnostics. The "wrong number, correct unit" distractors (e.g., "24 square inches" for 5×4) are PROCEDURAL diagnostics.

## 7F. GRIDS WITHOUT UNIT LABELS (Flag for Task 3 Practice)

All Lesson grids display unit labels on at least one cell (e.g., "1 square inch" visible on a grid cell). This is appropriate for the Lesson, where students are learning to associate the visual with the name. However, Task 3 Practice must include **at least 1-2 problems where grids are shown WITHOUT unit labels on individual cells** — students must identify the unit by SIZE alone (reading the dimension labels on the rectangle edges, or recognizing the grid scale from visual comparison to the Unit Square Reference Display). This tests whether students can identify units independently, not just read a label.

---

## 7G. PREDICTION QUESTION (Deferred from Lesson 2.4 to Synthesis)

The original Lesson interaction 2.4 asked a prediction question: "If I measured this rectangle with EVEN SMALLER tiles, would the number go up or down?" This was replaced with a second concrete comparison pair (TVP pair #3) because the prediction is developmentally ambitious for Grade 3 — it requires extending the inverse relationship to a hypothetical case. Task 3 Synthesis should consider including this prediction question as a forward-looking reflection: "You've seen that smaller units give bigger numbers. What would happen with an even SMALLER unit?" This works well in Synthesis's reflective context (Type A dominant) and bridges toward M6 (larger units = smaller numbers).
