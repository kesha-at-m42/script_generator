### EC.1: Area in Square Inches [CREATE] [Type B]

* **Purpose:** Can the student build a correct multiplication expression AND select the correct unit label for an inch grid? Tests Section 3 skill: reading dimensions, placing factors, selecting "square inches" from the unit palette.
* **Visual: Grid Rectangles (Display) + Equation Builder (unit-label slot).** A 3×3 square with inch-scale grid. Full grid visible. Dimension labels: "3 in" and "3 in." Unit label: "1 square inch" visible on one grid cell. EB: `[___] × [___] = [auto] [unit label]`. **Number tile palette: {2, 3, 4, 6, 9}.** **Unit label palette: {square inches, square centimeters, inches, centimeters}.**
* **Guide:** "Here's a rectangle. Build the multiplication expression and pick the unit."
* **Prompt:** "Build the multiplication expression and select the unit label."
* **Student Action:** Drag-to-place (Equation Builder) — select 2 number tiles + 1 unit label tile
  * **Expected:** 3 and 3 into factor slots (either order). Product 9 auto-computes. Select "square inches" for unit label.
* **Correct Answer:** 3 × 3 = 9 square inches
* **On Correct:** "9 square inches."
* **Remediation:** Pipeline

> **Design Note:** Type B — brief instruction, minimal teaching. Guide says only "Build the multiplication expression and pick the unit" — no reminders about what unit to select. This matches the Lesson Section 3 independence level (3.4's Guide was "One more. Build the expression."). Number palette {2, 3, 4, 6, 9}: off-by-one distractors (2 and 4 for factor 3), 6 (sum 3+3), and 9 (the product — tests whether student confuses a factor with the area). Unit label palette is identical to Lesson Section 3. The critical M5 test: does the student select "square inches" (not "inches")? Feedback is brief per EC Playbook §3E: "9 square inches." — confirms the complete answer with unit.

> **Design Note (EB auto-compute with wrong factors):** Including the product (9) in the factor palette is a diagnostic choice. If a student drags 9 into a factor slot alongside any other tile, the auto-computed product will be visibly wrong (e.g., 9 × 3 = 27, 9 × 2 = 18). The EB's auto-compute provides immediate implicit feedback: the student sees an unexpected product and can self-correct before submitting. This is consistent with the Lesson's EB behavior where auto-compute serves as a built-in reasonableness check. If the student submits with a wrong product, the error is classified as PROCEDURAL per §7B (wrong area, regardless of unit correctness).

### EC.2: Area in Square Centimeters — Same Rectangle [CREATE] [Type B]

* **Purpose:** Can the student build a correct multiplication expression with the correct unit label for a cm grid showing the SAME rectangle — WITHOUT a cell label cue? Tests Section 3 skill AND Section 2 setup (same shape, different grid). The student must switch from "square inches" (EC.1) to "square centimeters" by reading dimension labels ("8 cm"), not by copying a cell label. This is the critical M5 diagnostic: does the student connect dimension units to area units independently?
* **Visual: Grid Rectangles (Display) + Equation Builder (unit-label slot).** The SAME square from EC.1, now shown with cm-scale grid. Full grid visible. Dimension labels: "8 cm" and "8 cm." **No unit label on grid cells** — student must read dimension labels to determine this is a centimeter grid. The inch-grid version (EC.1) is NOT visible. EB: `[___] × [___] = [auto] [unit label]`. **Number tile palette: {6, 7, 8, 9, 10, 16}.** **Unit label palette: {square inches, square centimeters, inches, centimeters}.**
* **Guide:** "Same rectangle — different grid. Build the expression and pick the unit."
* **Prompt:** "Build the multiplication expression and select the unit label."
* **Student Action:** Drag-to-place (Equation Builder) — select 2 number tiles + 1 unit label tile
  * **Expected:** 8 and 8 into factor slots (either order). Product 64 auto-computes. Select "square centimeters" for unit label.
* **Correct Answer:** 8 × 8 = 64 square centimeters
* **On Correct:** "64 square centimeters."
* **Remediation:** Pipeline

> **Design Note:** Type B — parallel to EC.1 with increased multiplication difficulty AND reduced scaffolding. "Same rectangle — different grid" echoes Lesson Section 2 language without re-teaching the inverse relationship. The student must: (1) read dimension labels "8 cm" to identify this as a centimeter grid (no cell label to copy), (2) place correct factors, (3) select "square centimeters" from the unit palette. **Cell labels are removed for EC.2** — this is the key scaffolding fade from EC.1. In EC.1, the student sees "1 square inch" on a grid cell, providing a cue for unit selection. In EC.2, that cue is gone: the student must connect "8 cm" dimension labels → centimeter grid → "square centimeters" unit label. This tests genuine unit identification, not label-matching. The unit-label switch from EC.1 to EC.2 (inches → centimeters) is the critical diagnostic: students who select "square inches" on a cm grid with no cell label reveal M8 (unit confusion) more cleanly than if the cell label were present. Number palette {6, 7, 8, 9, 10, 16}: off-by-one distractors (7, 9 for 8), nearby (6, 10), and 16 (sum 8+8 — tests factor vs. sum confusion). Feedback brief: "64 square centimeters."

### EC.3: Which Unit Is Which? [COMPARE] [Type B]

* **Purpose:** THE COMPARISON TEST. Given the two area measurements from EC.1 and EC.2 (9 and 64), student matches each to the correct unit. Tests whether the inverse relationship has been internalized: smaller unit → bigger number. This directly tests TVP EC Problem 3 spec.
* **Visual: Grid Rectangles (Display, side-by-side mode).** Left panel: The 3×3 inch-grid square from EC.1, labeled "Area = 9." Right panel: The same square with 8×8 cm grid from EC.2, labeled "Area = 64." Both panels show the SAME square outline. Unit labels on cells are REMOVED for this problem — student must determine units from grid SIZE and dimension labels only.
* **Guide:** "You measured this square two ways — the area is 9 one way, and 64 the other. Which is which?"
* **Prompt:** "Which measurement is square inches and which is square centimeters?"
* **Student Action:** MC selection
  * **Options:** "9 is square inches, 64 is square centimeters" / "64 is square inches, 9 is square centimeters" / "Both should be the same number" / "9 is wrong — it should be 64"
* **Correct Answer:** "9 is square inches, 64 is square centimeters"
* **Answer Rationale:**
  * "9 is square inches, 64 is square centimeters" = Correct — inch tiles are bigger, so fewer fit (9); cm tiles are smaller, so more fit (64). Applies inverse relationship.
  * "64 is square inches, 9 is square centimeters" = Reversed — student may think bigger number goes with bigger unit (secondary misconception: bigger number = bigger area)
  * "Both should be the same number" = M8 — assumes same shape must give same count regardless of unit
  * "9 is wrong — it should be 64" = M8 — assumes only one correct answer exists per shape
* **On Correct:** "9 square inches and 64 square centimeters. Same square, different units."
* **Remediation:** Pipeline

> **Design Note:** Type B — brief Guide + complete instruction. This is the TVP's EC Problem 3 spec: "Given SAME rectangle with two area measurements, student selects which is sq in and which is sq cm." The unit labels on individual grid cells are REMOVED for this problem (per Working Notes §7F flag — testing identification by size alone). Students must use the dimension labels ("3 in" / "8 cm") or the visual size difference to determine which grid is which. The MC distractors mirror Lesson 2.3's distractor structure: reversed (secondary misconception), "same number" (M8), and "one is wrong" (M8). Feedback is brief but includes both measurements with labels: "9 square inches and 64 square centimeters. Same square, different units."

> **Design Note (§7F implementation):** EC.2 and EC.3 both remove unit labels from grid cells. EC.2 tests unit identification from dimension labels ("8 cm" → square centimeters) in an EB production context. EC.3 tests it in a comparison/matching context with BOTH grids label-free. Together they implement the §7F flag — by the end of EC, the student has encountered label-free grids in two different interaction types. Practice will extend this further with single-unit label-free problems.

At least 1-2 Practice problems should show grids WITHOUT unit labels on individual cells. Students must identify the unit by:
- Reading dimension labels on rectangle edges ("3 in" → square inches)

### Type B Misconception Trap Format Template

The "bigger number = bigger area" misconception trap uses a specific MC format. Example:

- **Visual:** Two rectangles side by side. Rectangle A: cm grid, large area number. Rectangle B: inch grid, small area number. Rectangle B is physically larger than Rectangle A.
- **Format:** 4-option MC
- **Sample Problem:** "Rectangle A has an area of 80 square centimeters. Rectangle B has an area of 12 square inches. Which rectangle covers more space?"
- **Options:** "Rectangle A — it has the bigger number" / "Rectangle B — it's the bigger rectangle" / "They cover the same space" / "Can't tell without more information"
- **Correct:** "Rectangle B — it's the bigger rectangle"
- **Distractor Rationale:**
  - "Rectangle A — bigger number" = secondary misconception (bigger number = bigger area). Diagnostic for M8.
  - "They cover the same space" = confusion about unit equivalence
  - "Can't tell" = plausible but incorrect — students CAN tell from the visual
- **On Correct:** "Right — Rectangle B covers more space. The numbers are different because the units are different sizes."

> **Design Note:** The key diagnostic is whether the student relies on the NUMBER (80 > 12, so A is bigger) or the VISUAL/REASONING (B is physically larger; the numbers differ because the units differ). The visual must clearly show Rectangle B as physically larger — the student should be able to verify their reasoning against what they see. Problems should vary which unit type has the larger number to prevent pattern-matching.
