## §1.0 THE ONE THING

### Testable Core Claim
By the end of Task 1, students will recognize that **larger square units (square feet and square meters) are appropriate for larger areas**, and they can explain *why* using visual comparison and "too many" reasoning (e.g., "It would take too many square centimeters to measure a playground"). This claim is testable via EXIT CHECK problem (3): students must select the most appropriate unit for three real-world contexts AND provide reasoning.

### CRA Stage
**Representational.** This is the LAST module before grid fading begins in M7. Students work with full grids in Warm-Up, Early, Mid, and Late Lesson phases, and with full grids again in EXIT CHECK and Practice. Grids remain complete and countable throughout M6. The Synthesis drag-to-rank interaction uses SAME-SIZE unit squares on screen (visual anchoring removed), forcing pure reasoning about relative unit sizes.

### Critical Misconception Addressed
**M8: Square Unit Confusion** (PRIMARY) — Students may confuse square units or think that "more squares means more area" even when units change. M6 explicitly targets this by:
1. Presenting three units side-by-side in Warm-Up (sq cm, sq in, sq ft visual comparison) with sq m referenced in text ("A square meter is even BIGGER than a square foot")
2. Showing dense cm grid vs. clean ft grid on the SAME rectangle (same area, different unit counts)
3. Requiring unit-selection reasoning in MC and EXIT CHECK

**"More Squares = More Area" (SECONDARY)** — Reinforced from M5: same physical area can have different numbers depending on unit choice.

### Success Indicators
1. **Naming:** Student can name and define square foot and square meter (same convention as sq in/cm from M5).
2. **Unit Appropriateness:** Student can identify when larger units are needed ("A classroom floor would use square feet, not square centimeters—that would be too many!").
3. **Unit Selection Reasoning:** Student can justify unit choice with reference to area size and practicality (Tier 1: obvious; Tier 2: customary vs. metric; Tier 3: boundary cases).
4. **Multiplicative Calculation:** Student can multiply dimensions to find area in any standard unit (row × column = area).

### Biggest Risk
**Grid removal happens AFTER M6.** If students do not develop strong fluency with full grids in M6, they will struggle in M7 when partial grids are introduced. Additionally, if unit-selection reasoning is not internalized in M6, students may revert to "pick the smallest unit" or random choice in M7+. Task 1 must deliver thorough grid practice AND establish unit-selection reasoning as habitual.

## §1.4 MISCONCEPTIONS

### M8: Square Unit Confusion (PRIMARY)

**Definition (as applied to M6):**
Students confuse square units or apply inconsistent logic when comparing or selecting units. Manifestations in M6:
1. **Unit Naming Confusion:** "Square feet" and "square meters" are confused with linear feet/meters or used interchangeably.
2. **Comparison Confusion:** Student cannot rank units by size or thinks all units are "the same."
3. **Selection Confusion:** Student picks a unit based on habit (e.g., always "square feet") rather than context, or picks the smallest unit regardless of area size.
4. **Inverse Relationship Confusion:** When the same area is measured in different units, student interprets "different numbers" as "different areas."

**Why It Matters:**
Unit confusion breaks down the mathematical reasoning in M6 and creates downstream errors in M7+ when grids are removed and unit reasoning becomes the PRIMARY strategy.

**How It's Addressed in M6:**

| Strategy | Mechanism | Phase |
|----------|-----------|-------|
| **Side-by-side Comparison** | Warm-Up: Grid Rectangles showing 1×1 squares in sq cm, sq in, sq ft simultaneously. Text note: "A square meter is even BIGGER than a square foot." | Warm-Up |
| **"Too Many" Visual Contrast** | Mid Lesson: Same rectangle shown in dense cm grid (e.g., thousands of squares) vs. clean ft grid (e.g., 12 squares). No numerical counter; visual impact drives reasoning. | Mid Lesson |
| **Unit-Selection Reasoning (Tier scaffolding)** | MC options include both unit choice AND reasoning. Students practice articulating why a unit is appropriate. | Mid Lesson, Late Lesson |
| **Context Clues** | Late Lesson and Practice: Real-world contexts provide anchors (playground = bigger unit; postcard = smaller unit). | Late Lesson, Practice |
| **EXIT CHECK Problem (3)** | Students must select and justify unit for three contexts in writing. Reasoning is assessed explicitly. | EXIT CHECK |
| **Synthesis Drag-to-Rank** | Students rank four unit squares (all SAME SIZE on screen) by size. Visual anchoring removed; pure reasoning required. Reinforces that size can be inferred from naming convention, not perception. | Synthesis |

### "More Squares = More Area" (SECONDARY)

**Definition:**
Students equate the number of squares counted to the "size" of the area, failing to account for unit changes. This is a carryover from M5 but is intensified in M6 with four units in play.

**Manifestation in M6:**
"The playground has more squares when we measure in centimeters than in feet, so it's bigger" (confusing count with area).

**How It's Addressed:**

| Strategy | Mechanism | Phase |
|----------|-----------|-------|
| **Same-Area, Different-Unit Comparison** | "Too many" moment: same rectangle, two grids. Same physical area, vastly different square counts. Explicit comparison: "Is the rectangle bigger or smaller? No—it's the same! The only thing that changed is the unit we're using." | Mid Lesson |
| **Inverse Relationship Language** | "Fewer of a larger unit cover the same space." Repeated in Late Lesson and Practice with varied examples. | Late Lesson, Practice |
| **Multiplication, Not Counting** | Emphasize: "We multiply the dimensions (rows × columns) to find area. The unit just tells us what we're multiplying." | Early Lesson, EXIT CHECK |
| **EXIT CHECK Problem (1 & 2)** | Students practice the same-area measurement in two different units, reinforcing that the area doesn't change, only the count. | EXIT CHECK |

### TERTIARY INTERACTION: Multiple Choice (MC) with Tier Scaffolding

#### Purpose
Unit-selection MC items appear in Mid Lesson (after "too many" visual moment) and Late Lesson. MC options scaffold reasoning about appropriate units through tiered options.

#### Tier 1 Scaffolding (Most Supportive)
**Context Clue:** Very obvious area (e.g., playground, classroom floor)
**Options:** Each includes unit AND reasoning statement (per TVP: "MC options include the unit AND the reasoning, not bare unit names")
**Example:**
Q: "What unit would you use to measure a playground?"
A1: "Square centimeters — a playground is small, like a phone screen" (wrong unit, wrong reasoning)
A2: "Square inches — a playground is about the size of a book" (wrong unit, wrong reasoning)
A3: "Square feet — a playground is a big area, so we need a big unit like square feet" (correct)
A4: "Square meters — a playground is a big area, so we need a big unit like square meters" (also correct; both accepted at Tier 1)

#### Tier 2 Scaffolding (Medium)
**Context Clue:** Ambiguous between customary and metric for a large area (e.g., classroom floor)
**Options:** Two reasonable large-unit choices with practical-size reasoning; two clearly wrong
**Example:**
Q: "What unit would you use to measure a classroom floor?"
A1: "Square centimeters — a classroom floor is made of many small tiles" (wrong — too many to count practically)
A2: "Square feet — a classroom floor is a large, flat space that we can measure with floor-tile-sized squares" (correct for US/customary)
A3: "Square meters — a classroom floor is a large area, and square meters cover big spaces quickly" (also correct for metric)
A4: "Square inches — an inch is small enough to measure the floor carefully" (wrong — impractical number)

#### Tier 3 Scaffolding (Least Supportive, Reasoning-Focused)
**Context Clue:** Boundary case where two small-area units compete (e.g., notebook cover, book page)
**Options:** Two plausible small units with reasoning; two clearly wrong large units
**Example:**
Q: "What unit would you use to measure the cover of a notebook?"
A1: "Square feet — a notebook is about the size of a floor tile" (wrong — too big)
A2: "Square meters — a notebook could fit inside a big rug" (wrong — much too big)
A3: "Square inches — a notebook cover is small, and square inches are good for small flat things" (correct; US standard)
A4: "Square centimeters — a notebook cover is small, and square centimeters measure small areas well" (also correct; metric standard; both accepted)

#### Implementation Notes
- **All MC options must include both unit AND reasoning statement** (not just the unit name).
- **Tier progression:** Warm-Up has no MC (comparison only). Early Lesson has no MC. Mid Lesson begins with Tier 1 after "too many" visual. Late Lesson includes Tier 1 and Tier 2. Practice includes all tiers mixed.
- **Feedback:** For selected option, reveal reasoning and unit size. For incorrect options, provide teaching feedback (e.g., "Square centimeters are too small for a playground—that would take thousands of squares!").
- **Real-world images:** Each MC item must include a visual context image (photo or illustration of the area being measured).

### QUATERNARY INTERACTION: Drag-to-Rank (Synthesis)

#### Purpose
Synthesis includes TWO drag interactions plus a bridge statement, per TVP. First, students rank unit squares by size (reasoning from labels, not visuals). Second, students match each unit to a real-world context. Together these consolidate M6's core insight: choosing the right unit is a practical decision based on area size.

#### Configuration

| Element | Specification |
|---------|---------------|

## NOTES FOR NEXT AUTHOR & IMPLEMENTATION TEAM

### For Tool Designers (Grid Rectangles & Equation Builder)
1. **Grid Rectangles Dense CM Mode:** Requires two-display capability (dense cm grid + clean ft grid side-by-side). Same rectangle, no scaling.
2. **Unit Label Consistency:** "Square feet," not "Feet²" or "ft²" in student-facing interface. Allow toggle to abbreviations for practice labels.
3. **Equation Builder Unit Labels:** Ensure multiplication equations display units clearly. Example: "3 feet × 4 feet = 12 square feet" (not just numbers).

### For Content Authors
1. **Real-World Context Images:** Create or source 12–15 varied, classroom-familiar contexts:
   - Obvious large (playground, gymnasium, parking lot)
   - Obvious small (postcard, book cover, phone screen)
   - Medium/ambiguous (classroom floor, bulletin board, garden plot)
   - Boundary cases (notebook cover, desk surface)

   All images should be age-appropriate, diverse in representation, and clearly show the object being measured.

2. **MC Reasoning Options:** For each Tier level, provide three option sets (minimum 6 MC items across all Tier levels). Ensure reasoning statements are grade-3-appropriate and promote transfer.

3. **Practice Problem Set:** Design 15–20 problems mixing all four units, contexts, and grid sizes (factors 2–10, products ≤100). Include 3–4 unit-selection problems (context image + MC).

### For Instructional Designer
1. **Warm-Up Pacing:** Allow 5–7 minutes for three-way unit comparison + text note on sq m. Students may ask "how big is a meter?" — prepare real-world anchor (e.g., "a little taller than a yard").

2. **"Too Many" Moment:** This is pivotal for M8 misconception prevention. Do NOT rush. Allow students to visually process the contrast for 30–45 seconds before explaining. Visual impact without counter is the strategy.

3. **EXIT CHECK Timing:** 5–8 minutes for four problems. Allocate most time to problem (3) and (4) because reasoning is required. Provide sentence frames if needed: "I would use _____ because _____."

4. **Synthesis Bridge Language:** Use explicit language: "You can measure any rectangle now using any of these four units. But in the next lesson, you won't always have a grid to count. What will you do instead?" This primes grid-fading anticipation.

### For Assessment Designer
1. **Vocabulary Precision in Rubrics:** EXIT CHECK must use full unit names ("square feet," "square meter") in problem statements. Student responses should use full names or accept abbreviated forms only if clearly labeled elsewhere.

2. **Reasoning Rubric for Unit Selection:** Develop a 3-level rubric:
   - Level 3: Unit choice is appropriate AND reasoning is mathematically sound ("It's a big area, so I need a big unit").
   - Level 2: Unit choice is appropriate but reasoning is vague or incomplete.
   - Level 1: Unit choice is inappropriate or reasoning is absent.

3. **Mixed-Unit Assessment Items:** EXIT CHECK problem (4) should mix units across multiple problems (one sq ft, one sq m, one sq in, one sq cm). Do not cluster by unit.

### For M7 Transition Planning
1. **Grid Fading Readiness:** M6 EXIT CHECK and Practice data will indicate which students are ready for M7 grid fading. Students who struggle with grid counting or unit selection in M6 will struggle more in M7 with dimensions-only reasoning.

2. **Drag-to-Rank Outcome:** Use Synthesis drag-to-rank results to gauge unit ranking fluency. Low performance signals need for additional M6 practice before M7 entry.

3. **Vocabulary Anchors:** In M7, revisit "square feet," "square meters," etc. briefly, but assume mastery from M6. Do not re-teach naming conventions.

## APPENDIX: PHASE-BY-PHASE LEARNING SEQUENCE OVERVIEW

> **Note:** Phase timings below are advisory estimates for planning purposes. Final timing will be determined in Phase Playbooks.

### Warm-Up (5–7 min)
**Goal:** Activate prior knowledge (sq in, sq cm from M5); introduce new units (sq ft, sq m); begin unit comparison reasoning.
**Activities:** Three 1×1 grid squares side-by-side (sq cm, sq in, sq ft). Text note: "A square meter is even BIGGER." No MC; no calculation.
**Toys:** Grid Rectangles (Warm-Up config)
**Vocabulary Introduced:** square foot, square meter (by visual and label)

### Early Lesson (8–10 min)
**Goal:** Reinforce M5 procedure (count grids, multiply dimensions) applied to new units (sq ft, sq m).
**Activities:** 2–3 sq ft problems, 1–2 sq m problems. Full grids; rows/columns countable. Callback to M5: "Remember how we counted grids for square inches? We do the same for square feet."
**Toys:** Grid Rectangles (individual configs, one unit per problem)
**Vocabulary Reinforced:** square foot, square meter, area, dimensions, multiply

### Mid Lesson (10–12 min)
**Goal:** Develop understanding of "too many" / why larger units are needed. Introduce unit-selection reasoning (Tier 1 scaffold).
**Activities:**
- Dense cm grid vs. clean ft grid (same rectangle) — visual contrast, no counting.
- MC unit-selection (Tier 1: obvious context, scaffolded options).
**Toys:** Grid Rectangles (dense cm + clean ft config); MC interaction
**Vocabulary Reinforced:** square foot, square meter, standard unit, appropriate, practical

### Late Lesson (8–10 min)
**Goal:** Apply unit selection independently with context clues (no scaffolded options). Continue grid fluency.
**Activities:** Mixed unit types (any of four); context images guide selection; students count/multiply and record area.
**Toys:** Grid Rectangles (mixed units); context images
**Vocabulary Reinforced:** all four units, area, dimensions, choose, context

### EXIT CHECK (5–8 min)
**Goal:** Assess procedural fluency (new units) + unit-selection reasoning (formal assessment).
**Activities (4 problems per TVP — matches §1.5 configuration):**
1. Find area in square feet (1 rectangle, full grid).
2. Find area in square meters (1 rectangle, full grid).
3. Select best unit for three real-world contexts (1 compound problem, with written reasoning).
4. Context image + grid rectangle → identify unit, calculate, state with unit (1 combined problem).
**Toys:** Grid Rectangles; context images; open-ended response
**Vocabulary:** Full names required; reasoning statements evaluated.

### Practice (10–15 min, async)
**Goal:** Reinforce fluency with mixed units and unit-selection reasoning via spaced repetition.
**Activities:** 15–20 problems mixing all four units, contexts, grid sizes. Self-check or teacher-marked.
**Toys:** Grid Rectangles; context images
**Vocabulary:** Consistent with EXIT CHECK; unit abbreviations acceptable with clear labels.

### Synthesis (8–10 min)
**Goal:** Consolidate unit system; rank all four units; bridge to M7 (grid fading).
**Activities:**
- Interaction 1: Drag-to-rank four unit squares (all same size visually) from smallest to largest. Feedback uses real-world anchors per TVP.
- Interaction 2: Match each unit to a real-world context by dragging (sq cm → fingertip/sticky note, sq in → book cover, sq ft → floor tile/rug, sq m → playground).
- Reinforce inverse relationship: "Fewer of a larger unit = same area."
- Bridge statement: "You can find area with ANY unit now. But you've always had a grid to count. What if there's no grid?" [M7 preview]
**Toys:** Drag-to-rank interaction; Drag-to-match interaction
**Vocabulary:** Minimal text; visual/reasoning-focused.
