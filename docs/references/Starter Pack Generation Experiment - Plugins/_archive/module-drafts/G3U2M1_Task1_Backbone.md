## **1.0 THE ONE THING**

Students must understand that **area is the amount of space a shape covers, and we can measure area by counting how many unit squares fit inside the shape.**

**Critical Misconception:** #1 Gaps/Overlaps Acceptable — Students think tiling with gaps or overlaps is valid measurement. M1 previews this misconception; M2 formally addresses it. The comparison activities also surface the tendency to equate "taller" with "larger," but this is a design feature of the comparison sets (see §1.5.1 Guardrails), not a recurring misconception targeted across the unit.

**Success Indicator:** Given a rectangle and unit square tiles, the student tiles the shape completely, counts the tiles, and states the area in square units.

**Biggest Risk:** Rushing to formal measurement before students experience the *need* for it. If comparison activities don't create genuine uncertainty, the transition to tiling feels arbitrary rather than motivated.

## **1.4 MISCONCEPTIONS**

### **1.4.1 #1: Gaps/Overlaps Acceptable (PRIMARY — previewed in M1, formally addressed in M2)**

*Global ID: #1 from Misconceptions database*

**Trigger Behavior:** When asked to find area, student counts tiles even when they overlap or leave gaps.

**Why It Happens:** Students focus on the counting task and don't monitor placement quality. In physical tiling, gaps and overlaps happen naturally with imprecise motor control.

**Visual Cue:** In M1, gap/overlap feedback is NOT active (per Notion Spec — M1 allows gaps and overlaps without feedback). The Synthesis previews this by showing a "bad" tiling.

**Prevention Strategy:** In M1, this is only PREVIEWED, not formally addressed. The Synthesis shows square tiles vs. circular tiles (squares fit with no gaps) and previews a tiling with gaps/overlaps as the bridge to M2. M2 is where gap/overlap rules are formally taught. In M1, students may produce imperfect tilings — this is expected and acceptable.

*Note: The Module Mapping also lists "Students confuse 'larger' with 'taller' or 'wider'." This is a design feature of the M1 comparison activities (see §1.5.1 Guardrails — at least one set where taller shape has LESS area), not a standalone misconception targeted across the unit. It does not have a global ID in the Misconceptions database.*

| **Interaction (Comparison)** | MC selection — student clicks one shape or "They cover the same space" |
| **Interaction (Tiling)** | Not interactive — serves as target surface for Unit Square Tiles |
| **Fill** | Solid color fill, clear boundary. Color should not distract from area comparison. |
| **Scale** | Consistent visual scale within each comparison set (shapes are to-scale relative to each other) |

#### M1 Guardrails

| DO | DO NOT |
| :---- | :---- |
| Use varied polygon types in comparison activities | Use only rectangles for comparison (creates "area = rectangle thing" misconception) |
| Include at least one set where taller shape has LESS area | Assume students will discover the "taller ≠ larger" insight without a designed counterexample |
| Use rectangles-only as tiling targets in Mid and EC | Use non-rectangular tiling targets in Mid or EC (accidental gaps before M2 rules) |
| Include 1-2 rectilinear non-rectangles in Late tiling | Use curved shapes, angled shapes, or shapes that don't tile cleanly |
| Keep comparison sets authored and sequenced (not random) | Generate comparison shapes randomly (difficulty progression is load-bearing) |

#### Progression Within M1

| Phase | Configuration |
| :---- | :---- |
| **Warmup** | Comparison display only. 2 sets of 3 shapes. MC selection + "Same Space" option. |
| **Lesson (Early)** | Comparison display only. 2 more sets of 3 shapes. Same MC interaction. Escalating difficulty. |
| **Lesson (Mid)** | Not used. Rectangular tiling targets use Grid Rectangles (§1.5.3). |
| **Lesson (Late)** | Tiling target for 1-2 L/T shapes only. Area 6-20 sq units. No grid. Rectangular targets still use Grid Rectangles. |
| **Exit Check** | Not used. EC uses Grid Rectangles for rectangular tiling targets. |
| **Synthesis** | Comparison callback. Grid overlay revealed on warm-up shapes for area verification. Static circular tiles illustration for "why square tiles?" moment. |

#### Comparison Set Sequence (Authored)

| Set | Phase | Shape Types | Difficulty | Design Intent |
| :---- | :---- | :---- | :---- | :---- |
| **Set 1** | Warmup | Triangles (3 triangles, different sizes) | Easy | Same shape type, obvious size differences. Confidence builder. |
| **Set 2** | Warmup | Mixed polygons (hexagon, trapezoid, triangle) | Medium | Different shape types, different side counts. Cross-category comparison. |
| **Set 3** | Lesson Early | Parallelograms/quadrilaterals | Medium-Hard | Visually harder to compare. Students hesitate. Include taller-has-less-area case. |
| **Set 4** | Lesson Early→Mid | Rectangles (tall/narrow vs. short/wide vs. square-ish) | Hard | Genuinely close in area. Creates the "our eyes can trick us" moment. Motivates measurement. |

*Design Note: At least one set must have "They cover the same space" as the correct answer. This prevents students from learning to always eliminate that option.*

### **1.5.2 Unit Square Tiles**

**Notion Spec:** [Unit Square Tiles](https://www.notion.so/ocpgg/Unit-Square-Tiles-2fb5917eac52801185a0f9e4e942bf83)
**Changes from prior:** First appearance in Unit 2.

#### Module Configuration (M1)

| Aspect | This Module |
| :---- | :---- |
| **Unit Type** | Generic — "square unit" (not sq cm, sq in, etc.) |
| **Grid Mode (Target Surface)** | No grid — empty shape outline only. Students CREATE coverage through tiling. |
| **Snap Mode** | Quarter-tile snap (half linear measure). Semi-freeform: students must place deliberately, alignment is not automatic, but extreme misplacement is prevented. |
| **Gap Handling** | Allowed — no feedback. M1 focus is covering and counting, not placement precision. |
| **Overlap Handling** | Allowed — no feedback. Gaps/overlaps previewed at end but formally taught in M2. |
| **Tile Supply (Early/Guide demos)** | Exact match — bank contains exactly the tiles needed. Empty bank = shape fully covered. |
| **Tile Supply (Mid-Late)** | Modest surplus — bank contains more tiles than needed. Student judges completeness. |
| **Interaction** | Drag-to-place (not tap) per Decision #6. Dragging reinforces the covering/tiling concept. |
| **Side Length Labels** | Available via tap/click but not proactively shown. Available for future reference. |

#### M1 Guardrails

| DO | DO NOT |
| :---- | :---- |
| Use drag-to-place for all tiling interactions | Use tap-to-place (Decision #6: dragging is more effective for area understanding) |
| Start with exact tile supply, progress to surplus | Provide unlimited/infinite tiles (prevents aimless dragging) |
| Let students count tiles one-by-one | Expect or prompt structured counting (M3) |
| Allow imperfect tilings (gaps/overlaps) without feedback | Show gap/overlap error feedback (that's M2) |
| Guide vocabulary introduction AFTER first tiling experience | Introduce "unit square" / "square unit" before students have tiled a shape |

#### Progression Within M1

| Phase | Configuration |
| :---- | :---- |
| **Lesson (Mid — Introduction)** | Exact tile supply. Rectangles only (2×5, 3×4 range). Guide demonstrates first, then student tiles. No grid on target. |
| **Lesson (Late — Independent)** | Modest surplus tile supply. Rectangles + 1-2 L/T shapes. No grid. Student tiles and counts independently. Area ≤ 20 sq units. |
| **Exit Check** | Rectangles only. Student tiles and states area. Some problems partially pre-tiled (SUPPORT tier — triggers after 2 incorrect). |

#### Counter (Remediation Only)

**The Counter is NOT a standard scaffold in M1.** Per SME decision: "Students count tiles themselves — the counting is the learning, and manual counting creates the M3 motivation for structured counting."

Counter is available as **guide-activated remediation** for students who demonstrate repeated inability to track a running count (2+ miscounts on consecutive problems where tiling is correct but stated area is wrong). When activated:
- Guide maintains counting connection verbally
- Logged for diagnostics (flags potential working-memory support need)
- NOT available during exit checks

*Design Note: Students who count SLOWLY are building the M3 motivation — do not remediate. Students who count INACCURATELY despite correct tiling have a tracking issue — the counter removes an unrelated barrier.*

#### Data Constraints

| Constraint | Value |
| :---- | :---- |
| Area range (all M1 activities) | 6–20 square units |
| Tiling targets (Mid, EC) | Rectangles only |
| Tiling targets (Late) | 2-3 rectangles + 1-2 rectilinear non-rectangles (L/T shapes). All right angles, integer dimensions. |
| No partial tiles | All shapes tile cleanly with unit squares |
| No non-rectilinear tiling targets | No curves, no diagonals |
| No decomposition guidance on non-rectangular shapes | Students tile and count only; decomposition belongs to M11 |
| Tile supply (early) | Exact match |
| Tile supply (mid-late) | Modest surplus (never infinite) |

### **1.5.3 Grid Rectangles (Secondary)**

**Notion Spec:** [Grid Rectangles](https://www.notion.so/ocpgg/Grid-Rectangles-2fb5917eac528035a19dc2b5b49aeeca)
**Changes from prior:** First appearance in Unit 2.

*Design Note: The TVP describes tiling targets as "Plane Figures," but Plane Figures are display-only polygons used for visual comparison. The rectangular tiling surfaces (where students place Unit Square Tiles) are Grid Rectangles in blank mode. Plane Figures handles the varied-polygon comparison activities; Grid Rectangles handles the rectangular tiling targets. This separation matters for engineering — the two toys have different rendering pipelines. For the 1-2 rectilinear non-rectangles in Late activities (L/T shapes), Plane Figures serves as the tiling target since Grid Rectangles only supports rectangles. Flag for author confirmation.*

#### Module Configuration (M1)

| Aspect | This Module |
| :---- | :---- |
| **Role** | Tiling target surface for rectangular shapes. Students place Unit Square Tiles onto Grid Rectangles. |
| **Grid State** | No grid visible (`grid_state: "none"` or equivalent blank mode). Students CREATE coverage through tiling — the grid is not pre-revealed. |
| **Unit Type** | Generic — "square units" |
| **Dimension Range** | Within M1 constraints: area 6–20 square units. Typical: 2×5, 3×4, 2×8, 4×5. |
| **Dimension Labels** | NOT shown. Students determine area by counting tiles, not by reading dimensions. |
| **Highlighting** | Not used in M1. Row/column highlighting begins in M3. |

#### M1 Guardrails

| DO | DO NOT |
| :---- | :---- |
| Use blank mode (no grid lines, no dimension labels) | Show grid lines or labels (students must tile to discover area) |
| Use only for rectangular tiling targets | Use for non-rectangular shapes (those use Plane Figures as target) |
| Keep within 6–20 sq unit area range | Exceed 20 sq units (countability ceiling for M1) |

#### Visibility Rules

Grid Rectangles appears in: Lesson Mid, Lesson Late (rectangles only), Exit Check.
Grid Rectangles does NOT appear in: Warmup, Lesson Early (those use Plane Figures for comparison), Synthesis (uses Plane Figures with grid overlay for callback).

#### [Module-Specific] Toy Spec Update Needed

The Grid Rectangles Notion spec does not currently document a fully blank mode (`grid_state: "none"` with no dimension labels). The most abstract state documented is "Labels Only." M1 needs a state below Labels Only — a blank rectangle outline with no internal markings and no edge labels, used as a tiling target surface. **Recommend adding a `blank` or `tiling_target` grid state to the spec.**

# OUTPUT 6: Section Plan

## Section Plan — How Content Distributes Across Phases

**Warmup (3-5 min):** Prior knowledge activation via visual comparison of plane figures. 2 authored comparison sets (triangles, then mixed polygons). Students select which covers more space from 3 shapes (A/B/C) or "Same Space." Quick success, builds confidence, activates informal "which is bigger" reasoning. Bridge teases that "bigger" isn't always what it seems.

**Lesson Section 1 — Escalating Comparison (3-4 min):** 2 more comparison sets (parallelograms, then rectangles). Difficulty escalates. Set 3 introduces the "taller has less area" counterexample — key misconception surface. Set 4 creates genuine visual uncertainty with rectangles of different proportions. The "close call" motivates measurement: "Our eyes can trick us. We need a TOOL." Major pedagogical move: creating the need for measurement before introducing the tool.

**Lesson Section 2 — Tiling Introduction (5-7 min):** Unit Square Tiles introduced with guide demonstration. Students drag tiles onto rectangular shapes. First tiling is guided (worked example with think-aloud), then 1-2 more with decreasing support. Vocabulary introduced AFTER first tiling experience: "square tiles" → "unit square" / "square unit" → "area." Key pedagogical move: vocabulary staging after visual experience.

**Lesson Section 3 — Independent Tiling (5-7 min):** Student tiles 4-5 shapes independently. Rectangles first, then 1-2 L/T shapes. States area in square units each time. No structured counting expected — one-by-one is appropriate. Tile supply shifts from exact to surplus. No decomposition guidance on non-rectangular shapes.

**Exit Check (3-5 min):** 3 problems testing tiling + counting. Rectangles only. Student tiles shape → counts → states area in square units. Cognitive types: CREATE (tiling) and IDENTIFY (stating area). Some partially pre-tiled problems as SUPPORT tier.

**Synthesis (5-7 min):** Callback to warm-up comparison — now verify with tile measurement. "Earlier we GUESSED. Now we can MEASURE." Vocabulary consolidation. Why square tiles? (square vs. circular tiles illustration). Bridge to M2: preview a "bad" tiling with gaps/overlaps.
