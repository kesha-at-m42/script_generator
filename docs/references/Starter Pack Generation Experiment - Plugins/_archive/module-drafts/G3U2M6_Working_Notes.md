## TABLE B: TVP (TEACHING & LEARNING PROGRESSION) — M6 SECTION (Verbatim)

**Learning Goal:** Use square feet and square meters to measure the area of a rectangle. Choose appropriate square units for real-world contexts.

### Key Teaching Points:
1. A square foot is a square with sides of 1 foot; a square meter is a square with sides of 1 meter — same naming convention as M5
2. Larger spaces need larger units — it would take TOO MANY square centimeters to measure a playground
3. THE BIG INSIGHT: Choosing the right unit is a practical decision based on area size. Square centimeters for a phone screen; square feet for a classroom.
4. Students already know sq in and sq cm from M5 — extends same principle to larger units
5. Inverse relationship reinforced: fewer of a larger unit cover the same space

### CRA Stage:
Representational — full grids continue (LAST module before grid fading begins in M7)

### Cognitive Focus:
- IDENTIFY (Conceptual) — recognizing when larger units are appropriate
- COMPARE (Conceptual) — understanding relative sizes of all four standard units
- APPLY (Transfer) — selecting appropriate unit for context
- CONNECT (Transfer) — linking all four standard units into a coherent system

### Misconceptions Targeted:
- M8: Square Unit Confusion (PRIMARY) — addressed by side-by-side comparison, "too many" counter experience, unit-selection reasoning
- "More squares = more area" (SECONDARY) — Reinforced from M5: same area, different units, different numbers

### What Students DO:

**WARM-UP: Grid Rectangles (1×1, side-by-side — three unit types: sq cm, sq in, sq ft)**
- Size comparison
- Text note: "A square meter is even BIGGER than a square foot."
- Scale note: True-to-scale sq cm to sq m impractical. Guide must acknowledge screen limitation.

**LESSON:**

*Early (Naming new units):*
- Grid Rectangles 1×1 foot/meter individually
- Same naming convention
- Callback to warmup
- 2-3 problems sq ft, 1-2 sq m

*Mid (Why larger units — "too many" experience):*
- Dense cm grid on large-context rectangle → clean ft grid on same rectangle
- Visual contrast is mechanism (no counter)
- MC unit-selection with reasoning in options (Tier 1/2/3 scaffolding)
- Unit-selection interaction with four options (sq cm, sq in, sq ft, sq m)

*Late (Independent practice all units):*
- Single unit type per problem, any of four units
- Context clues guide unit selection

**EXIT CHECK: 4 problems:**
1. Find area with foot grid, state in square feet
2. Find area with meter grid, state in square meters
3. Three real-world contexts → select most appropriate unit each
4. Context image + grid rectangle → identify unit, calculate, state with unit

**PRACTICE: Mixed all four unit types.**
- Unit-selection problems with context images

**SYNTHESIS:**
- Ranking interaction: Drag-to-rank four unit squares smallest→largest (sq cm → sq in → sq ft → sq m)
- ALL SAME SIZE ON SCREEN, differentiated only by labels (reasoning task, not perception)
- Match each unit to real-world context by dragging
- Reinforce inverse relationship
- Bridge to M7: "You can find area with ANY unit now. But you've always had a grid to count. What if there's no grid?"

### Data Constraints:
- Square foot: Factors 2–10, products within 100
- Square meter: Factors 2–10, products within 100
- Review problems (sq in, sq cm): Same as M5
- "Too many" moment: dramatic contrast (e.g., 4ft×3ft=12 sq ft vs same in cm = thousands)
- Unit selection contexts: Tier 1 (obvious large/small), Tier 2 (sq ft vs sq m), Tier 3 (sq cm vs sq in, both accepted)

### Tool Requirements:
- Side-by-side unit comparison (3 simultaneous Grid Rectangles at 1×1)
- Dense cm grid vs clean ft grid for "too many" moment
- Drag-to-rank standalone interaction for Synthesis

### Vocabulary:
- New: square foot, square meter
- Reinforced: square inch, square centimeter, standard unit, area, dimensions
- Accept "sq ft" informally but require full name on exit check

### Key Transitions:
- **M5→M6:** Two standard units → Four standard units + unit selection reasoning
- **M6→M7:** Complete unit system; full grids → Grid fading begins (partial → tick marks → dimensions only)

### SME Resolved Decisions:
- Unit selection depth: RESOLVED — MC options include unit AND reasoning (Tier system)
- Metric vs. customary: RESOLVED — 50-50 split
- All four units in one module: RESOLVED — appropriate consolidation
- Counter speed: RESOLVED — Counter removed; dense grid visual is the mechanism
- Borderline contexts: RESOLVED — accept both reasonable units for borderline cases
