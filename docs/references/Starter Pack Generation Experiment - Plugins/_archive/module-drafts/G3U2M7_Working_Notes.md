## 2. CROSS-REFERENCE TABLE B: TVP (Teaching & Vocabulary Progression — Full M7 Phase Flow)

### **LEARNING GOAL**
Determine the area of rectangles not displayed on a grid.

### **KEY TEACHING POINTS**

1. **You don't need to SEE every square to know how many there are** — side lengths tell you
2. **If you know 4 rows of 7, you know there are 28 squares WITHOUT counting each one**
3. **THE BIG INSIGHT: The grid was always just a visual confirmation of what the dimensions already told you. Dimensions alone are sufficient.** This is the moment "area = length × width" stops being a formula and starts being a TOOL.
4. **Students have been building toward this since M3 (rows/columns) and M4 (multiplication)** — M7 removes the visual scaffolding
5. **Portuguese azulejos (decorative tiles)** available as optional enrichment context — not core

### **CRA STAGE**
**Abstract** — grid fading is the central mechanism. First module WITHOUT full visual support. Progression: full grid (reference only) → partial grid → tick marks → dimensions only.

### **COGNITIVE FOCUS**
- **APPLY (Transfer)** — using multiplication to find area without counting
- **CONNECT (Transfer)** — linking visible partial structure to complete area
- **IDENTIFY (Conceptual)** — recognizing that dimensions encode the full grid structure

### **MISCONCEPTIONS TARGETED**
- **M5: Not Multiplying Side Lengths** (PRIMARY, HIGH) — Students read dimensions correctly but don't connect to multiplication. May add side lengths or leave as two numbers.
- **M8: Square Unit Confusion** (SECONDARY, MEDIUM) — Without grids, students may lose track of units.

### **PHASE-BY-PHASE LESSON FLOW**

#### **WARM-UP: Grid Rectangles (full grid, familiar)**
- Full grid, familiar rectangle (e.g., 4×6)
- Quick area calculation
- Then SAME rectangle with grid removed — only outline + dimension labels
- **Key Message:** "The rectangle didn't change. The area didn't change. We just can't SEE the squares anymore."

#### **EARLY PHASE (Partial Grid)**
- **Activity:** Grid Rectangles (partial grid mode — first row fully tiled, first column fully tiled, interior empty)
- **Student Action:** See one complete row and count rows from column. "I see 7 in each row and 4 rows, so 4 × 7 = 28."
- **Constraint:** One dimension ≤ 5 for partial grid
- **Volume:** 3-4 rectangles
- **Tool:** Equation Builder (multiplication, unlabeled slots)

#### **MID PHASE (Tick Marks)**
- **Activity:** Grid Rectangles (tick mark mode — evenly spaced marks along top and left edges, no interior)
- **Student Action:** Read dimensions from tick marks, multiply
- **Volume:** 3-4 rectangles increasing dimensions
- **Tool:** Equation Builder with unit labels
- **Critical Detail:** Unit labels required — all four unit types

#### **LATE PHASE (Dimensions Only)**
- **Activity:** Grid Rectangles (outline mode — rectangle outline with dimension labels, no grid, no ticks)
- **Key Pedagogical Beat:** Show tick-marked rectangle, add dimension labels, fade ticks. "The number already tells you what the tick marks showed."
- **Volume:** 4-5 rectangles, varied units
- **Text-Only Extension:** "A rectangle is 7 feet long and 4 feet wide. What is its area?"
- **Template (Rigid):** "A [object] is [number] [unit] long and [number] [unit] wide. What is its area?"

#### **EXIT CHECK: 3 Problems**
1. **Partial grid** → state area with correct unit
2. **Dimensions-only** → multiply and state area with correct unit
3. **Text-only** — dimensions in words, no visual → calculate area

### **PRACTICE**
- Mixed partial grid, tick marks, dimensions-only
- All four units
- Forward and reverse problems (50/50 ratio)
- Reverse tagged as TRANSFER

### **SYNTHESIS**
- Progression display: same rectangle in four states (full grid → partial → ticks → dimensions only)
- Callback to M3-M4 progression
- Vocabulary consolidation: side length, dimension
- **Bridge to M8:** "You found area when dimensions were GIVEN. What if you know the AREA and need to figure out what the dimensions could be?"

### **DATA CONSTRAINTS BY SECTION**

| Phase | Factor Constraints | Product Limit | Notes |
|-------|-------------------|---------------|-------|
| **Partial Grid (Early)** | One factor ≤ 5, other 3–9 | ≤ 45 | ≤5 factor should be column (number of rows) |
| **Tick Marks (Mid)** | Factors 3–10 | ≤ 100 | Any valid pairing |
| **Dimensions-Only (Late)** | Factors 2–10 | ≤ 100 | All four unit types |
| **Text-Only (Late/EC)** | Factors 2–10 | ≤ 100 | Rigid sentence template |

### **TOOL REQUIREMENTS**
- **Partial grid mode:** First row + first column filled, interior empty
- **Tick mark mode:** Evenly spaced marks along top and left edges only
- **Mode transitions:** Same rectangle transitioning between modes (animated)
- **Guide reveal:** Guide can activate grid overlay on outline-mode rectangles for remediation (NOT student-facing). 2+ reveals → flag for M3 reinforcement.

### **VOCABULARY SCOPE**
- **New:** side length, dimension
- **Reinforced:** area, square foot, square meter, square inch, square centimeter, multiply, expression, factor, product

### **MODULE TRANSITIONS**
- **M6 → M7:** Full grids with all units → Grid fading (partial → tick marks → dimensions only)
- **M7 → M8:** Students can find area from dimensions alone → Forward fluency + reverse reasoning (area → dimensions)

## 4. DESIGN CONSTRAINTS: IMPORTANT DECISIONS EXTRACTION

### **CRITICAL DECISION #3: PROGRESSIVE GRID REMOVAL (PRIMARY TO M7)**
- **Statement:** Grid support progressively fades across modules.
- **M7 Implementation (THE Decision #3 Module):** Grid fading begins and completes.
  - Warm-up: Full grid (reference only, familiar)
  - Early: Partial grid (first row/column filled, interior empty)
  - Mid: Tick marks (evenly spaced marks on edges only)
  - Late: Dimensions only (outline with labels, no grid, no ticks)
- **Why This Matters:** M7 is the first module without full visual support. Students make the cognitive leap from "counting visible squares" to "using dimensions to infer the full structure."
- **Gating Constraint:** All three transitions must complete within M7. M8 assumes no grid support; no backsliding permitted.

### **DECISION #1: UNIFIED PATH (CONFIRMED)**
- No separate tracks
- All students navigate: partial grid → tick marks → dimensions only

### **DECISION #2: EXPLICIT SPATIAL STRUCTURING (CONFIRMED)**
- M7's partial grid makes structure visible one final time before it's removed
- Essential bridge between M6 (full grids) and M8 (no grids)

### **DECISION #4: ARRAY KNOWLEDGE CONTINUES (CONFIRMED)**
- M7 reinforces rows/columns from M3 and M4 multiplication

### **DECISION #5: NO PERIMETER (CRITICAL FOR M7)**
- Do NOT introduce perimeter concept to differentiate from area
- Students may add side lengths by mistake → don't pivot to perimeter
- Must address via multiplication reasoning alone

### **DECISION #6: DRAGGING OVER TAPPING (CONFIRMED)**
- Equation Builder uses drag-to-place interaction

### **DECISION #7: STANDARD UNITS FULL SCAFFOLDING (M5/M6 COMPLETE)**
- M7 uses all four units but doesn't introduce new ones
- All four unit types required across mid and late phases

### **DECISION #8: MULTIPLICATION TABLE (NOT M7)**
- Table reference available but not taught in M7 (M10)

### **DECISION #9: APPLICATION MODULE (NOT M7)**
- No real-world application module in Unit 2 (M14 in later units)
