# BACKBONE

---

## **1.0 THE ONE THING**

Students must understand that area is the amount of space a shape covers, and we can measure area by counting how many unit squares fit inside the shape.

**Critical Misconception:** "Taller = Larger" — Students confuse height or length with area, believing the taller shape always covers more space. The comparison sets are designed to surface this and the tiling tool resolves it by providing exact measurement. Gaps/overlaps (#1) is previewed in M1 Synthesis but formally addressed in M2.

**Success Indicator:** Given a rectangle and unit square tiles, the student tiles the shape completely, counts the tiles, and states the area in square units.

**Biggest Risk:** Rushing to formal measurement before students experience the *need* for it. If comparison activities don't create genuine uncertainty, the transition to tiling feels arbitrary rather than motivated.

## **1.2 SCOPE BOUNDARIES**

### **✅ Must Teach**

- Area as an attribute of a shape — how much space it covers
- Informal visual comparison of area ("which covers more space?")
- The transition from estimation to measurement — creating the *need* for a measurement tool
- Unit square as a measurement tool (each tile = 1 square unit)
- Area = the count of unit squares covering a shape
- Drag-to-place tiling of rectangles (mid) and rectilinear shapes (late)
- Vocabulary: area, plane figure, cover, space, unit square, square unit, tile, measure
- Bridging term "square tiles" used informally during tiling, before formal vocabulary

### **❌ Must Not Include**

- Gaps/overlaps rules (conceptual — "why" gaps/overlaps cause wrong counts) — M2. Mechanical overlap feedback is always active.
- Structured counting, skip-counting, or row/column organization — M3
- Area as multiplication (rows × columns) — M5
- Standard units (square centimeters, square inches, etc.) — M5-M6
- Perimeter — NOT in Unit 2 (Decision #5)
- Decomposition strategies for non-rectangular shapes — M11
- Grid visibility on target shapes — students build coverage through tiling
- Pattern blocks — skipped per SME decision
- Formula language ("length times width") — M7+

> ⚠️ **TALLER ≠ LARGER PRINCIPLE:** At least one comparison set must include a case where the taller/longer shape has LESS area than the shorter/more compact shape. This surfaces the common conflation of height with area and motivates the need for measurement.

> ⚠️ **CRA STAGE — CONCRETE:** M1 is heavy manipulation and intuition-building. Students tile and count. No structured counting, no skip-counting, no multiplication connection. One-by-one counting is appropriate and expected.

## **1.4 MISCONCEPTIONS**

### **1.4.1 "Taller = Larger" (PRIMARY — M1's core teaching target)**

*No global ID in Misconceptions database. M1-specific.*

**Trigger Behavior:** Student selects the taller or longer shape as "covering more space" without considering overall area. May describe shapes as "bigger" based on a single dimension.

**Why It Happens:** Students' everyday experience reinforces height as the primary measure of size ("I'm taller so I'm bigger"). Before formal area concepts, they lack a framework for two-dimensional comparison.

**Visual Cue:** Comparison Set 3 includes a taller shape that has LESS area than a shorter, wider shape. Set 4 uses rectangles with different proportions (tall/narrow vs. short/wide) where visual estimation fails.

**Prevention Strategy:** The 4-set comparison escalation is designed to surface and resolve this:
- Sets 1-2 (Warmup): Build confidence with easier comparisons
- Set 3 (Lesson Early): Equal-area counterexample — parallelogram and irregular hexagon cover same space (hexagon decomposes to match parallelogram). First "Equal" correct answer. Previews conservation-of-area concept.
- Set 4 (Lesson Early→Mid): Genuinely ambiguous rectangles create the "our eyes can trick us" moment
- Lesson Mid: Unit Square Tiles provide the resolution — exact measurement replaces visual estimation

---

### **1.4.2 #1: Gaps/Overlaps Acceptable (SECONDARY — previewed in M1, formally addressed in M2)**

*Global ID: #1 from Misconceptions database*

**Trigger Behavior:** When asked to find area, student counts tiles even when they overlap or leave gaps.

**Why It Happens:** Students focus on the counting task and don't monitor placement quality.

**Visual Cue:** Semi-freeform placement with immediate overlap feedback (visual color change + audio cue) per TVP. The mechanical feedback is always active — students see when tiles overlap. The Synthesis previews the conceptual rule by showing a "bad" tiling.

**Prevention Strategy:** In M1, the overlap feedback provides mechanical correction (the tool shows overlaps), but the conceptual rules (why gaps and overlaps produce wrong counts) are not formally taught until M2. The M1 Synthesis shows square tiles vs. circular tiles (squares fit with no gaps) and previews a tiling with gaps/overlaps as the bridge to M2.

**Purpose:** Display-only shapes for visual area comparison. Students compare 3 shapes per set and select which covers more space. Used in Warmup and Lesson Early to build the comparison-to-measurement arc. Also used in Synthesis for comparison callback with grid overlay to verify earlier guesses.

#### Module Configuration (M1)

| Aspect | This Module |
| :---- | :---- |

| **Comparison Sets** | 4 authored sets of 3 shapes each (A, B, C). Sequenced by difficulty. |

| **Grid Overlay** | No grid during comparison. Grid overlay available for Synthesis callback (system-controlled reveal). |
| **Interaction** | MC selection — student clicks one shape or "They cover the same space" |
| **Fill** | Solid color fill, clear boundary. Color should not distract from area comparison. |
| **Scale** | Consistent visual scale within each comparison set (shapes are to-scale relative to each other) |

#### M1 Guardrails

| DO | DO NOT |
| :---- | :---- |
| Use varied polygon types in comparison activities | Use only rectangles for comparison (creates "area = rectangle thing" misconception) |
| Include at least one set where taller shape has LESS area | Assume students will discover the "taller ≠ larger" insight without a designed counterexample |
| Keep comparison sets authored and sequenced (not random) | Generate comparison shapes randomly (difficulty progression is load-bearing) |

#### Progression Within M1

| Phase | Configuration |
| :---- | :---- |
| **Warmup** | Comparison display only. 2 sets of 3 shapes. MC selection + "Same Space" option. |
| **Lesson (Early)** | Comparison display only. 2 more sets of 3 shapes. Same MC interaction. Escalating difficulty. |
| **Lesson (Mid–Late)** | Not used. Tiling activities use Unit Square Tiles (§1.5.2). |
| **Exit Check** | Not used. |
| **Synthesis** | Comparison callback. Grid overlay revealed on warm-up shapes for area verification. |

#### Comparison Set Sequence (Authored — mockups complete)

| Set | Phase | Shapes | Difficulty | Design Intent |
| :---- | :---- | :---- | :---- | :---- |
| **Set 1** | Warmup | 3 right triangles, clearly different sizes | Easy | Same shape type, obvious size differences. Confidence builder. |
| **Set 2** | Warmup | Regular hexagon (A), isosceles trapezoid (B), equilateral triangle (C) | Medium | Different polygon types, different side counts. Cross-category comparison. Correct = A (hexagon is 2× trapezoid area; triangle is taller distractor). |
| **Set 3** | Lesson Early | Parallelogram (A), irregular hexagon (B), parallelogram (C) | Medium-Hard | Equal-area counterexample. A and B have same area — hexagon decomposes to match parallelogram. Correct = Equal. Different shape types covering same space. |
| **Set 4** | Lesson Early→Mid | 3 rectangles with different proportions (tall/narrow, medium, short/wide) | Hard | Genuinely close in area. Creates the "our eyes can trick us" moment. Motivates measurement. |

MC options for all sets: A, B, C, Equal.

*Design Note: Set 3 (Lesson Interaction 1.1) has "Equal" as the correct answer. This prevents students from learning to always eliminate that option. Placed in the Lesson rather than Warmup so the "Equal" revelation has more pedagogical weight — students have seen 2 Warmup sets where one shape clearly wins, making the Equal outcome surprising.*

> ⚠️ **AUTHOR FLAG (Conflict 4):** The TVP and handwritten sketches consistently use 3 shapes per set. The Plane Figures Notion spec says 2 shapes per comparison. Recommend following TVP and updating spec. Needs author confirmation.

### **1.5.2 Unit Square Tiles**

**Notion Spec:** [Unit Square Tiles](https://www.notion.so/ocpgg/Unit-Square-Tiles-2fb5917eac52801185a0f9e4e942bf83) | **Changes from prior:** First appearance in Unit 2.

**Purpose:** The measurement tool students use to determine area. Unit Square Tiles renders its own tiling target surface (shape outline) — it does not require a separate toy for the target destination. Introduced in Lesson Mid after comparison activities create the need for measurement. Students drag tiles onto shapes, count them, and state area in square units. Establishes the foundational tiling experience that M2-M6 build upon.

#### Module Configuration (M1)

| Aspect | This Module |
| :---- | :---- |
| **Unit Type** | Generic — "square unit" (not sq cm, sq in, etc.) |
| **Target Surface** | Unit Square Tiles renders its own shape outline. No grid visible — students CREATE coverage through tiling. |
| **Target Shapes (Mid, EC)** | Rectangles only (2×5, 3×4, 2×8, 4×5 range). |
| **Target Shapes (Late)** | Rectangles + 1-2 rectilinear non-rectangles (authored shapes). All right angles, integer dimensions. |
| **Snap Mode** | Quarter-tile snap (half linear measure). Semi-freeform placement. |
| **Overlap Feedback** | Active — immediate visual color change + audio cue on overlap. Same tiling behavior as M2; consistent interaction across all tiling modules. |
| **Gap Feedback** | No active gap feedback in M1. Students may leave small gaps; the conceptual rule (no gaps for accurate measurement) is formally taught in M2. |
| **Tile Supply (Early/Guide demos)** | Exact match — bank contains exactly the tiles needed. Empty bank = shape fully covered. |
| **Tile Supply (Mid-Late)** | Modest surplus — bank contains more tiles than needed. Student judges completeness. |
| **Interaction** | Drag-to-place (not tap) per Decision #6. Dragging reinforces the covering/tiling concept. |
| **Side Length Labels** | Available via tap/click but not proactively shown. |

#### M1 Guardrails

| DO | DO NOT |
| :---- | :---- |
| Use drag-to-place for all tiling interactions | Use tap-to-place (Decision #6: dragging is more effective for area understanding) |
| Start with exact tile supply, progress to surplus | Provide unlimited/infinite tiles (prevents aimless dragging) |
| Let students count tiles one-by-one | Expect or prompt structured counting (M3) |
| Keep overlap feedback active (mechanical correction) | Turn off overlap detection (it's always on; the conceptual rule waits for M2) |
| Guide vocabulary introduction AFTER first tiling experience. Two-question pattern: "How many unit squares?" then "What is the area in square units?" | Introduce "unit square" / "square unit" before students have tiled a shape. Single-question format that conflates tool and measurement terms. |

#### Progression Within M1

| Phase | Configuration |
| :---- | :---- |
| **Lesson (Mid — Introduction)** | Exact tile supply. Rectangles only (2×5, 3×4 range). Guide demonstrates first, then student tiles. No grid on target. |
| **Lesson (Late — Independent)** | Modest surplus tile supply. Rectangles + 1-2 rectilinear shapes. No grid. Student tiles and counts independently. Area ≤ 20 sq units. |
| **Exit Check** | Rectangles only. Student tiles and states area. Some problems partially pre-tiled (SUPPORT tier — triggers after 2 incorrect). |

#### Counter (Remediation Only)

**The Counter is NOT a standard scaffold in M1.** Per SME decision: "Students count tiles themselves — the counting is the learning, and manual counting creates the M3 motivation for structured counting."

Counter is available as **guide-activated remediation** for students who demonstrate repeated inability to track a running count (2+ miscounts on consecutive problems where tiling is correct but stated area is wrong). When activated:
- Guide maintains counting connection verbally
- Logged for diagnostics (flags potential working-memory support need)
- NOT available during exit checks

*Design Note: Students who count SLOWLY are building the M3 motivation — do not remediate. Students who count INACCURATELY despite correct tiling have a tracking issue — the counter removes an unrelated barrier.*

## **1.6 WARMUP (3-5 minutes)**

### **Purpose**

Activate students' informal intuitions about relative size by comparing plane figures. Students decide which shape "covers more space" — building the perceptual reasoning that the Lesson will then challenge (when visual estimation fails) and resolve (with unit square tiling).

### **Warmup Parameters**

| Element | Specification |
| :---- | :---- |
| **Time** | 3-5 minutes |
| **Interactions** | 2 comparison tasks + bridge (3 total) |
| **Format** | Authored comparison sets — MC selection from 3 shapes + "Equal" |
| **Cognitive Load** | 20-30% (accessible, judgment-based) |
| **Remediation** | Pipeline |
| **Tone** | Medium-High energy, curious, inviting |
| **Engagement Anchors** | Choice/Agency (MC selection requiring genuine visual judgment), Narrative Setup ("which covers more space" — real comparison framing that serves the concept) |

### **Constraints**

| USE | AVOID |
| :---- | :---- |
| Informal language ("covers more space," "bigger," "takes up more room") | Formal vocabulary ("area," "unit square," "square unit," "measure") |
| Observable acknowledgments ("You picked the one that covers the most space") | Assumed thinking ("You thought about that carefully") |
| "Plane figure" used once, casually, not defined | Teaching or defining "plane figure" |
| Authored comparison sets (Sets 1-2 from §1.5.1) | Randomly generated shapes or unsequenced sets |
| MC options: A, B, C, Equal | Binary yes/no, open response, or text input |

### **Warmup Type Rationale**

**Standard Playbook Types:** Binary Choice, WODB, Number Talk, Notice & Wonder, Estimation 180, Visual Patterns, Mystery Reveal

**M1 Selection:** Binary Choice variant — Multi-Shape Comparison (3 shapes + "Equal")

**Why this serves the concept:**

- M1's core arc is comparison-to-measurement: students first use visual estimation, then experience its limits, then get a measurement tool. The Warmup activates the "visual estimation" baseline.
- Playbook recommends Binary Choice or Notice & Wonder for Module 1-3. This is a Binary Choice variant expanded to 3 shapes + "Equal" per the TVP comparison set design.
- Comparison sets are authored and sequenced (easy → medium) to build confidence before the Lesson escalates difficulty. Sets 3-4 in the Lesson surface the "taller = larger" misconception.
- Low cognitive load: one action per interaction, familiar shapes, clear question ("which covers more space?").

**Test:** If we removed this Warmup, would students lose mathematical preparation for the Lesson? YES — they would miss the confidence-building comparison experience that establishes the perceptual baseline the Lesson then disrupts.

**Interaction W.1: Comparison Set 1 — Same Shape, Different Sizes** — [Type B]

* **Purpose:** Confidence builder. Student compares 3 shapes of the same type where size differences are obvious. Establishes the comparison interaction pattern (MC from A/B/C/Equal).
* **Visual: Plane Figures (Display).** Set 1: 3 right triangles labeled A, B, C. Clearly different sizes. Solid color fill, clear boundaries. MC options visible: A, B, C, Equal.
* **Guide:** "Check out these plane figures. Which one do you think covers the most space? Pick the one you think is biggest."
* **Prompt:** "Which shape covers the most space? Select A, B, C, or Equal."
* **Student Action:** MC selection
  * **Options:** A, B, C, Equal
* **Correct Answer:** B
* **Answer Rationale:**
  - B = Correct (clearly largest triangle — obvious size difference, confidence builder)
  - A = Smallest triangle; selected if student reverses "most" and "least"
  - C = Medium triangle; reasonable if student confuses which is B vs. C
  - Equal = Incorrect (triangles are clearly different sizes; tests whether student considers "Equal" before eliminating it)
* **On Correct:** "That's the one. B covers more space than the others."
* **Remediation:** Pipeline

> **Design Note:** Set 1 is intentionally easy — same shape type with obvious size differences. This ensures early success, establishes the interaction pattern, and activates "bigger/smaller" reasoning. "Plane figures" used casually per §1.3 vocabulary staging — no definition, just naming.

---

**Interaction W.2: Comparison Set 2 — Different Shape Types** — [Type B]

* **Purpose:** Cross-category comparison. Student compares shapes of different polygon types, requiring judgment about space coverage rather than relying on shape familiarity. Medium difficulty — harder than Set 1, still visually determinable.
* **Visual: Plane Figures (Display).** Set 2: Regular hexagon (A), isosceles trapezoid (B), equilateral triangle (C). Different polygon types, different side counts. Solid color fill, clear boundaries. MC options visible: A, B, C, Equal.
* **Guide:** "Here's a trickier set. These shapes are all different types. Which one covers the most space — or do some of them cover the same amount?"
* **Prompt:** "Which shape covers the most space? Select A, B, C, or Equal."
* **Student Action:** MC selection
  * **Options:** A, B, C, Equal
* **Correct Answer:** A
* **Answer Rationale:**
  - A = Correct (the regular hexagon covers the most space — it is exactly double the area of the isosceles trapezoid)
  - B = Distractor (isosceles trapezoid — wide shape may attract "widest = biggest" reasoning, but hexagon is 2× its area)
  - C = Distractor (equilateral triangle is taller, so students who equate height with area may select it — but it covers less space than A)
  - Equal = Incorrect (areas are not equal in this set)
* **On Correct:** "The hexagon covers the most. The tall triangle might've looked big, but the hexagon actually covers more space."
* **Remediation:** Pipeline

> **Design Note:** Set 2 introduces cross-category comparison with a clear correct answer (A). The equilateral triangle (C) is taller than the hexagon, which surfaces early "taller = bigger" intuition — a preview of the misconception that Set 3 in the Lesson will fully exploit. The hexagon is exactly 2× the trapezoid's area, making A unambiguous for the authored visual. The "Equal" option is exercised as correct in Lesson Set 3 (Interaction 1.1), not here.

---

**Interaction W.3: Bridge to Lesson**

* **Purpose:** Create anticipation for Lesson without teaching. Tease that visual comparison has limits — the Lesson will reveal cases where "our eyes can trick us" and introduce a tool for measuring exactly.
* **Visual: Plane Figures (Display).** Set 2 remains visible (or transition to clear state).
* **Guide:** "Nice work. You used your eyes to figure out which shapes cover more space. But here's the thing — sometimes shapes can fool you. Ready to see what I mean?"
* **No student action.**

> **Voice Note:** The bridge should land with genuine curiosity, not teaching. "Sometimes shapes can fool you" creates a question in the student's mind (how? when?) that Lesson Set 3's equal-area reveal answers. Do not tip off the equal-area insight — let the Lesson surface it through the designed counterexample.

### **Core Purpose**

Build the conceptual understanding that area is measurable by counting unit squares that cover a shape. Students move from visual comparison (where estimation can fail) to exact measurement with unit square tiles, learning that area = the number of square units covering a shape.

**Scope Note:** This module combines OUR L1 (informal comparison) and L2 (formal measurement with unit squares). The Lesson's extended duration reflects this dual scope.

### **Pedagogical Flow**

| Standard CRA | M1 Implementation |
| :---- | :---- |
| **Pre-CRA Motivation** | Section 1: Visual comparison escalation creates the NEED for measurement before introducing the tool |
| **Concrete** | Section 2 (2.1–2.3b): Hands-on tiling with unit squares — students cover shapes and count tiles |
| **Relational** | Section 2 (2.4a–2.4b): Side-by-side comparisons of completed tilings — 2.4a: same count = same space (3×4 vs 2×6); 2.4b: more tiles = more space (2×5 vs 3×4). Tile count BECOMES the measurement |
| **Abstract** | Section 2 (2.5): Formal vocabulary introduction — area, unit square, square unit — attaches to concrete + relational hooks |
| **Application** | Section 2 (2.6, pre-tiled vocabulary application) + Section 3: Independent tiling with vocabulary, increasing autonomy, surplus tiles, non-rectangular shapes |

### **Lesson Structure**

| Section | Focus | Time |
| :---- | :---- | :---- |
| Section 1 | Escalating Comparison — Surfacing the Need for Measurement | 3-4 min |
| Section 2 | Tiling Introduction — Concrete → Relational → Abstract | 6-8 min |
| Section 3 | Independent Tiling — Application | 4-5 min |

### **1.7.1 LESSON SECTION 1: Escalating Comparison — Surfacing the Need for Measurement**

**Purpose:** Continue the comparison arc from Warmup with harder sets that surface the "taller = larger" misconception and create genuine visual uncertainty. The escalation motivates the transition to measurement — students experience that their eyes are unreliable before the Guide introduces a tool. Section 1 is pre-CRA motivation: it creates the need for the Concrete experience that follows.

**Interaction 1.1: Comparison Set 3 — The Equal Surprise** — [Type C]

*Scaffolding: Stage 1 (comparison with substantial teaching)*

* **Purpose:** Surface the misconception that different-looking shapes can't cover the same space. Students compare 2 parallelograms and an irregular hexagon. The hexagon and one parallelogram have equal area — the hexagon can be cut across the middle and repositioned to match the parallelogram identically. This is the "Equal" set — the first time Equal is the correct answer.
* **Visual: Plane Figures (Display).** Set 3: Parallelogram (A), irregular hexagon (B), parallelogram (C). A and B have equal area (B can be decomposed by cutting across the middle and rearranged to match A exactly). C is a different size. Solid color fill, clear boundaries. MC options visible: A, B, C, Equal.
* **Guide:** "Here are three more shapes to compare. Which one covers the most space — or do some of them cover the same amount?"
* **Prompt:** "Which shape covers the most space? Select A, B, C, or Equal."
* **Student Action:** MC selection
  * **Options:** A, B, C, Equal
* **Correct Answer:** Equal
* **Answer Rationale:**
  - Equal = Correct (A and B have the same area — the irregular hexagon can be cut and rearranged to match the parallelogram exactly)
  - A = Parallelogram selected — students may pick the more familiar shape as "biggest"
  - B = Hexagon selected — irregular shape with more sides may look like it covers more
  - C = Second parallelogram selected — distractor
* **On Correct:** "Two of those cover the same space — even though they look completely different. The parallelogram and the hexagon actually cover the same amount."
* **Remediation:** Pipeline
  * **Remediation Note:** After response (correct or 2nd error), show visual comparison overlay revealing that A and B have equal area. If engineering supports it, animate the hexagon decomposition (cut across middle, reposition halves to form the parallelogram). This is the first moment students experience that shape type doesn't determine area.

> **Design Note:** Set 3 is the "Equal" set and does double duty: it exercises the Equal option (preventing students from learning to always eliminate it) AND introduces the idea that shapes covering the same space can look completely different. The hexagon-to-parallelogram decomposition previews the conservation-of-area concept central to the unit — different shapes CAN cover the same space. The Equal reveal is more surprising here than it would be in the Warmup because students have seen two sets (W.1, W.2) where one shape clearly won. Per §1.5.1: Sets are authored and sequenced; difficulty progression is pedagogically load-bearing.

### **1.7.2 LESSON SECTION 2: Tiling Introduction — Concrete → Relational → Abstract**

**Purpose:** Introduce Unit Square Tiles as a measurement tool through worked example with think-aloud (Concrete). After two tilings, two Relational interactions explicitly connect tile count to area measurement — first with an equal-area pair (same tiles = same space), then with an unequal pair (more tiles = more space). Formal vocabulary is then introduced (Abstract) with the full weight of concrete AND relational experience behind it.

**CRA Progression within Section 2:**
- 2.1–2.3b = **Concrete** (tile, count, tile again, count again)
- 2.4a–2.4b = **Relational** (2.4a: equal-area pair compared — same tiles, same space; 2.4b: unequal pair compared — more tiles, more space)
- 2.5 = **Abstract** (formal vocabulary: unit square, square unit, area)
- 2.6 = **Application bridge** (pre-tiled vocabulary application — isolates vocabulary as single new demand)

**Interaction 2.1: Guide Demonstrates Tiling [WORKED EXAMPLE + THINK-ALOUD]** — [Type A]

*Scaffolding: Stage 1 — Full worked example (Guide demonstrates, student observes)*

* **Visual: Unit Square Tiles (Tiling Mode).** Rectangle outline (2×5 = 10 sq units). Tile bank with exactly 10 tiles (exact match). No grid visible on target shape.
* **Guide:** [Guide_Think_Aloud] "Here's how this works. I've got these square tiles and a rectangle to cover. [PLANNING] First I ask myself — how do I cover this whole shape? [ATTENTION] I'll start in this corner and drag tiles one at a time. One... two... three... each tile fits right next to the last one... four... five... that fills one row. Next row — six... seven... eight... nine... ten. [SELF-CHECK] Let me check — is the whole shape covered? Yes. I used 10 square tiles."
* **No student action.**

> **Design Note:** First worked example (full demonstration). Think-aloud tags per Lesson Playbook §3A: [PLANNING] = models planning behavior, [ATTENTION] = models strategic focus, [SELF-CHECK] = models verification. Guide counts one-by-one — modeling the counting students will do. Vocabulary stays informal ("square tiles") — formal terms wait until 2.5.

**Interaction 2.4a: Relational Comparison — Same Count, Different Shapes [SHOW-THEN-STATE]** — [Type C]

*CRA Stage: RELATIONAL (Part 1) — "I tiled shapes" → "the tile count tells me how much space"*

* **Purpose:** Connect tile count to area measurement using an equal-area pair. Both completed tilings from 2.2 and 2.3 displayed side by side. Guide draws attention to the surprise: different-looking shapes, same tile count, same space covered. Establishes the core principle that counting tiles tells you how much space a shape covers.
* **Visual: Unit Square Tiles (Tiling Mode).** BOTH completed tilings displayed side by side — 3×4 rectangle (12 tiles) and 2×6 rectangle (12 tiles). Tile counts visible or labeled.
* **Guide:** "Look at these two rectangles. Different shapes — but you used 12 tiles on both." [Visual beat: both tile counts highlighted] "The number of tiles tells you how much space a shape covers. Same number of tiles, same space. So which one covers more space — the first or the second?"
* **Prompt:** "Which rectangle covers more space: the first, the second, or are they the same?"
* **Student Action:** MC selection
  * **Options:** First rectangle, Second rectangle, They cover the same space
* **Correct Answer:** They cover the same space
* **Answer Rationale:**
  - They cover the same space = Correct (both have 12 tiles; same tile count = same space covered)
  - First rectangle = May associate the wider/shorter shape with "more" based on visual appearance
  - Second rectangle = May associate the taller/narrower shape with "more" based on visual appearance
* **On Correct:** "Same space. 12 tiles each."
* **Remediation:** Pipeline

> **Design Note:** Relational Part 1 uses the equal-area pair (12 tiles each, different shapes) — the counterintuitive case. Visually different shapes, same measurement. This is the strongest demonstration that appearance alone doesn't tell you area. The Guide states the core principle ("the number of tiles tells you how much space") BEFORE asking the student to apply it. Per Lesson Playbook §4B: explicit instruction, not discovery-first.

**Interaction 2.4b: Relational Comparison — Different Count, Different Space [SHOW-THEN-STATE]** — [Type C]

*CRA Stage: RELATIONAL (Part 2) — "more tiles = more space"*

* **Purpose:** Confirm the measurement principle with an unequal-area pair. The 2×5 rectangle from the Guide demo (2.1) is shown alongside the 3×4 rectangle from the student's first tiling (2.2). Different tile counts make the "more tiles = more space" claim visible and verifiable. Together with 2.4a, the student now has evidence for both directions: same count → same space, different count → different space.
* **Visual: Unit Square Tiles (Tiling Mode).** TWO completed tilings displayed side by side — 2×5 rectangle (10 tiles) and 3×4 rectangle (12 tiles). Tile counts visible or labeled.
* **Guide:** "Now look at these two. This one used 10 tiles, and this one used 12." [Visual beat: both tile counts highlighted] "The shape with more tiles covering it takes up more space. Which one covers more space?"
* **Prompt:** "Which rectangle covers more space: the one with 10 tiles, the one with 12 tiles, or are they the same?"
* **Student Action:** MC selection
  * **Options:** The 10-tile rectangle, The 12-tile rectangle, They cover the same space
* **Correct Answer:** The 12-tile rectangle
* **Answer Rationale:**
  - The 12-tile rectangle = Correct (12 tiles > 10 tiles; more tiles = more space covered)
  - The 10-tile rectangle = May confuse the 2×5's longer/narrower shape with "more"
  - They cover the same space = May not yet connect tile count to space comparison
* **On Correct:** "More tiles, more space. 12 is more than 10."
* **Remediation:** Pipeline

> **Design Note:** Relational Part 2 completes the logical pair: 2.4a showed same count → same space; 2.4b shows different count → different space. The 2×5 rectangle is from the Guide's demo (2.1), not a new shape — reusing it grounds the comparison in the student's existing experience. The "more tiles = more space" claim is now backed by visible evidence on screen, not asserted in the abstract. Both Relational beats together give students the full principle: tile count IS the measurement of space.

**Interaction 2.5: Vocabulary Introduction [TEACHING ONLY]** — [Type A]

*CRA Stage: ABSTRACT — Formal terms attach to Concrete (tiling) + Relational (tile count = measurement) hooks*

* **Purpose:** Introduce formal vocabulary AFTER two tilings AND the Relational insights (2.4a–2.4b). Terms now attach to both concrete experience (hands-on tiling) and relational understanding (tile count = space covered).
* **Visual: Unit Square Tiles (Tiling Mode).** Both tilings still visible (or the 2×6 tiling with "12" labeled).
* **Guide:** "These square tiles have a special name. Each tile is called a UNIT SQUARE because it covers one SQUARE UNIT of space. And the total square units covering a shape? That's the AREA. So the area of each rectangle is 12 square units."
* **No student action.**

> **Design Note:** Vocabulary staging per §1.3: "square tiles" (used informally 2.1–2.3) → "unit square" (the tile object, formally named here) → "square unit" (the measurement label) → "area" (the concept). The bridging sentence ("These square tiles have a special name") connects the informal term to the formal one. The SME-approved causal chain — tile is called a unit square BECAUSE it covers one square unit — grounds the name in the covering action, not in linear measurement (side lengths are M7+). All three formal terms introduced AFTER the student has tiled two shapes, counted both, AND experienced both Relational insights (2.4a: same count = same space; 2.4b: more tiles = more space). Per Lesson Playbook §4A: vocabulary after visual + conceptual grounding. Per Module Mapping: "Unit square = the tile itself. Square unit = the measurement unit."

### **Module-Specific Lesson Guidance**

**Two-Toy Architecture:** Plane Figures (display-only comparison) is used in Section 1 only. Unit Square Tiles (tiling + counting) is used in Sections 2-3. Unit Square Tiles renders its own tiling target surface — no separate toy is needed for the shape outline.

**Overlap Feedback:** Mechanical overlap feedback (visual color change + audio cue) is always active per TVP. Students see when tiles overlap. The conceptual rule (why overlaps cause wrong area counts) is NOT taught until M2. Do not reference overlap rules in M1 Guide dialogue.

**Tile Supply Progression:** Section 2 uses exact tile supply (bank has exactly the tiles needed; empty bank = shape fully covered). Section 3 uses modest surplus (more tiles than needed; student judges completeness). Never infinite/unlimited.

**Counting Expectation:** One-by-one counting is appropriate and expected throughout M1. Do not prompt skip-counting, row-counting, or any structured counting strategies. Slow, deliberate counting builds the M3 motivation for efficiency.

**Two-Step Tiling Interactions:** For the first two tilings (2.2/2.2b, 2.3/2.3b), tiling and counting are separated explicitly to scaffold the counting-after-tiling pattern. Interaction 2.6 uses a pre-tiled figure to isolate vocabulary application as the single new demand. ## **1.10 KEY DESIGN DECISIONS SUMMARY**

**[REQUIRED]**

Master list of cross-cutting design decisions for this module. References phase-level Design Notes where relevant. Numbered for easy reference in reviews and feedback.

1. **Two-Toy Architecture.** Plane Figures (display-only comparison, Warmup + Lesson Section 1) and Unit Square Tiles (tiling + counting, Lesson Sections 2-3, EC, Practice, Synthesis). Plane Figures create the need for measurement; Unit Square Tiles provide it. The toys serve complementary roles and never overlap in function. (§1.5)

2. **Counter Removed from Standard Path.** Per SME decision and TVP: students count tiles themselves. The counting IS the learning — manual one-by-one counting creates the M3 motivation for structured counting (rows × columns). Counter available only as guide-activated remediation for persistent miscounting (correct tiling, wrong area on 2+ consecutive problems). Not available during EC. (§1.5.2, Conflict Log #2)

3. **Comparison Set Design (3 Shapes + Equal).** TVP specifies 3 shapes per comparison set (A, B, C) with "Equal" as 4th MC option, overriding the Plane Figures spec's 2-shape constraint. Sets are authored by the SME and sequenced with deliberate difficulty progression: easy same-type (Warmup Set 1) → cross-category with clear winner (Warmup Set 2: hexagon, trapezoid, triangle) → equal-area surprise (Lesson Set 3: parallelogram = irregular hexagon via decomposition) → close call (Lesson Set 4). "Equal" is correct in Set 3 only, placed in the Lesson for maximum surprise after two Warmup sets where one shape clearly wins. The sequence is pedagogically load-bearing — reordering breaks the motivational arc. (§1.5.1, §1.6, §1.7.1, Conflict Log #4)

4. **Vocabulary Staging: Informal Before Formal.** "Square tiles" (informal) used through all of Section 2's concrete tiling experiences. Formal terms — "unit square," "square unit," "area" — introduced ONLY after 2+ tiling experiences AND both Relational bridges (2.4a–2.4b). This follows the Lesson Playbook's vocabulary-after-grounding principle: students need concrete referents before abstract labels. The staging is: square tiles → unit square/square unit → area. (§1.3, §1.7.2 Interactions 2.1-2.5)

5. **CRA Sequence with Explicit Relational Phase.** Section 2 follows Concrete → Relational → Abstract → Application. The Relational phase (2.4a–2.4b) is a pair of side-by-side comparisons of completed tilings: 2.4a uses an equal-area pair (same tiles = same space) and 2.4b uses an unequal pair (more tiles = more space). Together they establish that tile count IS the measurement of space. This bridges "I tiled and counted" (Concrete) to "the count IS the measurement" (Abstract). Without this bridge, formal vocabulary risks being disconnected from the tiling experience. (§1.7.2)

6. **Overlap Feedback Always Active, Conceptual Rules Deferred to M2.** Per TVP (overriding Notion Spec): the mechanical overlap feedback (visual color change + audio cue) is always on in M1. Students see when tiles overlap. But the conceptual rules — WHY gaps/overlaps produce wrong measurements — are not taught until M2. M1 Synthesis previews a "bad" tiling to create curiosity for M2. (§1.5, §1.9, Conflict Log #6)

7. **Tile Supply Progression: Exact → Surplus.** Section 2 (guided tiling) uses exact-match tile supply — the bank has exactly the tiles needed. This removes a variable during initial learning. Section 3 (independent tiling), EC, and Practice use modest surplus — student must judge completeness without the empty-bank cue. The progression scaffolds judgment: first learn the process, then learn to self-monitor. (§1.5)

8. **Dimension Reuse and Pre-Tiling as Deliberate Scaffold.** Interaction 2.6 uses a pre-tiled 5×2 rectangle (same area as demo 2.1's 2×5, rotated orientation) — pre-tiled because the student is focused on vocabulary application, and familiar area reduces cognitive load during the vocabulary transition. EC uses novel dimensions (2×4, 2×7, 3×5) per SME — EC tests application to new configurations, not recall of Lesson shapes. All decisions documented with explicit Design Notes. (§1.7.2, §1.8)

9. **Rectilinear Shapes in Lesson Only, Not EC.** Section 3 includes two non-rectangular rectilinear shapes: L-shape (10 sq units, 3.3) and Stairs (15 sq units, 3.4), both authored UXisFine assets. But EC tests rectangles only — per TVP and EC Playbook, EC assesses the core skill at representative middle difficulty, not edge cases. Rectilinear shapes return in Practice, which has access to the full UXisFine shape library (Ghost, Heart, Tree, L-shape, Stairs, Strawberry). (§1.7.3, §1.8)

10. **Section 1 as Pre-CRA Motivation, Not Assessment.** Lesson Section 1's comparison sets serve a motivational purpose — creating the emotional need for measurement — not an assessment purpose. Even students who guess correctly on the counterexample (Set 3) or close call (Set 4) benefit from the experience of uncertainty. On Correct feedback is deliberately understated to preserve the transition beat. (§1.7.1)

11. **Think-Aloud Protocol in Worked Example.** Interaction 2.1 uses a full worked example with inline think-aloud tags: [PLANNING] (what's my approach?), [ATTENTION] (what am I doing now?), [SELF-CHECK] (did I cover everything?). This models the cognitive process students will internalize through guided and independent practice. Tags are for script writers — not visible to students. (§1.7.2)

12. **No Purpose Frame Omission — KDD Not Needed.** The Purpose Frame is present and recommended. It uses only known vocabulary ("compare shapes," "measure exactly how much space") and connects backward (Warmup comparisons) and forward (measurement tool). No KDD needed for omission because it's included. (§1.7)

13. **Synthesis Closes the Narrative Arc.** S.1 callbacks to the Warmup comparison and resolves it with measurement — closing the "guessing → measuring" arc that spans the entire module. S.2 (why square tiles?) builds conceptual understanding beyond procedure. The Identity-Building Closure previews M2's gap/overlap focus through a "bad tiling" visual, creating forward motivation. (§1.9)

14. **Practice Pipeline Scope.** §1.8.5 provides Pipeline inputs (toy constraints, used dimensions, distribution guidance, cross-module references) without overstepping into skill decomposition or problem sequencing, which are Pipeline responsibilities. (§1.8.5)
