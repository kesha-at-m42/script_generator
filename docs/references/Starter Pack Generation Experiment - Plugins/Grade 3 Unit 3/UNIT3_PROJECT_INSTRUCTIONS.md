## Key Design Decisions (Unit-Level)

These are resolved in the TVP. Do not re-litigate in individual module chats unless new evidence surfaces.

- **D1 — Paired-tier sequencing.** Addition module at each complexity tier, followed by subtraction counterpart. NOT all-addition-then-all-subtraction.
- **D2 — Fuson's structural methods for algorithms.** Addition: "New Groups Below" with configurable indicator position. Subtraction: "Ungroup All First" — do ALL ungrouping before ANY subtracting.
- **D3 — Estimation integration.** Informal reasonableness checking threads M2–M9 ("Does this make sense?"). Formal estimation in dedicated Tier 3 (M10–M12). Estimation habit carries into M13–M15 word problems.
- **D4 — Word problem representation tool.** UX builds all three (number line hybrids, tape/block diagrams, equation-only) as equal competitors. SP authors should use whichever fits the problem type best.
- **D5 — Strategy Visualizer.** Design modules as if it exists; provide text-based fallback in each interaction's Visual line.
- **D7 — Number lines for computation.** Number lines enter in early computation modules (M2–M3 strategies), not just rounding.
- **D10 — Fluency gating.** Algorithm every time is acceptable if correct. Strategy flexibility is valued but not gated.

Track in `UNIT3_WORKING_NOTES.md` as each module completes. The bridge chain follows the tier structure:

- M1 → M2: Place value foundations → "Now let's USE place value to add"
