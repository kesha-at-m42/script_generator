#  **EXIT CHECK PHASE PLAYBOOK**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━  
 Phase: Exit Check  
 Duration: \~3-5 minutes  
 Purpose: Quick mastery checkpoint after Lesson  
 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## **1\. PHASE IDENTITY**

**Core Purpose:**  
 Quick assessment checking if student understood what was just taught in Lesson. This is assessment, not teaching.

**Position in Flow:**

* After Lesson → Exit Check → Practice  
* Transition FROM Lesson: "You've got the idea. Let's see what you know."  
* Transition TO Practice: "You're ready. Let's practice."

**Duration & Scope:**

* 3-5 minutes  
* 3 problems standard (expand to 5 if needed for thorough check)  
* All problems test the SAME core concept from Lesson

**Energy & Tone:**

* Medium-Low energy (calm, neutral assessment)  
* Low-stakes checkpoint, not high-stakes test  
* Brief feedback (see Remediation System doc)

**What Exit Check Is NOT:**

* Not extended practice (that's Practice phase)  
* Not a teaching moment (that was Lesson—keep feedback minimal)  
* Not standalone assessment (must map directly to Lesson content)

---

## **2\. STRUCTURAL REQUIREMENTS**

**Problem Count & Sequencing:**

**Standard 3 problems** (most modules):

* Problem 1: CREATE or IDENTIFY (simplest)  
* Problem 2: IDENTIFY or CREATE (moderate)  
* Problem 3: COMPARE (if comparison taught in Lesson)

**Expand to 5 when needed:**

* Problem 4: Alternate cognitive type  
* Problem 5: Final confirmation

**Sequencing principle:** Simple to complex cognitively, all testing SAME concept from Lesson.

**Note:** Exit Check uses only CREATE, IDENTIFY, and COMPARE. APPLY and CONNECT are reserved for Practice phase.

---

## **3\. PHASE-SPECIFIC REQUIREMENTS**

### **3A. The Three Cognitive Types for Exit Check**

Exit Check uses **three cognitive types only**—simpler than Practice phase, which adds APPLY and CONNECT for extended application.

**CREATE** \- Constructing or building representations

* *Definition:* Student constructs a fraction using visual tools  
* *Example:* "Shade 3/4 of the rectangle"  
* *When to use:* Testing if student can execute the skill taught in Lesson

**IDENTIFY** \- Recognizing or selecting correct representations

* *Definition:* Student recognizes fractions and their properties  
* *Example:* "Select the shape that shows 2/3" or "Point to the numerator"  
* *When to use:* Testing if student can recognize what was taught in Lesson

**COMPARE** \- Determining relationships between fractions

* *Definition:* Student analyzes relationships and makes judgments  
* *Example:* "Which is larger: 2/3 or 3/4?"  
* *When to use:* Testing if student understands relationships taught in Lesson

**Why Only These Three:**  
 Exit Check assesses basic understanding immediately after teaching. APPLY (contextual problems) and CONNECT (linking multiple representations) require transfer and extended thinking—better suited for Practice phase.

**Module Progression Guidelines:**

**Modules 1-3 (Foundational):**

* Focus: CREATE and IDENTIFY only  
* Typical sequence: CREATE → IDENTIFY → CREATE  
* Why: Students are learning basic construction and recognition

**Modules 4-6 (Building):**

* Add: COMPARE when comparison was taught in Lesson  
* Typical sequence: CREATE → IDENTIFY → COMPARE  
* Why: Students can now handle simple relationships

**Modules 7-12 (Complex):**

* All three types available when taught in Lesson  
* Typical sequence: CREATE → COMPARE → IDENTIFY (or vary as appropriate)  
* Why: Students have foundation for comparing representations

**Reality Check:** Only use cognitive types for skills explicitly taught in THIS Lesson. If Lesson didn't teach comparison, don't test it in Exit Check.

---

### **3B. Stay Within Lesson Boundaries**

**CRITICAL:** All Exit Check problems must use ONLY what was taught in Lesson.

**Visual Models:**

* ✓ Use visual models shown in Lesson  
* ✗ Do NOT introduce new visual models  
* Example: If Lesson used rectangle bars, Exit Check uses rectangle bars (not circles)

**Orientations:**

* ✓ Use orientations demonstrated in Lesson  
* ✗ Do NOT change orientations  
* Example: If Lesson showed horizontal bars, Exit Check uses horizontal bars (not vertical)

**Interaction Methods:**

* ✓ Use interaction methods practiced in Lesson  
* ✗ Do NOT add new interaction types  
* Example: If Lesson had students shade parts, Exit Check uses shading (not dragging or splitting)

**Fraction Complexity:**

* ✓ Use similar fractions to Lesson examples  
* ✗ Do NOT increase complexity  
* Example: If Lesson taught 1/2, 2/3, 3/4, Exit Check uses similar complexity (not 5/8 or 7/12)

**Why This Matters:**  
 Exit Check assesses understanding of what was taught, not ability to transfer to new formats. Adding new visuals, orientations, or complexity during assessment is unfair and inaccurate.

---

### **3C. Assessment Alignment (THE CRITICAL RULE)**

**Every Exit Check problem must test EXACT Lesson content—not adjacent skills, not prerequisites, not next steps.**

**Verification Process:**

```
STEP 1: Review Lesson - what did guide explicitly teach?

STEP 2: List specific skills taught (be precise)

STEP 3: Design problems testing ONLY those skills

STEP 4: Verify with ✓/✗ format:

LESSON TAUGHT:
"The numerator shows how many parts are shaded."

EXIT CHECK (ALIGNED):
✓ "How many parts are shaded? That's the numerator."
   → Tests exactly what was taught

EXIT CHECK (NOT ALIGNED):
✗ "Create a fraction with numerator 3."
   → We taught recognizing, not creating
```

**Common Misalignment Errors:**

❌ Testing application when Lesson taught recognition  
 ❌ Testing creation when Lesson taught identification  
 ❌ Using different visual models than Lesson  
 ❌ Using more complex fractions than Lesson  
 ❌ Testing vocabulary not introduced in Lesson

**Module-Specific Alignment:**

* **Modules 1-3:** Strict alignment (test exactly what was taught)  
* **Modules 4-8:** Can assume prior module concepts  
* **Modules 9-12:** Can combine concepts if both taught in THIS Lesson

---

### **3D. Problem Design**

**Appropriate Difficulty Level:**

Exit Check problems should be **representative middle difficulty**:

* **Not too easy:** Avoid giving false confidence with overly simple problems  
* **Not too hard:** Avoid creating anxiety with edge cases or tricks  
* **Just right:** "If a student can do this, they understood the Lesson"

**Clear Success Criteria:**

Each problem needs **objective, binary success:**

* Correct answer is unambiguous  
* Success/failure is immediately clear  
* No judgment calls needed

**Good success criteria:**

* "Shade 2 out of 3 parts" (clear visual check)  
* "Select the fraction that shows 3/4" (one correct answer)  
* "Which is larger?" (definitive answer)

**Unclear success criteria:**

* "Show a fraction close to 1/2" (what counts as "close"?)  
* "Make a reasonable fraction" (too subjective)

**Cognitive Type Selection by Module:**

**Modules 1-3:** Use CREATE and IDENTIFY only

* Focus on construction and recognition  
* Don't use COMPARE unless explicitly taught

**Modules 4-6:** Add COMPARE when taught

* Can include simple comparison if Lesson taught it  
* Still focus primarily on CREATE and IDENTIFY

**Modules 7-12:** All three types available

* Use CREATE, IDENTIFY, and COMPARE as appropriate to Lesson  
* Balance across the three types

**Problem Checklist:**

□ Tests exactly what Lesson taught (nothing more, nothing less)  
 □ Uses same visual models as Lesson  
 □ Uses same orientations as Lesson  
 □ Has clear, objective success criteria  
 □ Takes 30-60 seconds to complete  
 □ Student can succeed using only Lesson knowledge  
 □ Appropriate cognitive type for module level (see 3A)  
 □ Different cognitive type than other Exit Check problems

---

### **3E. Feedback**

**Success:** Brief, specific (5-10 words)

* "That's right—2 shaded parts is the numerator."  
* "Yes, you identified the larger fraction."

**Struggle:** Minimal support (see Remediation System doc)

* Brief redirect (don't re-teach)  
* Warm but neutral tone

**What Feedback Should NOT Do:**  
 ✗ Re-teach the concept  
 ✗ Extended scaffolding  
 ✗ Multiple approaches  
 ✗ Overpraise

---

## **4\. CRITICAL SUCCESS CHECKS**

**Must-Haves:**

□ Each problem maps to specific Lesson content  
 □ All problems test SAME core concept  
 □ 3-5 problems, simple to complex  
 □ Same visual models as Lesson  
 □ Brief, neutral feedback  
 □ No new concepts introduced

**Red Flags:**

| Warning Sign | Fix |
| ----- | ----- |
| Testing content not in Lesson | Map to Lesson lines explicitly |
| Using new visual models | Use exact Lesson models |
| Extended teaching in feedback | Brief redirect only |
| 6+ problems | Use 3-5 maximum |
| All same cognitive type | Vary: identify, create, compare |
| High-stakes tone | Keep warm, low-pressure |

---

## **5\. QUICK REFERENCE: WORKFLOW**

```
1. Review Lesson 
   → What was taught? 
   → What visuals/orientations used?
   → What cognitive types are appropriate? (CREATE/IDENTIFY/COMPARE)

2. Design 3 problems (prepare up to 5)
   → Modules 1-3: CREATE and IDENTIFY only
   → Modules 4-6: Add COMPARE if taught
   → Modules 7-12: All three types available

3. Verify alignment 
   → Map to Lesson lines (use ✓/✗)
   → Check visuals match Lesson
   → Check orientations match Lesson

4. Write each problem 
   → Clear prompt, brief feedback

5. Check whole phase
   → Low-stakes feel? 
   → Same visuals/orientations? 
   → Brief feedback?
```

**Quick Decisions:**

* **How many?** → 3 standard, expand to 5 if needed  
* **What to test?** → Exactly what Lesson taught (✓/✗ verification)  
* **Cognitive types?** → CREATE, IDENTIFY, COMPARE only (see 3A for module guidance)  
* **Feedback?** → Brief, specific, neutral (5-10 words)  
* **Visual models?** → Same as Lesson (no new models or orientations)
