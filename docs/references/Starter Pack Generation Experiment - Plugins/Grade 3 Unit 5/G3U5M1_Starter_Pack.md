# BACKBONE

## 1.0 THE ONE THING

Students learn that fractions require EQUAL parts and can name them, practicing the creation and recognition of equal partitions of shapes (2, 3, 4, 6, 8 parts). The fundamental understanding is that "equal parts" is not negotiable—all pieces must be exactly the same size for a fraction to exist.

**CRA Stage:** Concrete (This is the first module introducing fractions through physical partitioning)

**Critical Misconception:** Believing parts can be different sizes and still count as equal fractions, or "close enough" equality is acceptable

**Success Indicator:** Student consistently creates and identifies equal partitions, verbally rejects unequal parts, and can name them with both words and notation

**Biggest Risk:** Accepting "close enough" rather than true equality, or moving to notation before sufficient concrete experience with partitioning

* Partitioning shapes into 2, 3, 4, 6, 8 equal parts

* Comparing fraction sizes (Modules 10+)
* Adding or subtracting fractions (Unit 6)
* Equivalent fractions (Module 8)
* Number lines (Module 4)
* Circles (Module 2 — hexagons complete Module 1 shape progression)

## 1.4 MISCONCEPTIONS

### 1.4.1 Unequal Parts (PRIMARY) — Tracker #1

**Trigger Behavior:** Student accepts or creates partitions where parts are visibly different sizes; suggests "sort of equal" or "close enough" is acceptable for fractions

**Why It Happens:** Informal everyday language ("I'll split this") does not require precision; visual estimation without measurement tools; lack of understanding that fractions are mathematically based on exact equality

**Visual Cue:** Use highlighted outlines to overlay and directly compare part sizes; show grid arrays where equal parts fill identical grid units; place unequal partition alongside correct partition to create stark visual contrast

**Prevention Strategy:** Start with grid rectangles where equality is built-in through counting grid units; use direct comparison activities throughout Section 1; explicitly reject unequal partitions with language like "Those parts are different sizes, so they don't work for fractions"; build vocabulary "equal" early and reinforce constantly

### 1.4.2 Misidentifying the Whole (SECONDARY) — Tracker #2

**Trigger Behavior:** Student loses track of what constitutes "one whole"; compares parts from different-sized wholes; treats a part as the whole or the whole as a part

**Why It Happens:** No clear boundary established for "the whole" before partitioning; attention shifts to parts and loses sight of complete object; context switching between different shapes without resetting "whole" reference

**Visual Cue:** Highlight the entire shape with a border before partitioning; show complete grid array or hexagon with a highlighted outline; use consistent shape sizes throughout the lesson

**Prevention Strategy:** Always establish "this is our whole" before any partitioning; reference "the whole" repeatedly; keep whole shape visible even after partitioning; build vocabulary around "the whole shape"; use the Same Whole Principle consistently (all shapes in a comparison are identical size)

## 1.6 WARMUP (3-5 minutes)

### Core Purpose

**Key Function:** Activate recognition of equal vs. unequal parts using simplest cases, establishing the foundational visual pattern before formal partitioning

**Why this serves the concept:**
* Retrieves prior knowledge of "same size" and "equal" from Grade 2 partition work
* Creates cognitive load of 20-30% by using pre-made visuals (no toy manipulation)
* Establishes visual vocabulary: equal parts look the same, unequal parts are obviously different
* Plants the insight that "equal" is about size comparison, not just counting

**Test:** If we removed this Warmup and went straight to the Lesson, students would lack the visual reference for "equal parts" and might treat "sort of equal" as acceptable. The Warmup creates a shared baseline definition before partitioning begins.

### Warmup Parameters

| Element | Specification |
| :---- | :---- |
| **Time** | 3-5 minutes |
| **Interactions** | 3 (two identification tasks + bridge) |
| **Cognitive Load** | 20-30% |
| **Remediation** | Pipeline |
| **Vocabulary** | Informal only — no formal introduction of "partition," "fraction," "whole" |
| **Student Action Types** | Click-to-select binary or ternary choices only |

---

### Warmup Type Rationale

This Warmup is ACTIVATION type: it retrieves the visual pattern of "equal vs. unequal" that students learned in Grade 2, making it available for the new context of fractions. Pre-partitioned rectangles eliminate tool cognitive load, keeping focus on the concept of equality. No student manipulation of the toy keeps this phase light and fast.

---

### Interaction W.1: Hook + First Identification [ACTIVATION]

* **Purpose:** Activate equal vs. unequal distinction with the simplest case (2 parts), creating a shared visual definition of "equal"
* **Visual: Pre-Partitioned Rectangles.** Horizontal orientation. Two rectangles side by side: Rectangle A divided into 2 equal parts (vertical center line), Rectangle B divided into 2 unequal parts (one piece visibly larger). No toy manipulation.
* **Guide:** "When you share something like a sandwich with a friend, you want both pieces to be the same size, right? Look at these two rectangles. One has pieces that are the same size, and the other has pieces that are different sizes. Which one has equal pieces?"
* **Prompt:** "Select the rectangle with equal parts."
* **Student Action:** Click-to-select (binary choice)
* **Correct Answer:** Rectangle A
* **Answer Rationale:**
  - Rectangle A: Correct — two parts that are visually identical in size; matches the definition of "equal"
  - Rectangle B: #1 — Student may focus on the number of parts (both have 2) without checking whether parts are the same size; signals Misconception #1 (Unequal Parts)
* **On Correct:** "You found it. Those two pieces are exactly the same size. That's what equal means."
* **Remediation:** Pipeline

---

### Interaction W.2: Escalated Identification [ACTIVATION]

* **Purpose:** Confirm pattern recognition with added complexity. Tests attention to both the count of parts AND whether the parts are equal.
* **Visual: Pre-Partitioned Rectangles.** Horizontal orientation. Three rectangles: Rectangle A divided into 4 equal vertical strips (correct), Rectangle B divided into 4 unequal parts (equality distractor), Rectangle C divided into 5 equal parts (count distractor). No toy manipulation.
* **Guide:** "Now there are three rectangles. Check how many pieces each one has AND whether the pieces are the same size."
* **Prompt:** "Select the rectangle with four equal pieces."
* **Student Action:** Click-to-select (ternary choice)
* **Correct Answer:** Rectangle A
* **Answer Rationale:**
  - Rectangle A: Correct — exactly 4 parts, all equal size
  - Rectangle B: #1 — Equality distractor; has 4 parts but they are unequal. Student who counts parts without checking size will select this (Misconception #1)
  - Rectangle C: Count distractor — has equal parts but 5, not 4; student who focuses on equality but not the specified count will select this
* **On Correct:** "Four pieces, all the same size. You spotted both the number and the equality."
* **Remediation:** Pipeline

---

### Interaction W.3: Bridge to Lesson [BRIDGE]

* **Purpose:** Create anticipation for Lesson without teaching the content that comes next
* **Visual: Pre-Partitioned Rectangles.** The correct rectangle from W.2 remains visible
* **Guide:** "You spotted the equal pieces both times. Here's the cool thing: when pieces are equal like that, mathematicians have a special way to name them and write them down with symbols. Ready to learn how?"
* **No student action.**

## 1.7 LESSON (10-14 minutes)

### Core Purpose

This Lesson develops deep understanding that fractions are built on equal parts. Students move from concrete partitioning actions (making and verifying equal parts) through relational pattern recognition (seeing that the same fraction can look different) to abstract notation (writing 1/4 to mean "one out of four equal parts"). The three sections mirror the CRA progression, building misconception prevention at each stage.

### Pedagogical Flow

Section 1 (Concrete) establishes partitioning as the core action and introduces fraction words grounded in that action. Students cut shapes, verify equality, and immediately shade to maintain the link between "creating equal parts" and "naming them as fractions." Section 2 (Representational) introduces notation as a symbolic shorthand for what students have already done concretely. Section 3 (Abstract) transfers the skill to a new shape (hexagons) using new cutting techniques (radial), demonstrating that the principle "equal parts are fractions" is universal, not shape-specific.

### Lesson Structure

| Section | Focus | Time | CRA Stage |
| :---- | :---- | :---- | :---- |
| Section 1 | Concrete Partitioning and Fraction Words | 5-7 min | Concrete |
| Section 2 | Notation Introduction and Consolidation | 2-3 min | Representational |
| Section 3 | Hexagon Extension and Radial Cutting | 3-4 min | Relational/Abstract |

---

### Required Phrases (Must Appear in Script)

* "This entire shape, all of it together, is our WHOLE"
* "When we split one whole into equal pieces, we call that PARTITIONING"
* "For fractions to work, all pieces must be exactly the same size"
* "These equal parts can be named with FRACTIONS"
* "The bottom number tells us how many equal parts we made. The top number tells us how many parts we selected"
* All five fraction words: "halves," "thirds," "fourths," "sixths," "eighths"

### Forbidden Phrases (Create Misconceptions)

* ❌ **"Sort of equal"** or **"close enough"** — Reinforces misconception that approximate equality works for fractions. Parts must be exactly equal.
* ❌ **"Just count"** — Undermines the equal parts verification strategy. Counting parts without checking equality is the primary misconception (#1).
* ❌ **"Numerator/denominator"** — Module 2 vocabulary. Use "top number/bottom number" in M1. (§1.3 Vocabulary Architecture)
* ❌ Any reference to **circles** — Module 2 content. Shapes in M1 are rectangles and hexagons only.
* ❌ **"Today we will learn..." / "Today you're going to..."** — Non-session-relative and creates performance pressure. Use concrete behavioral framing. (Voice Script §3)
* ❌ **"Yesterday" / "Today" / "Tomorrow"** — Non-session-relative. Use "last time" / "this time" / "next time." (Voice Script §3)
* ❌ **"Perfect!" / "Excellent!" / "Amazing!" / "Great job!"** — Generic overpraise. Use observable acknowledgments: restate the factual answer or describe the action. (Voice Script §1)
* ❌ **"You carefully..." / "You understood..." / "You're thinking..."** — Assumed internal states. Describe observable actions only. (Voice Script §1)
* ❌ **"You have to..." / "You need to..."** — Controlling language. Use "Try..." or "You can..." (Voice Script §1)
* ❌ **"Formula"** (standalone) — Suggests rote memorization. Fractions are built from partitioning actions, not formulas.

## 1.7.1 LESSON SECTION 1: Concrete Partitioning and Fraction Words

### Interaction 1.1: Establishing the Whole and Toy Orientation [WORKED EXAMPLE]

* **Purpose:** Establish "the whole" as the reference point before any partitioning begins; demonstrate basic Grid Arrays interface (cut and erase actions)
* **Visual: Grid Arrays.** Horizontal orientation. One complete undivided rectangle visible, with boundary clearly marked. No partitions yet.
* **Guide:** "This rectangle is our WHOLE. It's the entire shape, all of it together. When we divide the whole into parts, each part is a piece of THIS whole. And here's something useful: if you make a cut you don't like, you can erase it. Watch!"
* **Visual: Grid Arrays.** Demonstration of eraser action on a single line. Click-to-erase.
* **Guide:** "I made a line, and then I clicked to erase it. The rectangle is whole again. Try it yourself: I'll put a line here, and you erase it."
* **Prompt:** "Click on the line to erase it."
* **Student Action:** Click-to-erase (demonstration action only)
* **Correct Answer:** Student successfully erases the partition line
* **On Correct:** "Now you know you can erase if you need to."
* **Remediation:** Pipeline
* **Voice Note:** Keep tone warm and encouraging; this is a tool demonstration, not an assessment

---

### Interaction 1.2: Partition into Halves [GUIDED PRACTICE]

* **Purpose:** First partitioning action — create equal parts for halves
* **Visual: Grid Arrays.** Horizontal orientation. One complete undivided rectangle, same size as 1.1
* **Guide:** "Now, can you divide this rectangle into 2 equal parts? You'll click once in the middle to split it into two pieces that are exactly the same size."
* **Prompt:** "Click once in the middle to split the rectangle into 2 equal parts."
* **Student Action:** Click-to-partition (place one vertical or horizontal line at midpoint)
* **Correct Answer:** Two equal sections (system accepts either vertical center line or horizontal center line)
* **On Correct:** "You made two parts. Let me check: are they equal? Yes, both parts are the same size. That's two halves."
* **Remediation:** Pipeline

---

### Interaction 1.3: Shade One Half [GUIDED PRACTICE]

* **Purpose:** Maintain partition→shade sequence to link the action of creating equal parts to the meaning of fractions
* **Visual: Grid Arrays.** The rectangle partitioned into halves from 1.2 remains visible
* **Guide:** "Now, let's shade just one of those parts. When we shade one part, we're showing that we selected one out of the two equal pieces."
* **Prompt:** "Click on 1 of the 2 parts to shade it."
* **Student Action:** Click-to-shade (select one section)
* **Correct Answer:** One of the two sections highlighted/shaded
* **On Correct:** "You selected one part out of two equal parts. And here's a new word: when we divide something into equal pieces like you just did, we call it PARTITIONING. You partitioned this rectangle into two halves."
* **Remediation:** Pipeline
* **Design Note:** This interaction immediately follows partitioning to preserve the cognitive connection between the action and the meaning.

---

### Interaction 1.4: Partition into Fourths — Cut Flexibility Introduction [WORKED EXAMPLE + GUIDED PRACTICE]

* **Purpose:** Introduce the critical insight that "four equal parts" can be created in multiple valid ways; prevent students from learning a single "right way" to cut
* **Visual: Grid Arrays.** Fresh undivided rectangle, same size

**DEMONSTRATION SEQUENCE (all three configurations shown sequentially, approximately 45-60 seconds total):**

---

**Configuration A — Vertical Cuts (1×4):**

* **Guide:** "Here's one way to partition a rectangle into four equal parts: three cuts going down."
* **Visual Demo:** Show rectangle divided into 4 vertical strips
* **Guide:** "Four tall rectangles. Each one is exactly the same width and height. Four equal parts."

---

**Configuration B — Horizontal Cuts (4×1):**

* **Visual Demo:** [CLEAR rectangle to show blank again]
* **Guide:** "But watch: here's another way to partition this same rectangle into four equal parts."
* **Visual Demo:** Show rectangle divided into 4 horizontal strips
* **Guide:** "Four wide rectangles. Still four equal parts, just oriented differently."

---

**Configuration C — Horizontal and Vertical Cuts (2×2):**

* **Visual Demo:** [CLEAR rectangle again]
* **Guide:** "And here's one more way."
* **Visual Demo:** Show rectangle divided into 2×2 grid (one vertical center line, one horizontal center line, creating four small squares)
* **Guide:** "One cut across and one cut down. Four small squares. Still four equal parts."

---

**KEY INSIGHT (REQUIRED):**

* **Guide:** "Three completely different ways to cut, but all three show fourths. What matters isn't HOW you cut: it's that you end up with four pieces that are exactly the same size. All three are correct."

---

**STUDENT TURN:**

* **Visual: Grid Arrays.** Fresh undivided rectangle, same size. Click-to-partition.
* **Guide:** "Now it's your turn. Partition this rectangle into four equal parts. You can use any of the three ways you just saw, or even a different way: as long as you've got four pieces that are exactly the same size."
* **Prompt:** "Partition the rectangle into 4 equal parts."
* **Student Action:** Click-to-partition (place lines to create four equal sections)
* **Correct Answer:** Four equal sections (any valid configuration: 1×4, 4×1, 2×2, or other valid combinations)
* **On Correct:** "Four equal parts: fourths. No matter which way you cut, as long as the pieces are equal, you've made fourths."
* **Remediation:** Pipeline
* **Critical Design Note:** Accept ANY of the three configurations (or valid combinations). Do NOT express preference for one over another. Accepting all valid solutions is essential for preventing the misconception that only one "right way" exists.

### Interaction 1.5b: Notation Introduction [WORKED EXAMPLE]

* **Purpose:** Introduce fraction notation (1/4) after students have concrete experience creating and shading equal parts. Connect the abstract symbol to the visual action students just performed.
* **Visual: Grid Arrays.** Student's shaded fourths from 1.5 remain visible. Fraction notation "1/4" displays nearby with clear visual annotation.

## 1.7.3 LESSON SECTION 3: Hexagon Extension and Shape Transfer

### Interaction 3.0: Radial Cutting Introduction [WORKED EXAMPLE]

* **Purpose:** Introduce the radial cutting technique required for hexagons before student attempts partitioning; prevent misconception that "cutting" works the same way across all shapes
* **Visual: Hexagons.** One complete undivided hexagon with center point marked clearly
* **Guide:** "You've been cutting rectangles with lines that go straight across (up and down, or left and right). This shape is called a hexagon. Hexagons are six-sided shapes, and for hexagons, we've got to work differently to make our parts equal."
* **Guide:** "See this point in the center? On hexagons, our cuts start from the center and go out to the corners (also called vertices). It's like the spokes on a wheel."
* **Visual: Hexagons.** Show one cut being drawn from center to a vertex
* **Guide:** "I start at the center point, and I draw a line straight out to that corner. That's how we'll partition hexagons. Follow along with me as I do a second cut."
* **Visual: Hexagons.** Show a second cut from center to the opposite vertex, creating two equal halves
* **Guide:** "When we make two cuts from the center that travel to opposite vertices of the shape, we create two equal parts."
* **No student action.**
* **Design Note:** Without explicit radial cutting instruction, students will attempt to draw horizontal or vertical lines across hexagons (like bars), which won't create equal parts and will be rejected by the system. This interaction prevents that misconception by modeling the correct technique.

### Misconception Prevention

**#1.4.1 / Tracker #1 (Unequal Parts — PRIMARY):**
* Throughout Section 1, include visual verification prompts: "Let me check — are all the parts exactly the same size?"
* In Interaction 1.4 (Fourths), explicitly demonstrate that multiple valid configurations all create equal parts, preventing fixation on one "right way"
* In Interactions 1.6 onward, when a student creates unequal parts, the system provides immediate visual feedback highlighting the size mismatch

**#1.4.2 / Tracker #2 (Misidentifying the Whole — SECONDARY):**
* Interaction 1.1 establishes "the whole" explicitly before any partitioning begins
* Every partition interaction references "this whole" or "the whole shape"
* In Section 3, the Shape Transfer activities (hexagons) maintain consistent whole size, preventing students from treating parts as separate wholes

## 1.9 SYNTHESIS (6-8 minutes)

### Purpose

Consolidate learning by making connections across representations and discovering the underlying principle that unifies all fraction work. Move students from procedural knowledge ("I can partition a rectangle") to conceptual understanding ("All fractions are about equal parts of a whole") and plant seeds for next module ("Equal parts are the foundation for naming fractions formally").

---

### Opening Frame

* **Purpose:** Signal shift from learning and assessment to reflection and connection
* **Visual: Grid Arrays and Hexagons.** Display multiple completed fraction models from the Lesson (various shapes with different fractions shaded)
* **Guide:** "You've learned so much about equal parts. You can create fractions on rectangles and hexagons, and you can name them with words and symbols. Now let's look at how these ideas connect to bigger patterns in math."
* **No student action.**

---

### Interaction S.1: Size Observation — More Parts = Smaller Pieces [PATTERN DISCOVERY]

* **Purpose:** Surface the relationship between number of parts and part size through guided visual comparison; reinforce Same Whole Principle
* **Visual: Grid Arrays.** Two identical-sized rectangles displayed side by side: Rectangle A partitioned into 2 equal parts with 1 shaded (1/2), Rectangle B partitioned into 4 equal parts with 1 shaded (1/4). Both rectangles are clearly the same size.
* **Guide:** "Look at these two rectangles. They're the same size, but one is partitioned into halves and one is partitioned into fourths. Here's the thinking question: why is it important that both rectangles are exactly the same size? Think about that while you look."
* **Prompt:** "Which shaded part is bigger? Click the rectangle with the bigger shaded part."
* **Student Action:** Click-to-select
* **Correct Answer:** Rectangle A (the half is bigger than the fourth)
* **Answer Rationale:**
  - Rectangle A (1/2): Correct — 2 parts means each part is larger; the shaded half is visibly bigger than the shaded fourth
  - Rectangle B (1/4): Student may confuse "more parts" with "bigger pieces," or focus on the number 4 being larger than 2 without connecting to part size
* **On Correct:** "You spotted it. The shaded part is bigger when there are fewer total parts."
* **Connection:** "This is a big idea in fractions: when we split one whole into more parts, each individual part gets smaller. Two parts means each piece is bigger. Four parts means each piece is smaller. But here's something equally important: we could only compare these parts because both rectangles started the same size. If one rectangle were bigger than the other, we couldn't just look at the number of total parts to know which piece is bigger. That's why the Same Whole Principle matters: when comparing fractions, the whole must be the same size."
* **Remediation:** Pipeline

---

### Interaction S.2: Pattern Discovery — Equal vs. Unequal [REPRESENTATION TRANSFER]

* **Purpose:** Apply the equal parts rule to identify which shapes follow the rule and which don't; reinforce misconception prevention
* **Visual: Grid Arrays and Hexagons.** Display four shapes in a grid: Hexagon A showing 2 equal parts (correct), Grid array B showing 3 equal parts (correct), Hexagon C showing 3 equal parts (correct), Grid array D showing 3 unequal parts (breaking the rule).
* **Guide:** "Three of these shapes follow our rule about fractions. One breaks it. Look at all four shapes. Which one breaks our equal parts rule?"
* **Prompt:** "Click on the shape that breaks our pattern."
* **Student Action:** Click-to-select
* **Correct Answer:** Grid array D (the one with unequal parts)
* **Answer Rationale:**
  - Grid array D: Correct — parts are visibly different sizes, violating the equal parts rule. Directly tests Misconception #1 (Unequal Parts)
  - Hexagon A (2 equal parts): Incorrect — student may think halves are "too simple" to count as fractions
  - Grid array B (3 equal parts): Incorrect — #1 — student may count parts without checking size equality (Misconception #1)
  - Hexagon C (3 equal parts): Incorrect — student may think hexagons can't have valid fractions, or confuse radial symmetry with inequality
* **On Correct:** "That grid array has unequal parts. Those parts are different sizes, so they don't work for fractions. You used the equal parts rule to spot the problem."
* **Connection:** "Fractions have one non-negotiable rule: parts must be equal. If the parts aren't equal, it's not a fraction. This is what makes fractions powerful and useful."
* **Remediation:** Pipeline

---

### Interaction S.3: Real-World Reasoning — Sharing and Size [REAL-WORLD BRIDGE]

* **Purpose:** Apply mathematical knowledge to practical context; reinforce "more parts = smaller pieces" through real-world scenario
* **Visual: Grid Arrays.** Three identical-sized rectangles representing chocolate bars: Bar A partitioned into 4 equal parts with 1 shaded, Bar B partitioned into 3 equal parts with 1 shaded, Bar C partitioned into 6 equal parts with 1 shaded.
* **Guide:** "Imagine these rectangles are chocolate bars, and you're sharing them with your friends. If these were real chocolate bars that someone was dividing among people, which bar gives each person the biggest piece? Think about the math we just learned."
* **Prompt:** "Click on the bar where each person would get the biggest piece."
* **Student Action:** Click-to-select
* **Correct Answer:** Bar B (thirds — fewest parts means biggest pieces)
* **Answer Rationale:**
  - Bar B (1/3): Correct — 3 parts is the fewest, so each piece is the largest
  - Bar A (1/4): Student may think 4 is a "normal" sharing number and default to it, or confuse part count with part size
  - Bar C (1/6): Student may think more parts means more chocolate (conflating quantity of parts with size of parts) — this is the "more = bigger" misconception
* **On Correct:** "When you share with fewer people, each person gets a bigger piece. Three parts means bigger pieces than six parts or four parts."
* **Connection:** "You're using math to solve real problems. In real life, when we share things (pizza, chocolate, snacks), we're making fractions. The fewer people we share with, the bigger each share becomes. That's fractions working in the real world."
* **Remediation:** Pipeline

---

### Interaction S.4: Metacognitive Reflection — Strategy Awareness [METACOGNITIVE REFLECTION]

* **Purpose:** Build awareness of reasoning strategies; make thinking visible
* **Visual: Grid Arrays.** Return to the three chocolate bar rectangles from S.3 (Bars A, B, C).
* **Guide:** "Take a moment to think about how you figured out which bar had the biggest pieces. Did you compare the sizes of the shaded parts with your eyes? Or did you think about the math: more parts means smaller pieces? Or did you think about sharing with friends? There's no right answer: I'm curious how your brain worked."
* **No student action.**
* **Guide:** "Whatever way you figured it out is smart thinking. Some people see it visually, some people think about the math idea, some people use real-world thinking. All of those are good math strategies."

---

### Interaction S.5: Generalization — Core Principle [CONCEPTUAL CHECK]

* **Purpose:** Articulate the one principle that unifies all fraction work in this module
* **Visual: Grid Arrays and Hexagons.** Display four partitioned and shaded shapes: Hexagon showing 1/2, Grid array showing 1/4, Hexagon showing 1/3, Grid array showing 1/6. All clearly show equal parts.
* **Guide:** "Look at all these fractions we've made (on rectangles and hexagons, different denominators, different shapes). What's ONE rule that's ALWAYS true about fractions? What's the thing that never changes?"
* **Prompt:** "What is always true about fractions?"
* **Student Action:** Click-to-select (multiple choice)
* **Options:**
  - A: The parts must be equal
  - B: There must be four parts
  - C: One part must be bigger than the others
  - D: All shapes must be rectangles
* **Correct Answer:** A (The parts must be equal)
* **Answer Rationale:**
  - A: Correct — Every fraction shown requires equal parts; this is the universal rule that applies across all shapes, all denominators, all representations
  - B: Incorrect — We made halves (2 parts), thirds (3), fourths (4), sixths (6), eighths (8), so the number of parts varies
  - C: Incorrect — #1 — Directly contradicts the equal parts principle (Misconception #1: Unequal Parts)
  - D: Incorrect — We showed hexagons work too; shape is not the limiting factor
* **On Correct:** "Every single fraction we made has one thing in common: the parts are equal. Whether it's halves, thirds, fourths, sixths, eighths, rectangles, or hexagons, equal parts are what make fractions work."
* **Remediation:** Pipeline

### Identity-Building Closure

* **Purpose:** Specific, behaviorally-focused affirmation connecting this session's work to future learning
* **Visual: Grid Arrays and Hexagons.** Display a selection of the student's completed work from throughout the lesson (partitioned shapes with shaded sections)
* **Guide:** "You've done something important. You started with equal pieces and built all the way to naming fractions on two different shapes. You partitioned, you shaded, you matched notation to visuals, and you spotted the pattern that more parts means smaller pieces. That's what makes fractions work: equal parts of a whole. You now know what a fraction is at its core. Next time, you'll build on this foundation. You'll learn formal names for the top and bottom numbers of a fraction, and you'll see how fractions work with brand-new shapes like circles. But this idea of equal parts? That stays with you as the most important thing you've learned. You've earned it."
* **No student action.**

## 1.10 KEY DESIGN DECISIONS

### KDD-1: Cut Orientation Flexibility at Fourths

Fourths is the intentional teaching moment for demonstrating that multiple valid configurations exist. We demonstrate all three configurations (1×4, 4×1, 2×2) before students attempt, establishing that "what matters is equal parts, not how you cut." After fourths, we accept any valid configuration for thirds, sixths, and eighths without re-demonstrating alternatives. This prevents students from developing a misconception that there is one "right way" to partition. Accepting all valid solutions teaches mathematical flexibility and generalizable thinking.

### KDD-2: Partition-Then-Shade Sequence is Non-Negotiable

Every partitioning interaction is immediately followed by a shading interaction. Shading maintains the conceptual link between "the action of creating equal parts" and "the meaning of a fraction as 'one of N equal parts.'" Separating these across time or contexts would break this connection and increase risk of the misconception that fractions are arbitrary symbols. The sequence is: partition → verify equality → shade → name. Disrupting this breaks the learning.

### KDD-3: Notation Introduced After Concrete Grounding

Fraction notation (1/4) is introduced only after students have created concrete partitions for at least halves AND fourths using the Grid Arrays toy. This respects CRA sequencing: concrete experience (making equal parts) → relational pattern recognition (seeing multiple ways to make the same number of parts) → abstract notation (symbols that represent the visual action). Front-loading notation would risk creating symbol manipulation without meaning. We wait until the visual experience is robust before introducing the abstract representation.

### KDD-4: Hexagon Radial Cutting Requires Explicit Instruction

Unlike Grid Arrays where cuts can go in multiple directions and still create equal parts, hexagons require radial cuts from center to vertices to produce equal sections. Interaction 3.0 provides explicit modeling of this technique before any student attempt. Without this instruction, students attempt linear cuts on hexagons (like they learned with rectangles), which fail to create equal parts and trigger system rejection. This is a toy affordance decision, not a misconception, but it requires proactive teaching. We embed the instruction as a worked example before practice.

### KDD-5: Same Whole Principle as Hard Constraint

All visual comparisons throughout the module use identical shapes and sizes. In S.1 (size observation), both rectangles are explicitly the same size so students can compare fractions fairly. In S.2 (pattern discovery), all four shapes are visually distinct in appearance but consistent in "wholeness." This principle prevents the misconception that fraction size is absolute rather than relative to the whole. A student who hasn't internalized "the whole must be the same size to compare fractions" will misinterpret 1/6 of a large rectangle as "bigger" than 1/2 of a small rectangle. We enforce this throughout and explicitly name it in the Synthesis connection to S.1.

### KDD-6: Eighths Constraint (8×1 Horizontal Not Supported)

The Grid Arrays toy cannot support 8 horizontal strips in a single row (7 cuts on the short side exceed the 5-cut maximum pixel constraint). Scripts must not demonstrate or expect this configuration. The supported configurations for eighths are 2×4 (two rows, four columns), 4×2 (four rows, two columns), and 1×8 wide (one row, eight columns). We make this explicit in the toy specs and in the remediation guidance for Interaction 1.10 so that if a student attempts 8×1 tall, the system naturally redirects to a supported configuration.

### KDD-7: Warmup Uses Pre-Partitioned Rectangles, No Toy Manipulation

The Warmup activates recognition of equal vs. unequal parts using pre-made visuals, not the Grid Arrays toy. This keeps cognitive load at 20-30% and avoids teaching tool usage before the Lesson. Tool introduction happens in Lesson Interaction 1.1 (establishing the whole and demonstrating erase function). This staging prevents cognitive overload in the first 3-5 minutes and keeps the Warmup focused on concept retrieval rather than tool mastery.

### KDD-8: No Circles in Module 1

Circles are Module 2 content, where they build on the hexagon work from Module 1. Keeping Module 1 to Grid Arrays and Hexagons provides a clean conceptual progression: rectangles with linear cuts (Section 1) → hexagons with radial cuts (Section 3) → circles (Module 2, which extends radial cutting to round shapes). Introducing circles here would compress the progression and create confusion about when radial cutting applies.

### KDD-9: Synthesis Connection to "More Parts = Smaller Pieces"

S.1 and S.3 both surface the relationship between number of parts and part size in fraction work. This insight is not explicit in the learning goals but is a critical conceptual bridge that prepares students for comparing fractions in Modules 10-12. By naming this pattern during Synthesis (rather than leaving it for students to discover later), we plant a seed that will make fraction comparison much more accessible. The Same Whole Principle is explicitly reinforced in S.1's connection statement so students connect "more parts = smaller pieces" with "only when the whole is the same."

### KDD-10: Purpose Frame Included

Although the Warmup bridge creates anticipation, it only teases that equal parts have special names. A Purpose Frame at the Lesson opening orients students to what they'll learn (partitioning, naming, and writing fractions) without front-loading vocabulary. This provides cognitive scaffolding while honoring the pedagogical principle that students should not hear the vocabulary they're about to learn in context before they've earned it through experience.
