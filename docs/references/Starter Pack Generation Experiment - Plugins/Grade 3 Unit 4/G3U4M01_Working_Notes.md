## Cross-Reference Table B — TVP Extraction

TVP: MODULE 1
===============

**Title:** What is Division? (Partitive & Quotitive)
**Source:** OUR Lessons 1, 2, 3 (partial) | Standards: 3.OA.A.2 (interpret quotients), 3.OA.A.3 (multiply/divide word problems)

**Learning Goal:** Solve "how many groups?" and "how many in each group?" problems in a way that makes sense. Understand that division situations involve finding either an unknown number of groups or an unknown number in each group.

**Key Teaching Points:**
1. Division means making equal groups from a total — but there are TWO ways to think about it
2. Partitive (sharing): "I have 12 items and 3 bags. How many in each bag?" — the number of groups is known, find the size of each group
3. Quotitive (grouping/measurement): "I have 12 items, 4 go in each bag. How many bags?" — the size of each group is known, find the number of groups
4. THE BIG INSIGHT: Both types start with a total and end with equal groups — the question is just about which part is unknown. Students must experience both as equally valid ways to divide, not "real division" vs. "other division."
5. Students know equal groups from Unit 1 multiplication (3 groups of 4 = 12). Division reverses the question: start with 12, end with groups.
6. CRA Stage: Concrete throughout — per SME: "most of this module needs to stay concrete." Drawings/symbolic representation deferred to M2.

**Cognitive Focus:**
- IDENTIFY (Conceptual) — recognizing both division types in word problem contexts
- COMPARE (Conceptual) — distinguishing partitive from quotitive situations
- CREATE (Procedural) — building equal groups representations using the Equal Groups tool
- CONNECT (Transfer) — linking division situations to familiar Unit 1 equal groups knowledge

**Misconceptions Targeted:**
- U4.1: "Division always makes things smaller" (PRIMARY) — Introduce division through meaningful contexts; avoid any language that reinforces "smaller." The concrete dealing/measuring animations show division as organizing, not reducing.
- U4.2: Partitive/quotitive confusion (PRIMARY) — Both types taught from Day 1 with 50-50 balance (per SME D2). Distinct visual treatments: partitive shows empty containers from the start (dealing into); quotitive shows group-size reference and containers forming during measurement.
- U4.4: Interpreting remainder in context (MONITOR) — All M1 problems divide evenly, but students may encounter situations where adjustments don't reach target cleanly. Address conversationally if it surfaces. Do not formalize.

**What Students DO:**

WARM-UP — Multiplication Reactivation:
- Equal Groups (Mode 1: Structural Recognition, concrete bags/boxes with pictures) → Display familiar Unit 1 multiplication situation: 3 bags with 5 apples each
- Students count groups, count items per group — familiar actions from Unit 1
- Guide narrates the multiplication reading: "3 groups of 5. You know this — 3 × 5 = 15."
- Then the reversal: "What if I told you there were 15 apples... and 3 bags... and asked you to figure out how many go in each bag? That's what we're learning today."
- Purpose: Activate Unit 1 equal groups knowledge, then create the conceptual pivot — same structure, reversed question.

LESSON — Early activities (Partitive introduction — sharing):
- Equal Groups (Mode 3: Partitive Division, concrete bags/boxes with pictures) → Partitive dealing animations. System shows ungrouped items and known number of empty containers; items deal one-at-a-time (round-robin for first exposure).
- Key beat — the question comes AFTER the process: "How many ended up in each bag?" Students observe the dealing process BEFORE being asked what happened.
- Key beat — structural organizer introduced: "Do you know the number of groups, or the number of items in each group? For this problem, you KNEW the groups — 3 bags. You figured out what was IN each group."
- 3 partitive problems. Scaffolding fades from guide-narrated demonstration → guided observation → independent observation.
- Language: "sharing equally," "dealing out," "same number in each group"

LESSON — Mid activities (Quotitive introduction — grouping):
- Equal Groups (Mode 4: Quotitive Division, concrete bags/boxes with pictures) → Two-stage quotitive introduction:
  - Stage 1 — Boundary grouping: System shows ungrouped items and group-size reference. Student hits "Next" → boundary forms around first set of items. Guide narrates: "1 group of 3." Remaining boundaries appear sequentially. Student answers MC: "How many groups of 3?" Items never move — group structure revealed within existing arrangement.
  - Stage 2 — Measuring out: System shows ungrouped items and group-size reference. Student hits "Next" → first set of items slides out to form a new group. Remaining groups measure out sequentially. Same two-beat interaction, different animation — items move to new locations.
- Key beat — visual contrast with partitive: "This time, we know how many go in each bag — 5. But we don't know how many bags we'll need." No containers visible at start — they form during measuring.
- Scaffolding logic: Stage 1 is perceptual (see structure emerge). Stage 2 is procedural (see measuring action). Stage 1 on first quotitive exposure; Stage 2 follows within same lesson.
- 3 quotitive problems. First uses Stage 1, subsequent use Stage 2. Same scaffolding fade as Early.
- Language: "measuring out," "groups of [size]," "how many groups"
- Framing question callback: "Do you know the number of groups, or the number of items in each group? THIS time, you knew what was in each group. You figured out HOW MANY groups."
- Starter Pack Note: For quotitive measuring interaction, consider card-sort-style pattern where student actively forms groups. See UX card sorting work in progress.

LESSON — Late activities (Abstraction step + side-by-side contrast):
- Problems shift from bags/boxes with pictures to circles with dots. Guide: "Division works the same whether it's apples in bags or dots in circles."
- Equal Groups (Mode 3 then Mode 4, circles with dots) → Same total, same numbers, presented as partitive then quotitive back-to-back.
- Key beat — "Both times we started with 20. Both times we ended with equal groups. But the QUESTION was different. And the way we made the groups was different."
- 2–3 paired-contrast problems. Late focuses on two demands only: the abstraction step and the paired contrast. Type identification deferred to Synthesis.

EXIT CHECK:
3 problems:
1. Partitive: Word problem → dealing animation (circles with dots) → identify quotient. (Tests: partitive comprehension with semi-abstract visuals)
2. Quotitive: Word problem → measuring animation (circles with dots) → identify quotient. (Tests: quotitive comprehension)
3. Framing question application: Word problem text → student identifies what's known vs. unknown using the framing question, then solves. (Tests: structural organizer internalized)

PRACTICE:
- Mixed partitive and quotitive problems with Equal Groups animations
- System tracks partitive vs. quotitive balance — target 50-50; flag at 45%, hard floor at 40% (per D2)
- Mix of container-based and circle-based grouping
- At least 1 problem per session where student builds representation
- Error patterns to watch: reporting divisor as quotient (early A1 signal); defaulting to one division type

SYNTHESIS:
- Callback to warm-up: "That was multiplication — you KNEW the groups and found the total. Today you started with the total and found the groups. That's division."
- Framing question consolidation: 2 word problems (text only, one of each type). "What information do you know — the number of groups, or the number of items in each group? What do you need to find?" MC options include values from the problem to prevent guessing and catch A1 errors.
- Vocabulary consolidation: divide, equal groups, equal shares, share, group, partition
- Bridge to M2: "Today you watched and counted. Tomorrow you'll start DRAWING division situations and learning a new symbol for division."

**Scaffolding Progression:**
CRA Note: M1 stays concrete by design — per SME directive. Early and Mid use bags/boxes with picture items (familiar from Unit 1 M7). Late transitions to circles with dots — still concrete, but a step toward semi-abstract representation. Drawings, symbolic notation, and the ÷ symbol all deferred to M2. The CRA progression across M1–M2–M3 is: Concrete (M1: Equal Groups animations, bags→circles) → Representational (M2: student-created drawings + ÷ symbol) → Representational/Abstract (M3: arrays + equations with unknowns).

## Data-Level Constraint Audit

M1's data constraints are straightforward and internally consistent:

**Audit 1: Dividend range vs. divisor range**
- Dividends: 10–30
- Divisors: 2, 3, 4, 5 only
- All combinations must divide evenly
- Valid dividend/divisor pairs within range: 10÷2=5, 10÷5=2, 12÷2=6, 12÷3=4, 12÷4=3, 14÷2=7, 15÷3=5, 15÷5=3, 16÷2=8, 16÷4=4, 18÷2=9, 18÷3=6, 20÷2=10, 20÷4=5, 20÷5=4, 21÷3=7, 24÷2=12(⚠️exceeds items-per-group max of 10 for partitive), 24÷3=8, 24÷4=6, 25÷5=5, 27÷3=9, 28÷4=7, 30÷2=15(⚠️exceeds), 30÷3=10, 30÷5=6
- ⚠️ Partitive constraint: items per group result 2–10. This eliminates: 24÷2=12, 30÷2=15, 30÷3=10 (borderline). For partitive, restrict divisor 2 to dividends ≤20, divisor 3 to dividends ≤30 (result=10 is borderline — keep or exclude?).
- ⚠️ Quotitive constraint: number of groups result 2–6. This eliminates some larger dividends with small group sizes: 30÷3=10 groups (exceeds 6), 28÷4=7 groups (exceeds 6), 27÷3=9 groups (exceeds 6), 24÷3=8 groups (exceeds 6), 24÷4=6 groups (OK), 30÷5=6 groups (OK).

**Resolution:** The constraints as stated produce a workable but restricted set. For M1 problems:
- Partitive safe pairs (groups 2-5, items per group 2-10): large set available
- Quotitive safe pairs (group size 2-5, number of groups 2-6): more restricted, must choose dividends carefully
- Recommend working from the intersection to ensure both types use similar number ranges

**Audit 2: Paired-contrast problems (Late)**
- Requirement: Same total for both types. Include same-answer AND different-answer pairs.
- Same-answer example: 20 ÷ 4 = 5 (partitive: 20 into 4 bags = 5 each; quotitive: groups of 4 from 20 = 5 groups) ✓
- Different-answer example: 24 (partitive: 24 into 4 bags = 6 each; quotitive: groups of 6 from 24 = 4 groups) ✓
- Both examples use dots in circles (Late abstraction level) ✓
- Check: All results within constraints? 20÷4=5 ✓ (items per group ≤10, groups ≤6). 24÷4=6 ✓ (items ≤10). 24÷6=4... ⚠️ divisor 6 is OUTSIDE the allowed divisor range (2, 3, 4, 5 only).
- Alternative: Use 24 with divisor 4: partitive 24÷4=6 each, quotitive groups of 4 from 24 = 6 groups. But this is SAME answer again.
- Better different-answer pair: 15 → partitive: 15 into 3 bags = 5 each; quotitive: groups of 5 from 15 = 3 groups. Results: 5 and 3 (different) ✓. Both divisors (3, 5) in range ✓.

### 1. What Went Well

**Source extraction was clean.** The Module Mapping workbook and TVP docx provided strong starting material. Cross-Reference Tables A and B (extraction) required minimal correction. The SME comments in the TVP were especially useful — they pre-resolved several design decisions (D2 balance target, D4 concrete-only CRA, D8 no formal equations) that would otherwise have surfaced as author questions at Gate 2.

This was a legitimate constraint violation that manual review missed during Task 3 drafting. Without the evaluation pipeline, it would have reached SME review. The fix (30÷5=6) required checking freshness against all Lesson values and divisor variety across EC — exactly the kind of cross-referencing that benefits from systematic verification.

**Author review cadence was productive.** Three design questions at Gate 2 (vocab staging, warmup pivot, paired-contrast animation) were resolved quickly because they were framed as specific tradeoffs with options rather than open-ended questions. ### 2. What Caused Friction

**L2 agent over-prescription on lesson-eval.** The lesson-eval agent flagged 7 MAJORs at Gate 3, but every one was either a confirmed false positive (Required/Forbidden ordering — agent had the wrong template model), a pre-publish QA item (metacognitive tags), or a design decision already documented in KDD. The agent applies a prescriptive standard that exceeds actual template requirements. At Gate 4, pedagogy-eval re-raised the Warmup bridge explicitness issue (PE1.5/PE2.3) even though the author had already resolved it at Gate 2 (PE2.4). There's no mechanism for agents to "see" prior gate resolutions — each gate's L2 agents start fresh.

**Recommendation:** Add a "Prior Gate Resolutions" section to the SP or Working Notes that L2 agents can read. Format: Finding ID → Author Decision → Gate. This would prevent re-flagging of resolved design decisions and reduce author fatigue. Alternatively, the Cowork Guidance could specify that carry-forward agents (those that ran at an earlier gate) don't re-run at later gates unless the relevant section changed.

**L1 false positive burden is stable but annoying.** Three false positives persisted from Gate 3 through Gate 4 across every run:
- ST11 ×2: Checker's ordering model disagrees with the template. Required/Forbidden Phrases appear AFTER sections in the template, but the checker expects them BEFORE Purpose Frame.
- V4 ×1: Checker does exact-string match for "divide" but EC uses inflected form "divided" inside [vocab] tags.
- TC2 ×2 (Gate 4 only): Checker looks for exact §1.5 subsection header strings ("Partitive Animation (Mode 3) Specifications") in Visual: lines, but interactions use the standard "Equal Groups with Pictures or Dots (Mode 3 — Partitive)" format.

**Recommendation:** Fix ST11 ordering model to match Template v3 (Required/Forbidden after sections, not before Purpose Frame). Fix V4 to match inflected forms (divide/divided/dividing/divides). Fix TC2 to match toy names rather than §1.5 header strings.

**EC.2 value selection required two iterations.** The initial value (21÷3=7) passed manual review but violated the quotitive groups cap. The gate1-eval agent's recommended replacement (18÷3=6) was also wrong — 18÷3 is used in Lesson interaction 3.1. The correct fix (30÷5=6) required checking: (a) dividend 10–30, (b) divisor 2–5, (c) quotitive groups 2–6, (d) not used in any Lesson interaction, (e) distractor set works (group size ≠ quotient so U4.2 distractor is distinct), (f) divisor variety across EC (div 4, div 5, div 2 — all different).

**Recommendation:** Add a "Value Selection Checklist" to the EC drafting process: constraint compliance, Lesson freshness check, distractor viability, divisor variety. This is the kind of cross-referencing where a systematic approach prevents one-off errors.

**Notion push hit a size limit.** The full SP (96K characters) exceeded what could be passed through a single MCP tool call. Only §1.0 was pushed to the Notion page; the rest requires manual copy-paste or incremental push.

**Recommendation:** For M2+, push content in sections (Backbone, Warmup, Lesson, EC+Practice, Synthesis+KDD) rather than as a single payload. Or investigate whether the Notion MCP tool has a file-upload mode.
