## 1.0 THE ONE THING

**[REQUIRED]**

Division situations can be represented in multiple connected ways — as drawings, as ÷ expressions, and as word problems — and the same division expression can describe both a sharing (partitive) AND a grouping (quotitive) situation.

**CRA Stage:** Concrete → Representational bridge. Students still manipulate concrete representations (Equal Groups tool) but shift from observing to constructing, and connect these representations to the new ÷ symbol. Per D4: abstract notation (÷) appears alongside concrete visuals, not after.

**Critical Misconception:** A1 (dividend/divisor confusion) — students default to "big number ÷ small number" as a position rule rather than grounding each number's placement in the situation's meaning ("What's the total? What's the group information?").

**Biggest Risk:** Students learn to write ÷ expressions procedurally ("put the big number first") without connecting the expression to what it means — which number is the total, which describes the groups, and why the same expression serves both division types.

**Success Indicator:** Given a word problem with no visual support, the student writes a correct ÷ expression AND can explain what each number represents in the context of the problem (not by citing a position rule).

## 1.1 LEARNING GOALS

**[REQUIRED]**

*Verbatim — Script Must Achieve These*

**L1:** "Interpret and relate drawings and descriptions of division situations. Understand that a division situation may involve finding an unknown number of groups or finding an unknown number of objects in each group."

**L2:** "Interpret division expressions. Understand that the same division expression can be used to represent both types of division situations."

**L3:** "Solve 'how many groups?' and 'how many in each group?' problems. Write division expressions to represent division situations."

### 1.1.2 Module Bridges

**From M1 (What is Division?):** Students experienced both partitive (sharing) and quotitive (grouping) division concretely through Equal Groups animations. They can identify quotients from animations, distinguish the two types using the framing question ("Do you know the number of groups, or the number of items in each group?"), and use informal vocabulary (divide, equal groups, equal shares, share, group, partition). They have NOT seen the ÷ symbol, written any equations, or created their own representations.

**This Module:** Students move from watching to constructing — they build Equal Groups representations using stepper controls, learn the ÷ symbol and "division expression," practice translating in both directions (visual ↔ symbolic), and write division expressions independently from word problems. The key insight is that one expression (e.g., 20 ÷ 4 = 5) represents both division types.

**To M3 (Division as Unknown Factor):** Students arrive with ÷ fluency — they can read, write, and interpret division expressions for both division types. M3 connects ÷ to ×: "15 ÷ 3 = 5 is the same as ? × 3 = 15." This unknown-factor framing reframes division as "what number times 3 gives me 15?" — leveraging multiplication facts students already know. Arrays replace Equal Groups as the primary tool, enabling the visual bridge between multiplication and division. Formal vocabulary (dividend, divisor, quotient) is introduced in M3.

## 1.2 SCOPE BOUNDARIES

**[REQUIRED]**

### ✅ Must Teach

* Student construction of Equal Groups representations using stepper controls (both division types)
* The ÷ symbol as notation for division: "[total] ÷ [groups or group size] = [result]"
* "Division expression" / "division equation" as vocabulary
* Two-directional translation: situation/drawing → expression AND expression → situation/drawing
* The key insight: one ÷ expression represents BOTH partitive and quotitive situations
* Reading ÷ expressions aloud: "[total] divided by [groups/group size] equals [result]"
* Word problem → ÷ expression writing (both division types, ~50-50 balance per D2)
* Reinforcement of M1 vocabulary: divide, division, equal groups, equal shares, share, group, partition
* Reinforcement of M1 framing question: "Do you know the number of groups, or the number of items in each group?" — now operationalized as a tool interaction (which stepper to set first)

### ❌ Must Not Include

* Formal vocabulary: dividend, divisor (M3 — per SME: "defer, too much to introduce here")
* Quotient as a formal term requiring student use (M3 — informal Guide use of "the answer to a division problem" is acceptable)
* Unknown factor framing: ? × 3 = 12 (M3)
* Arrays as a division tool (M3 — Equal Groups is the M2 tool)
* Fact families or inverse relationship framing (M4)
* Problems with remainders — all M2 problems divide evenly
* Formal division type labels requiring student use ("partitive," "quotitive" — students use the framing question, not labels)
* Multiplication expressions or equations as a primary focus (× appears only in M2 Synthesis bridge to M3)

⚠️ **MEANING OVER POSITION:** Students must never learn "put the big number first" as a rule for writing ÷ expressions. Every expression must be grounded in situational meaning: "What's the total? That's the first number. What's the group information? That's the second number." This prevents A1 (dividend/divisor confusion) from becoming a rigid positional habit.

## 1.4 MISCONCEPTIONS

**[REQUIRED]**

### 1.4.1 #U4.2: Partitive/Quotitive Confusion (PRIMARY)

**Trigger Behavior:** Student correctly writes ÷ expressions for one division type but struggles with the other. May reinterpret all problems as sharing (or all as grouping). When told that 20 ÷ 4 = 5 can mean both "share 20 among 4 groups" AND "measure groups of 4 from 20," student shows confusion or disbelief.

**Why It Happens:** Most prior instruction (and home experience) reinforces sharing as "real division." The insight that one expression represents two fundamentally different actions is genuinely conceptually challenging. Students may also have M1 experiences that were inadvertently partitive-heavy.

**Visual Cue:** Side-by-side Equal Groups displays: same equation, same numbers, different arrangements (one partitive, one quotitive). The visual contrast makes "same expression, different meaning" concrete.

**Prevention Strategy:** (1) Maintain 50-50 partitive/quotitive balance across all phases (D2). (2) The "same equation, both types" key moment in S2 is the primary intervention — show two different arrangements both matching the same expression. (3) The framing question operationalized as stepper interaction in S1 forces students to distinguish types BEFORE building. (4) In EC, both types are tested — not just the "easier" one.

### 1.4.2 #A1: Dividend/Divisor Confusion (PRIMARY)

**Trigger Behavior:** Student writes 3 ÷ 12 instead of 12 ÷ 3 when shown a picture of 12 items in 3 groups. May state "you always put the big number first" as a rule without being able to explain why. When writing expressions from word problems, reverses the total and the group information.

**Why It Happens:** Division notation is new — students have no prior experience with the ÷ symbol's positional conventions. "Big number first" is a heuristic that happens to work for all Grade 3 problems, making it self-reinforcing. Without meaning-based grounding, notation becomes arbitrary symbol manipulation.

**Visual Cue:** Connection lines from visual elements to equation parts (total → first number, groups/group-size → second number). The animation makes the mapping between situation and notation visible.

**Prevention Strategy:** (1) Every ÷ expression is grounded in situational meaning — Guide always identifies what each number represents before or after writing the expression. (2) Never state or reinforce "big number first" as a rule. (3) EC.2 includes a "numbers-swapped distractor" — if student selects the arrangement matching the reversed expression, the pipeline catches A1. (4) The stepper construction in S1 embeds the structure: student sets the known quantity first, and the visual total updates — the equation maps directly to the construction process.

## 1.6 WARMUP (~2 minutes)

[REQUIRED]

### Core Purpose

Reactivate M1 concrete division experience and prime students for the shift from observation to construction — the cognitive pivot that defines M2.

**Key Function:** This Warmup callbacks to M1's partitive sharing animation (familiar context, familiar numbers) and then prompts students to identify what information they had to figure out. This activates the framing question ("Do you know the groups or the items per group?") as a thinking tool, priming the Lesson's stepper interaction where setting the correct stepper IS identifying the division type.

**Why this serves the concept:**

* Activates M1 division vocabulary ([vocab]divide[/vocab], [vocab]equal groups[/vocab]) in a familiar context — students hear these terms before M2 adds new notation
* Reactivates the "identify the unknown" reasoning that becomes a tool interaction in S1
* Creates the conceptual question ("What if YOU had to build this?") that the Purpose Frame and S1 immediately answer

**Test:** If we removed this Warmup and went straight to the Lesson, students would encounter the stepper controls without the mental frame that the key decision is "which quantity is known?" — they'd manipulate the tool without understanding what they're doing. The Warmup pre-loads this reasoning.

### Warmup Parameters

| Element | Specification |
| :---- | :---- |
| **Time** | ~2 minutes |
| **Interactions** | 3 (2 with student action + 1 bridge) |
| **Cognitive Load** | 20-30% |
| **Remediation** | Pipeline |
| **Vocabulary** | NEW terms: forbidden. ESTABLISHED M1 terms: [vocab]divide[/vocab], [vocab]division[/vocab], [vocab]equal groups[/vocab], [vocab]group[/vocab] activated for review. |

| MUST | MUST NOT |
| :---- | :---- |
| Use Equal Groups Mode 3 (Partitive) with familiar bag/picture visuals | Introduce ÷ symbol or "division expression" (save for Lesson S2) |

### Interaction W.1: Division Recall [ACTIVATION]

* **Purpose:** Reactivate M1 partitive division. System replays a brief sharing animation using M1-familiar values and visuals. Student identifies the quotient — confirming that division meaning is active.
* **Visual: Equal Groups (Mode 3 — Partitive).** Horizontal. 15 loose apples displayed. 3 bags visible, empty. Round-robin dealing animation plays (~10-15 sec): apples deal one-at-a-time into bags until all 15 are distributed. Post-animation state: 3 bags with 5 apples each.
* **Guide:** "Here's something you've done before. Apples going into bags, same number in each. Watch... How many ended up in each bag?"
* **Prompt:** "How many apples are in each bag?"
* **Student Action:** MC (single)
  * **Options:** 5, 3, 15, 12
* **Correct Answer:** 5
* **Answer Rationale:**
  - 5 = Correct (15 dealt into 3 bags = 5 per bag)
  - 3 = Reports number of bags, not items per bag (U4.2 precursor)
  - 15 = Reports the total, not the per-group result
  - 12 = 15 - 3 (subtraction error)
* **On Correct:** "Five in each bag. You [vocab]divided[/vocab] 15 into 3 [vocab]equal groups[/vocab]."
* **Remediation:** Pipeline

> **Engagement Anchors:** (1) Personalization — "something you've done before" callbacks to M1 experience. (2) Narrative Setup — "apples going into bags" uses a familiar sharing scenario that contextualizes the math.

---

### Interaction W.2: What Did You Figure Out? [ACTIVATION]

* **Purpose:** Activate the framing question reasoning from M1. Student identifies which piece of information was the unknown — priming the stepper interaction in S1 where the student must determine which quantity to set first.
* **Visual: Equal Groups (Mode 3 — Partitive).** Post-animation state from W.1: 3 bags with 5 apples each.
* **Guide:** "You started with 15 apples and 3 bags, and you figured out 5 go in each. Think about that. Which part wasn't given?"
* **Prompt:** "Which piece of information did you figure out?"
* **Student Action:** MC (single)
  * **Options:** How many in each bag, How many bags there were, How many apples altogether, How many apples to take away
* **Correct Answer:** How many in each bag
* **Answer Rationale:**
  - How many in each bag = Correct (total 15 and groups 3 were given; per-group amount was the unknown)
  - How many bags there were = Incorrect (number of bags was given — 3 visible from the start)
  - How many apples altogether = Incorrect (total was given — 15 stated)
  - How many apples to take away = Incorrect (subtraction frame — no apples were removed)
* **On Correct:** "You knew the total and the [vocab]groups[/vocab]. You figured out the in-each-[vocab]group[/vocab] part. That's [vocab]dividing[/vocab]."
* **Remediation:** Pipeline

> **Scaffolding Note:** W.2 activates the "identify the unknown" reasoning without teaching anything new — this IS what students learned in M1 through the framing question. The MC options mirror the structure of M1's framing question ("Do you know the groups or the items per group?") and prime the stepper decision in S1 where setting the correct stepper first IS identifying the known quantity.

### Interaction W.3: Bridge to Lesson [BRIDGE]

* **Purpose:** Create anticipation for Lesson without teaching. Preview the two new skills (building pictures + new symbol) that define M2.
* **Visual: Equal Groups (Mode 3 — Partitive).** Same post-animation state: 3 bags with 5 apples each.
* **Guide:** "Last time, you watched items get sorted. But what if YOU could build the picture yourself? And what if there were a way to write this down that goes with your pictures? That's exactly what you're about to learn."

> **Design Note:** The bridge creates two curiosity gaps — "build the picture yourself" (S1) and "write this down that goes with your pictures" (S2) — that the Lesson immediately addresses. No division vocabulary beyond established M1 terms. Session-relative language: "last time" not "yesterday."

## 1.7 LESSON (~12-14 minutes)

[REQUIRED]

### Core Purpose

Build understanding that division situations can be represented in multiple connected ways — as student-constructed Equal Groups drawings, as ÷ expressions, and as word problems — and that the same ÷ expression represents both partitive and quotitive situations. Students move from construction (building their own representations) to notation (learning the ÷ symbol) to independent expression writing, completing the Concrete → Representational bridge.

### Pedagogical Flow

Students progress through three sections that map to the CRA arc specified in the TVP:

**Section 1 — Creating and Interpreting Drawings (Concrete):** Students shift from M1's OBSERVATION to M2's CONSTRUCTION. Using stepper controls, they build Equal Groups representations from word problems. The framing question ("Do you know the groups or the items per group?") becomes a tool interaction — setting the correct stepper first IS identifying the division type. Reverse direction also practiced: completed display → student identifies the situation.

**Section 2 — Introducing ÷ and Connecting to Drawings (Concrete → Representational bridge):** The ÷ symbol is introduced AFTER S1's concrete grounding (Known Pattern #11). Equation Builder enters. Students match in both directions: display → equation (easier, first per R2) then equation → display (harder). The key insight — same expression represents both division types — is made explicit through a paired comparison.

**Section 3 — Writing Expressions Independently (Representational → early Abstract):** Students construct ÷ equations from word problems using Equation Builder with no visual upfront. Visual scaffolding fades: confirmation after → no confirmation → full loop (context → equation → drawing). This is the closest to Abstract that M2 reaches.

### 1.7.2 LESSON SECTION 2: Introducing ÷ and Connecting to Drawings

### Interaction 2.1: The Division Symbol [WORKED EXAMPLE]

* **Purpose:** Introduce the ÷ symbol and "division expression" vocabulary AFTER concrete grounding in S1 (Known Pattern #11). Guide names the notation alongside a familiar completed display, connecting each part of the expression to its meaning in the picture. Connection-line animation makes the mapping visible.
* **Visual: Equal Groups (Mode 3 — Partitive).** Horizontal. 3 bags with 5 apples each, total 15 displayed. Alongside: Equation Builder showing tiles assembling into 15 ÷ 3 = 5. Connection lines animate: total (15 loose apples at start) → "15," groups (3 bags) → "3," per-group (5 in each) → "5."
* **Guide:** "Mathematicians have a short way to write what you've been building. Watch. This picture shows 15 apples [vocab]divided[/vocab] into 3 bags, 5 in each. How would they write that? Like this: 15 [vocab]÷[/vocab] 3 = 5. That [vocab]÷[/vocab] sign means 'divided by.' Fifteen divided by three equals five. This whole thing, 15 [vocab]÷[/vocab] 3 = 5, is called a [vocab]division expression[/vocab]. See how it connects? The first number is the total. The number after [vocab]÷[/vocab] is the [vocab]group[/vocab] information. The answer comes after the equals sign."

> **Scaffolding Note:** This is a Type A (teaching-only) interaction. Three new terms introduced: ÷ (symbol), "divided by" (oral), "division expression" (name). ### 1.7.6 Success Criteria

By the end of the Lesson phase, a student who has met the Learning Goals will demonstrate:

1. **Construction fluency (L1):** Given a word problem specifying a partitive or quotitive division, the student uses stepper controls to build the correct Equal Groups representation — setting the known quantity stepper first and adjusting the other to match the target total — without Guide prompting on which stepper to use.

2. **Symbolic translation (L2):** Given a completed Equal Groups display, the student constructs the correct ÷ expression using Equation Builder, placing the total first and the group information after ÷, for both partitive and quotitive arrangements.

3. **Independent expression writing (L3):** Given a word problem with no visual support, the student constructs a correct ÷ expression, grounding each number's placement in the problem's meaning (not a positional rule).

4. **Dual-type recognition (L2):** When shown two different Equal Groups arrangements alongside the same ÷ expression, the student recognizes that one expression represents both division types.

> **Observable vs. Internal:** Criteria 1–3 are directly observable through student actions in the tool. Criterion 4 is assessed through the MC response in 2.4 and reinforced through the contextual disambiguation in 2.5. All criteria map to Exit Check items: EC.1 tests criterion 2, EC.2 tests criteria 2+4 (with A1 distractor), EC.3 tests criterion 3.

### Error-Pattern Monitoring

The following error patterns should trigger diagnostic flags in the Practice pipeline:

* **A1 — Dividend/divisor reversal in expression writing:** If a student consistently places the divisor first (e.g., writes 3 ÷ 18 instead of 18 ÷ 3), flag for A1 diagnostic. Indicates positional rule ("small number first") rather than meaning-based construction. Intervention: route to SUPPORT tier with labeled Equation Builder slots and explicit "total goes first" Guide prompts.
* **U4.2 — Division-type confusion in matching:** If a student consistently selects the wrong-type display when matching equation→display (e.g., selects partitive display for quotitive context or vice versa), flag for U4.2. Intervention: increase frequency of S5 same-expression problems to reinforce the insight that context determines type, not expression.
* **D2 — Division-type default in expression writing:** If a student succeeds on partitive word problems but fails quotitive (or vice versa), flag for D2 balance intervention. Intervention: increase frequency of the weaker type in S3 problems and add framing question overlay.
* **Stepper confusion:** If a student consistently sets the wrong stepper first in S1 problems (e.g., sets items-per-group stepper to the number of groups), flag for framing question remediation. This indicates the student can construct representations but can't parse the problem structure. Intervention: add framing question prompt before each S1 problem.

## 1.10 KEY DESIGN DECISIONS SUMMARY

[REQUIRED]

### KDD-1: Both Division Types Maintained at ~50-50 Balance (D2)

Partitive and quotitive appear with equal emphasis across all phases. Lesson: 7P/6Q/1Both = 50-43-7%; EC: 1Q/2P; Practice targets 50-50 per system tracking. The slight partitive lean in EC is acceptable — Practice corrects any cumulative imbalance. D2's system flag at 45%, hard floor at 40%, is enforced by the Practice pipeline.

### KDD-2: Vocabulary Introduced AFTER Concrete Grounding (Known Pattern #11)

The ÷ symbol and "division expression" are introduced in S2, after 5 S1 interactions where students construct Equal Groups representations concretely. The sequence is: build (S1) → name (S2) → use independently (S3). This ensures notation maps to a felt understanding rather than an abstract definition. Int 2.1 introduces three related terms together — ÷ (symbol), "divided by" (oral reading), "division expression" (name) — because all three name a single concept (the notation for an action students already understand). Splitting them across interactions would create artificial separation between a symbol, how to say it, and what to call it.

### KDD-3: D8 Nonstandard Equation Formats Limited to Synthesis (LIGHT)

Module Mapping says "equations presented in nonstandard formats from the start" — this is a unit-level directive. M2 calibrates locally: the ÷ symbol is brand new, students are learning Equation Builder for the first time, and two-directional translation is already cognitively demanding. Aggressive nonstandard exposure would overload M2. One observational instance in Synthesis S.2 (5 = 30 ÷ 6) provides light exposure without disrupting the core learning arc. Nonstandard exposure increases in M3+ where ÷ is familiar. See Conflict Log #6.

### KDD-4: Stepper Construction with Both Steppers Free (per SME Resolution)

In S1, both the groups stepper and items-per-group stepper are free from the start. Students must determine which quantity is known and set that stepper first — this decision IS the division type identification. Locking one stepper would hint the division type and undermine the framing question's pedagogical purpose.

### KDD-5: "Set Stepper" and "Drag to Build" Flagged as NEW-ACTION

The Equal Groups Stepper Mode interaction (students use +/- controls to set a target numeric value) does not map to any existing Student Action Vocabulary term. "Set stepper" was coined with the definition "use +/- controls to set a target numeric value." Used in interactions 1.2, 1.3, 1.4, 3.4b. Additionally, "Drag to build" applies the existing Student Action Vocabulary "drag" pattern to Equation Builder tile-construction (placing number and operator tiles into labeled slots). Both terms require engineering confirmation per Student Action Vocabulary extension rules. AF3 (§1.7.5) tracks the "Drag to build" confirmation specifically.

### KDD-6: Meaning-Based Equation Frame, NEVER Positional Rules

Every ÷ expression is grounded in "What's the total? That goes first. What's the group information? That goes after ÷." The phrase "put the big number first" is explicitly forbidden. This prevents A1 (dividend/divisor confusion) from solidifying as a positional habit. The meaning-based frame is stated in 2.1 (introduction), reinforced in 2.2–2.3 (Guide dialogue), tested in 2.6 (A1 distractor), and assumed in S3 (independent writing).

### KDD-7: S2 Expanded to 6 Interactions (4 Matching Problems) per TVP Spec

TVP specifies "4–5 matching problems across both directions." Original draft had 3 matching problems (2 display→equation + 1 equation→display). Expanded to 4 matching problems (2 display→equation + 2 equation→display) by adding interaction 2.5 (quotitive equation→display). This balances both translation directions at 2:2 and gives quotitive a second assessed touch in S2. S2 time estimate adjusts from ~4-5 min to ~5-6 min; total Lesson time adjusts from ~12-14 min to ~13-15 min.

### KDD-8: Quotient Variety Enforced Across Consecutive Interactions

Quotient variety enforced module-wide to prevent students from expecting "the answer is always 6." S1: changed 1.4 from 12 ÷ 2 = 6 to 16 ÷ 2 = 8 (sequence: 6, 6, 8, 5). S3: changed 3.2 from 24 ÷ 4 = 6 to 28 ÷ 4 = 7, breaking three consecutive quotient-6 problems (sequence: 6, 7, 6, 5). Practice should actively vary quotients across the full range — quotient 6 is structurally overrepresented in the Lesson's data range.

### KDD-9: R3 Remediation Skip on 3.4b Removed — Not Engineered

The original design included a conditional skip: "If student enters 3.4 after heavy remediation on 3.1–3.3, skip 3.4b (drawing construction)." This interaction-level conditional skipping is not a supported pipeline feature. The three-step chain (3.4a→3.4b) stands for all students; the Pipeline handles remediation within each sub-step independently.

### KDD-10: EC.3 Uses Lower Late Data Range per R4

EC.3 (word problem → equation, no visual) is the highest-demand assessment item. Values were restricted to the lower end of the Late range (dividend 18, divisor 3) to avoid confounding the conceptual assessment with computational difficulty. A student who understands context→expression translation shouldn't fail because 36 ÷ 6 requires harder arithmetic.

### KDD-11: Connection-Line Animation and Real-Time Cross-Tool Linking Flagged for Engineering (AF1, AF2)

Two TVP-specified features require engineering confirmation: (a) connection lines from visual elements to equation parts during the first ÷ equation introduction (2.1), and (b) real-time Equal Groups display updates as students place Equation Builder tiles (3.1 remediation). Both are new visual state types per Known Pattern #44. Fallbacks documented in §1.7.5 if either is not buildable.

### KDD-12: Consolidation Moment After First Equation Construction (2.2)

S2 bundles 5 novel interactions in a tight cognitive window around the key ÷ symbol introduction. A consolidation note after 2.2 (the student's first equation construction) gives the notation time to land before moving to quotitive in 2.3. The labeled Equation Builder slots in 2.2–2.3 serve as the consolidation scaffold, keeping the meaning-based frame visible.

### KDD-13: Warmup Activation Type, Not Binary Choice

M2 continues directly from M1 and needs M1 knowledge activated, not explored fresh. The Activation warmup re-establishes concrete division experience and primes the construction + notation shift. Per Warmup Playbook §4D: Activation best for modules with concrete prior-module knowledge that must be reactivated before extending.

### KDD-14: Think-Alouds Model Genuine Uncertainty

Worked-example think-alouds in 1.1 and 2.1 use hedging language ("Let me think about...", "Let me try...", "How would they write that?") rather than confident narration. This models the authentic reasoning process students should internalize — approaching a new tool or notation with strategic thinking, not procedural certainty.

### KDD-15: Animation Specifications Use Visual-Field Format, Not Pattern 3 Specification Tables

Consistent with M1 SP convention. Animation details (type, duration, post-animation state) are captured inline in the Visual field. No information is lost relative to the spec table format.

### KDD-16: Synthesis S.1 Uses Type D (Representation Transfer) for Early Module

Playbook §3C prefers Type A (Pattern Discovery) + Type C (Real-World Bridge) for early modules (M1-M3). S.1 uses Type D (Representation Transfer) instead because M2's core work IS representation transfer — students learn to move between story, picture, and ÷ expression. Testing this transfer in Synthesis is the natural consolidation task. Per Playbook §2 pedagogical override principle: when synthesis guidelines conflict with effective teaching, choose effective teaching.

### KDD-17: Two Worked Examples Sufficient for ÷ Notation Introduction

Playbook §2 recommends ≥3 worked examples when introducing brand-new concepts. M2's ÷ notation introduction uses 2 (Int 1.1 stepper think-aloud, Int 2.1 symbol introduction with connection-line animation) because: (a) M1 provides 5+ concrete division interactions as foundational priming — the underlying concept is established, only the notation is new; (b) 2.1 is a high-density worked example with connection-line animation making the visual-to-symbolic mapping explicit; (c) 2.2's labeled-slot Equation Builder with pre-populated tiles functions as heavily scaffolded guided practice at near-worked-example level. Adding a 3rd worked example would delay student action and slow the section without proportional learning gain.
