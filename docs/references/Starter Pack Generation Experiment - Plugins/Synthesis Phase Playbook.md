# **SYNTHESIS PHASE PLAYBOOK**

Version: January 2026

## **1\. PHASE IDENTITY**

**Core Purpose:**  
 Consolidation phase where students connect concepts to broader mathematical understanding through pattern recognition, representation transfer, and metacognitive reflection. This is NOT more practice—it's connection-making and generalization.

**Position in Flow:**

* Follows Practice (mastery-building phase)  
* Precedes Recap or End of Module  
* Transition FROM Practice: "You worked through \[X problems\]. Now let's connect this to bigger ideas."  
* Transition TO Recap: "You've connected these ideas beautifully. Let's wrap up what you learned today."

**Typical Duration:**  
 6-8 minutes (see Section 2 for timing breakdown; prioritize quality connections over rigid timing)

**Interaction Count:**

* **Typical range:** 3-5 interactions per synthesis  
* **Minimum:** 3 meaningful connections (2 or fewer insufficient for consolidation)  
* **Maximum:** 6 interactions (beyond this, you're practicing not synthesizing)  
* **Module-specific guidance:** See Section 3C for detailed breakdown by module level

**Let pedagogical need determine count.** Guidelines are tendencies, not mandates.

**Primary Focus:**

* Recognizing patterns across multiple examples  
* Transferring concepts between visual representations  
* Connecting mathematics to real-world contexts  
* Reflecting on thinking strategies  
* Generalizing "what's always true"

**Pedagogical Rationale:**  
 Practice builds procedural fluency—students learn to DO the skill consistently. But understanding isn't just execution—it's recognizing invariant patterns across contexts, connecting to broader mathematical ideas, and developing metacognitive awareness. Synthesis reveals and cements that deeper understanding. Students who can only execute procedures in Practice haven't fully internalized concepts.

**Energy & Tone:**

* **Energy Level:** Medium (reflective, not exhausted)  
* **Voice:** Warm, affirming, discovering together  
* **Example language:** "Let's explore what patterns you discovered..."  
* **Balance:** Genuine excitement about mathematical connections

**Cognitive Load Consideration:**  
 Students reaching Synthesis have completed Warmup, Lesson, Exit Check, and Practice (potentially 25-30 minutes of work). They've demonstrated mastery but may be fatigued. Keep synthesis:

* **Efficient:** 6-8 minutes, not extended  
* **Varied:** Change task types to maintain engagement  
* **Rewarding:** Emphasize discovery and growth, not more assessment  
* **Lighter cognitively:** Reflection and connection are less taxing than problem-solving

**What Makes Synthesis Different from Other Phases:**

| Element | Practice | Synthesis |
| ----- | ----- | ----- |
| **Purpose** | Build fluency through repetition | Connect and generalize concepts |
| **Interactions** | 10-12 similar problem variations | 3-5 diverse discovery tasks |
| **Cognitive Work** | Apply procedure consistently | Recognize patterns, transfer, reflect |
| **Student State** | Building mastery | HAS mastery, extending it |
| **Guide Role** | Support independent work | Facilitate discovery and reflection |
| **Remediation** | Full support as needed | Minimal—mastery assumed |
| **Energy** | Medium, task-focused | Medium, reflective, connecting |

**Prerequisites:**  
 Students reach Synthesis after passing Practice (70%+ with transfer gate). See Section 5 workflow for verification steps.

**What Synthesis Is NOT:**

* Not additional practice problems (that was Practice phase)  
* Not assessment (that was Exit Check)  
* Not new teaching (if introducing new procedures, it's wrong)  
* Not review of specific Practice problems (generalize instead)

## **2\. STRUCTURAL REQUIREMENTS**

**Required Elements:**

Every Synthesis must include:

1. **Opening Frame** (30-45 seconds)

   * Signal shift from practice to reflection/connection  
   * Set tone for discovery  
   * Example: "You've practiced \[concept\] with \[tools\]. Now let's see what patterns emerge."  
2. **Connection Tasks** (4-5 minutes)

   * 3-5 diverse discovery activities  
   * Each task reveals different aspect of understanding  
   * Must involve connection, not just application  
   * Visual support for every task  
3. **Metacognitive Reflection** (1-2 minutes)

   * Student reflects on thinking or strategies  
   * Builds awareness of mathematical reasoning  
   * Example: "What strategy helped you most today?"  
4. **Identity-Building Closure** (30 seconds)

   * Specific, growth-oriented affirmation  
   * Connect to future learning  
   * Example: "You discovered \[specific pattern\]. That thinking will help when we \[future application\]."

**Sequencing Rules:**

The order builds cognitive progression:

```
OPENING → CONNECTION TASKS (3-5) → REFLECTION → CLOSURE
```

1. **Opening:** Brief (30-45 sec) frame that signals consolidation work  
2. **Tasks:** Sequence from simpler recognition to complex generalization  
3. **Reflection:** Metacognitive moment after tasks but before closure  
4. **Closure:** Identity affirmation, not generic praise

**Task Progression Guidelines:**

* Start with recognition or pattern identification (easier)  
* Move to transfer between representations (moderate)  
* End with generalization or real-world connection (harder)  
* Each task should be cognitively distinct (not variations)

**Pedagogical Override Principle:**

When synthesis guidelines conflict with effective teaching:

1. Choose effective teaching  
2. Document the decision  
3. Explain pedagogical reasoning

**Examples:**

* Module needs 6 interactions for thorough consolidation → Use 6, not force into 5  
* Standard task types feel artificial → Design custom consolidation task that serves learning  
* Time extends to 9 minutes for deep reflection → Allow it if connections are genuine  
* Module has thin conceptual depth → Use 3 interactions minimum, don't artificially inflate

**Quality learning always supersedes structural compliance.**

## **3\. PHASE-SPECIFIC REQUIREMENTS**

### **3A. The Five Types of Cognitive Work in Synthesis**

Synthesis does cognitive work that Practice doesn't. Students must demonstrate these five capabilities:

| Type | Definition | Example | Why It Matters |
| ----- | ----- | ----- | ----- |
| **Pattern Recognition** | Identifying commonalities across multiple examples | "Look at 2/4, 3/6, and 4/8. What do ALL of them share?" | Seeing what stays the same across examples reveals mathematical structure |
| **Representation Transfer** | Showing same concept using different visual tools | "Show 2/4 on a bar. Now show 2/4 on a number line. What stayed the same?" | Concepts transcend specific representations—this tests true understanding |
| **Real-World Connection** | Linking abstract mathematics to concrete, familiar contexts | "Where do you see thirds outside of math class?" | Mathematics describes patterns in the real world—making this explicit cements understanding |
| **Metacognitive Reflection** | Thinking about thinking processes and strategies | "What helped you know these fractions were equal?" | Awareness of reasoning processes enables students to apply thinking to new situations |
| **Generalization** | Articulating principles that are "always true" | "What's ALWAYS true about fractions that equal 1/2?" | Moving from examples to principles is the hallmark of mathematical understanding |

**The Test:**  
 If a Synthesis task could be in Practice with just different numbers, it's not Synthesis—it's more practice. Synthesis tasks MUST involve CONNECTION, not just APPLICATION.

**Example Contrast:**

| Practice Task | Synthesis Task |
| ----- | ----- |
| "Which fraction equals 1/2?" | "Look at 2/4, 3/6, 4/8. What pattern makes them ALL equal 1/2?" |
| "Shade 3/4 on this bar" | "Show 3/4 on a bar, then on a circle. What's the mathematical relationship that stayed the same?" |
| "Is 2/3 larger than 1/3?" | "You compared fractions with same denominators. When is that strategy useful?" |

### **3B. Four Synthesis Task Types**

**CRITICAL: These are STRUCTURAL TEMPLATES showing connection patterns, not scripts to copy.**

Each task type below includes a complete worked example to illustrate the pattern. When generating synthesis for YOUR module:

* **Preserve the STRUCTURE:** 4-grid comparison, 4-step progression, 3-scenario bridge, 2-tool transfer  
* **Replace ALL CONTENT:** Fractions → your domain's concepts, tools, and contexts  
* **Keep SPECIFICITY:** Use concrete values and examples, not generic placeholders  
* **Focus on CONNECTION:** Synthesis ties concepts together, reveals patterns, bridges representations—it does NOT teach new procedures

**DO NOT copy example fractions/contexts into non-fraction modules.**  
 **DO NOT introduce new teaching—consolidate what students already know.**

---

Choose 3-5 tasks from these four types based on module complexity and learning goals:

---

#### **TYPE A: PATTERN DISCOVERY**

**Structure:** Present 4 examples in a grid (3 follow pattern, 1 breaks it)

**Student Task:** "Which three go together? Why?"

**Cognitive Demand:** Identifying commonality and recognizing outliers

**When to Use:** Early in Synthesis, or when students need to recognize mathematical structures

**Example (Module 5: Comparing Fractions):**

```
Four comparisons appear:
┌─────────────┬─────────────┐
│  2/4 < 3/4  │  1/3 < 2/3  │
│  [Bar vis]  │  [Bar vis]  │
├─────────────┼─────────────┤
│  3/4 > 2/4  │  1/2 < 3/4  │
│ [DIFFERENT] │  [Bar vis]  │
│  [Circle!]  │             │
└─────────────┴─────────────┘

Prompt: "Look at all four comparisons. Which three go together?"

Guide: "Interesting choice! Here's what other students noticed..."
• "The bottom left uses a circle instead of bars"
• "Three compare same denominators, one compares different denominators"
• "One shows 'greater than,' three show 'less than'"

Guide: "See how the same idea—comparing fractions—works across different setups? That flexibility shows you truly understand size relationships."
```

**Design Principles:**

* All examples visible simultaneously  
* Multiple valid "doesn't belong" choices (rich discussion)  
* Guide validates different observations  
* Connects to broader understanding at end

---

#### **TYPE B: PROGRESSIVE CHALLENGE**

**Structure:** 4-task sequence building from recognition to generalization

**Cognitive Progression:**

1. **Recognition** (Low demand) \- "Which shows \[concept\]?"  
2. **Creation** (Medium demand) \- "Now YOU create one"  
3. **Comparison** (Med-High demand) \- "How are these similar? Different?"  
4. **Generalization** (High demand) \- "What's ALWAYS true?"

**When to Use:** Mid-to-late Synthesis, when students are ready for scaffolded complexity progression

**Example (Module 3: Non-Unit Fractions):**

```
Task 1 - RECOGNITION:
Guide: "Which bar shows 2/3?"
[Three bars appear, student selects]
Guide: "Yes! Two out of three equal parts are shaded."

Task 2 - CREATION:
Guide: "Now shade 2/3 on this blank bar."
[Blank bar with three sections appears]
Guide: "Perfect! You created your own example of 2/3."

Task 3 - COMPARISON:
Guide: "Look at both—the bar and the circle. Both show 2/3. How are they similar? How are they different?"
[Student selects from options]
Guide: "Right! The shape changed, but the relationship—2 out of 3 equal parts—stayed the same."

Task 4 - GENERALIZATION:
Guide: "What's ALWAYS true about any fraction that shows 2/3?"
[Student considers, selects response]
Guide: "Exactly! No matter the shape or tool, 2/3 means 2 equal parts out of 3 equal parts total. That's the mathematical pattern behind the fraction."
```

**Design Principles:**

* Each task builds on previous  
* Cognitive demand increases gradually  
* Final task requires principle extraction  
* Guide connects back to core concept

---

#### **TYPE C: REAL-WORLD BRIDGE**

**Structure:** Present 3 real-world scenarios, student identifies which matches module concept

**Student Task:** "Which of these uses the same mathematical idea?"

**When to Use:** After students have mastered abstract concept, to cement understanding through familiar contexts

**Example (Module 2: Naming Unit Fractions):**

```
Guide: "We've been working with unit fractions like 1/2, 1/3, 1/4. Where might you see these outside of math?"

Three scenarios appear:
┌──────────────────────────────────────┐
│ 🍕 Scenario 1: Pizza cut into 8      │
│    slices, 1 slice highlighted       │
│                                      │
│ 📏 Scenario 2: Ruler showing inches │
│    divided into halves               │
│                                      │
│ 🎂 Scenario 3: Birthday cake cut    │
│    into 6 equal pieces, 1 on plate  │
└──────────────────────────────────────┘

Prompt: "All of these show unit fractions! Which one shows 1/6?"

[Student selects Scenario 3]

Guide: "Exactly! The birthday cake is cut into 6 equal pieces, and you get 1 piece—that's 1/6. Fractions aren't just numbers on paper. Every time something gets divided into equal parts and you take one, that's a unit fraction. You see them at birthday parties, sharing pizza, cooking—they're everywhere!"
```

**Design Principles:**

* Age-appropriate contexts (Grade 3\)  
* Visual support for each scenario  
* Explicit connection from concrete to abstract  
* Emphasizes mathematics in real world  
* Create variety in context, but maintain relevancy for target audience.

---

#### **TYPE D: REPRESENTATION TRANSFER**

**Structure:** Same concept shown using 2+ different visual tools

**Student Task:** Show concept on Tool A, then Tool B. Reflect on what changed vs. stayed the same.

**When to Use:** When student has used multiple visual tools in Lesson/Practice, to consolidate understanding

**Example (Module 6: Equivalent Fractions):**

```
Task 1:
Guide: "Here's 2/4 on a fraction bar. Two out of four equal parts are shaded."
[Bar showing 2/4]

Task 2:
Guide: "Now let's show 2/4 on a number line. Place the marker at 2/4."
[Student drags marker on number line]
Guide: "Great! 2/4 is right there—two jumps of 1/4 each."

Task 3 - REFLECTION:
[Both bar and number line appear side by side]
Guide: "Look at both representations. What stayed the same? What changed?"

[Student selects: "The amount stayed the same, the tool changed"]

Guide: "Exactly! The bar and the number line LOOK different, but they both show the same mathematical idea: 2/4 means halfway between 0 and 1. That's what makes math powerful—the same concept works across different tools."
```

**Design Principles:**

* Use tools student already mastered  
* Show simultaneously for comparison  
* Explicit reflection on invariance  
* Connects to mathematical power

### **3C. Task Selection Strategy by Module Level**

**Early Modules (1-3) \- Foundation:**

* **Interaction Count:** 3-4 typical  
* **Cognitive Focus:** Simple recognition, basic connections  
* **Preferred Tasks:** Type A (Pattern Discovery), Type C (Real-World Bridge)  
* **Cognitive Work:** Identify, Compare \- "Which shows the pattern?"  
* **Example:** "Which shows 1/2?" \+ "Where do you see halves at home?"  
* **Avoid:** Complex generalization or multi-step transfer

**Middle Modules (4-7) \- Development:**

* **Interaction Count:** 4-5 typical  
* **Cognitive Focus:** Transfer between 2-3 representations  
* **Preferred Tasks:** All types A-D appropriate  
* **Cognitive Work:** Apply, Connect \- "How are these related?"  
* **Example:** "Show 2/4 on bar, then number line" \+ "What stayed the same?"  
* **Include:** At least one Type D (Representation Transfer)

**Late Modules (8-12) \- Mastery:**

* **Interaction Count:** 4-6 typical  
* **Cognitive Focus:** Abstract generalization, complex patterns  
* **Preferred Tasks:** Type B (Progressive), Type D (Transfer) emphasized  
* **Cognitive Work:** Synthesize, Generalize \- "What's always true?"  
* **Example:** "What makes ANY two fractions equivalent?" \+ Progressive sequence  
* **Include:** Generalization task requiring principle extraction

**Flexibility Note:**  
 Module Starter Pack requirements override these defaults. Always check module-specific guidance first.

**Selection Checklist:**

□ Chosen 3-5 tasks appropriate to module level  
 □ Tasks are cognitively distinct (not just variations)  
 □ Sequenced easier → harder  
 □ At least one involves pattern recognition or transfer  
 □ At least one includes metacognitive reflection  
 □ No task is just "more practice" (all involve connection)

### **3D. Feedback Approach**

**Success Responses:**

Synthesis feedback should be warm and affirming, highlighting the CONNECTION students made:

**Pattern Recognition:**

* "You spotted that relationship\!"  
* "That's the mathematical structure underneath."  
* "You're seeing the pattern across all these examples."

**Transfer:**

* "You bridged between those representations beautifully."  
* "The tool changed, but you saw what stayed the same."  
* "That's flexibility in mathematical thinking."

**Generalization:**

* "You extracted the principle—that's sophisticated thinking."  
* "You moved from examples to the 'always true' rule."  
* "That generalization will serve you well."

**What Success Feedback Should NOT Do:** ✗ Generic praise ("Great job\!")
 ✗ Over-enthusiasm ("Amazing\! Fantastic\!")
 ✗ Praise without specificity ("You're so smart\!")
 ✗ New information not covered by the remediation path (students on heavy remediation skip On Correct)
 ✗ Forward bridges to the next task (put those in the next interaction's Guide)

**No New Information Rule:** On Correct targets 10–20 words. Students who receive heavy remediation do not see On Correct — they proceed directly to the next task. On Correct must not introduce new vocabulary, patterns, or connections that the next task's Guide doesn't independently re-establish.

**Struggle Responses:**

**Minimal remediation only.** Students in Synthesis have demonstrated mastery in Practice.

**Light Remediation Format:**

* Brief redirect (10-20 words)  
* Focus attention on key element  
* Warm but minimal

**Examples:**

* "Check the denominators again—same pieces or different?"  
* "Look at the shaded amounts. Do they take up the same space?"  
* "Count the total parts one more time."

**When Student Needs More Than Light Remediation:**

**STOP** and flag: "Student requires extended support. Not ready for Synthesis. Recommend Practice phase repeat."

**Do NOT provide extended teaching in Synthesis.** This phase assumes mastery. If remediation goes beyond brief redirects, the student wasn't ready for Synthesis yet.

### **3E. Metacognitive Reflection Requirements**

**Purpose:** Help students recognize strategies and connections—build awareness of what they DID and what they NOTICED.

**Timing:** After connection tasks, before closure (1-2 minutes)

**Research Note:** 8-9 year olds are effective at PLANNING reflection ("What did I do?") but less effective at MONITORING reflection ("How is my understanding changing?"). Focus questions on strategies used and connections made rather than self-assessment of understanding depth.

---

**Four Reflection Types (Use ONE per Synthesis):**

**Type 1: Strategy Identification** *(PLANNING-focused — strongest for this age)*

*Purpose:* Surface and validate the approaches students used

*Example:*

```
Guide: "What helped you most when working with [concept] today?"

Options appear:
- Counting carefully
- Looking at the picture/bar/line
- Thinking about equal parts
- Checking my answer

[After selection]
Guide: "[Strategy-specific validation]"
```

*Response examples:*

* "Counting carefully" → "Careful counting really pays off with fractions."  
* "Looking at the picture" → "The visual makes it click, doesn't it?"  
* "Thinking about equal parts" → "Equal parts—that's the key."  
* "Checking my answer" → "Checking is what strong mathematicians do."

---

**Type 2: Connection Recognition** *(CONNECTION-focused — appropriate for this age)*

*Purpose:* Help students see relationships between concepts

*Example:*

```
Guide: "Today's work reminded me of something we did before. Did you notice a connection?"

Options appear:
- Yes—this is like [previous concept]
- Kind of—some parts are similar
- Not really—this felt new

[After selection]
Guide: "[Connection-specific response]"
```

*Response examples:*

* "Yes" → "You're right\! \[Explicit connection\]. Seeing patterns like that helps."  
* "Kind of" → "Good noticing. The \[specific element\] is similar."  
* "Not really" → "That's okay—it IS new\! Watch for connections as we keep going."

---

**Type 3: Tool/Approach Preference** *(PLANNING-focused — concrete and accessible)*

*Purpose:* Students identify which tools/approaches worked for them

*Example:*

```
Guide: "We used [Tool A] and [Tool B] today. Which one made more sense to you?"

Options appear:
- [Tool A] helped more
- [Tool B] helped more
- Both helped in different ways
- I'm not sure yet

[After selection]
Guide: "[Tool-specific validation]"
```

*Response examples:*

* Tool A → "The \[Tool A\] really shows \[what it shows well\]. Good to know what works for you."  
* Tool B → "\[Tool B\] makes \[specific thing\] clear. That's useful to remember."  
* Both → "Different tools for different moments—that's flexible thinking."  
* Not sure → "Still figuring it out is fine. You'll notice over time."

---

**Type 4: Key Takeaway** *(SYNTHESIS-focused — concrete output)*

*Purpose:* Students identify what they want to remember

*Example:*

```
Guide: "If you had to remember ONE thing from today, what would it be?"

Options appear:
- [Core concept statement A]
- [Core concept statement B]
- [Procedure statement]
- Something else I noticed

[After selection]
Guide: "[Validation + brief reinforcement]"
```

*Note:* Options should be module-specific and concrete. "Something else" allows for unexpected but valid takeaways.

---

**Module-Level Guidance:**

| Module Level | Preferred Types | Rationale |
| ----- | ----- | ----- |
| Early (1-4) | Type 1, Type 3 | Planning-focused, concrete |
| Middle (5-8) | Type 1, Type 2, Type 3 | Add connection recognition |
| Late (9-12) | All types | Full range appropriate |

---

**Reflection Guidelines:**

* Use ONE reflection type per Synthesis (don't overdo)  
* **Prefer Types 1 and 3** (planning-focused) for Modules 1-6  
* **Introduce Types 2 and 4** (connection/synthesis-focused) for Modules 7-12  
* Connect to module content specifically  
* Validate ANY response student gives (no wrong answers in reflection)  
* Keep brief (1-2 minutes total)  
* Guide response is ONE sentence—don't over-explain

---

**AVOID These Question Types:**

| ❌ Avoid | Why | ✅ Use Instead |
| ----- | ----- | ----- |
| "How has your understanding changed?" | Requires monitoring (weak at this age) | "What helped you most?" |
| "What would you feel confident teaching?" | Hypothetical \+ self-assessment | "Which tool made more sense?" |
| "Rate your understanding 1-5" | Abstract self-assessment | "What's the one thing to remember?" |
| "What was hardest?" | Negative framing \+ monitoring | "What clicked today?" |
| "What would you do differently?" | Counterfactual (not developed) | "What would you try first next time?" |

### **3F. Identity-Building Closures**

**Purpose:** Affirm growth in specific, meaningful ways

**The Golden Rule:** Be specific, not generic

**Avoid Generic Praise:** ✗ "Great job\!"  
 ✗ "You did it\!"  
 ✗ "Excellent work\!"  
 ✗ "I'm so proud of you\!"

**Use Specific, Growth-Oriented Language:**

**Pattern: Discovery**

```
Template: "You discovered that [specific pattern]. That thinking will help you with [future application]."

Example: "You discovered that all equivalent fractions represent the same amount, just with different piece sizes. That thinking will help you when we simplify fractions next."
```

**Pattern: Strategy Use**

```
Template: "The way you [specific strategy] shows you're developing [mathematical habit]."

Example: "The way you checked each fraction on the number line before comparing shows you're developing precision—a key habit for advanced math."
```

**Pattern: Question Asking**

```
Template: "You asked yourself [metacognitive question]. That's how concepts stick."

Example: "You asked yourself, 'Which pieces are bigger?' before comparing. That's how mathematical thinkers approach problems—with strategic questions."
```

**Connection to Future Learning:**

End with brief preview:

* "Next time we'll see how \[concept\] helps us \[next challenge\]."  
* "You're ready for \[future application\] because you understand \[foundation\]."

**Example (Module 2 → Module 3):**

```
Guide: "You named unit fractions like 1/2, 1/3, and 1/4 today. Next time, we'll use this power to name fractions with MORE than one part—like 2/3 or 3/4. You're ready because you understand how denominators name the parts."
```

### **3G. Edge Cases and Adaptations**

**When Standard Task Types Don't Fit:**

Synthesis works best when modules have rich conceptual depth, multiple representations, and clear patterns. Sometimes this isn't the case. Here's how to handle edge cases:

**Module has only ONE visual representation:**

* Type D (Representation Transfer) won't work—skip it  
* Can still use Type A (Pattern Discovery), Type B (Progressive), Type C (Real-World)  
* Consider: Same tool, different orientations or contexts (horizontal bar → vertical bar)  
* Flag if pattern persists across modules—may indicate curriculum design issue

**No clear patterns to discover:**

* Don't force Type A (Pattern Discovery) with artificial patterns  
* Use Type B (Progressive Challenge) to build from recognition to generalization  
* Focus on Type C (Real-World Connection) and metacognitive reflection  
* Consider: Does this module genuinely benefit from synthesis, or is it checkbox compliance?

**Concept is entirely procedural with thin conceptual depth:**

* Reduce to 3 interactions minimum (don't artificially inflate)  
* Focus on strategy reflection and when/why to use the procedure  
* May use Type C (Real-World) to show application contexts  
* If synthesis feels forced, document concern—some modules may not need formal synthesis

**Students already deeply familiar with real-world contexts:**

* Type C (Real-World Bridge) becomes redundant  
* Use Type D (Representation Transfer) or Type B (Progressive) instead  
* Can adapt Type C to novel/surprising contexts rather than obvious ones

**Module uses multiple tools but they don't transfer cleanly:**

* Type D works only if tools show SAME concept differently  
* If tools serve different purposes, use Type A or B instead  
* Don't force artificial "what's the same?" comparisons

**Pedagogical Override Applies:**

If standard synthesis patterns don't serve learning, adapt or reduce scope. The goal is consolidation and connection, not compliance with a template. Mark decision and document pedagogical reasoning.

If multiple modules show these edge cases, flag for curriculum review—synthesis phase may need reconceptualization for this content domain.

### **3H. CONSOLIDATION MOMENT CHECK**

**The Principle:**  
If the module taught multiple strategies or concepts, Synthesis must include explicit side-by-side review. Don't assume students will consolidate automatically.

**When This Applies:**

- Module taught 2+ comparison strategies (e.g., same denominator AND same numerator)  
- Module introduced multiple representations (e.g., shapes AND number lines)  
- Module built on prior module's strategy (e.g., M11 builds on M10)

**Required Elements:**

When consolidation applies:

- [ ] Show BOTH/ALL strategies or concepts together visually  
- [ ] Explicitly name when to use each ("Same denominator → count pieces; Same numerator → compare piece sizes")  
- [ ] Name the meta-skill ("You learned to CHOOSE which strategy based on what matches")

**Example Structure:**

Guide: "Let's look at the two strategies you learned."

\[Visual shows side-by-side:\] Strategy 1: Same Denominator → Compare numerators Example: 2/5 vs 4/5 → "More fifths \= greater"

Strategy 2: Same Numerator → Compare piece sizes  
 Example: 3/4 vs 3/8 → "Fourths are bigger pieces"

Guide: "The key is asking: What's the SAME? That tells you which strategy to use."

**Why This Matters:**  
Students who learn strategies separately may not see them as a connected toolkit. Explicit consolidation prevents fragmented understanding and supports strategy selection in novel problems.

## **4\. CRITICAL SUCCESS CHECKS**

**Must-Haves:**

- [ ] Includes all required elements from Section 2 (Opening, Connection Tasks, Reflection, Closure)  
- [ ] 3-5 diverse connection tasks (cognitively distinct, minimum 2 different task types)  
- [ ] At least one task involves pattern recognition or transfer  
- [ ] Metacognitive reflection moment included  
- [ ] Identity-building closure (specific, not generic \- see Section 3F)  
- [ ] Visual support for every task  
- [ ] Total time appropriate (see Section 2 timing breakdown)  
- [ ] Zero new teaching or procedures  
- [ ] Remediation minimal (light redirects only \- see Section 3D)   
- [ ] **Active task check:** At least one task requires placement, matching, or creation (not all multiple-choice)   
- [ ] **Consolidation check:** If module taught multiple strategies, explicit side-by-side review included (see Section 3G)

**Red Flags:**

| ⚠️ Warning Sign | Fix |
| ----- | ----- |
| Tasks are just problem variations | Replace with diverse task types (A-D) from Section 3B |
| Introducing new procedures | Remove—see Section 1 for what Synthesis is NOT |
| Total time exceeds Section 2 guidance | Cut one task or streamline transitions |
| Extended teaching in remediation | See Section 3D: Light redirects only |
| Generic praise ("Great job\!") | See Section 3F: Specific behavioral observation |
| No pattern recognition or transfer | Add Type A, B, or D task from Section 3B |
| Abstract discussion with no visual | Add visual representation to every task |
| Only 2 interactions | Add one more connection task |
| Referencing specific Practice problems | Generalize: "fractions like 2/3" not "problem 7" |
| Missing metacognitive reflection | See Section 3E: Add reflection moment before closure |

   ├─ Minimum 2 different task types (see Section 4)
   └─ Sequence: easier → harder cognitive demand

4. WRITE SYNTHESIS STRUCTURE
   ├─ Follow Section 2 required elements and timing
   ├─ Opening → Connection Tasks → Reflection → Closure
   └─ Each task cognitively distinct

   ├─ Visual support for ALL tasks

   ├─ Check against Section 2 timing breakdown

**"Which task types?"**

## **6\. AUTHENTICITY CHECK**

Before submitting, scan your Synthesis script:

**Specificity Test:**

* \[ \] Could any line work for a DIFFERENT module/concept?  
   → If YES: Make it specific to THIS concept

**Discovery Test:**

* \[ \] Is every task about CONNECTION, not just APPLICATION?  
   → If NO: Replace with true synthesis task (Types A-D)

**Behavioral Observation Test:**

* \[ \] Did you reference what student SPECIFICALLY discovered?  
   → Must include: "You noticed \[X\]" or "You connected \[Y\]"

**Pattern Variety:**

* \[ \] Would a child notice if we used this same Synthesis for different topic?  
   → If YES: Add more concept-specific language

**Voice Test:**

* \[ \] Does it sound like a real teacher reflecting with a student?  
   → If NO: Remove robotic/compliance language, add genuine warmth

**Enthusiasm Test:**

* \[ \] Does the guide sound genuinely excited about mathematical patterns?  
   → Voice should convey authentic discovery, not checkbox completion

**Closure Test:**

* \[ \] Is closure specific to what THIS student discovered?  
   → If NO: Replace generic praise with specific growth affirmation

---

**Quality Litmus Test:**

Before finalizing, ask:

**"Does this synthesis help the student see connections they couldn't see alone, or is it just repeating Practice with different numbers?"**

If the answer is "just more practice," the synthesis has failed its purpose. Redesign tasks to focus on:

* Pattern recognition across examples  
* Transfer between representations  
* Real-world connections  
* Metacognitive reflection  
* Generalization of principles

Synthesis reveals and cements deeper understanding—it doesn't drill procedures.
