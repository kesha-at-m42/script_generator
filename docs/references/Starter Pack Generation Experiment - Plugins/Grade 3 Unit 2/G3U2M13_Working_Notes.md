## Cross-Reference Table B — TVP Extraction

```
TVP: MODULE 13
===============
Title: Finding Missing Side Lengths
OUR Lesson: L14
Standard: 3.MD.C.7.d (figures with missing side lengths)

FRAMING NOTE (TVP):
⚠️ NEW REASONING STEP, NOT NEW TOOL. M13 adds missing-side reasoning to the
composite figures workflow from M11–M12. What's new is that students must
determine side lengths that aren't labeled. This is an extension of spatial
reasoning, not a new area concept.

WORKFLOW PROGRESSION WITHIN M13:
Early (L-shapes, principle teaching): Students learn the "sides add up"
principle on L-shapes with 1 missing side, using Highlight Related Sides with
full scaffolding. Critically, for L-shapes with 1 missing side, there is
always a decomposition that does NOT require the missing side — so the area
calculation proceeds via that decomposition. Early isolates the geometric
principle without pretending the area calculation depends on it. Students
practice finding missing sides as a geometric skill, then decompose and
calculate area independently.

Mid (T-shapes, principle becomes necessary): T-shapes introduce figures where
the missing side IS required regardless of decomposition choice. The principle
taught in Early now has genuine purpose. Decompose-first workflow: student
decomposes → discovers they need a missing dimension → uses "sides add up"
to find it → calculates → adds.

Late (all figure types, full independence): Decompose → find needed missing
sides → calculate → add. No scaffolding. The decompose-first workflow is
the target state.

The progression is: learn the tool (Early) → need the tool (Mid) →
use the tool independently (Late).

KEY INSIGHT:
In rectilinear figures, horizontal sides constrain each other and vertical
sides constrain each other. If the long bottom edge is 8 and one horizontal
segment is 5, the other must be 3 — because the horizontal pieces MUST add
up to the total horizontal length. Same for vertical.

WHY THIS IS HARD FOR THIRD GRADERS:
The "add up" relationship requires attending to the WHOLE figure's structure,
not just the rectangle they're working on. Students must hold the global shape
in mind while reasoning about local dimensions. This is why misconception M7
(wrong missing side length) is HIGH priority — the error is about which
values to subtract, which requires spatial orientation that's genuinely
challenging at this age.

LEARNING GOAL:
Calculate the area of ungridded figures composed of rectangles including
figures with missing side lengths.

KEY TEACHING POINTS:
1. Some figures don't have every side length labeled. You need to FIGURE OUT
   the missing ones before you can find the area.
2. THE KEY INSIGHT: In any rectilinear figure, horizontal pieces add up to
   the total horizontal length, and vertical pieces add up to the total
   vertical length. If you know the total and one piece, you can find the
   other: total − piece = missing piece. This is subtraction in service of
   geometry — students aren't learning subtraction, they're using it as a tool.
3. After finding missing sides, the workflow is identical to M12: decompose →
   identify dimensions per rectangle → calculate → add.
4. Early (L-shapes): Finding missing sides is taught as a geometric principle —
   "related sides add up." For L-shapes with 1 missing side, students practice
   finding the missing dimension, but the area calculation uses a decomposition
   that doesn't require it. The principle is learned on its own terms, not as
   a prerequisite for area.
5. Mid–Late (T-shapes and beyond): Decompose first, THEN find the missing
   sides each sub-rectangle needs. T-shapes are where the missing-side skill
   becomes genuinely necessary — no decomposition avoids the missing dimension.
6. The workflow shift is deliberate: learn the tool (Early) → need the tool
   (Mid) → use the tool independently (Late).
7. Visual support: The Highlight Related Sides teaching action (internal tool
   name — not student-facing vocabulary) shows which sides constrain each
   other. Color-matched sides with an equation: [known] + [?] = [total].
   The guide uses plain directional language: "Look at the horizontal sides" /
   "Look at the vertical sides." No named term is taught — students learn
   the PROCEDURE without needing to NAME the relationship.
8. Multiple missing sides appear in Mid/Late — students may need to find 2–3
   missing dimensions before they can calculate area.

COGNITIVE FOCUS:
- IDENTIFY (Conceptual) — recognizing which sides are missing and which known
  sides relate to them
- APPLY (Transfer) — using subtraction to find missing dimensions from the
  "sides add up" principle
- APPLY (Transfer) — executing the full find-missing → decompose → calculate
  → add workflow

MISCONCEPTIONS TARGETED:
M7: Wrong Missing Side Length (PRIMARY, HIGH)
  Observable: given an L-shape with sides 8, 5, and "?" on the horizontal,
  student writes ? = 13 (added) or subtracts wrong pair (uses vertical
  dimension to find horizontal missing side).
  Addressed by: (1) Highlight Related Sides making constraint explicit,
  (2) consistent "which direction?" reasoning, (3) color-coding related sides.

M12: Multiplying All Given Numbers (SECONDARY, from M12)
  Missing sides add MORE visible numbers after found, increasing pool of
  dimensions for indiscriminate multiplication.
  Addressed by: maintaining decompose → identify dimensions per rectangle →
  calculate → add workflow with per-rectangle dimension highlighting.

M11: Completing the Rectangle (TERTIARY)
  May surface with bounding box dimensions. Less prominent — practiced across
  M11–M12.

PHASE-BY-PHASE FLOW:

WARM-UP:
- Composite Figures (L-shape, labels only, all dimensions labeled) → Quick
  M12-style solve. Decompose, calculate, add. ~1 problem. Should feel routine.
- SAME figure redisplayed with ONE dimension label replaced by "?"
- "This is the same shape. But now one side length is missing. Can you figure
  out what it is?"
- System activates Highlight Related Sides: missing side and related known
  sides light up in same color. Equation appears with actual values
  (e.g., 8 − 5 = ?).
- Guide: "Remember — related sides add up. The long bottom is 8. This piece
  is 5. So the other piece must be... 3!"
- Student does NOT recalculate the area — already computed it on same figure.
- Guide connects: "You already found the area of this shape. Even with a
  missing side, you had enough information to figure it out. That's what
  we're going to practice today."
- Purpose: Establish that missing sides are findable, not blockers. The
  "related sides add up" principle from M12 is the tool — M13 extends it
  from verification to problem-solving. No recalculation needed. The warmup
  is a confidence moment, not a calculation moment.

LESSON — EARLY (L-shapes, 1 missing side, guided):
- Composite Figures (L-shapes, labels only, 1 missing side "?") + Equation
  Builder
- System activates Highlight Related Sides automatically: missing side and
  two related known sides color-matched. Constraint equation appears with
  actual values (e.g., 8 − 5 = ?).
- Student selects missing dimension (MC from 4 options: correct value, total
  instead of difference, wrong-direction subtraction, random distractor).
- Key beat — directional reasoning: Guide connects to M12's verification:
  "Remember when we checked that related sides add up? Now you're going to
  use the SAME idea to find a missing side."
- Guide explicitly: "Look at the horizontal sides. The long bottom is 8. One
  piece is 5. What's the other piece?"
- After missing side found, system fills in dimension label. ALL sides labeled.
- ⚠️ Area calculation uses decomposition that does NOT require the missing
  side. For L-shapes with 1 missing side, there is always a decomposition
  where both sub-rectangles have all dimensions from originally-labeled edges.
  Student decomposes (student-placed) and calculates area using only
  originally-given dimensions.
- WHY: If we forced students to use decomposition requiring the missing side,
  we'd create a false dependency. Teaching the principle honestly (as geometric
  skill with its own value) is better pedagogy. Genuine need arrives in Mid
  with T-shapes.
- If student CHOOSES decomposition using the missing side: fine — they found
  it already. Don't redirect. Constraint is on Guide framing, not student
  choice.
- 3–4 problems. All L-shapes, 1 missing side each. Highlight Related Sides
  automatic throughout.

LESSON — EARLY — Directional Error Check (after 2nd problem):
- System displays figure with missing horizontal side. Guide: "A student
  looked at this figure and subtracted the vertical side from the horizontal
  side. Would that work?"
- Student examines: no — vertical side doesn't relate to horizontal missing
  piece.
- "Horizontal sides add up with horizontal sides. Vertical sides add up with
  vertical sides."
- Directly addresses directional component of misconception M7. Cost: ~30 sec.

LESSON — MID (T-shapes, 1–2 missing sides, fading support):
- ⚠️ FIGURE TYPE SHIFT: T-shapes, where missing side IS required regardless
  of decomposition choice. First time students NEED the principle for area.
- ⚠️ WORKFLOW SHIFT: Decompose-first ordering. Guide makes explicit: "You
  know how to find missing sides. You know how to decompose. Let's put them
  together — decompose first, then figure out what you need."
- Student draws decomposition line (student-placed, same as M12).
- Student discovers sub-rectangle missing a dimension. Guide: "You need this
  rectangle's width to find its area. Can you figure out the missing side?"
  System prompts MC.
- Highlight Related Sides: Guide-controlled (reactive — activated after wrong
  MC answer, not automatic). Matches ghost grid pattern from M12.
- After missing sides resolved, dimension highlighting per rectangle activates.
  Student calculates each area and adds.
- 3–4 problems. All T-shapes. 1–2 missing sides each.
- At least one problem requires double subtraction.
- At least one requires addition of segments.

KEY BEAT — "you couldn't have skipped that" (after 1st or 2nd T-shape):
- After student finds missing side and calculates area, guide reflects:
  "Notice something different? With L-shapes, you could sometimes find the
  area without the missing side. With this shape, you needed it. The missing
  side was part of one of your rectangles."
- Makes geometric distinction L-shapes vs T-shapes explicit. Brief (~15 sec).

KEY BEAT — sequencing and multi-segment reasoning:
- Some figures require subtracting TWO known segments from a total (double
  subtraction), not just one.
- Concrete example — T-shape: Bottom width: 9 − 4 − 2 = 3 cm (double
  subtraction). Left stem height: same as right side = 5 cm. Right top
  vertical = 3 cm. All three independently solvable — no dependency.
- Concrete example — L-shape with notch: Bottom width: 3 + 5 = 8 ft
  (addition of segments). Right side height: 8 − 2 = 6 ft (single
  subtraction). Both independently solvable. Note: bottom uses ADDITION
  while right side uses SUBTRACTION.
- DATA CONSTRAINT (updated): M13 missing sides are independently solvable —
  no dependency chains where one missing side requires another to be found
  first. Complexity comes from: (a) double subtraction, (b) recognizing
  whether to add or subtract, (c) identifying which sides are "related" on
  multi-edge figures. True dependency chains are M14 territory.
- MC distractors for double-subtraction: correct answer (9 − 4 − 2 = 3),
  single-subtraction error (9 − 4 = 5 or 9 − 2 = 7), addition error
  (4 + 2 = 6).

LESSON — LATE (all figure types, full independence):
- Composite Figures (L-shapes, T-shapes, U-shapes, labels only, 1–3 missing
  sides) + Equation Builder
- Full independence. Highlight Related Sides available only as remediation
  (guide-activated on wrong MC answer).
- Decompose → find needed missing sides → calculate → add. Full workflow,
  no scaffolding.
- Figure variety: 2 L-shapes, 1 T-shape, 1 U-shape.
- 3–4 problems.

KEY BEAT — the complete workflow:
- Guide frames on first Late problem: "Here's what we do: First, find any
  missing side lengths. Then, decompose into rectangles. Then, find each
  rectangle's area. Then, add them up. Four steps."
- Culminating workflow for entire M11–M13 arc.

EXIT CHECK:
3 problems:
1. One missing side, L-shape: Student finds missing side (MC), then calculates
   total area. (Tests: basic missing-side reasoning + area calculation)
2. Two missing sides, T-shape: Student finds both missing sides in correct
   order, then calculates total area. (Tests: multi-step reasoning +
   sequencing + area calculation)
3. Misconception check — directional error: Figure with missing horizontal
   side. MC options: correct answer (horizontal subtraction), wrong-direction
   (vertical subtraction), addition error, random distractor. (Tests: "same
   direction" principle)

Assessment ratio: 1 basic : 1 multi-step : 1 misconception-targeted.

PRACTICE:
- Mixed figure types: L-shapes, T-shapes, U-shapes
- All ungridded (labels only), 1–3 missing sides
- Highlight Related Sides: guide-controlled scaffold
- Decompose → find missing → calculate → add (Mid–Late workflow)
- ⚠️ CRITICAL — error cascading guidance (MUST be prominent in SP): If student
  correctly decomposes but gets missing side wrong then correctly multiplies
  with wrong value, guide acknowledges process: "Your steps were right —
  decompose, multiply, add. But let's go back and check the missing side.
  That's where the error started." Address ROOT cause, not downstream effect.

SYNTHESIS:
- Full workflow display: figure with missing sides → find missing → decompose
  → calculate → add
- "Every composite figure problem is the same four steps: find any missing
  sides, break into rectangles, find each area, add them up."
- Capstone statement for M11–M13 arc.
- "Sides add up" principle reinforcement: L-shape with horizontal sides
  highlighted. "The horizontal pieces always add up to the total horizontal
  length. The vertical pieces always add up to the total vertical length."
- Vocabulary consolidation: missing side (new). Reinforce: decompose, composite
  figure, dimensions, multiply, add.
- Bridge to M14: "You've learned to find area of all kinds of shapes — with
  grids, without grids, even with missing measurements. Now let's use all of
  that to design a room!"

TRANSITION IN (from M12):
M12's Synthesis closure: "What if some sides didn't have measurements? Could
you use that same idea to figure out the missing ones?" — "related sides add
up" introduced as verification in M12, becomes problem-solving tool in M13.

TRANSITION OUT (to M14):
"You've learned to find area of all kinds of shapes — with grids, without
grids, even with missing measurements. Now let's use all of that to design
a room!" — preview of application context.
```

## Design Constraints Extraction — Important Decisions

```
DESIGN CONSTRAINTS (from Important Decisions)
=============================================
Decision 1: Unified Path with Adaptive Pacing
  Rule: Use a single instructional path following CRA progression.
        Differentiate through pacing and representation duration, not
        different approaches.
  Applies to M13? YES
  What it constrains: M13 follows the single CRA path. The Highlight
  Related Sides scaffold fades (automatic → guide-controlled → remediation
  only), which is pacing differentiation, not a different approach. All
  students learn the same "sides add up" principle and workflow.

Decision 2: Explicit Spatial Structuring as Central Focus
  Rule: Make row-by-column "seeing" an explicit instructional target.
  Applies to M13? PARTIAL
  What applies: Spatial structuring is assumed mastery by M13 (mastered at
  L5-6 per Conceptual Spine). M13's spatial demand is different: attending
  to the WHOLE figure's edge structure to identify related sides. This is
  spatial reasoning but not row-by-column structuring specifically.
  What doesn't: M13 doesn't include row-column activities.

Decision 3: Progressive Grid Removal (Concreteness Fading)
  Rule: M13-M14: Dimensions only with missing sides (full abstraction)
  Applies to M13? YES — HARD CONSTRAINT
  What it constrains: NO grids in M13. All figures are labels-only with
  missing sides. No partial grids, no ghost grids as primary scaffold
  (ghost grid may exist as deep remediation only). M13 operates at full
  abstraction level.

Decision 4: Array Knowledge is Helpful but Not Required
  Rule: Design Unit 2 to be self-contained. No Unit 1 dependencies.
  Applies to M13? YES (unit-wide)
  What it constrains: No references to Unit 1 arrays. Multiplication is
  used as a tool (side × side = area), not re-taught.

Decision 5: Perimeter Kept Separate — Do Not Introduce
  Rule: Do NOT introduce perimeter. Do not proactively address perimeter-
        area confusion.
  Applies to M13? YES — HARD CONSTRAINT
  What it constrains: No mention of perimeter anywhere. The "sides add up"
  principle might superficially resemble perimeter reasoning to an adult,
  but the SP must NOT draw this connection. Students are reasoning about
  directional edge constraints, not computing perimeter.

Decision 6: Dragging Over Tapping for Manipulatives
  Rule: Prioritize drag-to-place for manipulatives over tap-to-place.
        Exception: Simple selection tasks (MC, identifying) can use tap.
  Applies to M13? YES
  What it constrains: Decomposition line drawing uses drag. Missing side
  MC selections use tap (selection exception). Consistent with M12.

Decision 7: Standard Units Get Full Scaffolding
  Rule: Keep sq inches/cm and sq feet/meters as separate modules.
  Applies to M13? NO — M13 does not introduce new units.

Decision 8: Include Multiplication Table Connection
  Rule: Include L11 as synthesis/connection module.
  Applies to M13? NO — already completed (M10).

Decision 9: Keep Application Module Standalone
  Rule: Include L15 as standalone application module.
  Applies to M13? PARTIAL
  What applies: M14 is the application module. M13 must NOT absorb
  application content that belongs in M14. M13's scope is the missing-side
  reasoning skill; real-world application is M14's territory.

Missing side found as geometric skill; area uses decomposition NOT requiring missing side. New cognitive demand: "sides add up" principle applied to problem-solving.
- Section 2 — Mid (T-shapes, principle necessary, ~4–5 min): 3–4 problems. CRA stage: Abstract. Highlight Related Sides guide-controlled. Workflow shift to decompose-first. Student discovers need for missing side through decomposition. Double subtraction and addition-of-segments introduced. Metacognitive beat: "you couldn't have skipped that."
- Section 3 — Late (mixed figures, full independence, ~3–4 min): 4 problems (2 L, 1 T, 1 U; includes 1 student-discovered sequencing problem). No scaffolding (remediation only). Decompose-first workflow consolidated. 3-missing sides is stretch (most 1–2).

- Distribution across Early/Mid/Late skill levels

- 5 interactions (single-concept extension module with moderate conceptual depth: 1 opening + 2 connection tasks + 1 reflection + 1 closure)

- "Sides add up" principle reinforcement

- Task types: Workflow Consolidation, Principle Reinforcement, Metacognitive Reflection, Identity-Building Closure
