TVP: MODULE 10
===============
Title: Area and the Multiplication Table
OUR Lesson 11 | Standards: 3.MD.C.7.b (area-multiplication connection); 3.OA.D.9 (patterns in multiplication table)

⚠️ SYNTHESIS MODULE — This module does not teach a new skill. It teaches students to SEE their existing skills inside a new representation.

Decision 8 note (UPDATED): This module is required. SME confirmed the multiplication table as area representation is too valuable to skip. Rectangle region marking makes the insight spatial and physical. The "cuttable" flag from Important Decision 8 is removed.

Learning Goal: Explore connections between area and the multiplication table. Recognize that the multiplication table organizes the area of every rectangle with whole-number dimensions.

Key Teaching Points:
1. The multiplication table is like a giant collection of rectangles — every cell IS an area
2. Row number = one dimension, column number = other dimension, product = area
3. This is NOT new math. Students have experienced 4 × 6 = 24 (area of a 4 by 6 rectangle) multiple times across this unit. The table ORGANIZES all those facts in one place. Students may not have all products memorized — the table itself supports fact-finding while building toward fluency.
4. THE BIG INSIGHT: You can "see" rectangles INSIDE the multiplication table. Shade the region from the top-left corner down to row 4, across to column 6 — that region IS a 4 by 6 rectangle, and the product in the corner (24) IS the area.
5. Patterns in the table connect to area patterns: products grow as dimensions grow, the diagonal contains square numbers (areas of squares), commutative pairs (3×4 and 4×3) mirror across the diagonal
6. Factor-pair reasoning from M8 now has a visual home: "What rectangles have area 24?" becomes "Where is 24 in the table?"

⚠️ Unit-level note: This unit must be accompanied by regular fast math / fact fluency practice outside the module lessons.

CRA Stage: Abstract — the multiplication table is a purely symbolic representation. But the rectangle region marking CONNECTS it to the concrete/representational work students did with grids in M1–M6.

Cognitive Focus:
- CONNECT (Transfer) — linking concrete area experience (grids, rectangles) to abstract table representation
- IDENTIFY (Conceptual) — recognizing patterns in the multiplication table that correspond to area relationships
- APPLY (Transfer) — using the table as a tool for finding area and exploring factor pairs

Misconceptions Targeted:
- M2: Counting One-by-One (PRIMARY) — Students count cells rather than reading products. Detection: extended response time, off-by-one errors.
- Table as memorization tool only (SECONDARY) — Students see table as "just facts to memorize" rather than meaningful area representation. Rectangle region marking addresses this directly.

PHASE-BY-PHASE FLOW:

WARM-UP:
- Grid Rectangles (full grid, display) → 3×5 rectangle displayed, area = 15
- Next to it: Multiplication Tables (10×10, fully revealed)
- System demonstrates rectangle region marking: shades (1,1) to (3,5), highlights corner product 15
- "See the rectangle IN the table? 3 rows, 5 columns — just like the grid. And the number in the corner? 15. That's the area."
- Second demonstration: different rectangle (e.g., 4×6=24). Two examples establish the pattern.
- Purpose: "Watch this" moment — students observe, not interact. Wow factor.
- Note: Warmup uses fully revealed table. Lesson begins fresh with partially revealed table. State does NOT carry.

LESSON:
Early activities (Student marks rectangles → reads area):
- Multiplication Tables (10×10, 2s/5s/10s pre-revealed, rest hidden) + Grid Rectangles (outline + dimensions, display)
- Student sees Grid Rectangle with labeled dimensions (e.g., 4×7). Task: "This rectangle is 4 by 7. What's its area?"
- Prediction step: Before touching the table, student calculates area via MC (e.g., 28/24/32/21).
- Location prediction: "Where will you find that in the table?"
- Student marks rectangle region from (1,1) to (4,7). System validates. Both 4×7 and 7×4 accepted.
- On correct: corner cell reveals product 28. Connection Animation fires: visual link between Grid Rectangle and shaded table region.
- Revealed cells persist across Early activities.
- 4–5 problems: 2×4, 3×7, 5×6, 4×9, 6×8
- Key beat after 3rd or 4th problem: "Every time you marked a rectangle, the corner number was the area. That's because the multiplication table is really a collection of ALL the rectangles you could make."

Mid activities (Reverse — given product, find and mark rectangle):
- Accumulated table state from Early
- System highlights a product cell (e.g., row 3, col 8 showing 24). "This cell shows 24. That means there's a rectangle hiding here with area 24. Mark it."
- Student marks (1,1) to (3,8). Grid Rectangle (3×8, area 24) appears as confirmation.
- 3–4 problems
- Key beat: multiple rectangles for same product. "24 shows up in the table more than once. Can you find another 24?"
- Student locates 24 at row 4, col 6. System highlight-by-value shows all 24s.
- Guide distinguishes: "3 by 8 and 8 by 3 are the same rectangle turned sideways. But 3 by 8 and 4 by 6 are genuinely DIFFERENT rectangles with the same area."
- Commutative mirror observation: diagonal line, above/below.

Late activities (Pattern discovery):
- Fully revealed table
- Row patterns: highlight row 5. MC: "They're all multiples of 5" / "They end in 0 or 5" / "They go up by 5" (all valid)
- Column patterns: same for a column. Connection to commutativity.
- Square numbers on diagonal: 1, 4, 9, 16, 25, 36, 49, 64, 81, 100. "Both sides are the same — it's a SQUARE." Light-touch.
- Using patterns to find unknown products: "You know 6×5=30. How can you find 6×6?" MC: "Add one more row of 6, so 30+6=36." 1–2 problems. Light bonus.

EXIT CHECK:
3 problems:
1. Rectangle → table: "A rectangle is 7 by 4. Mark this rectangle in the multiplication table. What is its area?" (mark region, read corner product 28)
2. Table → rectangle: System highlights product 40 at row 5, col 8. "This product is in row 5, column 8. What are the dimensions of the rectangle it represents?" MC: "5 by 8" / "4 by 10" / "5 by 10" / "8 by 4." Correct: "5 by 8." (Row/column anchoring eliminates commutativity confound.)
3. Pattern/connection: "Jada found that 18 appears at row 2, column 9. Name a DIFFERENT rectangle that also has area 18." MC: "3 by 6" / "2 by 8" / "3 by 9" / "4 by 5." Correct: "3 by 6."

Assessment ratio: 1 forward : 1 reverse : 1 synthesis (factor pairs).

PRACTICE:
- Mixed rectangle-to-table and table-to-rectangle with rectangle region marking
- Additional pattern explorations
- Factor-pair hunting with highlight-by-value confirmation
- Some problems WITHOUT table visible — test internalization. Multi-select MC preferred.
- Example: "Would 35 appear in the 5s row?" Multi-select with misconception-targeted distractors.
- All table-reasoning problems without visual support must have four MC options.

SYNTHESIS:
- Multiplication table as "area atlas" — map of every rectangle with whole-number sides 1–10
- "You started this unit by covering shapes with squares. Then you counted rows and columns. Then you multiplied. Now you can see ALL of that — every rectangle, every area — organized right here in the multiplication table."
- Vocabulary consolidation: multiplication table (new), factor, product, dimensions, area (reinforced)
- Bridge to M11: "You've been finding the area of SINGLE rectangles. What if a shape is made of TWO rectangles stuck together? How would you find ITS area?" — display L-shape briefly, don't solve.

Scaffolding Progression:
- Warmup → Lesson: table resets
- Within Lesson: cell reveals persist (Early → Mid → Late)
- Lesson → EC: table resets (clean assessment)

Tool Requirements:
1. Mark Rectangle Region (NEW — major): Student selects rectangular region anchored at (1,1), endpoint at (R,C). Shaded fill, corner cell revealed. Both orientations accepted. Hidden cells within region get shaded fill but do NOT auto-reveal values — only corner cell reveals.
2. Targeted vs. Exploratory click mode (NEW — moderate)
3. State persistence scope (NEW — moderate): activity / lesson_phase
4. Highlight-by-value (NEW — minor): highlights all cells containing specified product
5. Connection Animation: visual link from Grid Rectangle to shaded table region
6. Co-display: Multiplication Table (10×10) + Grid Rectangle + guide + MC simultaneously

No Equation Builder. Table IS the representation.

Data Constraints:
- Early: Factors 2–9, products within 100. Examples: 2×4=8, 3×7=21, 5×6=30, 4×9=36, 6×8=48. Avoid 1×n.
- Mid: Products with ≥2 unique factor pairs within 1–10. Good targets: 12, 18, 24, 36, 20.
- Commutative pairs: table shows both orientations. Guide distinguishes "same rectangle turned sideways" vs "DIFFERENT rectangles, same area."
- Pattern problems: rows 2, 5, 10 (skip-counting families), diagonal (squares), multi-occurrence products.
- Table size: 10×10 (factors 1–10). No 12×12.

Vocabulary:
- New: multiplication table
- Reinforced: factor, product, area, dimensions, multiply, row, column, square units
- Language pattern: "Mark the rectangle: ___ rows, ___ columns. The corner product is ___. That's the area of a ___ by ___ rectangle."
- Vocab teaching note from Module Mapping: "The multiplication table is like a collection of all the rectangles we could make!"

TRANSITION IN (from M9):
What M9 establishes: Students can use area to solve varied problem types in context. Fluent with forward and reverse reasoning.
What M10 builds: The multiplication table AS an area model — every product is an area, every cell is a rectangle.
Bridge: "You've been using multiplication to find the area. Now let's see how the multiplication TABLE connects to the area — you can find areas right inside the table."
Pedagogical Shift: Applied area problem-solving (contextual) → Structural synthesis (seeing the area-multiplication system as a whole)
Why this sequence: M9 showed area skills work in real contexts. M10 zooms out to show those skills are part of a larger mathematical structure. "Seeing the forest after walking through the trees."

TRANSITION OUT (to M11):
What M10 establishes: Full multiplication-area connection — every product is an area, every cell is a rectangle.
What M11 builds: Composite figures — shapes made of MULTIPLE rectangles. Area is additive. Full grids return.
Bridge: "You've been finding the area of SINGLE rectangles. What if a shape is made of TWO rectangles stuck together? How would you find ITS area?"
⚠️ Engineering alert: grids come back in M11 (CRA reset for new concept).

## SECTION PLAN

**Warmup (3–5 min):** Activation type — "Watch This" demonstration. Two system-driven demonstrations of rectangle region marking (3×5=15, 4×6=24). Students observe the core insight: rectangles can be seen INSIDE the multiplication table. Grid Rectangle co-displayed for visual comparison. Bridge: "Now it's your turn to find rectangles in the table." No student action in warmup — pure demonstration of the new interaction.

**Lesson (~15–18 min, 3 sections):**
- **Section 1 (Early — Rectangle → Table, ~7 min):** Students mark rectangle regions in the table given a Grid Rectangle with labeled dimensions. Prediction-first pattern: calculate area via MC BEFORE marking the table. Connection Animation links Grid Rectangle to shaded table region. 4–5 problems covering different table regions. Key teaching beat: "The multiplication table is a collection of ALL rectangles." CRA: Abstract (table is symbolic) connected back to Concrete (grid rectangles).
- **Section 2 (Mid — Table → Rectangle, ~5 min):** Reverse direction. System highlights a product; student marks the rectangle region. Then: multiple rectangles for same product (factor-pair visual home). Commutative vs. genuinely different pairs distinguished. 3–4 problems. Extends M8's factor-pair reasoning.
- **Section 3 (Late — Patterns, ~4 min):** Fully revealed table. Row patterns, diagonal (square numbers), using known products to find unknown products. Light-touch pattern discovery. 2–3 activities. Previews fluency strategies.

**Exit Check (3–5 min):** 3 problems testing the three core skills: (1) mark rectangle in table + read area, (2) read table position → identify dimensions, (3) factor-pair reasoning within table context.
