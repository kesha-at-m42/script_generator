## Cross-Reference Table B — TVP Extraction

```
TVP: MODULE 12
===============
Title: Composite Figures Without Grids
OUR Lesson: L13
Standard: 3.MD.C.7.d (ungridded composite figures — area as additive with multiplication)

CRA CONTEXT:
M12 is to composite figures what M7 was to single rectangles: the grid fading module. M11 used full grids because decomposition was new. Now that students can decompose, calculate, and add, the grid fades. The decompose-calculate-add workflow is identical — only the visual support changes.

Parallel to M7: M7 faded grids for single rectangles (partial grid → tick marks → dimensions). M12 fades grids for composite figures (partial grid → dimensions with labels). The composite figure version SKIPS the tick mark stage — tick marks on a multi-edged composite shape create visual complexity that doesn't serve the learning goal. Partial grid is the bridge; dimension labels are the target.

Learning Goal:
Calculate the area of ungridded figures made of rectangles using multiplication and addition.

Key Teaching Points:
1. Same decompose-calculate-add strategy from M11 — nothing new about the PROCESS
2. What's new: the grid is gone. Students read dimensions from labels, not squares.
3. Late addition — subtraction decomposition: After students are fluent with additive decomposition on ungridded figures, M12 Late introduces a second strategy: imagine the bounding rectangle, calculate its area, then subtract the missing piece. This reframes the M11 misconception check — the bounding rectangle isn't wrong, it's just incomplete. Students solve the same figure both ways (additive and subtractive) and verify they get the same answer. Additive decomposition remains the primary strategy; subtraction decomposition is a taught alternative, not a replacement.
4. THE KEY SKILL SHIFT: In M11, students could count squares to verify. In M12, they must TRUST the multiplication. The grid was a safety net; now it's removed. Same trust-building as M7 for single rectangles.
5. All labeled dimensions are given (no missing sides — that's M13). Student's job: (1) decompose, (2) identify which dimensions belong to which rectangle, (3) multiply + add correctly.
6. Dimension assignment is the new challenge. When a composite figure has 6+ dimension labels, students must determine which pairs of dimensions define each component rectangle AFTER decomposition. This is where misconception M12 (multiplying all numbers indiscriminately) surfaces. The decomposition line determines which dimensions pair together.
7. Real-world contexts: patio paving (sustained context). "How many square feet of pavers do we need?"
8. Parenthesized expressions continue: (a × b) + (c × d) = total, with color-coding linking each term to its rectangle.

CRA Stage: Representational → Abstract transition. Partial grid (Early) → dimensions only (Mid/Late).

Cognitive Focus:
- APPLY (Transfer) — decompose-calculate-add workflow in a new representation (ungridded)
- IDENTIFY (Conceptual) — determining which dimensions belong to which component rectangle after decomposition
- CONNECT (Transfer) — linking M7's single-rectangle abstraction to M11's composite decomposition

Misconceptions Targeted:
- M12: Multiplying All Given Numbers (PRIMARY, HIGH) — When grids are removed, students see multiple dimension labels and multiply some random combination instead of decomposing first. Addressed by: (1) always decomposing FIRST, (2) color-coding linking dimensions to rectangles, (3) explicit "which dimensions go with which rectangle?" reasoning step.
- M11: Completing the Rectangle (SECONDARY) — Without grids, students may be MORE tempted to treat L-shape as full rectangle. Addressed by maintaining decomposition-first workflow + MC distractors based on bounding-box multiplication.
- M6: Overlapping Decomposition (TERTIARY) — Less visible without grids. System validation continues to prevent mechanically.

PHASE-BY-PHASE FLOW:

WARM-UP:
- Composite Figures (L-shape, full grid) → Quick solve using M11 workflow. Students decompose, calculate, add. Should feel routine.
- Then: SAME figure displayed again with grid removed — only outline + dimension labels on all edges.
- "The shape didn't change. The area didn't change. But the grid is gone. Can you still find the area?"
- Guide walks through: "You already drew a decomposition line. The blue rectangle is 4 by 3 — the dimensions say so. The pink rectangle is 5 by 2 — the dimensions say so. (4 × 3) + (5 × 2) = 12 + 10 = 22. Same answer."
- Purpose: Establish that dimensions replace the grid — same information, different form. Mirrors M7's warm-up structure.

LESSON:

Early activities (Partial grid — structural bridge):
- Composite Figures (L-shapes and simple rectilinear, partial grid — first row + first column of each component rectangle visible, dimension labels on all edges)
- Partial grid is the bridge state: students can see enough structure to verify reasoning, but can't count every square. M7 partial-grid concept applied to composite figures.
- ⚠️ Early figures must use DIFFERENT dimensions than the warm-up figure.
- Student decomposes (student-placed), system color-codes rectangles and shows dimensions for each.
- Key beat — dimension assignment: After decomposition, guide explicitly asks: "Look at the blue rectangle. Which dimensions belong to it?" System highlights dimension labels on blue rectangle's edges. Student identifies. Then same for pink rectangle. This is the NEW reasoning step replacing counting.
- Student calculates each area (MC), adds (MC). Equation Builder displays color-coded expression.
- 3–4 problems with partial grid. All L-shapes or simple 2-component rectilinear.

Dimension-pairing check (after 2nd problem):
- System displays decomposed figure with all dimensions labeled. Guide poses incorrect pairing: "A student looked at this figure and multiplied 5 × 4. Is that one of the rectangles?" Student examines: no, 5 and 4 are on different rectangles. "5 belongs to the blue rectangle and 4 belongs to the pink rectangle. You can't just grab any two numbers."
- Directly addresses misconception M12. Cost: ~30 seconds.

Mid activities (Dimensions only — grid removed):
- Composite Figures (L-shapes and simple rectilinear, labels only — no grid, all dimension labels visible) + Equation Builder (color-coded)
- Full abstraction step. No grid at all. All edges have dimension labels.
- Student decomposes (student-placed). System color-codes rectangles. Dimensions highlighted per rectangle.
- Student calculates each area (MC), adds (MC).
- 3–4 problems. L-shapes and 2–3 component rectilinear figures.

Key beat — related sides add up (after 1st or 2nd labels-only problem):
- After correct completion, guide adds verification step: "Let's check something. Look at the horizontal sides." System highlights horizontal edges. "This piece is 5, this piece is 3, and the total across the bottom is 8. Does 5 + 3 = 8? Yes! Related sides always add up. That's how you know your dimensions are right."
- Repeat on next problem with vertical sides.
- By Late, prompt student to self-check: "Can you check — do the related sides add up?"
- Vocabulary note: "Related sides" introduced here informally. Becomes formal vocabulary in M13. Same preview-then-formalize pattern as "rows of" (M3) → "×" (M4).

Patio paving context introduced (Mid, problem 3 or 4):
- "The Garcia family is paving their patio. The patio is this shape. How many square feet of pavers do they need?"
- L-shaped patio with dimension labels in feet.
- Same workflow, real-world context. First real-world context for composite figures. Sustained through Late and Practice.
- Guide: "Knowing the total area tells you how many square feet of pavers to buy."

Late activities (Independent, mixed contexts):
- Composite Figures (L-shapes, T-shapes, U-shapes, labels only, 2–3 components) + Equation Builder
- Full independence. No partial grid. No ghost grid unless remediation triggered.
- Cognitive load sequencing: First Late problem = 2-component figure with many dimension labels (6+ edges labeled) — testing dimension-pairing complexity WITHOUT adding components. Then T-shapes and U-shapes (3 components, 8–10 labeled edges). Separates two increases: more labels first, then more components.
- Mix of abstract (generic square units) and patio paving context (square feet/square meters).
- 3–4 problems.

Key beat — multiple dimensions, one figure:
- Some Late figures have 6–8 dimension labels visible on exterior edges. After decomposition, student must select correct 2 dimensions for each component rectangle from among all visible labels.
- MC options for each rectangle's area should include plausible wrong answers from incorrect dimension pairing.

Key beat — subtraction decomposition (after 2nd Late problem):
- System displays L-shape student just solved additively (or new L-shape with clean numbers).
- Guide: "You found the area by breaking the shape into rectangles and adding. Some mathematicians look at this a different way. Watch."
- System shows bounding rectangle (dotted outline, same visual as M11's misconception check). "Remember this? In Module 11, we said this doesn't work BY ITSELF. But what if we kept going?"
- System highlights bounding rectangle dimensions: "The full rectangle would be 6 by 8. That's 48." Then highlights missing corner: "But this piece isn't part of our shape. It's 3 by 4 — that's 12." Equation Builder: (6 × 8) − (3 × 4) = 48 − 12 = 36.
- "48 minus 12 is 36 — the same answer you got by adding! Two different strategies, same area."
- Guide: "Breaking into parts and adding. Or imagining the full rectangle and subtracting what's missing. Both work. You get to choose which feels easier for each shape."
- 1 additional problem where student tries subtraction decomposition themselves: L-shape with clear bounding rectangle and simple missing corner. Student identifies bounding rectangle dimensions (MC), calculates bounding area, identifies missing corner dimensions (MC), calculates missing area, subtracts.
- ⚠️ Scope constraint: Subtraction decomposition taught ONLY for figures with single rectangular missing piece (L-shapes). T-shapes and U-shapes use additive decomposition only.
- Frame as strategic choice, not better method.

Multiple decompositions — light touch:
- 1 problem where guide notes: "You decomposed it one way. There's another way too — and you'd get the same total." Brief, not solved. M11 insight recalled, not re-taught.

EXIT CHECK:
3 problems:
1. Partial grid composite: L-shape with partial grid and dimension labels. Student decomposes, calculates total area. (Tests: can student work with partial visual support for composite figures)
2. Labels-only composite: L-shape or T-shape, dimensions only, no grid. Student decomposes, identifies correct dimensions per rectangle, calculates total area. (Tests: can student find area without ANY grid, correctly pairing dimensions)
3. Misconception check — multiplying all numbers: Figure with 6 dimension labels. MC includes: correct answer, bounding-rectangle answer (M11 misconception), and a "multiply random dimensions" distractor (M12 misconception). (Tests: does student decompose properly and avoid grabbing numbers indiscriminately)

Assessment ratio: 1 scaffolded (partial grid) : 1 independent (labels only) : 1 misconception-targeted.

PRACTICE:
- Mixed figure types: L-shapes, T-shapes, U-shapes
- All ungridded (labels only) — Practice at target abstraction level
- Ghost grid available as remediation scaffold
- Mix of abstract and patio/room contexts with named units (sq ft, sq m)
- Some problems require writing complete parenthesized expression before calculating (Equation Builder production)
- ⚠️ After every decomposition, guide should ask/prompt: "Which dimensions go with the [blue/pink] rectangle?" — consistent step, not just in Early.
- Error patterns to watch: multiplying all visible numbers; bounding-box dimensions; pairing dimensions from different rectangles; forgetting to add; addition errors

SYNTHESIS:
- CRA progression display for composite figures: same figure in three states — full grid (M11) → partial grid (M12 Early) → dimensions only (M12 target)
- "Same shape. Same area. Three different ways to see it. You started by counting every square inside each rectangle. Now you just need the dimensions and your decompose-calculate-add strategy."
- Mirrors M7's synthesis display (four states for single rectangles).
- Patio paving callback: "Knowing area helps you figure out how many pavers to buy, how much paint you need, how much carpet to order. And now you can find the area of shapes that aren't just simple rectangles."
- Vocabulary consolidation: no new terms. Reinforce: decompose, composite figure, dimensions, multiply, add, parentheses.
- Bridge: "You learned that related sides add up — and you used that to check your work. What if some sides DIDN'T have measurements? Could you use that same idea to figure out the missing ones?" — display figure with some sides labeled and others marked "?". Don't solve it. Leave as teaser.

TRANSITION IN (from M11):
Pedagogical Shift: Full-grid composite figures (concrete/representational) → Ungridded composite figures (abstract)
What carries forward: The decompose-calculate-add workflow, two-tap decomposition interaction, color-coding, parenthesized expressions, multiple valid decompositions
What's new: No grid — dimensions only. Dimension-pairing as explicit reasoning step. Real-world contexts (patio paving). Ghost grid as remediation.
Why this works now: Students proved in M11 they can decompose, calculate, and add. They proved in M7 they can find area from dimensions alone. M12 combines these two established skills. Neither skill is new — the combination is.

CRA parallel:
Single rectangles: M5–M6 (full grid) → M7 (partial → tick → dimensions) → M8+ (dimensions only)
Composite figures: M11 (full grid) → M12 (partial → dimensions) → M13+ (dimensions with missing sides)

TRANSITION OUT (to M13):
What M12 establishes: Students can find area of composite figures from dimension labels alone, using decompose-calculate-add with correct dimension pairing.
What M13 builds: Some dimension labels are MISSING. Students must infer missing side lengths from relationship between visible sides before calculating area. Additional reasoning step: find missing dimensions → THEN decompose-calculate-add.
Bridge: "Today every side had a measurement. What if some sides DIDN'T have measurements? Could you figure out the missing ones?"
Engineering note: M13 uses Composite Figures in "Missing Sides" mode — some edges labeled with "?" instead of values. "Highlight Related Sides" teaching action becomes critical.
```

### Scaffolding Progression (TVP)

```
Scaffolding strategy: The decomposition interaction is constant (student draws throughout).
What changes is the visual support — from partial grid to no grid. This isolates one
variable: students aren't learning a new decomposition method AND losing the grid
simultaneously. They learned decomposition in M11; M12 removes the grid on the SAME skill.

Ghost grid as remediation: Composite Figures supports a ghost grid mode (faint grid lines
throughout figure). Guide-activated, not student-accessible. Used when student can't work
from dimensions alone. Ghost grid appears, student recalculates, ghost grid fades.
If student triggers ghost grid on 2+ problems in Mid/Late, flag for M11 reinforcement.

Ghost grid diagnostic: If needed once = normal transition friction. If needed twice =
guide says "Let's slow down — the grid is helpful. Let's practice with grids a bit more
before we take them away." Signals adaptive system to loop back to M11-level problems.
```

### Tool Requirements (TVP)

```
Critical capability — Dimension highlighting after decomposition: After student draws
decomposition line and rectangles are color-coded, system must highlight which dimension
labels belong to each rectangle. When blue rectangle is selected/active, only its
dimension labels are emphasized (others dim). Primary defense against M12 misconception.
Early: automatic. Late: on-demand (student taps rectangle to see dimensions highlighted).

segment dimensions activate. (Example: L-shape with "8" on full edge and "5"/"3" on

Critical capability — Partial grid for composite figures: Shows first complete horizontal
row across WIDEST part + first complete vertical column down TALLEST part. Defined relative
to whole figure, not pre-decomposed components. Where figure's shape creates absent
positions, those positions are visibly empty (reinforcing composite shape).

Critical capability — Ghost grid remediation: Guide-activated faint grid overlay on any
labels-only composite figure. Teaching/remediation action, not student toggle. Logged for
diagnostic purposes.

Equation Builder color-coding: Continues from M11 — each parenthesized term matches color
of its corresponding rectangle. Even more important in M12 because color link is primary
visual connection between abstract expression and ungridded figure.

No new tools. M12 uses same Composite Figures and Equation Builder from M11. Change is in
configuration (grid states), not tooling.
```

### Data Constraints (TVP)

```
Early figures (partial grid):
- 2-component L-shapes with all dimensions labeled
- Side lengths 2–8; component areas ≤ 40; total area ≤ 60
- Examples: L-shape 4×3 + 5×2 = 22; 6×3 + 2×4 = 26; 5×4 + 3×3 = 29
- ⚠️ Must use different dimensions than warm-up figure

Mid figures (labels only):
- 2–3 component L-shapes and simple rectilinear
- Side lengths 2–10; component areas ≤ 50; total area ≤ 80
- Examples: L-shape 5×4 + 3×6 = 38; T-shape with 3 components totaling 45
- Total area raised from M11's ≤ 50 because students trust multiplication
- Include figures where multiple dimensions visible on same edge (e.g., "8" full edge +
  "5"/"3" segments). After decomposition, full-edge total dims, segment dimensions activate.

Late figures (labels only, mixed context):
- 2–3 component figures, L/T/U shapes
- Side lengths 2–10; total area ≤ 100 (with real-world contexts)
- Patio paving example: L-shaped patio, 6 ft × 8 ft overall with 3 ft × 4 ft notch.
  Area = (6 × 4) + (3 × 4) = 24 + 12 = 36 sq ft.
- Multiple dimension labels on edges: at least 2 Late problems with 6+ visible labels
- MC distractor design for M12 misconception:
  * Correct answer (decompose + multiply + add)
  * Bounding-box error (multiply longest horizontal × longest vertical)
  * Random dimension multiplication (two dims from different rectangles)
  * Partial calculation (only one rectangle's area)

Patio paving context constraints:
- All dimensions in feet or meters (no mixing within one problem)
- L-shaped patios for Early/Mid; T-shaped or U-shaped for Late
- Keep contexts simple: "How many square feet of pavers?" No cost calculations.

Avoid in M12:
- Missing dimensions (all sides labeled — that's M13)
- Subtraction decomposition on T-shapes or U-shapes (only L-shapes with single missing piece)
- Figures with more than 3 components
- Figures where decomposition creates 1×n rectangles
```

## Section Plan

**Warmup (3–4 min):** Activation + Discovery hybrid (mirrors M7 warmup structure). Two parts: (1) Quick M11-workflow solve on gridded L-shape — routine, confirms decompose-calculate-add. (2) SAME figure with grid removed — dimensions only. Guide walks through: dimensions replace the grid, same information, same answer. Establishes that grid removal doesn't change the math. Anticipated warmup type: Activation + Discovery (same as M7, same as M11). Bridge target: "The grid is gone but the dimensions tell you everything you need."

**Lesson (~12–14 min, 3 sections):**
- **Section 1 (Early — Partial Grid, ~4 min):** CRA bridge. Partial grid composite figures with dimension labels. Student draws decomposition line (student-placed from the start — M11 established this). NEW reasoning step: dimension-pairing ("Which dimensions go with which rectangle?"). 3–4 problems. Dimension-pairing misconception check after 2nd problem. CRA stage: Representational.
- **Section 2 (Mid — Labels Only, ~4 min):** Full abstraction. No grid. All edges have dimension labels. Same workflow, new representation. "Related sides add up" verification principle introduced. Patio paving context introduced (problem 3 or 4). 3–4 problems. CRA stage: Abstract.
- **Section 3 (Late — Independent + Subtraction Decomposition, ~4 min):** Independence with mixed contexts and figure types (L/T/U). Cognitive load sequencing: more labels first, then more components. Subtraction decomposition taught (1 demo + 1 student practice, L-shapes only). Multiple decompositions light-touch (1 problem, brief). 3–4 problems plus subtraction beat. CRA stage: Abstract + strategy extension.

**Exit Check (3–5 min):** 3 problems testing the module's CRA progression. EC.1: partial grid composite (scaffolded). EC.2: labels-only composite (independent). EC.3: misconception-targeted MC with diagnostic distractors. Cognitive types: CREATE (EC.1, EC.2), IDENTIFY (EC.3).

**Practice Inputs:** Mixed L/T/U shapes, all ungridded. Ghost grid as remediation scaffold. Patio/room contexts. Dimension-pairing step reinforced throughout. BASELINE/STRETCH/SUPPORT/CONFIDENCE tiers.

**Synthesis (5–7 min):** CRA progression display (3 states: full grid → partial grid → dimensions only). Patio paving callback. Vocabulary consolidation (no new terms). M13 bridge: "What if some sides didn't have measurements?" with "?" figure teaser. Anticipated task types: Pattern Discovery (CRA progression), Connection/Reflection.

**Timing estimate:** Warmup 3–4 + Lesson 12–14 + EC 3–5 + Synthesis 5–7 = 23–30 min total module block. This is above the ~15 min atomic block target (Pattern #43). However, the Lesson alone is 12–14 min — at the edge but manageable because: (1) Early uses student-placed decomposition from the start (no I-do phase needed — established in M11), (2) the subtraction decomposition beat in Late could be deferred to Practice if timing is tight. **Flag for author review:** If the subtraction decomposition beat pushes the atomic block beyond acceptable range, it can be moved to Practice Inputs as a STRETCH-level problem type.
