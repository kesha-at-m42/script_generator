# **MODULE 12: Composite Figures Without Grids**

# BACKBONE

---

## **1.0 THE ONE THING**

**[REQUIRED]**

The grid is gone, but the math hasn't changed. Students already know how to decompose composite figures into rectangles, multiply to find each area, and add — M11 established that workflow with full grid support. M12's single new demand is trusting dimension labels instead of counting squares. The decompose-calculate-add process is identical; the representation shifts from concrete (gridded) to abstract (dimensions only). The critical skill that replaces grid-counting is dimension-pairing: after drawing a decomposition line, students must identify which labeled dimensions belong to which component rectangle. The decomposition line is the organizing principle — it determines which dimensions pair together. When students read dimensions correctly and pair them to the right rectangles, the rest is multiplication and addition they've been doing since M4.

**CRA Stage:** Representational → Abstract transition. M12 parallels M7's role for single rectangles: M7 faded grids from full → partial → tick marks → dimensions for single rectangles; M12 fades grids from full (M11) → partial → dimensions for composite figures. Tick marks are skipped for composites because they create visual complexity on multi-edged shapes without proportionate pedagogical benefit (resolved SME question). Partial grid is the bridge; dimension labels are the target representation. By Late, students work entirely from dimension labels — the same abstraction level as M8–M9 for single rectangles, now applied to composite figures.

**Critical Misconception:** #12 — Multiplying All Given Numbers. When grids are removed, students see multiple dimension labels on a composite figure (e.g., 3, 5, 2, 4 on an L-shape) and multiply some combination indiscriminately instead of decomposing first and pairing dimensions to specific rectangles. The grid's absence removes the visual verification — students can't count squares to catch the error. The primary defense is the dimension-pairing workflow: always decompose FIRST, then identify which dimensions belong to which rectangle, then multiply. Color-coding that links dimensions to rectangles and the explicit "which dimensions go with which rectangle?" reasoning step make the pairing visible.

**Success Indicator:** Given a composite figure with dimension labels only (no grid), the student independently draws a valid decomposition line, correctly identifies which dimensions belong to each component rectangle, calculates each area by multiplying, adds the areas, and arrives at the correct total — demonstrating that they can execute the full decompose-calculate-add workflow without visual grid support.

**Biggest Risk:** Students execute the dimension-pairing procedure correctly on simple 2-component L-shapes but break down when figures have 6+ dimension labels on their edges. The cognitive load isn't the multiplication or the decomposition — it's the visual parsing of WHICH numbers go with WHICH rectangle when many labels are visible simultaneously. If the Lesson doesn't progressively increase label density (2-component with few labels → 2-component with many labels → 3-component), students will hit a wall in Late and Practice when T-shapes and U-shapes appear with 8–10 labeled edges.

### **1.1.2 Module Bridges**

**From M11 (Area is Additive):**
M11 introduces composite figures and the decompose-calculate-add workflow with full grid support. Students learn to see inside corners as cut points, draw decomposition lines, handle multiple valid decompositions, and express the process with parenthesized notation: (a × b) + (c × d) = total. The full grid provides a safety net — every square is visible, and students can count to verify their multiplication. M11's closure teases M12: "What if the grid disappeared and you only had the side lengths?"

**This Module (M12: Composite Figures Without Grids):**
M12 removes the grid and asks students to trust their multiplication. The decompose-calculate-add workflow is identical to M11; the representation changes from gridded to dimension-labeled. The new cognitive demand is dimension-pairing — identifying which labeled dimensions belong to which component rectangle after decomposition. M12 also introduces real-world contexts (patio paving), the "related sides add up" verification principle (previewing M13's problem-solving tool), and subtraction decomposition as an alternative strategy for L-shapes. The CRA progression parallels M7: partial grid → dimensions only.

**To M13 (Missing Side Lengths):**
M13 adds one new reasoning step: some dimension labels are missing ("?"), and students must use the "related sides add up" principle to find them before they can decompose and calculate. The decompose-calculate-add workflow from M12 is unchanged — M13 adds a prerequisite step. M12's informal introduction of "related sides add up" as a VERIFICATION tool becomes M13's PROBLEM-SOLVING tool, reducing M13's cognitive load by one variable.

## **1.2 SCOPE BOUNDARIES**

**[REQUIRED]**

### **✅ Must Teach**

* **Grid fading for composite figures — partial grid → dimensions only:** The core CRA transition. Students move from M11's full-grid composites through a partial-grid bridge (Early) to dimensions-only (Mid/Late). This parallels M7's grid fading for single rectangles. Per Decision #3: M12 = grid fading for composites.
* **Dimension-pairing as explicit reasoning step:** After decomposition, students identify which labeled dimensions belong to which component rectangle. This replaces grid-counting as the verification mechanism. The guide consistently asks: "Which dimensions go with the [blue/pink] rectangle?" This is the module's signature cognitive demand.
* **Decompose-calculate-add workflow at abstract level:** Same workflow from M11 — decompose into rectangles, multiply dimensions for each area, add totals. No new procedural skill; the representation changes.
* **Subtraction decomposition (Late only, L-shapes only):** Taught alternative strategy: imagine the bounding rectangle, calculate its area, subtract the missing piece. Completes the "by itself" foreshadowing from M11's misconception check. Scoped to L-shapes with a single rectangular missing piece — T-shapes and U-shapes use additive decomposition only. Framed as strategic choice, not better method.
* **"Related sides add up" verification principle:** Informal introduction in Mid. After a problem, guide demonstrates: "This piece is 5, this piece is 3, and the total is 8. Does 5 + 3 = 8? Related sides always add up." Used as a verification tool in M12; becomes a problem-solving tool in M13. Same preview-then-formalize pattern as "rows of" (M3) → "×" (M4).
* **Real-world contexts — patio paving:** Sustained context from Mid through Late and Practice. "How many square feet of pavers?" Context wraps the math; it doesn't extend it. No cost calculations.
* **Proactive M12 misconception prevention — dimension-pairing check:** After 2nd Early problem, guide poses an incorrect dimension pairing and asks student to evaluate it. Directly addresses misconception #12 before it takes root.
* **Figure types:** L-shapes (Early through Late), T-shapes and U-shapes (Late only, labels-only). All 2–3 component figures. Progression: 2-component with few labels → 2-component with many labels → 3-component.
* **Multiple valid decompositions — light touch:** 1 Late problem where guide notes another valid decomposition exists. Brief recall of M11 insight, not re-taught or assessed.
* **Vocabulary reinforcement:** decompose, composite figure, dimensions, area, multiply, add, square feet, square meters, square units, expression, parentheses. No new active vocabulary — this is a consolidation module.

### **❌ Must Not Include**

* **Perimeter** — Per Decision #5. Not mentioned, even for contrast.
* **Missing dimensions ("?" sides)** — Finding missing side lengths begins at M13. M12 labels ALL dimensions explicitly.
* **Subtraction decomposition on T-shapes or U-shapes** — Subtraction decomposition is taught only for L-shapes with a single rectangular missing piece. More complex shapes use additive decomposition exclusively.
* **Figures with more than 3 components** — M13–M14 handle complex multi-component figures. M12 uses 2–3 components only.
* **Figures where decomposition creates 1×n rectangles** — Technically valid but uninteresting; same constraint as M11.
* **Cost calculations or multi-step word problems** — The patio context asks "How many square feet of pavers?" NOT "Pavers cost $3 per square foot — how much?" Context wraps the math; it doesn't extend it.
* **Grid Rectangles** — M12's primary tool is Composite Figures. Grid Rectangles are not used in any phase (unlike M11's Warmup activation).
* **Equation Builder in student-construction mode** — Students confirm values via MC selection within the Equation Builder, not by typing or constructing expressions independently.
* **Division language** — Decomposition uses spatial language ("break into parts"), not arithmetic language.
* **Order of operations language** — Parentheses remain descriptive notation. Beyond Grade 3 scope.
* **New vocabulary** — No new active or receptive terms. "Related sides" is previewed informally but not taught as formal vocabulary (that's M13).
* **Full grids** — Except in the Warmup's initial activation problem (which uses a gridded L-shape to mirror M11 and set up the grid-removal reveal). All Lesson, EC, and Synthesis interactions use partial grid or labels only.

### **Vocabulary Notes**

**"Dimensions" (reinforced — heightened emphasis):** In M12, "dimensions" carries more weight than in any prior module. Without a grid, dimensions are the ONLY source of area information. The Guide should model precise language: "The dimensions of the blue rectangle are 4 and 3" rather than "it's 4 and 3." This precision is critical when multiple dimension labels are visible on the same figure. Module Mapping teaching note: "Reinforce: dimensions."

**"Related sides" (new informal preview):** Introduced in Mid after the 1st or 2nd labels-only problem. The Guide demonstrates that segment dimensions add up to the full edge dimension. This is a verification concept in M12 ("That's how you check your work") that becomes a problem-solving concept in M13 ("That's how you find the missing side"). Same preview-then-formalize pattern as "rows of" (M3) → "×" (M4) and "length/width" (M4 Mid) → "length × width" (M4 Late). NOT assessed in M12.

**Parenthesized expression notation:** Continues from M11. (a × b) + (c × d) = total, with color-coding linking each term to its corresponding rectangle. The Equation Builder generates this automatically. Even more important in M12 because the color link is the primary visual connection between the abstract expression and the ungridded figure. For subtraction decomposition: (a × b) − (c × d) = total.

**Patio paving language:** "Square feet" and "square meters" used in real-world contexts. "Pavers" is a context word, not mathematical vocabulary. "How many square feet of pavers do we need?" Keep context language simple — the context wraps the math, it doesn't extend it.

**Language patterns for M12:**

* "This figure has dimensions labeled on every side."
* "After decomposing: the [blue/pink] rectangle is __ by __. Its area is __ square [unit]."
* "(__ × __) + (__ × __) = __ + __ = __ square [unit]."
* "The patio needs __ square feet of pavers."
* "Which dimensions go with the [blue/pink] rectangle?"
* "Related sides add up — that's how you check your work."

### **Terms to Avoid (Save for Later Modules)**

* **"Perimeter"** — Do not use, even for contrast. Per Decision #5.
* **"Formula"** (standalone) — Suggests rote memorization. Same constraint as M7–M11.
* **"Divide" / "Division"** — Not introduced in Unit 2.
* **"Order of operations"** — Parentheses are descriptive notation. Grade 3 does not teach formal operation ordering.
* **"Missing side"** — No "?" dimensions in M12. All sides are labeled. Missing side-length problems begin at M13.
* **"Memorize"** — Decomposition is a strategy, not a fact to memorize.
* **"Subtract"** (in decomposition naming context) — The guide uses "minus" and "subtracting what's missing" in the Late subtraction decomposition beat, but does not label the strategy as "subtraction decomposition." Student-facing language: "imagining the full rectangle and taking away the missing piece."

## **1.4 MISCONCEPTIONS**

**[REQUIRED]**

### **1.4.1 #12: Multiplying All Given Numbers (PRIMARY)**

**Trigger Behavior:** When grids are removed and a composite figure has multiple dimension labels visible (e.g., 3, 5, 2, 4 on an L-shape), the student multiplies some random combination of numbers instead of decomposing first and pairing dimensions to specific rectangles. The student treats all visible numbers as inputs to a single multiplication, ignoring that each component rectangle has its own pair of dimensions.

**Why It's a Barrier:** This misconception is the signature error of ungridded composite figures. The grid's absence removes the visual check — students can't count squares to catch the error. Unlike #11 (completing the rectangle), which produces a specific wrong answer (bounding rectangle area), #12 can produce many different wrong answers depending on which numbers the student grabs, making it harder to diagnose from the answer alone.

**Prevention Strategy (M12-specific):**

* **Primary defense — dimension-pairing workflow:** Always decompose FIRST, then identify which dimensions belong to which rectangle. The decomposition line determines dimension pairing. This is taught explicitly in Early, not left implicit.
* **Color-coding reinforcement:** After decomposition, system highlights dimension labels per rectangle. When the blue rectangle is active, only its dimension labels are emphasized. This makes the dimension-to-rectangle mapping visible.
* **Proactive dimension-pairing check (Early, after 2nd problem):** Guide poses an incorrect pairing and asks student to evaluate: "A student looked at this figure and multiplied 5 × 4. Is that one of the rectangles?" Student examines: no, those dimensions are on different rectangles. Costs ~30 seconds, addresses M12 before it takes root.
* **MC distractor design:** EC and Late MC options include a "random dimension multiplication" distractor — an answer produced by multiplying two dimensions from different rectangles. This diagnoses the specific error.
* **Dimension label density progression:** Early uses figures with few visible labels (4–5). Late uses figures with 6–8 labels. The progression builds dimension-parsing fluency before overwhelming with label density.

**Where It Surfaces:** Mid (first labels-only problems — the most likely moment), Late (complex figures with many labels), EC problem 3 (targeted MC).

### **1.4.2 #11: Completing the Rectangle (SECONDARY)**

**Trigger Behavior:** Student treats an L-shape or composite figure as a complete rectangle by multiplying the longest horizontal side by the longest vertical side. Produces the bounding rectangle area, not the composite figure area. Without grids, students may be even MORE tempted to "complete" the shape since they can't see the missing squares.

**Why It's a Barrier:** If students default to bounding-rectangle multiplication, they see no need for decomposition — the entire module's workflow becomes unnecessary in their eyes. The error is self-reinforcing: the bounding rectangle answer is always LARGER than the correct answer but still plausible.

**Prevention Strategy (M12-specific):**

* **Decomposition-first workflow:** The consistent expectation to decompose BEFORE calculating makes the bounding-rectangle shortcut structurally impossible within the workflow.
* **Subtraction decomposition reframes the bounding rectangle:** In Late, the bounding rectangle is reintroduced — not as a mistake, but as the first step of a valid strategy (bounding area minus missing piece). This completes M11's "by itself" foreshadowing. The bounding rectangle isn't wrong; it's just incomplete.
* **MC distractors:** EC problem 3 includes a bounding-rectangle distractor (same as M11's EC.3 strategy, now without grid support).

**Where It Surfaces:** EC problem 3 (targeted), Mid/Late independent work (may emerge when students face labels-only figures without the grid's visual reminder that squares are "missing").

### **1.4.3 #6: Overlapping Decomposition (TERTIARY)**

**Trigger Behavior:** When decomposing, student draws a line that creates overlapping regions and counts some area twice.

**Why It's a Barrier:** Violates the "non-overlapping" requirement of 3.MD.C.7.d. Less visible without grids (can't see overlapping colored squares), which makes it harder to catch visually but also less likely to occur if students use the decomposition tool correctly.

**Prevention Strategy (M12-specific):**

* **System validation continues:** The Composite Figures tool validates decomposition lines mechanically, preventing invalid decompositions.
* **Color-coding:** After decomposition, each rectangle highlights in a distinct color. Even without grid squares, the color boundary makes overlap visible.

**Where It Surfaces:** Mid (when dimension-only figures may confuse students about valid cut lines), though system validation catches it mechanically.

# **PHASE SPECIFICATIONS**
## **1.6 WARMUP (3–4 minutes)**

**[REQUIRED]**

### **Core Purpose**

**One-Line Summary:**
Establish that dimensions on a figure carry the same information as grid squares—same shape, same area, different form.

**Key Function:**
This Warmup reactivates the M11 decomposition workflow (identify rectangles, calculate area, add), then presents the critical threshold moment: the grid is removed, but the area method stays the same. Students connect "I just counted 12 squares in that rectangle" with "the dimensions 4 and 3 tell me the same thing." This transition anchors the entire module—without grids, students still have everything they need.

**Why This Serves the Concept:**
* Composites don't require grids to have area
* Dimensions are the alternative, equally reliable way to find a rectangle's area
* Same decomposition strategy works with or without visual counting
* Lowers cognitive load for the Lesson (students won't need to *learn* decomposition; they'll only practice it without grids)

**Test:**
If we removed this Warmup, would students be ready for the Lesson? **Yes, conditionally.** Without it, students would encounter grid-less figures and might assume the strategy requires squares to count. This Warmup erases that assumption and establishes that they already have the tools.



### Warmup Type Rationale

**Type selected:** Activation + Reveal (custom hybrid per Warmup Phase Playbook)

**Why this type fits M12:** M12's warmup has two jobs that no single Playbook type covers cleanly. First, students must reactivate the M11 decomposition workflow (decompose, multiply, add) on a familiar gridded figure — that's pure Activation (§4A). Second, the grid is removed from that same figure and the teacher demonstrates that dimension labels carry the same information — that's a Discovery/Reveal moment (§4D). The combination ensures students both confirm their existing skill AND confront the module's threshold shift (grid → no grid) before the Lesson begins.

**Why not other types:**
* Mystery Reveal alone — M10's type. M12's reveal (grid removal) isn't counterintuitive the way "rectangles live inside the multiplication table" was; it's a representation change on a familiar task. The activation step is essential because students need to solve with the grid FIRST to anchor the "same answer" comparison.
* Notice & Wonder — Could work for W.2 ("What changed? What stayed the same?"), but the threshold moment needs teacher narration to connect grid-counting to dimension-reading. Open observation risks students noticing surface features instead of the mathematical invariance.
* Number Talk — M12 is not about mental math strategies; it's about representation transfer.
* Visual Patterns — The pattern is structural (grid = dimensions), not sequential.
* Binary Choice — No meaningful binary to present; the insight is "these two representations are equivalent," not "which one is correct."

### Design Notes & Voice Notes

> **Design Note (Grid-Removal Pacing):**
> The deliberate unhurried pace in W.2 is critical. The teacher does not rush through the dimension-reading. Each step—"The blue rectangle is 4 by 3. > The "grid removal" serves as the emotional and narrative anchor for the Warmup. It's not a problem or obstacle; it's a transition framed as empowerment ("the numbers on the edges do the same job"). > **Voice Note (Engagement Anchor: Strength Prompt):**
> The repeated "you already know" language in the Guide and Bridge positions students as capable, not confused. The grid is not a crutch being removed; it's a training wheel being set aside because the student is ready. This tone prevents defensive learning and supports risk-taking in the Lesson.

> **Design Note (Figure Consistency):**
> The L-shape (4×3 + 5×2 = 22) is deliberately simple and carries no extraneous complexity. > **Voice Note (Session-Relative Language):**
> "Last time" (W.1 Guide) refers to M11's explicit work; "now" (throughout) is present-session language. No references to calendar time or external events. All anchors are procedural ("the strategy you used last time") or immediate ("watch," "try," "you just did").

### **Misconception Prevention**

**Misconception #12: Multiplying All Given Numbers**

*Risk:* Student sees a figure with dimensions 6, 4, 3, 5 and multiplies 6 × 4 × 3 × 5 or adds them carelessly.

*Prevention in Early (Interaction 1.2):*
After student decomposes the figure, system color-codes each rectangle. Guide asks: "Look at the blue rectangle. Which dimensions belong to it?" Student identifies exactly 2 dimensions. System highlights those and asks: "Multiply these two. What do you get?" Repeat for each rectangle. Then: "Add all the areas."

*Prevention in Early (Interaction 1.5 — Dimension-Pairing Check):*
Teacher delivers an incorrect pairing: "A student looked at this shape and said they'd multiply 5 × 4. Is that one of the rectangles?" Student examines: no. System shows why: those two dimensions aren't on the same rectangle. "The numbers on the edges always go with their rectangle. You always multiply just two numbers—the width and height of one rectangle."

*Prevention in Mid & Late:*
Every MC answer choice includes a "multiplied-all-numbers" distractor. "Correct" answer, bounding-box error, random-dim error, partial-calculation error all present. This keeps dimension-pairing active.

**Misconception #11: Completing the Rectangle**

*Risk:* Student draws a bounding rectangle around the composite figure and reports that area, ignoring missing corners.

*Prevention in Mid (Interaction 2.4 — Related Sides Verification):*
Guide demonstrates: "This vertical side says 8. But look: 5 and 3 are on this side. Do 5 and 3 add up to 8?" Student checks: yes. "So all the numbers match. That tells you the rectangles fit together perfectly—no extra space, no missing space. That's how you know your decomposition works."

*Prevention in Late (Interaction 3.3 — Subtraction Decomposition Introduction):*
Teacher shows bounding rectangle (dotted outline, 6 × 8 = 48). "You learned that this doesn't work by itself. But here's another way to think about it: this shape IS inside the big rectangle. There's just a piece missing. Watch: 6 × 8 = 48. Missing corner: 3 × 4 = 12. 48 − 12 = 36. Two strategies, same area." Explicitly reframes: "The bounding rectangle is a tool, not the answer."

**Misconception #6: Overlapping Decomposition**

*Risk:* Student draws decomposition lines that overlap rectangles, counting one piece twice.

*Prevention:*
System validation in all Early and Mid interactions. After student decomposes, system displays color-coded rectangles. If overlap detected: "That line crosses through a rectangle. Let's try again." No area calculation proceeds; student re-decomposes. By Late, students have internalized non-overlapping decomposition and system relies on self-check language: "Do the rectangles touch but not overlap?"

### Interaction 2.3: Related Sides Verification [CONCEPTUAL CHECK]

* **Answer Rationale:** In a composite figure made of non-overlapping rectangles, the sum of partial edge dimensions must equal the full edge dimension. This is a consistency check that validates the decomposition.

4. **Diagnostic log:** If this is the 2nd+ ghost grid trigger across Mid/Late, system flags for M11 reinforcement loop-back. Guide acknowledges: "We're going to come back to grids for a bit so you can build that bridge more slowly."

### Interaction 3.2: Composite T-Shape — New Shape Type [INDEPENDENT PRACTICE]

* **Purpose:** Introduce T-shapes (and later U-shapes). No new method; decomposition is the only tool. A T-shape typically decomposes into 2–3 components (top bar + stem, or left wing + center + right wing). This tests transfer: "If the method works for L-shapes, does it work for T-shapes?" Answer: yes. Students decompose, match dimensions, calculate. No grid. 9 labeled edges (higher complexity than 3.1's 7).

* **Visual: Composite Figures (labels only).** T-shaped composite (symmetric): top bar 8 wide × 3 tall, stem 4 wide × 5 tall centered beneath. Overall bounding box: 8 wide × 8 tall. All edges labeled: top (8), bar-left (3), bar-right (3), left overhang (2), right overhang (2), stem-left (5), stem-right (5), bottom (4), full left height (8 = 3 + 5, related sides). No grid. Shape suggests 2-part decomposition (top bar + stem) or 3-part (left wing + center column + right wing). Students choose their own strategy.

* **Guide:** "New shape. Same strategy. Find the total area."
* **Prompt:** "Find the total area of this T-shape."

* **Student Action:** 1. Decomposes the T (e.g., top bar 8 × 3 = 24 + stem 4 × 5 = 20, or three vertical strips).
2. Identifies dimensions for each piece.
3. Multiplies and adds.

* **Correct Answer:** 44 square units (top bar 8 × 3 = 24, stem 4 × 5 = 20; 24 + 20 = 44). Alternative decomposition: left wing 2 × 3 = 6, center column 4 × 8 = 32, right wing 2 × 3 = 6; 6 + 32 + 6 = 44.

* **Answer Rationale:** T-shapes decompose exactly like L-shapes: split horizontally or vertically into non-overlapping rectangles. The two decompositions yield the same total (44), reinforcing the "multiple valid decompositions" concept seeded in the Design Note. Related-sides check: 3 + 5 = 8 on the left or right full edge confirms non-overlapping decomposition.

* **On Correct:** "Excellent. A new shape, and the same steps work: decompose, multiply, add."

* **Remediation:** Pipeline
  > **Remediation Note:** If student is unsure how to decompose the T:
1. **Guide option A:** "You can split it horizontally, top bar and bottom part."
2. **Guide option B:** "Or split it vertically: left, middle stem, right."
3. **Direct:** "Choose one and decompose it."
4. **Verify:** Student draws or indicates a valid decomposition.
5. **Reconnect:** "Now find the dimensions and multiply."

### Interaction 3.3: Subtraction Decomposition Introduction [WORKED EXAMPLE — SUBTRACTION METHOD]

* **Purpose:** Introduce subtraction decomposition as a *strategic alternative* to addition. Teacher demonstrates: bounding rectangle (6 × 8 = 48) minus missing corner (3 × 4 = 12) equals the composite area (48 − 12 = 36). This addresses Misconception #11 (Completing the Rectangle) by reframing the bounding box as a tool, not an error. Crucially, this is positioned as "two strategies, same area," not "the better method." The scope is explicit: subtraction applies cleanly to L-shapes with a single missing corner; more complex shapes revert to addition decomposition.

* **Visual: Composite Figures (labels only).** L-shaped composite on a labeled diagram. First, the bounding rectangle is shown as a dotted outline (not filled). Dimensions of bounding box: 6 × 8. Then, the missing corner is highlighted and labeled: 3 × 4. The L-shape is highlighted as the remainder.

* **Guide:** "You found the area by breaking into rectangles and adding. Some mathematicians use a different strategy. Watch. [PLANNING] Instead of breaking the L into two pieces, I could think of it as a big rectangle with a piece missing. [ATTENTION] The big rectangle is 6 by 8. That's 48 square units. But this corner is missing, it's 3 by 4. That's 12 square units. So the area of the L is: 48 minus 12. That's 36 square units. [SELF-CHECK] Let me verify: by adding, we got 36. By subtracting, we got 36. Same answer. That tells me both strategies work. Two strategies, same area. Both work. You choose which one makes sense to you."
* **No student action.**

---

> **Scope Note (3.3 — Subtraction Limitation):**
> Subtraction decomposition is only demonstrated and practiced on L-shapes with a single rectangular missing corner. For T-shapes, U-shapes, or shapes with multiple missing regions, guide explicitly directs students back to addition decomposition: "This shape has more than one missing piece, so addition is easier here." This scope containment prevents cognitive overload and keeps the focus on mastery of one strategy at a time.

### Interaction 3.4: Subtraction Decomposition Practice [GUIDED PRACTICE]

* **Purpose:** Student applies subtraction decomposition to an L-shape. Figure is similar to 3.3 (single missing corner), and guide provides minimal prompting: "Find the area using subtraction." Student identifies the bounding rectangle, calculates its area, identifies the missing corner, calculates its area, and subtracts. This is a low-scaffolding practice problem that consolidates the new strategy. By the end, students know both strategies and can choose. Equation Builder displays: "Bounding rectangle: ___ × ___ = ___, Missing corner: ___ × ___ = ___, Area: ___ − ___ = ___"

* **Visual: Composite Figures (labels only).** L-shaped composite. Bounding rectangle shown as a dotted outline (or lightly filled). Missing corner clearly delineated. All dimensions labeled on outer edges and for the missing corner.

* **Guide:** "This time, find the area using subtraction. Start with the bounding rectangle. What's its area? Then subtract the missing corner."

* **Prompt:** "Find the total area of this L-shape using subtraction (bounding rectangle minus missing corner)."

* **Student Action:** 1. Identifies the bounding rectangle dimensions (e.g., 7 × 6 = 42).
2. Identifies the missing corner dimensions (e.g., 4 × 2 = 8).
3. Calculates 42 − 8 = 34 square units.

* **Correct Answer:** 34 square units (or the correct result for the given figure).

* **Answer Rationale:** The bounding rectangle minus the missing corner equals the composite area. This is an alternative decomposition strategy that yields the same answer as addition.

* **On Correct:** "You used a different strategy and got the same area. That's proof both strategies work."

* **Remediation:** Pipeline
  > **Remediation Note:** If student struggles to identify the bounding rectangle:
1. **Guide:** "The bounding rectangle is the smallest rectangle that touches all the corners of your shape. What are its width and height?"
2. **Verify:** Student identifies dimensions.
3. **Reconnect:** "Calculate that area. Then subtract the missing corner."

If student miscalculates subtraction:
1. **Recount:** "You have 42. You subtract 8. Let's count down: 42, 41, 40... or count up: 8 to 42 is... What do you get?"
2. **Verify:** Student recalculates.

---

> **Design Note (3.4 — Strategy Choice Framing):**
> The on-correct line emphasizes "proof"—the fact that both strategies yield the same answer. This builds mathematical confidence and validates choice. It's not "one is better"; it's "you have options."

### Interaction 3.5: Shape with Mixed Dimensions — Patio Context [INDEPENDENT PRACTICE]

* **Purpose:** Late figure with full independence and patio context. This problem tests synthesis: composite figure (L, T, or U), labels-only, real-world scenario (patio paving), and a choice of decomposition strategies (addition or subtraction). Guide provides no scaffolding. By this point, students should fluently apply the method. MC distractors include bounding-box error, random multiplication, and incomplete calculation (e.g., one rectangle's area only).

* **Visual: Composite Figures (labels only).** L-shaped patio in overhead view. "The Martinez family is building an L-shaped patio." Overall 9 ft wide × 10 ft tall. ### **1.7.5 Success Criteria**

By the end of the Lesson, students demonstrate mastery if they can:

1. **Decompose composite figures** into non-overlapping rectangles without grid support, using either addition decomposition (2–3+ pieces) or subtraction decomposition (L-shapes with single missing corner).

2. **Match dimensions correctly** to rectangles, identifying which two numbers on the edges belong to each component piece.

3. **Calculate individual rectangle areas** using multiplication (width × height) without relying on visual counting.

4. **Verify decomposition validity** by checking that related sides (adjacent partial dimensions) add up to the full edge dimension, catching overlaps/gaps before calculation.

5. **Add multiple rectangle areas** to find total composite area, managing multi-step arithmetic accurately.

6. **Apply the method independently** to ungridded figures with 6–10 labeled edges, including complex shapes (T, U, irregular rectilinear).

7. **Apply the method in context** (patio paving) without additional scaffolding, maintaining accuracy across realistic scenarios.

8. **Choose between decomposition strategies** (addition vs. subtraction for L-shapes) based on the figure's structure, demonstrating strategic flexibility.

## **1.10 KEY DESIGN DECISIONS SUMMARY**

**[REQUIRED]**

Master list of cross-cutting design decisions for this module. References phase-level KDDs where relevant. Numbered for easy reference in reviews and feedback.

1. **W.1 has the student draw the decomposition line rather than pre-placing it.** The TVP specifies "Students decompose, calculate, add. Should feel routine" for the Warmup. Pre-placing the decomposition line would skip the skill that needs reactivation before M12's harder context. Drawing the line (drag interaction per Decision #6) reactivates the full M11 workflow, ensuring the student's first M12 decomposition in Early doesn't happen cold.

2. **W.2 is teacher-led with deliberate pacing; 2.1 is "we-do" — together they manage two representational shifts without consecutive passivity.** The grid-removal reveal (W.2) requires teacher narration to connect "I counted 12 squares" with "4 × 3 = 12" — rushing this makes it performative rather than transformative. But having TWO consecutive pure observations (W.2 and 2.1) would create excessive passivity. The we-do structure for 2.1 (teacher models decomposition + dimension identification, student performs multiplication + addition) preserves the modeling benefit for the representational shift while giving students active ownership of the familiar arithmetic.

3. **Early maintains partial grid through all 5 interactions with progressive fading — grid is not removed until Mid.** The Backbone's CRA specification ("Early = partial grid + dimension labels") applies to ALL Early interactions, not just the first. Removing the grid within Early would create a premature representational shift and conflate two learning demands (dimension-pairing AND grid removal). The partial grid fades within Early (from visible first-row/first-column in 1.1 to faintly visible in 1.3/1.5) but never disappears. Full removal happens at the Mid boundary.

4. **Misconception check (1.5) is sequenced after 1.4 (complete full workflow), per pedagogical best practice and KDD specification.** The TVP specifies the dimension-pairing misconception check appears "after 2nd Early problem." Interactions 1.1–1.3 constitute one problem broken into steps; 1.4 is the second full workflow. Placing the misconception check (1.5) after the complete problem gives students the strongest grounding for evaluating an incorrect pairing: they have just executed the full decompose-multiply-add workflow and can apply it to judge whether an attempted pairing makes sense. Sequencing reordered at present assembly to prioritize the full workflow before the misconception check. (Noted as AF-4 in Working Notes; resequenced per pedagogical intent.)

5. **Ghost grid remediation appears in 2.2, 2.5, and 3.1 — threshold is "2+ in same section" for M11 loop-back.** Ghost grid (guide-activated faint grid overlay) is M12's primary remediation mechanism for the representational gap. It appears as the final tier in remediation pipelines for the first student-solves labels-only problem (2.2), the last Mid problem (2.5), and the first Late problem (3.1). The "2+ in same section" threshold (AF-1) means a student who needs ghost grid once in Mid and once in Late is acclimating normally, but a student who needs it twice within Mid signals genuine struggle requiring M11 reinforcement.

6. **Dimension highlighting switches from automatic (Early/Mid) to on-demand (Late).** In Early and Mid, the system automatically color-codes rectangles and highlights their dimension labels after decomposition — this reduces cognitive load during the transition to labels-only. In Late, highlighting is on-demand (student taps a rectangle to see its dimensions highlighted). This shift matches the module's independence target: by Late, students actively identify dimension pairings rather than receiving them.

7. **Subtraction decomposition is NOT tested in EC.** The TVP introduces subtraction decomposition in Late as a "strategic alternative." The EC Playbook requires "representative middle" difficulty. Subtraction decomposition is a Late-introduced extension, not the module's core workflow (additive decomposition). Testing it in EC would overweight a skill students practiced once, and would also require COMPARE cognitive type (comparing strategies) when the EC's purpose is checking the core decompose-calculate-add competency.

8. **Two CREATE problems in EC (EC.1 and EC.2) despite the Playbook's "different cognitive type" preference.** M12's core assessment variable is the representational level (partial grid vs. labels only), not the cognitive type. EC.1 tests CREATE at partial-grid level; EC.2 tests CREATE at labels-only level. These are genuinely different assessments — the same workflow tested against two visual support levels. Using IDENTIFY for EC.1 (e.g., pre-placing the decomposition) would skip the decomposition step, which is the skill M12 needs to assess. The pedagogical value of testing at both abstraction levels outweighs the type-diversity preference.

9. **EC uses L-shapes only, carrying forward M11's scope decision.** T-shapes and U-shapes were introduced in Late (3.2, 3.6). The EC Playbook requires "representative middle" — L-shapes represent the full module span (Warmup through Late); T/U-shapes represent Late extension only. Same reasoning as M11's KDD-8.

10. **Purpose Frame does not preview subtraction decomposition.** The original Purpose Frame mentioned "a completely different strategy." This was removed because naming the subtraction strategy before any Lesson interactions primes student attention toward "What's the different strategy?" throughout Early and Mid, when focus should be on dimension-pairing. The subtraction strategy arrives as a discovery moment in Late (3.3), which is more pedagogically effective than a 10-minute-early preview.

11. **Related-sides verification follows a three-beat thread: introduction (2.3) → reinforcement (2.4) → internalization (3.5).** The TVP specifies horizontal check, then vertical check, then Late self-check prompt. The SP threads "related sides add up" across three interactions: 2.3 introduces it on horizontal edges, 2.4 reinforces on vertical edges (after patio problem), 3.5 prompts the student to self-check. This mirrors the preview-then-formalize pattern used throughout the unit ("rows of" M3 → "×" M4; "related sides" M12 → formal vocabulary M13).

12. **Synthesis uses Type D (Representation Transfer) as primary task because M12's defining achievement is the CRA progression itself.** M12's central insight is that the same mathematics works across three visual representations (gridded, partial grid, dimensions only). Type D is the natural synthesis structure for this — "what stayed the same?" across representations. Type A (Pattern Discovery) would work for figure-type invariance (M11's approach), but M12's invariant is the REPRESENTATION, not the figure type.

13. **M13 bridge uses "related sides" as the conceptual hinge.** "Related sides add up" was introduced informally in M12 (Mid, 2.3) as a verification tool. The Synthesis closure transforms it into an anticipation tool: "What if some sides didn't have measurements? Could you use that same idea to figure out the missing ones?" This completes the preview-then-formalize pattern — M12 establishes the concept, M13 formalizes and extends it. The bridge language matches the TVP transition out exactly.
